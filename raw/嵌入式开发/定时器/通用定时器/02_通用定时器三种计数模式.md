# 通用定时器三种计数模式 笔记

---

## 一、计数模式总览

| 模式 | CRS 位设置 | DIR 位 | 说明 |
|------|-----------|:------:|------|
| **向上计数** | CMS = 00 | 0 | 从 0 递增到 ARR（默认） |
| **向下计数** | CMS = 00 | 1 | 从 ARR 递减到 0 |
| **中央对齐** | CMS = 01/10/11 | — | 先增到 ARR-1 再减回 0 |

> CMS 和 DIR 均在控制寄存器 **CR1** 中配置。

---

## 二、向上计数模式（默认）

### 2.1 计数过程

```
CNT: 0 → 1 → 2 → ... → ARR → [溢出] → 0 → 1 → 2 → ...
                              ↓
                       产生更新事件 (UEV)
                       UIF = 1（若开启中断则产生中断）
```

### 2.2 时序图（ARR = 36，预分频 = 2）

```
定时器时钟:  ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐
             └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘
              ↑   ↑   ↑   ↑   ↑   ↑       ↑
CNT:         34  35  36  37→溢出 0   1     ...
                            ↓
                      更新事件 (UEV)
                      CNT 归零
                      UIF = 1
```

### 2.3 关键说明

| 项目 | 说明 |
|------|------|
| 起始值 | 0 |
| 递增到 | ARR |
| 溢出条件 | CNT 超过 ARR（ARR + 1 个计数周期后溢出） |
| 溢出后 | CNT 归零，产生更新事件 |
| 更新事件次数 | 每 ARR + 1 个计数周期产生一次 |

### 2.4 CR1.DIR 位

```
DIR = 0 → 向上计数（复位默认值）
```

## 三、向下计数模式

### 3.1 计数过程

```
CNT: ARR → ARR-1 → ... → 1 → 0 → [溢出] → ARR → ARR-1 → ...
                            ↓
                     产生更新事件 (UEV)
                     UIF = 1
```

### 3.2 时序图（ARR = 36，预分频 = 2）

```
定时器时钟:  ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐
             └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘
              ↑   ↑   ↑   ↑   ↑       ↑   ↑
CNT:          2   1   0  溢出→36     35  34
                    ↓
              更新事件 (UEV)
              CNT 重装载为 ARR
              UIF = 1
```

### 3.3 关键说明

| 项目 | 说明 |
|------|------|
| 起始值 | ARR（从 ARR 值开始递减） |
| 递减到 | 0 |
| 溢出条件 | CNT 减到 0 后再减（ARR + 1 个计数周期后溢出） |
| 溢出后 | CNT 重装载为 ARR，产生更新事件 |

### 3.4 与向上计数对比

| 对比项 | 向上计数 | 向下计数 |
|--------|----------|----------|
| 起始值 | 0 | ARR |
| 终点 | ARR | 0 |
| 溢出方向 | ARR → 0（归零） | 0 → ARR（重装载） |
| 计数次数 | ARR + 1 | ARR + 1 |
| CR1.DIR | 0 | 1 |

## 四、中央对齐模式（向上/向下交替）

### 4.1 计数过程

```
CNT: 0 → 1 → ... → ARR-1 → [向上溢出] → ARR → ARR-1 → ... → 1 → [向下溢出] → 0 → ...
```

### 4.2 时序图（ARR = 6，预分频 = 1）

```
定时器时钟:  ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐
             └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘

CNT:         00  01  02  03  04  05  06  05  04  03  02  01  00  01
              │   ↑   ↑   ↑   ↑   ↑   ↓   ↓   ↓   ↓   ↓   │   ↑
              │   向上递增 ──────────→ │ ←── 向下递减        │   ...
              │                       │                     │
              │                   向上溢出               向下溢出
              │                  (UEV, UIF=1)          (UEV, UIF=1)
              │                  CNT 设为 ARR           CNT 设为 0
              │
              └── 开始计数

一个完整周期：0 → 1 → 2 → 3 → 4 → 5 → 6 → 5 → 4 → 3 → 2 → 1 → 0
              共 12 个计数周期 = 2 × ARR
```

### 4.3 关键说明

| 项目 | 说明 |
|------|------|
| 向上阶段 | 从 0 递增到 **ARR - 1**（不是 ARR！） |
| 向上溢出 | 到达 ARR-1 后，下一个时钟周期产生向上溢出 |
| 溢出后 CNT | 设为 **ARR**（不是归零） |
| 向下阶段 | 从 ARR 递减到 **1**（不是 0！） |
| 向下溢出 | 到达 1 后，下一个时钟周期产生向下溢出 |
| 溢出后 CNT | 设为 **0**（不是 ARR） |
| 一个完整周期 | 2 × ARR 个计数周期 |

### 4.4 为什么不需要减 1

```
向上计数模式：
  设 ARR = 10 → CNT 从 0 计到 10 → 共 11 次 → 需要 ARR = 9

中央对齐模式：
  设 ARR = 10 → 向上到 9（ARR-1）→ 向下到 0 → 共 10 次 → 直接设 10

中央对齐模式天然有减 1 处理，ARR 值就是实际计数次数
```

### 4.5 中央对齐模式的 CMS 配置

| CMS 值 | 模式 | 说明 |
|:------:|------|------|
| 00 | 向上/向下计数禁用 | 使用 DIR 选择向上或向下 |
| 01 | 中央对齐模式 1 | 仅在向下计数时产生比较匹配 |
| 10 | 中央对齐模式 2 | 仅在向上计数时产生比较匹配 |
| 11 | 中央对齐模式 3 | 向上和向下都产生比较匹配 |

> CMS ≠ 00 时，DIR 位无效，计数方向由硬件自动控制。

## 五、三种模式对比

| 对比项 | 向上计数 | 向下计数 | 中央对齐 |
|--------|:--------:|:--------:|:--------:|
| 计数方向 | 单向递增 | 单向递减 | 交替增减 |
| 起始值 | 0 | ARR | 0 |
| 溢出条件 | 超过 ARR | 减到 0 以下 | 到 ARR-1（上溢）/ 到 1（下溢） |
| 更新事件频率 | 每 ARR+1 次 | 每 ARR+1 次 | 每 2×ARR 次 |
| CR1.DIR | 0 | 1 | 无效 |
| CR1.CMS | 00 | 00 | 01/10/11 |
| 常用程度 | **最常用** | 较少 | PWM 电机控制 |

## 六、更新事件产生的位置

```
向上计数：
  0 → 1 → ... → ARR → [溢出] → 0
                    ↑
               此处产生更新事件

向下计数：
  ARR → ARR-1 → ... → 0 → [溢出] → ARR
                        ↑
                   此处产生更新事件

中央对齐：
  0 → ... → ARR-1 → [向上溢出] → ARR → ... → 1 → [向下溢出] → 0
              ↑                                    ↑
         此处产生更新事件                      此处产生更新事件

  中央对齐模式在一个完整周期内产生 2 次更新事件
```

## 七、CR1 寄存器相关位

| 位 | 名称 | 功能 |
|:--:|------|------|
| 0 | **CEN** | 计数器使能 |
| 1 | UDIS | 禁止更新 |
| 2 | URS | 更新请求源 |
| 3 | OPM | 单脉冲模式 |
| 4 | **DIR** | 计数方向（0=向上，1=向下） |
| 6:5 | **CMS** | 中央对齐模式选择 |
| 7 | ARPE | 自动重装载预装载使能 |

```
DIR 和 CMS 的关系：
  CMS = 00 时：由 DIR 决定方向（向上或向下）
  CMS ≠ 00 时：DIR 无效，硬件自动控制方向（中央对齐）
```

---

## 八、应用场景

| 模式 | 典型应用 |
|------|----------|
| **向上计数** | 定时中断、PWM 输出、输出比较、输入捕获 |
| **向下计数** | SysTick 类似功能、特定定时需求 |
| **中央对齐** | **电机控制**（FOC）、对称 PWM 波形生成、三相逆变器控制 |

> 日常使用中，向上计数模式最为常见，能满足绝大多数定时和 PWM 需求。
> 中央对齐模式主要在电机控制等需要对称波形的场景中使用。

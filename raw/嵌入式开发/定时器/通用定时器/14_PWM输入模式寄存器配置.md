## 三、寄存器配置详解

### 3.1 CCMR1 — 通道模式配置

```
CCMR1 寄存器（16位）：
  低 8 位 [7:0]  → 通道 1 配置
  高 8 位 [15:8] → 通道 2 配置
```

#### 通道 1：CC1S = 01（直连）

```
CC1S[1:0] = 01

含义：IC1 映射到 TI1 上
通路：TI1 → 滤波 → 边沿检测 → TI1FP1 → IC1 → CCR1
      （直连 / Direct Mode）

CCR1 捕获：上升沿 → 周期 T
```

#### 通道 2：CC2S = 10（交叉）

```
CC2S[1:0] = 10

含义：IC2 映射到 TI1 上（交叉连接）
通路：TI1 → 滤波 → 边沿检测 → TI1FP2 → IC2 → CCR2
      （交叉 / Indirect Mode）

⚠️ CCxS 的定义因通道而异：
  对于通道 1：
    01 = IC1 映射到 TI1（直连）
    10 = IC1 映射到 TI2（交叉）
  对于通道 2：
    01 = IC2 映射到 TI2（直连）
    10 = IC2 映射到 TI1（交叉）← 本实验用这个

CCR2 捕获：下降沿 → 高电平时间 t
```

#### CCMR1 完整配置

| 位域 | 值 | 功能 |
|------|:--:|------|
| CC1S[1:0] | **01** | 通道 1 直连（TI1→IC1） |
| IC1F[3:0] | 0000 | 不滤波 |
| IC1PSC[1:0] | 00 | 不分频 |
| CC2S[1:0] | **10** | 通道 2 交叉（TI1→IC2） |
| IC2F[3:0] | 0000 | 不滤波 |
| IC2PSC[1:0] | 00 | 不分频 |

### 3.2 CCER — 极性和使能配置

| 位 | 通道 1 | 通道 2 | 说明 |
|:--:|:------:|:------:|------|
| CCxP | **0** | **1** | 上升沿 / **下降沿** |
| CCxE | **1** | **1** | 都使能 |

```c
// 配置代码
TIM4->CCER &= ~TIM_CCER_CC1P;    // 通道 1 上升沿
TIM4->CCER |=  TIM_CCER_CC2P;    // 通道 2 下降沿 ← 新增
TIM4->CCER |=  TIM_CCER_CC1E;    // 通道 1 使能
TIM4->CCER |=  TIM_CCER_CC2E;    // 通道 2 使能 ← 新增
```

### 3.3 SMCR — 触发源与从模式配置

| 位域 | 值 | 功能 |
|------|:--:|------|
| TS[2:0] | **101** | 触发源选择 TI1FP1 |
| SMS[2:0] | **100** | 从模式选择复位模式 |

```c
// 配置代码
TIM4->SMCR |= (0b101 << 4);   // TS = 101（TI1FP1）
TIM4->SMCR |= (0b100 << 0);   // SMS = 100（复位模式）
```

## 四、两个关键寄存器总结

### 4.1 CCMR1 — "怎么接线"

```
CCMR1 控制的是信号的通路选择：
  CC1S → 通道 1 的信号来源（直连/交叉）
  CC2S → 通道 2 的信号来源（直连/交叉）
  ICxF → 滤波器
  ICxPSC → 输入预分频

类比：多路选择开关，决定信号走哪条路
```

### 4.2 SMCR — "触发源 + 从模式"

```
SMCR 控制的是触发行为：
  TS[2:0] → 选择哪个信号作为触发输入（8选1）
  SMS[2:0] → 选择触发后执行什么动作（8种模式）

类比：先选信号源（遥控器），再选执行动作（开机/关机/复位）
```

### 4.3 两者的协作关系

```
CCMR1 配置的信号 → 经过极性选择(CCER) → TI1FP1
                                          │
                                          ├──→ CCR1 捕获（直连）
                                          │
                                          └──→ TRGI（SMCR.TS 选择）
                                                  │
                                                  ↓
                                            从模式控制器
                                            （SMCR.SMS 配置）
                                                  │
                                                  ↓
                                              CNT 复位清零
```

## 六、不需要的配置

| 配置项 | 原因 |
|--------|------|
| **中断（DIER.CC1IE）** | 硬件自动复位，无需中断 |
| **NVIC** | 不用中断，不需要配置 NVIC |
| **全局变量** | CCR1/CCR2 硬件自动更新 |

## 七、读取结果

```c
// 无需中断，主循环直接读取
uint32_t ccr1 = TIM4->CCR1;    // 周期（μs）
uint32_t ccr2 = TIM4->CCR2;    // 高电平时间（μs）

double period    = (double)ccr1 / 1000.0;           // ms
double frequency = 1000000.0 / (double)ccr1;        // Hz
double dutyCycle = (double)ccr2 / (double)ccr1;     // 0.0~1.0
```

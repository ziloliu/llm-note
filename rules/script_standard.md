# 脚本编写与存放规范

## 定位
`scripts/` 文件夹统一存放所有自动化脚本，保持知识库根目录整洁。

## 一、脚本存放规则

### 1.1 统一路径
所有脚本一律放入 `scripts/`，禁止散落在知识库根目录或其他文件夹。

### 1.2 子文件夹（可选）
当脚本数量较多时，可按用途分文件夹：
```
scripts/
├── raw/            # 处理 raw 入库相关的脚本
├── extract/        # 提炼相关的脚本
├── index/          # 索引生成相关的脚本
└── utils/          # 通用工具脚本
```
脚本数量少时（<10个）不必分子文件夹，平铺即可。

---

## 二、脚本命名规范

- 使用小写字母与下划线
- 文件名能清晰表达功能
- 格式：`动词_名词.py`（或 `.sh` 等）
- 示例：
  ```
  ✅ rename_and_sort.py        – 重命名并排序 raw 文件
  ✅ extract_to_core.py         – 从 raw 提炼至 core
  ✅ generate_index.py          – 生成 index 索引
  ✅ check_broken_links.py      – 检查失效链接

  ❌ script1.py                 – 含义不明
  ❌ do_stuff.py                – 太模糊
  ```

---

## 三、脚本编写基本要求

### 3.1 解释器声明
首行指定解释器：
```python
#!/usr/bin/env python3
```

### 3.2 文档字符串
每个脚本开头必须用多行注释说明：
```python
"""
用途：对 raw/ 下文件进行重命名规范化检查，确保符合 序号_标题.md 格式
用法：python rename_check.py [--dry-run] [--path raw/]
参数：
  --dry-run   仅检查不实际修改，打印建议
  --path      指定检查路径，默认 raw/
"""
```

### 3.3 注释习惯
- 关键步骤前加简要注释
- 非显而易见的逻辑需要解释
- AI 生成代码时保持注释风格一致

## 四、脚本说明书（重要）

### 4.1 说明书文件
在 `scripts/` 下维护一个 `README.md`，记录**每个脚本的关键信息**。

### 4.2 必须记录的内容
每个脚本一条记录，包含以下字段：

| 字段 | 说明 |
|------|------|
| **脚本名** | 文件名 |
| **用途** | 一句话说明脚本做什么 |
| **使用方式** | 命令行示例或调用方式 |
| **输入** | 读取哪个文件夹/文件 |
| **输出** | 生成什么文件或效果 |
| **依赖** | 需要哪些库或环境（Python版本、pip包等） |
| **状态** | 见下方状态标记 |
| **最后更新** | 日期 |
| **备注** | 任何需要补充的信息 |

### 4.3 状态标记
| 状态 | 含义 |
|------|------|
| ✅ 可用 | 测试过，正常使用中 |
| ⚠️ 待验证 | 已生成但未充分测试 |
| ❌ 已废弃 | 不再使用，保留仅供参考 |
| 🔧 开发中 | 正在编写或调试 |

### 4.4 说明书格式示例

文件：`scripts/README.md`

```markdown
# 脚本目录说明

> 最后更新：2026-06-02

## 脚本清单

### rename_check.py
- **用途**：检查 raw/ 下文件命名是否符合规范，支持自动修复
- **使用方式**：`python rename_check.py --path raw/ --fix`
- **输入**：raw/ 文件夹
- **输出**：重命名后的文件 + 终端报告
- **依赖**：Python 3.8+
- **状态**：✅ 可用
- **最后更新**：2026-05-28
- **备注**：使用 --dry-run 可预览改动

### extract_to_core.py
- **用途**：读取 raw/ 笔记，提炼核心内容写入 core/
- **使用方式**：`python extract_to_core.py --topic 机器学习`
- **输入**：raw/ 下指定主题的笔记
- **输出**：core/ 下对应主题的 md 文件
- **依赖**：Python 3.10+, openai
- **状态**：⚠️ 待验证
- **最后更新**：2026-06-01
- **备注**：需要配置 API KEY 环境变量

### old_format_converter.py
- **用途**：早期命名格式转换为新格式（已不再需要）
- **使用方式**：`python old_format_converter.py`
- **输入**：raw/ 下的旧格式文件
- **输出**：转换后的文件
- **依赖**：Python 3.6+
- **状态**：❌ 已废弃
- **最后更新**：2025-12-10
- **备注**：所有文件已转换完毕，保留供参考
```

## 五、脚本与日志的关系

- 脚本执行时应输出操作摘要
- AI 调用脚本后，必须在 `log/` 中记录执行情况
- 记录格式：`[时间戳] SCRIPT [状态] scripts/脚本名 / 摘要`

---

## 六、维护原则

- **脚本说明书必须与脚本同步更新**：新增脚本 → 立即在 README 中追加条目；废弃脚本 → 更新状态标记，不删除记录
- **说明书的修改视同 rules 修改**，修改后记录日志
- **定期清理**：每季度检查一次，确认标记为废弃的脚本是否可以移除

# STM32 系统滴答定时器（SysTick） 笔记

---

## 一、SysTick 基本概念

### 1.1 归属与定位

| 特性 | 说明 |
|------|------|
| 归属 | **ARM Cortex-M3 内核**（非 STM32 外设） |
| 位置 | 内嵌在 **NVIC**（嵌套向量中断控制器）中 |
| 位宽 | **24 位**计数器 |
| 计数方向 | **向下递减** |
| 前面课程 | 已在基础篇实现过 `Delay_us()` / `Delay_ms()` |

### 1.2 与其他定时器的区别

```
SysTick（系统滴答定时器）
  ├── 归属：Cortex-M3 内核
  ├── 位宽：24 位
  └── 计数方向：向下递减

TIM1~TIM8（外设定时器）
  ├── 归属：STM32 外设
  ├── 位宽：16 位
  └── 计数方向：可向上/向下/中央对齐
```

## 二、工作原理

### 2.1 基本流程

```
① 给 LOAD 寄存器设置一个初始装载值
② 计数器从装载值开始向下递减
③ 每次滴答（时钟周期）减 1
④ 减到 0 时：
   → COUNTFLAG 标志位置 1
   → 若开启了中断，产生一次中断
   → 自动重新装载初始值，继续递减（循环往复）
```

### 2.2 计数时间间隔

```
系统时钟频率 = 72 MHz（8MHz 晶振 × PLL 9 倍频）

每次滴答时间间隔 = 1 / 72MHz = 1/72 μs ≈ 0.0139 μs

即：每 1/72 微秒，计数器减 1
```

### 2.3 定时时间计算

```
定时时间 = 装载值 × (1 / 系统时钟频率)

例：
  要定时 1 μs → 装载值 = 72    → 72 × (1/72)μs = 1μs
  要定时 10 μs → 装载值 = 720   → 720 × (1/72)μs = 10μs
  要定时 1 ms  → 装载值 = 72000 → 72000 × (1/72)μs = 1000μs = 1ms

通用公式：
  装载值 = 目标时间(μs) × 72
```

## 四、中断机制

### 4.1 在中断向量表中的位置

```
Cortex-M3 内部异常（中断向量表前 16 项）：

  位置 0:  初始 SP
  位置 1:  Reset
  位置 2:  NMI
  位置 3:  HardFault
  位置 4:  MemManage
  位置 5:  BusFault
  位置 6:  UsageFault
  位置 7~10: 保留
  位置 11: SVCall
  位置 12: Debug Monitor
  位置 13: 保留
  位置 14: PendSV
  位置 15: SysTick  ← 系统滴答定时器中断
```

### 4.2 中断触发条件

```
计数器从 1 递减到 0 时：
  → COUNTFLAG 位自动置 1
  → 如果 TICKINT 位 = 1（开启中断）→ 触发 SysTick 异常
  → 如果 TICKINT 位 = 0（关闭中断）→ 仅置标志位，不触发中断
```

## 五、SysTick 的主要用途

### 5.1 操作系统心跳（时基）

| 用途 | 说明 |
|------|------|
| **时基（Time Base）** | 操作系统的底层时钟节拍 |
| **心跳（Heartbeat）** | 维持系统运行节奏 |
| **任务调度** | RTOS 中用于时间片轮转 |

```
CubeMX 配置中的 Time Base 源：
  SYS → Parameter Settings → Timebase Source
  默认选择 SysTick

作用：
  → HAL_Delay() 依赖 SysTick
  → 操作系统任务调度依赖 SysTick
  → 超时检测依赖 SysTick
```

### 5.2 操作系统心跳的作用

```
操作系统运行时：
  ├── 等待外部请求 → 需要心跳维持响应
  ├── 程序跑飞检测 → 心跳异常可发现
  ├── 睡眠/唤醒 → 心跳节拍控制
  └── 任务切换 → 基于心跳的时间片轮转

如果没有心跳：
  → 无法感知时间流逝
  → 无法调度多个任务
  → 系统可能陷入死循环或睡眠
```

### 5.3 其他用途

| 用途 | 说明 |
|------|------|
| **延时函数** | `Delay_us()` / `Delay_ms()`（基础篇已实现） |
| **闹钟** | 定时触发事件 |
| **时间测量** | 测量某段代码或信号的执行时间 |
| **周期性任务** | 周期性触发中断，执行特定操作 |

## 六、SysTick 与其他定时器功能对比

| 功能 | SysTick | 基本定时器 | 通用定时器 | 高级定时器 |
|------|:-------:|:---------:|:---------:|:---------:|
| 定时 | ✅ | ✅ | ✅ | ✅ |
| 产生中断 | ✅ | ✅ | ✅ | ✅ |
| 输入捕获 | ❌ | ❌ | ✅ | ✅ |
| 输出比较 | ❌ | ❌ | ✅ | ✅ |
| PWM 输出 | ❌ | ❌ | ✅ | ✅ |
| 脉冲计数 | ❌ | ❌ | ✅ | ✅ |
| 死区控制 | ❌ | ❌ | ❌ | ✅ |
| 紧急刹车 | ❌ | ❌ | ❌ | ✅ |
| 驱动 DAC | ❌ | ✅ | — | — |
| 操作系统时基 | ✅ | — | — | — |

> SysTick 功能最简单，只做定时和中断，但它是操作系统的心跳来源。

## 七、SysTick 定时总结

```
SysTick = 24 位向下计数器
          + 自动重装载
          + 可选中断
          + 操作系统时基

基本参数：
  位宽：24 位（最大装载值 = 2^24 - 1 = 16,777,215）
  时钟：72 MHz
  最小定时：1/72 μs ≈ 0.014 μs
  最大定时：16,777,215 / 72 μs ≈ 233 ms

工作模式：
  单次定时（不重装载）
  循环定时（自动重装载）
```

# PWM 输入模式 — 同时测量周期和占空比原理详解 笔记

---

## 一、核心需求

```
PWM 方波的三个关键参数：
  ① 周期 T = 连续两个上升沿之间的时间
  ② 频率 f = 1 / T（与周期互为倒数）
  ③ 占空比 D = 高电平时间 t / 周期 T

已知：周期可以通过输入捕获 + 上升沿检测实现
目标：在测周期的基础上，增加测量占空比的能力
```

---

## 二、占空比测量的核心思路

### 2.1 需要测量的两个量

```
PWM 信号：  ┌──────┐                ┌──────┐
            │  t   │                │  t   │
       ─────┘      └────────────────┘      └──────
            ←── T ──→←────── T ─────→
            
  T = 周期（两个上升沿之间）
  t = 高电平时间（上升沿到紧接着的下降沿之间）
  占空比 = t / T
```

### 2.2 测量方案

| 测量目标 | 触发边沿 | 捕获到的寄存器 | 配置 |
|----------|:--------:|:--------------:|------|
| 周期 T | **上升沿** | CCR1 | 通道 1，直连，上升沿 |
| 高电平时间 t | **下降沿** | CCR2 | 通道 2，交叉，下降沿 |

### 2.3 前提条件

```
① 每个上升沿到来时，CNT 必须清零（复位）
   → 计数器从零开始，测量才有意义

② 下降沿到来时，CCR2 捕获当前 CNT 值
   → 即高电平时间 t

③ 下一个上升沿到来时，CCR1 捕获当前 CNT 值
   → 即周期 T，同时 CNT 再次清零
```

## 三、信号通路设计

### 3.1 同一信号，两个通道，两种极性

```
CH1 引脚 (PB6) 输入 PWM 方波
    │
    ↓
输入滤波器（不滤波）
    │
    ├──────────────────────────────────────┐
    │                                      │
    ↓                                      ↓
上升沿检测 (CC1P=0)               下降沿检测 (CC2P=1)
    │                                      │
    ↓                                      ↓
  TI1FP1                                TI2FP2
    │                                      │
    ├→ 直连 (CC1S=01) → IC1 → CCR1       │
    │   （测周期 T）                       │
    │                                      │
    └→ 触发输入 (TS=101) → TRGI           │
         │                                │
         ↓                                │
    从模式控制器 (SMS=100)                 │
         │                                │
         ↓                                ↓
    CNT 清零（复位）          交叉 (CC2S=10) → IC2 → CCR2
                                      （测高电平时间 t）
```

### 3.2 关键配置点

| 配置点 | 作用 | 说明 |
|--------|------|------|
| **直连 vs 交叉** | 决定 CCR 的信号来源 | CH1 直连→CCR1, CH2 交叉→CCR2 |
| **极性选择** | 决定检测哪种边沿 | 通道 1=上升沿, 通道 2=下降沿 |
| **触发源 TS=101** | 选择 TI1FP1 作为 TRGI | 上升沿信号控制复位 |
| **从模式 SMS=100** | 选择复位模式 | TRGI 上升沿→CNT 清零 |

## 四、为什么需要触发输入和从模式

### 4.1 仅用输入捕获的问题

```
问题：通道 1 检测上升沿，通道 2 检测下降沿
     → 两个通道各自独立，都是检测"相邻同种边沿"
     → 通道 1：连续上升沿 = 周期 ✅
     → 通道 2：连续下降沿 = 也是周期 ❌（不是高电平时间）

解决：必须在每个上升沿时将 CNT 清零
     → 这样通道 2 检测到的下降沿 CNT 值 = 高电平时间
     → 通道 1 检测到的上升沿 CNT 值 = 周期
```

### 4.2 中断清零 vs 硬件复位

| 对比项 | 中断清零（之前方案） | 硬件复位（从模式） |
|--------|:-------------------:|:-----------------:|
| CNT 清零方式 | 软件（中断服务函数） | **硬件（TRGI 信号）** |
| 响应延迟 | 若干时钟周期 | **瞬间（1 个周期）** |
| 是否需要中断 | ✅ | **❌** |
| 是否需要全局变量 | ✅ | **❌** |
| 精度 | 受中断延迟影响 | **更高** |
| CPU 占用 | 有中断开销 | **几乎无** |

### 4.3 硬件复位的时序保证

```
同一个上升沿同时触发两个硬件动作：

① 输入捕获事件：CCR1 = CNT（快照）  ← 先执行（硬件瞬间）
② 复位模式：     CNT = 0（清零）      ← 后执行

硬件保证：① 先于 ②
→ CCR1 保存的是清零前的正确周期值
→ 不会出现 CCR1=0 的错误
```

## 五、完整时序分析

### 5.1 时序图

```
PWM 信号：  ┌──────┐                ┌──────┐
            │      │                │      │
       ─────┘      └────────────────┘      └──────
            ↑      ↑                ↑      ↑
            ①      ②                ③      ④

CNT：       0─→─→─→2               0─→─→─→2
                   ↓继续              ↓继续
              3─→─→4               3─→─→4

CCR1：           (上次周期)              4（周期）
CCR2：           2（高电平）             2（高电平）

硬件动作：
  ① 上升沿 → CCR1=上次周期 + CNT=0（复位）
  ② 下降沿 → CCR2=2（高电平时间）
  ③ 上升沿 → CCR1=4（周期）+ CNT=0（复位）
  ④ 下降沿 → CCR2=2（高电平时间）
```

### 5.2 每一步详解

| 步骤 | 事件 | 通道 1 动作 | 通道 2 动作 | CNT 状态 |
|:----:|------|:-----------:|:-----------:|:--------:|
| ① | 上升沿 | CCR1=上次周期 | 无 | **清零=0** |
| — | 计数中 | — | — | 0→1→2 |
| ② | 下降沿 | 无 | **CCR2=2** | 继续 |
| — | 计数中 | — | — | 3→4 |
| ③ | 上升沿 | **CCR1=4** | 无 | **清零=0** |
| — | 计数中 | — | — | 0→1→2 |
| ④ | 下降沿 | 无 | **CCR2=2** | 继续 |

### 5.3 为什么 CCR2 不会捕获到错误值

```
疑问：第二次下降沿（如果有）会不会改变 CCR2？

关键：上升沿的复位发生在第二次下降沿之前

时间线：
  上升沿① → 下降沿② → 上升沿③ → 下降沿④ → ...
     │          │          │
     CNT=0      CCR2=2     CCR1=4 + CNT=0

③ 的上升沿在 ④ 的下降沿之前到来
→ ③ 到来时 CNT 被复位为 0
→ ④ 的下降沿到来时，CCR2 再次捕获正确的高电平时间
→ CCR2 始终保存"从复位开始到第一个下降沿"的时间
```

## 六、涉及的关键模块

### 6.1 下方：输入捕获通道

```
两个通道共用一个输入引脚 (CH1)
通道 1（直连 CC1S=01）：
  → 检测上升沿（CC1P=0）
  → CCR1 捕获周期 T

通道 2（交叉 CC2S=10）：
  → 信号来自通道 1 引脚
  → 检测下降沿（CC2P=1）
  → CCR2 捕获高电平时间 t
```

### 6.2 上方：触发输入和从模式

```
触发输入源：TI1FP1（TS=101）
  → 来自通道 1 的上升沿信号
  → 作为 TRGI 给到触发控制器

从模式：复位模式（SMS=100）
  → TRGI 上升沿 → CNT 硬件自动清零
  → 同时产生更新寄存器信号
```

### 6.3 模块协作关系

```
┌─────────────────────────────────────────────┐
│               触发输入和从模式                 │
│                                              │
│  TI1FP1 → TRGI(TS=101) → 复位模式(SMS=100)  │
│                              │               │
│                              ↓               │
│                          CNT 清零            │
└──────────────────────────────────────────────┘
                    ↕ 控制
┌──────────────────────────────────────────────┐
│               输入捕获通道                     │
│                                              │
│  CH1引脚 → 上升沿 → CCR1 = 周期 T            │
│  CH1引脚 → 下降沿 → CCR2 = 高电平时间 t      │
│                                              │
│  占空比 = CCR2 / CCR1                        │
└──────────────────────────────────────────────┘
```

## 七、寄存器配置总览

### 7.1 SMCR（从模式控制寄存器）

| 位域 | 值 | 功能 |
|------|:--:|------|
| TS[2:0] | **101** | 触发源选择 TI1FP1 |
| SMS[2:0] | **100** | 从模式选择复位模式 |

### 7.2 CCMR1（捕获/比较模式寄存器 1）

| 位域 | 值 | 功能 |
|------|:--:|------|
| CC1S[1:0] | **01** | 通道 1 直连模式 |
| CC2S[1:0] | **10** | 通道 2 交叉模式 |
| IC1F[3:0] | 0000 | 不滤波 |
| IC2F[3:0] | 0000 | 不滤波 |
| IC1PSC[1:0] | 00 | 不分频 |
| IC2PSC[1:0] | 00 | 不分频 |

### 7.3 CCER（捕获/比较使能寄存器）

| 位 | 通道 1 | 通道 2 | 功能 |
|:--:|:------:|:------:|------|
| CCxP | **0** | **1** | 上升沿 / 下降沿 |
| CCxE | **1** | **1** | 两个通道都使能 |

### 7.4 时基配置

| 寄存器 | 值 | 功能 |
|--------|:--:|------|
| PSC | **71** | 72 分频 → 1MHz（1μs 精度） |
| ARR | **65535** | 最大计数范围，防溢出 |

## 八、CCMR 中多路选择的三种模式

### 8.1 CCxS 配置

| CCxS[1:0] | 模式 | 信号来源 | 本实验使用 |
|:---------:|------|----------|:----------:|
| **00** | 输出 | — | ❌ |
| **01** | 直连输入 | 本通道信号 → 本通道 CCR | ✅ 通道 1 |
| **10** | 交叉输入 | 其他通道信号 → 本通道 CCR | ✅ 通道 2 |
| 11 | TRC | 触发控制器输出 | ❌ |

### 8.2 本实验的多路选择

```
通道 1（CC1S=01，直连）：
  CH1 引脚 → 滤波 → 上升沿检测 → TI1FP1 → IC1 → CCR1
  
通道 2（CC2S=10，交叉）：
  CH1 引脚 → 滤波 → 下降沿检测 → TI2FP2 → IC2 → CCR2
  （注意：信号来源是通道 1 引脚，但经过通道 2 的边缘检测和交叉连接）
```

## 十一、为什么从模式比中断更好

### 11.1 中断方式的时序

```
上升沿到来
  → 硬件捕获：CCR1 = CNT（快照）     ← 瞬间
  → 硬件置位：SR.CC1IF = 1
  → 中断处理流程：                     ← 延迟
      NVIC 仲裁 → CPU 响应 → 保存现场
      → 跳转中断函数 → 执行 CNT = 0
      → 恢复现场 → 中断返回
  
延迟时间：不确定（受其他中断、优先级影响）
```

### 11.2 硬件复位方式的时序

```
上升沿到来
  → 硬件捕获：CCR1 = CNT（快照）     ← 瞬间
  → 硬件复位：CNT = 0（从模式）       ← 瞬间
  
两个动作都由硬件完成，时序固定且确定
无需任何软件干预
```

### 11.3 优势总结

| 对比项 | 中断清零 | 硬件复位（从模式） |
|--------|:--------:|:-----------------:|
| CNT 清零方式 | 软件 | **硬件** |
| 响应延迟 | 不确定 | **固定 1 周期** |
| 测量精度 | 受延迟影响 | **最高** |
| CPU 资源 | 占用 | **不占用** |
| 需要中断 | 是 | **否** |
| 需要全局变量 | 是 | **否** |
| 能测占空比 | 否 | **能** |

## 十二、计算公式

```
PSC = 71 → 计数频率 = 1MHz → 每次计数 = 1μs

CCR1 = 周期 T 的 μs 数
CCR2 = 高电平时间 t 的 μs 数

周期   = CCR1 / 1000.0            （ms）
频率   = 1000000.0 / CCR1          （Hz）
占空比 = CCR2 / CCR1 × 100%       （%）

例：
  CCR1 = 10000 → 周期 = 10.00ms, 频率 = 100.00Hz
  CCR2 = 6000  → 高电平 = 6.00ms
  占空比 = 6000/10000 = 60.00%
```

## 十三、配置流程（按模块）

```
① 时基配置
   PSC = 71（1MHz）
   ARR = 65535（最大）
   计数方向 = 向上

② 从模式配置
   SMCR.TS = 101（TI1FP1 作为触发源）
   SMCR.SMS = 100（复位模式）

③ 通道 1 配置（测周期）
   CCMR1.CC1S = 01（直连）
   CCER.CC1P = 0（上升沿）
   CCER.CC1E = 1（使能）

④ 通道 2 配置（测高电平时间）
   CCMR1.CC2S = 10（交叉）
   CCER.CC2P = 1（下降沿）
   CCER.CC2E = 1（使能）

⑤ 不需要中断
   DIER 不需要开启 CC1IE 或 CC2IE
```

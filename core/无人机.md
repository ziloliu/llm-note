## 一、空地融合（LCC Studio）

### 1.1 概念

空地融合是将**无人机航拍数据**与**手持激光扫描仪数据**整合为统一三维模型的技术，覆盖空中和地面全场景。

**设备要求**：
- 手持扫描仪：搭载 RTK 的灵光 K1 扫描仪
- 无人机：带 RTK 的测绘级无人机

📄 [原文01: LCC Studio空地融合教程](../raw/无人机/01_LCC Studio空地融合教程.md)

### 1.2 采集前勘察

1. 获取测区的**高清卫星影像**或**航拍俯视图**，了解环境与重点关注对象
2. 选取 **4–5 个空旷区域**，均匀分布于测区内，作为无人机起降点
3. 规划整体采集路径

📄 [原文01: LCC Studio空地融合教程](../raw/无人机/01_LCC Studio空地融合教程.md)

### 1.3 地面数据采集要点

- 坐标系统统一设置为 **WGS84**
- 采集路径需**经过每个起降点**
- 每个起降点**半径 2 米范围内完成两次回环采集**

📄 [原文01: LCC Studio空地融合教程](../raw/无人机/01_LCC Studio空地融合教程.md)

### 1.4 无人机数据采集

**核心要求**：
- 测区内所有关注的地面物体，需有**高于 85% 以上重叠度**的倾斜式影像照片
- 照片分辨率建议**不低于 1.0 厘米/像素**（分辨率越高，模型越清晰）

**飞行参数**：

| 参数 | 设置值 |
|------|--------|
| 坐标系统 | WGS84 |
| 航线模式 | 精灵智能摆拍航线 |
| 采集模式 | 倾斜采集 |
| 倾斜 GSD | ≤ 1 厘米 |
| 云台角度 | 45° |
| 旁向/航向重叠率 | 均 85% |
| 飞行高度 | 高于测区内建筑最高高度 |

📄 [原文01: LCC Studio空地融合教程](../raw/无人机/01_LCC Studio空地融合教程.md)

### 1.5 起降拍摄技巧

起降拍摄是空地融合的关键环节，用于连接地面数据和空中数据：

- 航线飞行完成后，前往每个起降点进行起降拍摄
- 每次起降时，将镜头对准**远处**、空中倾斜摄影和灵光扫描仪**都能看到的一个固定物体**
- 拍摄高度范围：灵光扫描仪地面采集高度 → 航线设置平行的高度
- 从低到高拍摄约 **30–40 张照片**，相邻照片**重叠度不低于 85%**
- 每个起降点选择**三个不同方向**进行起降拍摄，保证空地融合成功率

📄 [原文01: LCC Studio空地融合教程](../raw/无人机/01_LCC Studio空地融合教程.md)

### 1.6 关键参数速查

| 项目 | 要求 |
|------|------|
| 坐标系统 | WGS84（地面 + 无人机统一） |
| 照片重叠度 | ≥ 85% |
| 照片分辨率 | ≥ 1.0 cm/pixel |
| GSD 设置 | ≤ 1 cm |
| 云台角度 | 45° |
| 起降点数量 | 4–5 个 |
| 起降拍摄方向 | 每点 3 个方向 |
| 每组起降照片数 | 30–40 张 |
| 地面回环 | 每个起降点半径 2m 内回环 2 次 |

📄 [原文01: LCC Studio空地融合教程](../raw/无人机/01_LCC Studio空地融合教程.md)

## 二、K1 设备固件升级

### 2.1 升级前准备

- **电量**：电池电量需高于 **70%**（建议 4 格绿灯）
- **版本号与 SN**：核实当前固件版本，确保升级路径正确
- **数据线**：确保 USB-TypeC 数据线无损坏（损坏的线会导致固件包变小，升级失败）

📄 [原文02: K1设备固件升级指南](02_K1,K2设备固件升级指南.md)

### 2.2 升级路径

**核心规则：不可跳级，必须逐级升级。**

```
低于 1.2.4 → 1.2.4 → 2.4.1 → 3.0.2 → 3.1.1 → 3.2.1 → ...
```

版本 1.2.3 及更早的固件需进行多次逐级升级。

📄 [原文02: K1设备固件升级指南](02_K1,K2设备固件升级指南.md)

### 2.3 固件下载与传输

**固件下载地址**：[https://xgrids.cn/support/download?page=K1](https://xgrids.cn/support/download?page=K1)

**下载注意事项**：
- 不要解压缩固件文件
- 不要重命名、增减字符
- 若同一路径下已存在同名文件，需手动删除冲突项，确保文件名为原始名称

**传输步骤**：
1. 将仪器开机，通过 APP（如 Lixel GO）连接设备
2. 在 Lixel GO 设置中打开「U 盘模式」（或按电源键两次进入）
3. 使用 USB 线将仪器连接至电脑
4. 将固件压缩包放入 **U 盘根目录**，与 `model` 文件夹**同级**（不能放在 `model` 内部）

📄 [原文02: K1设备固件升级指南](02_K1,K2设备固件升级指南.md)

### 2.4 升级与验证

**升级流程**：
1. 在 Lixel GO 设置中关闭「U 盘模式」，拔出 USB 线
2. 关闭仪器电源
3. 重新开机，设备自动进入固件升级模式

**指示灯状态**：

| 状态 | 含义 |
|------|------|
| 红绿灯交替闪烁 | 正在升级中 |
| 绿灯常亮 | 升级成功 |

> 升级过程中**不可关机或拔出电池**。

升级后使用 APP 连接设备，确认固件版本号已更新为目标版本。

📄 [原文02: K1设备固件升级指南](02_K1,K2设备固件升级指南.md)

### 2.5 常见问题排查

若重启后未出现红绿交替闪烁，或升级后版本号未变动，按以下顺序排查：

| 检查项 | 说明 |
|--------|------|
| 版本路径 | 确认未跳级，逐级升级 |
| 固件包完整性 | 不得解压、重命名、增减字符 |
| 固件放置位置 | 必须在 U 盘根目录，与 `model` 同级 |
| U 盘模式 | 需先打开 U 盘模式再拷贝固件 |
| 电量 | 需高于 70%（4 格绿灯） |
| 数据线 | 更换其他线重试 |

若以上均确认无误仍无法解决，联系官方技术支持。

📄 [原文02: K1设备固件升级指南](02_K1,K2设备固件升级指南.md)

## 三、地面Mesh提取（Windows环境搭建）

### 3.1 前置条件

| 工具 | 最低版本 | 验证命令 |
|------|---------|---------|
| Git | 2.x+ | `git --version` |
| Python | **3.12**（3.13 不兼容 open3d） | `python --version` |
| winget | 1.x+ | `winget --version` |

> **关键约束**：`open3d 0.19.0` 无 Python 3.13 预编译包，必须使用 Python 3.12。

📄 [原文03: 地面Mesh提取Windows环境搭建](../raw/无人机/03_地面Mesh提取Windows环境搭建.md)

### 3.2 环境搭建流程

**1. 获取项目代码**：
```powershell
# 创建项目根目录
if (!(Test-Path "C:\projects")) { mkdir C:\projects }
cd C:\projects

# 从远程仓库拉取处理脚本
git clone http://192.168.18.200:53000/hezitian/3dgs_pcd_ground_mesh_pipeline
```

**2. Python 3.12 环境搭建**：
```powershell
# 安装 Python 3.12（推荐 winget）
winget install Python.Python.3.12

# 创建虚拟环境
cd C:\projects\3dgs_pcd_ground_mesh_pipeline
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1

# 安装依赖（配置清华镜像）
python -m pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
python -m pip install -r requirements.txt
```

**3. 验证环境**：
```powershell
python scripts/check_environment.py
```

预期输出（全部显示 `[ok]`）：
```

[ok] numpy: 2.4.6
```

📄 [原文03: 地面Mesh提取Windows环境搭建](../raw/无人机/03_地面Mesh提取Windows环境搭建.md)

### 3.3 Git Bash 配置

**为什么必须用 Git Bash**：
- 项目入口为 `.sh` 脚本，PowerShell 无法直接执行
- Windows 用 `\`，Git Bash 用 `/`
- Python 输出 Unicode 字符时，Windows 控制台会报 `gbk` 编码错误

**路径转换规则**：
```
Windows:  C:\projects\lcc\...\point_cloud.ply
Git Bash: /c/projects/lcc/.../point_cloud.ply

规则：盘符 C: → /c，反斜杠 \ → 正斜杠 /
```

**设置环境变量**：
```bash
export PYTHON_CMD='/c/projects/3dgs_pcd_ground_mesh_pipeline/.venv/Scripts/python.exe'
export PYTHONIOENCODING=utf-8
```

📄 [原文03: 地面Mesh提取Windows环境搭建](../raw/无人机/03_地面Mesh提取Windows环境搭建.md)

### 3.4 运行处理

**执行流程**：
```bash
# 1. 打开 Git Bash，进入项目目录
cd /c/projects/3dgs_pcd_ground_mesh_pipeline

# 2. 设置环境变量
export PYTHON_CMD='/c/projects/3dgs_pcd_ground_mesh_pipeline/.venv/Scripts/python.exe'
export PYTHONIOENCODING=utf-8

# 3. 运行（传入 PLY 文件的 Git Bash 格式路径）
./run_3dgs_to_pcd_ground_mesh.sh /c/projects/lcc/SU_E_0245_K1_260613_DJI_260613/ply-result/point_cloud/iteration_100/point_cloud.ply
```

**输出文件**：
| 文件 | 说明 |
|------|------|
| `ground_points.ply` | 地面点云（25–50 MB） |
| `ground_low_heightfield.ply` | 地面网格 |
| `ground_low_heightfield.obj` | OBJ 格式网格（200–400 MB） |
| `ground_low_heightfield.mtl` | 材质文件 |
| `texture_atlas.png` | 纹理图集（20–50 MB） |
| `obj_export_metadata.json` | 导出元数据 |
| `evaluation_ground_fit/` | 拟合评估结果 |

📄 [原文03: 地面Mesh提取Windows环境搭建](../raw/无人机/03_地面Mesh提取Windows环境搭建.md)

### 3.5 常见问题速查

| 现象 | 原因 | 解决 |
|------|------|------|
| `git clone` 超时或拒绝连接 | 无法访问内网 GitLab | 确认网络可达 `192.168.18.200:53000` |
| `No matching distribution found for open3d` | Python 版本为 3.13 | 降级到 3.12，重建虚拟环境 |
| `Input path does not exist: /mnt/c/...` | 路径格式错误（WSL 风格） | 使用 `/c/...` 而非 `/mnt/c/...` |
| `UnicodeEncodeError: 'gbk' codec can't encode` | 控制台编码不匹配 | `export PYTHONIOENCODING=utf-8` |
| `Could not install packages due to EnvironmentError` | 网络/镜像问题 | 换用阿里云镜像，或 `pip cache purge` |

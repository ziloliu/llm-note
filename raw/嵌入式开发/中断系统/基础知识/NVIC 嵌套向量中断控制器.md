## 一句话定义

NVIC (Nested Vectored Interrupt Controller) 是 Cortex-M3 内核的嵌套向量中断控制器，负责统一管理所有中断请求，进行优先级仲裁，决定中断服务程序的执行顺序。

## 核心内容

### NVIC 概述

| 项目 | 内容 |
|------|------|
| 全称 | Nested Vectored Interrupt Controller（嵌套向量中断控制器） |
| 位置 | 与 Cortex-M3 内核紧密耦合，可视为内核的一部分 |
| 核心职责 | 统一管理所有中断请求，进行优先级仲裁，决定中断服务程序的执行顺序 |

- 所有中断（内部异常 + 外部中断）最终都经 NVIC 提交给内核处理
- 可实现低延迟、高效率的中断响应
- 详细技术文档位于 **Cortex-M3 编程手册**（非 STM32 参考手册），参考资料中仅有英文版

### 中断的三种状态

每个中断源可独立配置使能/禁止，并具有以下状态：

| 状态 | 含义 | 说明 |
|------|------|------|
| **禁止（Disabled）** | 中断源被关闭 | 即使硬件信号到来，也不会产生中断请求 |
| **挂起（Pending）** | 中断请求已产生，尚未被响应 | 通过状态位置位表示挂起，清除表示已处理完毕 |
| **激活（Active）** | 正在执行对应的中断服务程序 | 内核当前正在处理该中断 |

状态转换流程：
```
中断源使能 → 硬件信号触发 → 置位 Pending → NVIC 仲裁 → 内核响应 → 进入 Active → 处理完毕 → 清除 Pending
```

### 中断信号触发方式

| 触发方式 | 说明 |
|----------|------|
| **电平触发** | 高电平或低电平有效 |
| **脉冲触发（更常见）** | 上升沿或下降沿检测，更适合捕捉突发异常事件 |

### 中断优先级机制

#### 基本原则

- **高优先级可抢占低优先级：** 高优先级中断到来时，可打断正在执行的低优先级中断服务程序（中断嵌套）
- **低优先级不可抢占高优先级：** 必须等待高优先级中断处理完毕后才能被响应

#### NVIC 的两级优先级划分

NVIC 将优先级分为两个层级：

| 层级 | 名称 | 功能 |
|------|------|------|
| 第一层 | **抢占优先级**（Preemption Priority） | 决定**能否打断**正在执行的中断 |
| 第二层 | **响应优先级**（Sub-Priority） | 决定多个挂起中断**被响应的先后顺序** |

#### 两级优先级的协作规则

**场景一：抢占（嵌套）**
- 抢占优先级**不同** → 高者可打断低者（支持中断嵌套）
- 抢占优先级**相同** → 互不打断，先处理完当前中断再响应下一个

**场景二：排队响应（多个中断同时挂起）**
- 抢占优先级相同，响应优先级**不同** → 响应优先级高者先被处理
- 抢占优先级相同，响应优先级**相同** → 按中断向量表中的默认硬件编号顺序依次处理（编号小者优先）

#### 规则总结表

| 比较条件 | 处理规则 |
|----------|----------|
| 抢占优先级 高 > 低 | 高者可抢占低者 |
| 抢占优先级相同，响应优先级 不同 | 响应优先级高者先处理（仅影响排队顺序，不抢占） |
| 抢占优先级相同，响应优先级相同 | 按中断向量表默认硬件编号顺序处理 |

### 关键概念辨析

| 概念 | 说明 |
|------|------|
| 抢占 | 一个中断能否**打断**另一个正在执行的中断 |
| 响应 | 多个中断同时挂起时，**先处理**哪一个 |
| 抢占优先级与响应优先级的关系 | 抢占优先级是前提条件；仅当抢占优先级相同时，响应优先级才发挥作用 |

### 补充说明

- 外部中断：GPIO 归并的 16 路 + PVD / RTC / USB / 以太网唤醒共 4 路 = 20 路（F103 无以太网，实际 19 路），经 EXTI 处理后送 NVIC
- 内部异常：直接送 NVIC
- 每个中断源可独立使能/禁止，不影响其他中断

## 注意事项 & 踩坑

- **NVIC 与内核耦合**: NVIC 与 Cortex-M3 内核紧密耦合，可视为内核的一部分
- **优先级配置**: 优先级值越小优先级越高，默认优先级值为 0（最高优先级）
- **中断嵌套**: 高抢占优先级可打断低抢占优先级中断
- **状态管理**: 每个中断源有禁止、挂起、激活三种状态
- **触发方式**: 脉冲触发更常见，适合捕捉突发异常事件

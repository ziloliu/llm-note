## 一句话定义

STM32 中断体系架构基于 Cortex-M3 内核，由嵌套向量中断控制器 (NVIC) 和外部中断/事件控制器 (EXTI) 组成，提供完整的中断管理和响应机制。

# STM32 中断体系架构笔记

---

## 一、总体结构

整个中断系统由三个核心模块组成，构成层级式的中断管理架构：

| 模块 | 全称 | 职责 |
|------|------|------|
| **Cortex-M3 内核** | — | 最终响应中断，打断当前程序，跳转执行中断服务程序 |
| **NVIC** | Nested Vectored Interrupt Controller | 管理所有中断源，进行优先级仲裁与排队，向内核提交中断请求 |
| **EXTI** | External Interrupt/Event Controller | 管理外部中断信号的归并、来源识别与触发配置 |

**信号流方向：** 中断源 → NVIC（优先级仲裁）→ 内核（响应并执行中断服务程序）

---

## 二、三大中断来源

### 三大中断来源对比

| 来源 | 位置 | 是否经过 EXTI | 说明 |
|------|------|:------------:|------|
| 内核内部异常 | Cortex-M3 内核中 | 否 | 如 SysTick、复位、硬故障等，直连 NVIC |
| 片上外设 | STM32 芯片内部 | 否 | 如 TIM、USART、SPI、DMA、ADC、CAN 等，直连 NVIC |
| 片外外设 | STM32 芯片外部 | **是** | 通过 GPIO 引脚输入，经 AFIO → EXTI → NVIC |

### 第一类：内核内部异常

- 不经过 EXTI，直接向 NVIC 提交请求
- 包含：
  - 复位（Reset）
  - 系统滴答定时器（SysTick）
  - 其余 10 种内中断/异常（如硬故障、NMI 等）
- 特点：属于内核自身产生的异常，响应链路最短

### 第二类：片上外设中断

- 不经过 EXTI，直接向 NVIC 提交请求
- 包含：
  - 通信模块：USART、SPI、I2C 等
  - 定时器：TIM1、TIM2~TIM8 等
  - DMA、ADC、CAN 等片内外设
- 特点：芯片内部集成的外设所产生的中断，每种外设有独立的中断请求线连接 NVIC

### 第三类：外部中断（片外外设）

- 需经过 **AFIO → EXTI → NVIC** 的链路
- 来源：通过 GPIO 引脚传入的片外设备中断请求

---

## 三、外部中断的归并机制

### 3.1 问题背景

- GPIO 共 7 组（A~G），每组 16 个引脚（0~15），合计 112 个引脚
- 若每个引脚单独连接 EXTI，内部走线达 112 条，芯片面积与布线复杂度过高

### 3.2 AFIO 复用归并（7 合 1）

**归并规则：** 7 组 GPIO 中**相同编号**的引脚，通过 AFIO 合并为 1 条中断请求线

| 合并后的线 | 对应引脚 |
|------------|----------|
| EXTI0 | PA0、PB0、PC0、PD0、PE0、PF0、PG0 |
| EXTI1 | PA1、PB1、PC1、PD1、PE1、PF1、PG1 |
| …… | …… |
| EXTI15 | PA15、PB15、PC15、PD15、PE15、PF15、PG15 |

- 归并后共 **16 条线** 连接 EXTI
- 互斥约束：同一编号的 7 个引脚**同一时刻仅允许配置 1 个**作为中断源，否则信号冲突
- 具体选择哪个引脚，通过 AFIO 的复用选择寄存器配置

### 3.3 EXTI 接收的额外外部中断源

除 GPIO 归并的 16 路外，EXTI 还接收 4 路独立外部中断：

| EXTI 线 | 中断来源 |
|---------|----------|
| EXTI16 | PVD（可编程电压检测） |
| EXTI17 | RTC 闹钟 |
| EXTI18 | USB 唤醒检测 |
| EXTI19 | 以太网唤醒 |

- EXTI 共处理 **20 路**外部中断（16 + 4）
- **注意：** F103 系列无以太网模块，实际仅有 **19 路**

### 3.4 EXTI 的核心工作

1. **来源识别：** 归并后需配置具体使用哪个 GPIO 引脚（PA0 还是 PB0 等）
2. **触发方式配置：** 上升沿、下降沿、双边沿触发
3. **中断/事件选择：** 可配置为产生中断请求（送 NVIC）或事件信号（触发其他外设）

## 四、EXTI → NVIC 的再次合并

EXTI 的信号线交给 NVIC 管理时，并非每根线都独立对应一个中断向量，部分线**合并管理**：

| 中断向量 | 对应 EXTI 线 | 合并情况 |
|----------|-------------|----------|
| EXTI0 | EXTI0 | 1 线 1 中断 |
| EXTI1 | EXTI1 | 1 线 1 中断 |
| EXTI2 | EXTI2 | 1 线 1 中断 |
| EXTI3 | EXTI3 | 1 线 1 中断 |
| EXTI4 | EXTI4 | 1 线 1 中断 |
| **EXTI9_5** | EXTI5 ~ EXTI9 | **5 线合 1 中断** |
| **EXTI15_10** | EXTI10 ~ EXTI15 | **6 线合 1 中断** |

### 合并后的影响

| 方面 | 说明 |
|------|------|
| 优先级配置 | 合并的线共享同一个优先级设置 |
| 中断入口 | 合并的线共用同一个中断服务程序入口 |
| 来源判断 | 需在中断服务程序中通过**请求挂起寄存器（PR）**的对应位判断具体是哪条 EXTI 线触发 |
| 引脚判断 | 需通过 EXTI 中的配置寄存器确认七合一中具体是哪个 GPIO 引脚（PAx / PBx / …） |

## 五、完整信号流

```
┌─────────────────────────────────────────────────────┐
│  第一类：内核内部异常                                  │
│  (SysTick / 复位 / 硬故障 等) ──────────────→ NVIC    │
├─────────────────────────────────────────────────────┤
│  第二类：片上外设中断                                  │
│  (USART / TIM / SPI / I2C / DMA 等) ───────→ NVIC    │
├─────────────────────────────────────────────────────┤
│  第三类：外部中断                                      │
│  GPIO(A~G) × 16引脚                                  │
│       ↓ AFIO归并(7合1)                               │
│  EXTI0 ~ EXTI15 (16路) ──┐                           │
│  PVD / RTC / USB / ETH   ──┤──→ EXTI ────→ NVIC     │
│       (4路独立)           ┘                           │
├─────────────────────────────────────────────────────┤
│                    NVIC (优先级仲裁)                   │
│                         ↓                            │
│                  Cortex-M3 内核                       │
│              (执行中断服务程序)                         │
└─────────────────────────────────────────────────────┘
```


## 六、NVIC 核心要点

| 要点 | 内容 |
|------|------|
| 地位 | 管理所有中断，紧耦合内核，视为内核的一部分 |
| 核心功能 | 优先级仲裁，决定中断响应顺序和嵌套关系 |
| 两级优先级 | 抢占优先级（能否打断）+ 响应优先级（同时挂起时的响应顺序） |
| 优先级值 | 越小越高，默认值为 0（最高） |
| 实际有效位 | 4 位（bit7~bit4），共 16 级 |
| 编程方式 | 调用标准库函数（core_cm3.c），不直接操作寄存器 |
| 分组模式 | 推荐模式 3（全抢占优先级，配置最简单） |

## 七、EXTI 核心要点

### 关键寄存器

| 寄存器 | 操作时机 | 说明 |
|--------|----------|------|
| 上升沿触发选择寄存器（RTSR） | 初始化时配置 | 检测上升沿 |
| 下降沿触发选择寄存器（FTSR） | 初始化时配置 | 检测下降沿 |
| 中断屏蔽寄存器（IMR） | 初始化时配置 | 对应位置 1 打开中断通路 |
| 软件中断事件寄存器（SWIER） | 一般不配 | 软件模拟中断触发（调试用） |
| 请求挂起寄存器（PR） | 中断服务程序中清零 | 硬件自动置位，软件写 1 清零 |

### 完整信号流

```
GPIO引脚信号变化
    ↓
边缘检测电路（依据RTSR/FTSR配置）
    ↓
请求挂起寄存器 PR 硬件自动置位
    ↓
与中断屏蔽寄存器 IMR 相与（AND）
    ├── IMR=0 → 屏蔽，不传递
    └── IMR=1 → 传递至 NVIC
                    ↓
              NVIC 优先级仲裁
                    ↓
              内核执行中断服务程序
                    ↓
              软件清零 PR 对应位
```

## 八、关键认知点

| 认知点 | 说明 |
|--------|------|
| 七合一发生在 GPIO → EXTI 阶段 | 减少引脚到 EXTI 的连线数量 |
| 再次合并发生在 EXTI → NVIC 阶段 | EXTI9_5 和 EXTI15_10 分别合并为一个中断向量 |
| 合并后需软件区分来源 | 通过 PR 寄存器判断哪条 EXTI 线触发，通过 EXTI 配置确认具体 GPIO 引脚 |
| 片上外设中断不经过 EXTI | 直连 NVIC，无需额外的引脚和归并操作 |
| 内核异常优先级在 SHPRx 中配 | 与外部中断的 NVIC_IPRx 寄存器分开管理 |

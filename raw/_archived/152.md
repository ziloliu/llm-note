| 功能 | 采集可变电阻器电压，串口打印 |

| 工作模式 | 连续转换 + 软件触发 |

## 二、CubeMX 配置步骤

### 2.1 基础配置

| 配置项 | 设置 |
|--------|------|
| Debug | Serial Wire（单线调试） |
| RCC | HSE + LSE 外部晶振 |
| 时钟树 | HSE → PLL ×9 → 72MHz, APB1 /2 → 36MHz |
| Connectivity | USART1 异步模式（串口打印） |

### 2.2 ADC 配置

```
位置：Analog → ADC1

通道选择：
  ✅ IN10（PC0 引脚自动亮起，复用为 ADC 通道 10）
```

### 2.3 ADC 参数配置

| 参数 | 值 | 说明 |
|------|:--:|------|
| Mode | **Independent mode** | 独立模式 |
| Data Alignment | **Right alignment** | 右对齐（默认） |
| Scan Conversion Mode | **Disable** | 单通道不需扫描 |
| Continuous Conversion Mode | **Enable** | 连续转换（单曲循环） |
| Discontinuous Conversion Mode | **Disable** | 不用间断模式 |
| Number Of Conversion | **1** | 1 个通道 |
| External Trigger | **Software trigger** | 软件触发（111） |
| Rank 1 Channel | **10** | 通道 10 |
| Sampling Time | **7.5 Cycles** | 采样时间 |

### 2.4 未使用的配置

| 配置项 | 状态 | 说明 |
|--------|:----:|------|
| Injected Conversion | 禁用 | 不使用注入通道 |
| Analog Watchdog | 未勾选 | 不使用模拟看门狗 |
| DMA | 未使用 | 不使用 DMA |
| NVIC 中断 | 未勾选 | 不使用中断 |

### 2.5 ADC 时钟分频（必须手动配置）

```
⚠️ CubeMX 不会自动配置 ADC 分频！

Clock Configuration 页签：
  → ADC Prescaler 默认是 /2（36MHz > 14MHz，报错）
  → 需手动改为 /6（72MHz / 6 = 12MHz < 14MHz ✅）

如果 CubeMX 弹出自动配置提示：
  → 点击 Cancel（不要自动调整）
  → 手动选择 6 分频
  → 自动调整可能改变 PLL 配置
```

## 三、CubeMX 生成的 ADC 初始化代码

```c
ADC_HandleTypeDef hadc1;

void MX_ADC1_Init(void)
{
    ADC_ChannelConfTypeDef sConfig = {0};

    hadc1.Instance = ADC1;
    hadc1.Init.ScanConvMode = ADC_SCAN_DISABLE;
    hadc1.Init.ContinuousConvMode = ENABLE;
    hadc1.Init.DiscontinuousConvMode = DISABLE;
    hadc1.Init.ExternalTrigConv = ADC_SOFTWARE_START;
    hadc1.Init.DataAlign = ADC_DATAALIGN_RIGHT;
    hadc1.Init.NbrOfConversion = 1;
    HAL_ADC_Init(&hadc1);

    sConfig.Channel = ADC_CHANNEL_10;
    sConfig.Rank = ADC_REGULAR_RANK_1;
    sConfig.SamplingTime = ADC_SAMPLETIME_7CYCLES5;
    HAL_ADC_ConfigChannel(&hadc1, &sConfig);
}
```

## 五、HAL 库关键函数

### 5.1 启动 ADC 转换

```c
HAL_ADC_Start(&hadc1);
```

```
参数：ADC 句柄地址
功能：启动规则通道 ADC 转换（轮询方式）
选项：
  HAL_ADC_Start()      → 轮询（本实验使用）
  HAL_ADC_Start_IT()   → 中断
  HAL_ADC_Start_DMA()  → DMA
```

### 5.2 获取转换结果

```c
uint32_t value = HAL_ADC_GetValue(&hadc1);
```

```
参数：ADC 句柄地址
返回值：DR 寄存器中的 12 位数字量（0~4095）
本质：读取 ADC->DR 寄存器

相关函数（不使用）：
  HAL_ADC_GetError()  → 获取错误信息
  HAL_ADC_GetState()  → 获取 ADC 状态
```

### 5.3 ADC 校准（关键！）

```c
HAL_ADCEx_Calibration_Start(&hadc1);
```

```
⚠️ 必须在启动转换之前调用！

不校准：最大电压只能到约 3.25V（有偏差）
校准后：最大电压可达 3.3V（准确）

函数名含义：
  HAL   = HAL 库
  ADCEx = ADC Extended（扩展功能，位于 adc_ex.c）
  Calibration_Start = 开始校准

内部操作：
  → 类似寄存器方式中 CR2.CAL = 1
  → 硬件自动执行校准
  → 校准完成后函数返回
```

## 六、主函数实现

### 6.1 完整代码

```c
#include "main.h"
#include "adc.h"
#include "usart.h"
#include <stdio.h>

int main(void)
{
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    MX_ADC1_Init();
    MX_USART1_UART_Init();

    printf("Hello World!\n");

    // 校准（必须！）
    HAL_ADCEx_Calibration_Start(&hadc1);

    // 启动 ADC 转换
    HAL_ADC_Start(&hadc1);

    while (1)
    {
        // 获取转换结果并计算电压
        double v = HAL_ADC_GetValue(&hadc1) * 3.3 / 4095;

        // 打印电压值
        printf("V = %.2f V\n", v);

        HAL_Delay(1000);  // 每秒打印一次
    }
}
```

### 6.2 电压计算公式

```c
double v = HAL_ADC_GetValue(&hadc1) * 3.3 / 4095;

  HAL_ADC_GetValue(&hadc1) → 12 位数字量（0~4095）
  × 3.3                     → 乘以参考电压
  / 4095                    → 除以最大数字量（2^12 - 1）
  → 实际电压值（0V ~ 3.3V）
```

## 七、校准问题详解

### 7.1 问题现象

```
未校准时：
  最大电压只能到约 3.25V（如 3.25V）
  最小电压可到 0.00V

校准后：
  最大电压可达 3.30V ✅
  最小电压可到 0.00V ✅
```

### 7.2 原因分析

```
ADC 转换器存在固有偏差：
  → 类似天平没有调零就开始称重
  → 读数会有系统性偏移
  → 需要先校准消除偏差
```

### 7.3 校准时机

```
正确顺序：
  ① MX_ADC1_Init()                      → 初始化
  ② HAL_ADCEx_Calibration_Start(&hadc1)  → 校准
  ③ HAL_ADC_Start(&hadc1)                → 启动转换
  ④ HAL_ADC_GetValue(&hadc1)             → 读取结果

校准必须在启动转换之前！
```

### 7.4 与寄存器方式的对应

| 寄存器方式 | HAL 库方式 |
|-----------|-----------|
| `ADC1->CR2 \|= ADC_CR2_CAL` | `HAL_ADCEx_Calibration_Start(&hadc1)` |
| `while(ADC1->CR2 & ADC_CR2_CAL)` | 函数内部自动等待 |
| 校准完成后 CAL 位硬件清零 | 函数自动返回 |

## 十、寄存器方式 vs HAL 库方式对比

| 对比项 | 寄存器方式 | HAL 库方式 |
|--------|-----------|-----------|
| 时钟配置 | 手动 RCC->CFGR | CubeMX 图形化 |
| GPIO 配置 | 手动 CRL/CNF | CubeMX 自动生成 |
| CR1 配置 | 手动 SCAN 位 | CubeMX 参数化 |
| CR2 配置 | 手动 CONT/ALIGN/EXTTRIG/EXTSEL | CubeMX 参数化 |
| SMPR 配置 | 手动 SMP10 位 | CubeMX 下拉选择 |
| SQR 配置 | 手动 SQ1 + L | CubeMX 参数化 |
| 校准 | 手动 CAL 位 + 等待 | `HAL_ADCEx_Calibration_Start()` |
| 启动 | 手动 SWSTART 位 | `HAL_ADC_Start()` |
| 读取结果 | 手动读 SR + DR | `HAL_ADC_GetValue()` |
| 代码量 | 较多 | **极少** |

## 十二、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 最大电压不到 3.3V | 未校准 | 添加 `HAL_ADCEx_Calibration_Start()` |
| 读数恒为 0 | 未启动 ADC | 添加 `HAL_ADC_Start()` |
| CubeMX 时钟飘红 | ADC 分频未配 | 手动设 ADC Prescaler = /6 |
| CubeMX 弹自动配置 | 时钟超 14MHz | Cancel 后手动选 6 分频 |
| 通道选择错误 | 引脚不对应 | 确认 PC0 = ADC123_IN10 |
| MicroLib 未勾选 | printf 无法使用 | Keil 中勾选 Use MicroLib |
| 函数找不到 | 在 adc_ex.c 中 | 使用 ADCEx 前缀的函数 |

## 十三、完整调用流程

```
初始化阶段（CubeMX 自动生成）：
  MX_GPIO_Init()           → GPIO 配置
  MX_ADC1_Init()           → ADC 配置
  MX_USART1_UART_Init()    → 串口配置

运行阶段（用户代码）：
  ① HAL_ADCEx_Calibration_Start(&hadc1)  → 校准
  ② HAL_ADC_Start(&hadc1)                → 启动转换
  ③ HAL_ADC_GetValue(&hadc1)             → 读取 DR
  ④ 电压 = 结果 × 3.3 / 4095             → 计算电压
  ⑤ printf 打印                           → 串口输出
  ⑥ HAL_Delay(1000)                      → 延时 1 秒
  ⑦ 循环 ③ ~ ⑥
```

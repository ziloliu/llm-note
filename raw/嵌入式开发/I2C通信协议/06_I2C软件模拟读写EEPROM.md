# I²C 通信案例 — 软件模拟 I²C 读写 EEPROM（AT24C02） 笔记

---

## 一、实验需求

| 项目 | 内容 |
|------|------|
| 功能 | STM32 通过 I²C 向 AT24C02 写入一段数据，再读出并通过串口打印验证 |
| 通讯方式 | **软件模拟 I²C**（不使用 STM32 硬件 I²C 模块） |
| 存储芯片 | AT24C02（EEPROM，256 字节） |
| 验证方式 | 将读出的字符串通过串口（printf）发送到电脑，与写入内容对比 |

### 数据流

```
STM32 → I²C 写入 → AT24C02 存储 → I²C 读出 → STM32 → 串口 → 电脑打印验证
```

## 二、AT24C02 芯片介绍

### 2.1 基本参数

| 参数 | 数值 | 说明 |
|------|------|------|
| 全称 | Electrically Erasable Programmable Read-Only Memory | 电可擦除可编程只读存储器 |
| 容量 | **256 字节**（2 Kbit） | 256 × 8 bit |
| 页大小 | **16 字节** | 写入操作的基本单位 |
| 页数 | 16 页 | 256 / 16 = 16 |
| 写入周期 | **5 ms** | 每次写入后需等待 5ms 才能进行下一次写入 |
| I²C 模式 | 标准模式 100kHz / 快速模式 400kHz | |
| 供电电压 | 1.7V ~ 5.5V | 开发板使用 3.3V |

### 2.2 内部地址结构

```
8 位地址（0x00 ~ 0xFF）表示 256 个字节

地址位划分：
┌──────────────┬──────────────┐
│  高 4 位       │  低 4 位       │
│  页地址        │  页内地址       │
│  (0~15)       │  (0~15)       │
└──────────────┴──────────────┘

例：地址 0x3A → 页地址 = 0x3，页内地址 = 0xA
```

## 三、硬件电路

### 3.1 AT24C02 引脚定义（8 脚 DIP 封装）

| 引脚 | 名称 | 连接 | 说明 |
|------|------|------|------|
| 1 | **A0** | GND | 设备地址 bit0 = 0 |
| 2 | **A1** | GND | 设备地址 bit1 = 0 |
| 3 | **A2** | GND | 设备地址 bit2 = 0 |
| 4 | **GND** | GND | 地 |
| 5 | **SDA** | I²C 数据总线 | → STM32 **PB11** |
| 6 | **SCL** | I²C 时钟总线 | → STM32 **PB10** |
| 7 | **WP** | GND | 写保护关闭（低电平允许写入） |
| 8 | **VCC** | 3.3V | 电源 |

### 3.2 设备地址

AT24C02 的 7 位 I²C 设备地址由两部分组成：

```
设备地址 = [1 0 1 0] [A2 A1 A0]
             高4位固定    低3位由引脚决定

本实验：A2=A1=A0=0（全接 GND）

设备地址 = 0b1010000 = 0x50

完整字节（含 R/W 位）：
  写操作：0x50 << 1 | 0 = 0xA0
  读操作：0x50 << 1 | 1 = 0xA1
```

### 3.3 总线连接

```
          3.3V
           │
          [R] 4.7KΩ 上拉电阻
           │
PB10(SCL)──┼────── AT24C02 SCL (Pin 6)
           │
          [R] 4.7KΩ 上拉电阻
           │
PB11(SDA)──┼────── AT24C02 SDA (Pin 5)
```

- SCL 和 SDA 各需一个**上拉电阻**（典型值 4.7KΩ）接 3.3V
- 空闲时总线为高电平（上拉电阻拉高）
- 设备输出低电平时总线被拉低（开漏输出 + 线与逻辑）

## 四、软件模拟 I²C 方案

### 4.1 为什么不使用硬件 I²C 模块

| 原因 | 说明 |
|------|------|
| 学习目的 | 软件模拟可深入理解 I²C 协议的每一个时序细节 |
| 灵活性 | 不受硬件 I²C 模块的引脚限制，可使用任意 GPIO |
| 可移植性 | 软件模拟的代码可移植到任何有 GPIO 的平台上 |

### 4.2 引脚配置

| 引脚 | 配置模式 | 说明 |
|------|----------|------|
| **PB10** (SCL) | **通用开漏输出** | 时钟线，由主机控制 |
| **PB11** (SDA) | **通用开漏输出/输入切换** | 数据线，收发共用，需动态切换方向 |

> SDA 使用开漏输出模式：
> - 输出 0 → 引脚接地 → 总线拉低
> - 输出 1 → 引脚高阻态 → 上拉电阻拉高总线
> - 读取时需切换为输入模式读取电平

### 4.3 需要实现的底层函数

```
① I²C 初始化（GPIO 配置）
② 起始信号（S）：SCL 高电平期间，SDA 高→低
③ 停止信号（P）：SCL 高电平期间，SDA 低→高
④ 发送一个字节（8 位，高位先行）
⑤ 接收一个字节（8 位，高位先行）
⑥ 发送应答（ACK：拉低 SDA）
⑦ 发送非应答（NACK：保持 SDA 高电平）
⑧ 等待应答（检测从设备是否 ACK）
```

## 五、I²C 时序详解

### 5.1 起始信号（S）

```
SCL:  ──────┐        ┌──────
             └────────┘
SDA:  ──┐              
        └──────────────
         ↑
      SCL 高电平期间
      SDA 下降沿（高→低）
```

**代码逻辑：**
1. SDA 先拉高（确保起始状态）
2. SCL 拉高
3. SDA 拉低（产生下降沿）
4. SCL 拉低（准备发送数据）

### 5.2 停止信号（P）

```
SCL:  ──────┐        ┌──────
             └────────┘
SDA:  ──────────────┐
                    └──────
                     ↑
                  SCL 高电平期间
                  SDA 上升沿（低→高）
```

**代码逻辑：**
1. SDA 拉低（确保起始状态）
2. SCL 拉高
3. SDA 拉高（产生上升沿）

### 5.3 发送一个字节

```
SCL: ─┐┌─┐┌─┐┌─┐┌─┐┌─┐┌─┐┌─┐
      └┘ └┘ └┘ └┘ └┘ └┘ └┘ └┘
SDA: ═╤═╤══╤══╤═╤══╤═╤══      （高位先行 D7→D0）
      D7 D6 D5 D4 D3 D2 D1 D0

每 1 个时钟周期传输 1 位：
  SCL 低电平 → SDA 设置数据（高或低）
  SCL 高电平 → 保持稳定，从设备采样
```

### 5.4 接收一个字节

```
与发送类似，但 SDA 由从设备驱动
主机在每个 SCL 高电平期间读取 SDA 电平
接收完 8 位后，主机发送 ACK 或 NACK
```

### 5.5 应答信号

| 类型 | 第 9 个时钟周期 SDA 电平 | 说明 |
|------|:------------------------:|------|
| **ACK** | 低电平 (0) | 接收成功 |
| **NACK** | 高电平 (1) | 接收失败或通讯结束 |

## 六、AT24C02 读写操作

### 6.1 写入一个字节

```
主机发送：[S] [设备地址+0(W)] [ACK] [字节地址] [ACK] [数据] [ACK] [P]

S          → 起始信号
0xA0       → 设备地址 0x50 + 写(0) = 0xA0
ACK        → AT24C02 应答
字节地址    → 要写入的 EEPROM 地址（0x00~0xFF）
ACK        → AT24C02 应答
数据        → 要写入的字节数据
ACK        → AT24C02 应答
P          → 停止信号

写入后需等待 5ms（写入周期）
```

### 6.2 读取一个字节（随机读）

```
第一步（伪写入，设置地址）：
主机发送：[S] [0xA0(W)] [ACK] [字节地址] [ACK]

第二步（重复起始 + 读取）：
主机发送：[S] [0xA1(R)] [ACK]
AT24C02发送：[数据]
主机发送：[NACK] [P]

NACK → 主机不再需要更多数据，通知从设备停止发送
P    → 停止信号，结束通讯
```

### 6.3 页写入（Page Write）

```
一次最多写入 16 字节（一页）

主机发送：[S] [0xA0] [ACK] [起始地址] [ACK]
          [数据1] [ACK] [数据2] [ACK] ... [数据N] [ACK] [P]

注意：页写入不能跨页，地址会自动在页内循环
写入后同样需等待 5ms
```

### 6.4 连续读取（Sequential Read）

```
读取完一个字节后，主机发送 ACK（而非 NACK）
AT24C02 自动将地址 +1，继续发送下一个字节
直到主机发送 NACK + P 结束
```

## 七、软件模拟 I²C 实现清单

### 7.1 需要实现的函数

| 函数名 | 功能 | 说明 |
|--------|------|------|
| `I2C_Init()` | GPIO 初始化 | PB10/PB11 开漏输出，上拉 |
| `I2C_Start()` | 发送起始信号 | SCL 高时 SDA 高→低 |
| `I2C_Stop()` | 发送停止信号 | SCL 高时 SDA 低→高 |
| `I2C_SendByte(uint8_t)` | 发送一个字节 | 高位先行，8 个时钟周期 |
| `I2C_ReceiveByte()` | 接收一个字节 | 高位先行，返回 8 位数据 |
| `I2C_WaitAck()` | 等待从设备应答 | 检测 SDA 是否为低 |
| `I2C_SendAck()` | 发送应答(ACK) | 主机拉低 SDA |
| `I2C_SendNack()` | 发送非应答(NACK) | 主机保持 SDA 高 |

### 7.2 EEPROM 操作封装

| 函数名 | 功能 |
|--------|------|
| `EEPROM_WriteByte(uint8_t addr, uint8_t data)` | 向指定地址写入一个字节 |
| `EEPROM_ReadByte(uint8_t addr)` | 从指定地址读取一个字节 |
| `EEPROM_WriteBytes(uint8_t addr, uint8_t *data, uint8_t len)` | 写入多字节 |
| `EEPROM_ReadBytes(uint8_t addr, uint8_t *data, uint8_t len)` | 读取多字节 |

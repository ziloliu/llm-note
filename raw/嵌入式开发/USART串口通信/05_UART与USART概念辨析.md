## 一句话定义

UART是通用异步收发器（硬件），USART是通用同步/异步收发器（硬件），它们是实现串口通讯协议的底层硬件电路。

## 核心内容

### 概念层级关系

```
串口通讯协议（软件协议层）
        ↓ 需要底层硬件实现
UART（通用异步收发器）── 硬件设备层
        ↓ 增加同步功能
USART（通用同步/异步收发器）── 硬件设备层
        ↓ 增加电气化标准
RS-232 / RS-485 / RS-422 ── 电气标准层（长距离传输）
```

### UART 与 USART 对比

| 项目 | UART | USART |
|------|------|-------|
| 全称 | Universal Asynchronous Receiver/Transmitter | Universal Synchronous/Asynchronous Receiver/Transmitter |
| 含义 | 通用异步收发器 | 通用同步/异步收发器 |
| 同步功能 | 无 | 有（带时钟信号线） |
| 异步功能 | 有 | 有 |
| 本质 | 实现串口通讯协议的**底层硬件电路** | UART 的功能扩展版 |

- **UART** 是实现经典串口通讯协议的硬件基础，负责串并转换、波特率控制、数据收发
- **USART** 在 UART 基础上增加了同步时钟信号，可工作在同步或异步模式下
- 当 USART 不使用同步功能时，等效于 UART

### STM32 中的五个串口外设

| 外设 | 类型 | 总线 | 最大速度 | 同步 | 硬件流控 | DMA | 智能卡 |
|------|------|------|---------|:----:|:-------:|:---:|:------:|
| **USART1** | USART | APB2（高速） | **4.5 Mbps** | ✓ | ✓ | ✓ | ✓ |
| **USART2** | USART | APB1（低速） | 2.25 Mbps | ✓ | ✓ | ✓ | ✓ |
| **USART3** | USART | APB1（低速） | 2.25 Mbps | ✓ | ✓ | ✓ | ✓ |
| **UART4** | UART | APB1（低速） | 2.25 Mbps | ✗ | ✗ | ✓ | ✗ |
| **UART5** | UART | APB1（低速） | 2.25 Mbps | ✗ | ✗ | ✗ | ✗ |

**关键差异：**

| 对比项 | USART1~3 | UART4~5 |
|--------|----------|---------|
| 同步模式 | 支持 | 不支持 |
| 硬件流控制 | 支持 | 不支持 |
| 速度 | USART1 最快（APB2），其余受 APB1 限制为 USART1 的一半 | 同 USART2/3 |
| DMA | 均支持 | UART4 支持，UART5 不支持 |
| LIN / 智能卡 / IrDA | 支持 | 不支持 |

### 三层概念区分

| 层级 | 名称 | 规定内容 | 说明 |
|------|------|----------|------|
| **协议层** | 串口通讯协议 | 数据帧格式、起始/停止位、校验、波特率 | 通用的异步串行通讯规则 |
| **硬件层** | UART / USART | 串并转换、波特率控制、收发电路 | 实现串口协议的芯片内硬件外设 |
| **电气层** | RS-232 / RS-485 / RS-422 | 电压电平、逻辑定义、传输距离 | 长距离传输的电气化标准 |

### RS-232 / RS-485 / RS-422 电气标准

| 标准 | 信号方式 | 电平定义 | 传输方式 | 线数 | 双工模式 |
|------|----------|----------|----------|------|----------|
| **RS-232** | 单端信号 | 负逻辑：1→负电压(-15~-5V)，0→正电压(+5~+15V) | 单端 | 2+地线 | 全双工 |
| **RS-485** | **差分信号** | 两线间电平差表示 0/1 | 差分 | 2 线 | **半双工** |
| **RS-422** | **差分信号** | 两线间电平差表示 0/1 | 差分（两组） | 4 线 | **全双工** |

### 不同场景下的术语习惯

| 场景 | 习惯称呼 | 说明 |
|------|----------|------|
| 单片机/嵌入式 | **UART / USART** | 直接使用 TTL 电平，不涉及额外电气标准 |
| 电脑端 | **COM 口**（如 COM1、COM2） | Windows 操作系统对串行端口的命名 |
| 电脑硬件接口 | **RS-232**（DB-9 九针接口） | 带电气标准的物理接口 |
| 泛指 | "串口" / "UART 协议" | 有时将协议与硬件设备混用，实际指串口通讯协议通过 UART 实现 |

## 注意事项 & 踩坑

- "UART 协议" 实际是通过 UART 硬件实现的串口通讯协议
- "COM 口" 是 Windows 系统对电脑串行端口的管理命名
- "RS-232 接口" 采用 RS-232 电气标准的物理接口，底层协议仍是串口协议
- 协议规定"数据长什么样"，硬件规定"怎么收发"，电气标准规定"用什么电平传输"
- 三者是不同层面的概念，但在实际使用中经常被混用

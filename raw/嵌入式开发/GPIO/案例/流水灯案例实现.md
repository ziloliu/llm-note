## 一句话定义

流水灯通过配置PA0、PA1、PA8为推挽输出模式,依次输出低电平点亮三个LED,实现流动的视觉效果。

## 核心内容

### 硬件配置
- **LED连接**:
  - LED1:PA0(黄色)
  - LED2:PA1(蓝色)
  - LED3:PA8(绿色)
- **电路设计**:
  - LED通过2kΩ限流电阻连接3.3V电源(共阳极接法)
  - MCU输出低电平时点亮
  - 实际PCB布局时应考虑LED间距均匀
- **工作模式**:通用推挽输出模式(GPIO_Mode_Out_PP)

### 实现原理
- **流水灯效果**:三个LED依次点亮,每个灯亮一段时间后熄灭,下一个灯点亮,循环往复
- **控制方式**:通过配置GPIO输出寄存器(ODR)改变引脚电平状态
- **延时实现**:使用系统滴答定时器实现精确延时

### 寄存器方式实现步骤
#### 工程准备
- **复制工程**:复制上一个点灯工程模板,避免重复配置基础文件
- **重命名工程**:重命名工程为"02_LED_Flow_Register"表明寄存器实现方式

#### 关键寄存器
- **CRL/CRH**:配置引脚工作模式
  - PA0-PA7用CRL(控制低8位)
  - PA8-PA15用CRH(控制高8位)
- **ODR**:直接控制引脚输出电平状态
- **BSRR**:原子操作设置/清除特定位(推荐替代直接操作ODR)

#### 实现流程
1. **使能GPIOA时钟**:通过设置RCC->APB2ENR寄存器开启GPIOA的时钟
2. **配置GPIO工作模式**:
   - PA0、PA1使用GPIOA->CRL(控制低8位)
   - PA8使用GPIOA->CRH(控制高8位)
   - 配置为通用推挽输出模式(50MHz)
3. **初始化LED**:初始化所有LED为熄灭状态(输出高电平)
4. **实现循环控制**:
   - 点亮当前LED(输出低电平)
   - 延时保持
   - 熄灭当前LED(输出高电平)
   - 切换至下一个LED

### 配置时钟
- **APB2总线使能**:通过设置RCC->APB2ENR寄存器来开启GPIOA的时钟
- **操作方式**:直接对寄存器进行赋值操作,确保对应GPIO端口的时钟使能位被置1

### 配置GPIO工作模式
#### 寄存器选择
- **PA0、PA1**:使用GPIOA->CRL(控制低8位)
- **PA8**:使用GPIOA->CRH(控制高8位)

#### 配置方法
```c
// 清除CNF位
GPIOA->CRL &= ~GPIO_CRL_CNF1;

// 设置MODE位(50MHz输出模式)
GPIOA->CRL |= GPIO_CRL_MODE1;

// 对于PA8同样操作
GPIOA->CRH &= ~GPIO_CRH_CNF8;
GPIOA->CRH |= GPIO_CRH_MODE8;
```

#### 模式参数
- **CNF=00**:通用推挽输出模式
- **MODE=11**:最大输出速度50MHz

### 初始输出状态配置
- **关灯逻辑**:
  - LED采用低电平点亮设计
  - 初始应输出高电平保持所有LED关闭状态
- **寄存器操作**:
  - 通过GPIOA->ODR寄存器进行位操作
  - 使用位或操作置1:GPIOA->ODR |= GPIO_ODR_ODR0等

### 流水灯实现逻辑
#### 效果设计原理
- **工作流程**:依次点亮每个LED→保持→关闭→点亮下一个LED
- **与常亮灯区别**:需要动态控制每个LED的状态变化

#### 实现步骤分解
- **点亮当前LED**(输出低电平)
- **调用延时函数**保持状态
- **关闭当前LED**(输出高电平)
- **切换到下一个LED**重复流程

#### 循环结构设计
- **主循环**:使用while(1)无限循环维持程序持续运行
- **执行顺序**:严格按照LED1→LED2→LED3的顺序处理
- **状态保持**:每个LED操作后都需要适当的延时

### 精准延时实现
#### 微秒延迟函数
```c
void Delay_us(uint32_t us)
{
    SysTick->LOAD = 72 * us;
    SysTick->CTRL = 0x05;  // 使用系统时钟,不产生中断,使能定时器
    while((SysTick->CTRL & SysTick_CTRL_COUNTFLAG) == 0){}
    SysTick->CTRL &= ~SysTick_CTRL_ENABLE;
}
```

#### 毫秒延迟函数
```c
void Delay_ms(uint32_t ms)
{
    for(uint32_t i = 0; i < ms; i++)
    {
        Delay_us(1000);
    }
}
```

#### 实现要点
- **时钟基础**:STM32系统时钟为72MHz,对应周期为1/72微秒
- **装载值计算**:每次滴答耗时1/72μs,因此延迟us微秒需要装载72×us次
- **CTRL寄存器配置**:0x05(二进制101)
  - 使用系统时钟(位2置1)
  - 计数结束不产生中断(位1置0)
  - 使能定时器(位0置1)
- **数值限制**:uint16_t最大值为65535,直接计算ms×1000会溢出
- **优化方案**:采用循环调用Delay_us(1000)实现毫秒延迟

### 软件设计注意事项
- **模式配置**:
  - 必须正确设置GPIO速度为50MHz(输出模式)
  - 推挽输出模式可提供较强的驱动能力
- **代码优化**:
  - 使用宏定义封装LED控制操作
  - 可采用位带操作提高代码可读性
- **调试技巧**:
  - 先单独测试每个LED的点亮功能
  - 使用逻辑分析仪监测GPIO波形
  - 检查寄存器值是否按预期变化

### 开发技巧总结
- **代码复用**:相同操作模式的LED配置可以复制修改
- **调试方法**:
  - 先实现单个LED的闪烁
  - 再扩展为多个LED的流水效果
- **注意事项**:
  - 确保GPIO端口时钟已使能
  - 输出模式配置必须完整(CNF+MODE)
  - 初始状态应为全关(高电平输出)

## 注意事项 & 踩坑

- PA0也可用作WKUP唤醒功能,PA1与UART2_TX复用,需避免功能冲突
- PA8属于GPIO高位,需使用CRH寄存器配置
- 必须先使能GPIOA时钟才能配置GPIO工作模式
- 推挽输出模式输出0为低电平(点亮),输出1为高电平(熄灭)
- 延时函数必须使用SysTick定时器实现,空循环延时不准确
- 工程复制后需重新配置Debug选项中的ST-Link调试器设置

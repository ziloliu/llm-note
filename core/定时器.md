## 一、定时器分类总览

STM32F103 系列共有 **11 个定时器**，分为 4 大类：

| 类型 | 数量 | 名称 | 位宽 | 特点 |
|------|:----:|------|:----:|------|
| **基本定时器** | 2 | TIM6, TIM7 | 16位 | 功能最少（低配版） |
| **通用定时器** | 4 | TIM2, TIM3, TIM4, TIM5 | 16位 | 功能较全（标准版） |
| **高级定时器** | 2 | TIM1, TIM8 | 16位 | 功能最全（高配版） |
| **系统滴答定时器** | 1 | SysTick | 24位 | 系统时钟，向下计数 |
| **看门狗定时器** | 2 | IWDG, WWDG | — | 故障检测，系统安全 |

### 1.1 三类定时器功能对比

```
功能丰富程度：

基本定时器（TIM6/TIM7）  ▰▰▱▱▱▱▱▱▱▱  低配版
通用定时器（TIM2~TIM5）  ▰▰▰▰▰▰▱▱▱▱  标准版
高级定时器（TIM1/TIM8）  ▰▰▰▰▰▰▰▰▰▰  高配版
```

| 功能 | 基本定时器 | 通用定时器 | 高级定时器 |
|------|:---------:|:---------:|:---------:|
| 定时中断 | ✅ | ✅ | ✅ |
| 向上计数 | ✅ | ✅ | ✅ |
| 向下计数 | ❌ | ✅ | ✅ |
| 中央对齐 | ❌ | ✅ | ✅ |
| 输入捕获 | ❌ | ✅ | ✅ |
| 输出比较 | ❌ | ✅ | ✅ |
| PWM 输出 | ❌ | ✅ | ✅ |
| 死区控制 | ❌ | ❌ | ✅ |
| 紧急刹车 | ❌ | ❌ | ✅ |
| 通道数 | 0 | 4 | 4 |

### 1.2 定时器命名与总线

| 编号 | 类型 | 总线 | 定时器时钟 |
|:----:|------|:----:|:----------:|
| TIM1 | 高级 | APB2 | 72 MHz |
| TIM2 | 通用 | APB1 | 72 MHz |
| TIM3 | 通用 | APB1 | 72 MHz |
| TIM4 | 通用 | APB1 | 72 MHz |
| TIM5 | 通用 | APB1 | 72 MHz |
| TIM6 | 基本 | APB1 | 72 MHz |
| TIM7 | 基本 | APB1 | 72 MHz |
| TIM8 | 高级 | APB2 | 72 MHz |
| SysTick | 内核 | — | 72 MHz |

> APB1 总线频率 36MHz，但定时器时钟 = 36MHz × 2 = 72MHz（因为 APB1 预分频系数 > 1）。

### 1.3 功能递进关系

```
基本定时器（TIM6/TIM7）
  │
  │  + 输入捕获 + 输出比较 + PWM + 脉冲计数 + 4通道
  ▼
通用定时器（TIM2~TIM5）
  │
  │  + 死区控制 + 紧急刹车 + 互补输出
  ▼
高级定时器（TIM1/TIM8）
```

## 二、SysTick 系统滴答定时器

### 2.1 基本概念

| 特性 | 说明 |
|------|------|
| 归属 | **ARM Cortex-M3 内核**（非 STM32 外设） |
| 位置 | 内嵌在 NVIC 中 |
| 位宽 | **24 位**计数器 |
| 计数方向 | **向下递减** |
| 时钟源 | 72MHz（AHB）或 9MHz（AHB/8） |
| 最大定时 | ≈ 233ms（72MHz）或 ≈ 1.86s（9MHz） |

**与外设定时器的区别**：
- SysTick 属于内核，不在 STM32 参考手册的外设章节
- TIM1~TIM8 属于外设，位宽 16 位，计数方向可配置

📄 [原文SysTick01: SysTick基本概念与工作原理](../raw/嵌入式开发/定时器/SysTick/01_SysTick基本概念与工作原理.md)

### 2.2 寄存器详解

SysTick 共有 **4 个寄存器**：

| 寄存器 | 全称 | 位宽 | 功能 |
|--------|------|:----:|------|
| **CTRL** | Control and Status | 32位 | 控制 + 状态（只用低3位+bit16） |
| **LOAD** | Reload Value | 32位 | 重装载值（用 bit[23:0]） |
| **VAL** | Current Value | 32位 | 当前计数值（用 bit[23:0]） |
| **CALIB** | Calibration | 32位 | 校准寄存器（只读，出厂固定） |

**CTRL 寄存器关键位**：

| 位 | 名称 | 功能 |
|:--:|------|------|
| 0 | ENABLE | 计数器使能 |
| 1 | TICKINT | 中断使能 |
| 2 | CLKSOURCE | 时钟源选择（0=AHB/8, 1=AHB） |
| 16 | COUNTFLAG | 计数到零标志（只读） |

**常用配置值**：

| 配置值 | ENABLE | TICKINT | CLKSOURCE | 含义 |
|:------:|:------:|:-------:|:---------:|------|
| 0x05（101） | 1 | 0 | 1 | 开启，不产生中断，72MHz |
| 0x07（111） | 1 | 1 | 1 | 开启，产生中断，72MHz |

**LOAD 寄存器**：
- 24 位重装载值，最大 16,777,215
- 定时时间 = 装载值 / 时钟频率
- 装载值 = 目标时间(μs) × 72（72MHz 时）

**VAL 寄存器**：
- 读操作：返回当前计数值
- 写操作：清零计数器（重新开始计时）

📄 [原文SysTick02: SysTick寄存器详解](../raw/嵌入式开发/定时器/SysTick/02_SysTick寄存器详解.md)

### 2.3 工作原理

```
① 给 LOAD 寄存器设置初始装载值
② 计数器从装载值开始向下递减
③ 每个时钟周期减 1
④ 减到 0 时：
   → COUNTFLAG 标志位置 1
   → 若开启中断，产生 SysTick 异常
   → 自动重新装载初始值，继续递减
```

**中断特性**：
- SysTick 是 Cortex-M3 内核异常（中断向量表第 15 项）
- 无需手动配置 NVIC，内核自动处理
- 中断服务函数名：`SysTick_Handler`

### 2.4 轮询方式 vs 中断方式

| 对比项 | 轮询方式 | 中断方式 |
|--------|----------|----------|
| CTRL 配置 | 0x05（101） | 0x07（111） |
| TICKINT | 0 | 1 |
| 等待方式 | `while(!(CTRL & COUNTFLAG))` | 中断服务函数自动执行 |
| CPU 占用 | 高（一直循环等待） | 低 |
| 是否阻塞 | 阻塞 | 不阻塞 |
| 适用场景 | 简单延时 | 定时任务、操作系统时基 |

**24 位计数器容量限制**：
- 72MHz 时，最大定时 ≈ 233ms
- 要定时 1 秒，需分段计数：每 1ms 中断一次，累计 1000 次

📄 [原文SysTick03: SysTick中断方式LED闪烁](../raw/嵌入式开发/定时器/SysTick/03_SysTick中断方式LED闪烁.md)

### 2.5 HAL 库实现

**CubeMX 配置**：
- SysTick 不在 Timers 菜单中配置
- `HAL_Init()` 内部自动完成 SysTick 初始化
- 默认配置：LOAD=71999, CTRL=0x07, 优先级=15

**HAL 库内部流程**：
```
HAL_Init()
  → HAL_InitTick(15)
    → HAL_SYSTICK_Config(72000)
      → SysTick_Config(72000)  // CMSIS 函数
        ├── SysTick->LOAD = 71999
        ├── SysTick->VAL  = 0
        ├── SysTick->CTRL = 0x07
        └── NVIC_SetPriority(SysTick_IRQn, 15)
```

**全局滴答计数器 uwTick**：
- 类型：`__IO uint32_t`，单位毫秒
- 每次 SysTick 中断 +1（通过 `HAL_IncTick()`）
- `HAL_Delay()` 基于 uwTick 实现
- 最大可运行约 49.7 天才溢出

📄 [原文SysTick04: SysTick HAL库实现](../raw/嵌入式开发/定时器/SysTick/04_SysTick HAL库实现.md)

### 2.6 延时函数实现

**延时函数层次**：
```
Delay_us(us)  ← 微秒级（基础，直接操作 SysTick）
Delay_ms(ms)  ← 毫秒级（循环调用 Delay_us(1000)）
Delays(s)     ← 秒级（循环调用 Delay_ms(1000)）
```

**Delay_us 实现（轮询方式）**：
```c
void Delay_us(uint16_t us)
{
    SysTick->LOAD = 72 * us;
    SysTick->VAL  = 0;
    SysTick->CTRL = 0x05;  // 72MHz + 无中断 + 使能
    while (!(SysTick->CTRL & (1 << 16)));  // 等待 COUNTFLAG
    SysTick->CTRL = 0x00;  // 关闭
}
```

**轮询 vs 中断方式的 LOAD 值差异**：
- 轮询方式：LOAD = 72000（不需要减 1）
- 中断方式：LOAD = 71999（需要减 1）
- 原因：中断方式在下一个时钟周期才进入中断，多 1 个周期

**HAL_Delay 实现（中断方式）**：
```c
__weak void HAL_Delay(uint32_t Delay)
{
    uint32_t tickstart = HAL_GetTick();
    uint32_t wait = Delay;
    if (wait < HAL_MAX_DELAY) wait += uwTickFreq;
    while ((HAL_GetTick() - tickstart) < wait) {}
}
```

| 对比项 | 轮询方式 | HAL_Delay |
|--------|----------|-----------|
| 计时源 | SysTick 硬件 | 全局变量 uwTick |
| SysTick 占用 | 函数内独占 | 与系统共享 |
| 能否嵌套 | 容易卡死 | 相对安全 |

## 三、基本定时器 TIM6/TIM7

### 3.1 基本特性

| 特性 | TIM6 | TIM7 |
|------|:----:|:----:|
| 位宽 | **16 位** | **16 位** |
| 计数方向 | **只能向上递增** | **只能向上递增** |
| 外部 IO | **无** | **无** |
| 连接总线 | APB1 | APB1 |
| 定时器时钟 | **72 MHz** | **72 MHz** |
| 互相独立 | 完全独立，不共享资源 | 完全独立，不共享资源 |

**与 SysTick 对比**：

| 对比项 | SysTick | 基本定时器 |
|--------|:-------:|:---------:|
| 位宽 | 24 位 | 16 位 |
| 计数方向 | 向下递减 | 向上递增 |
| 预分频器 | 无 | **有**（PSC） |
| 触发 DAC | 不支持 | **支持** |
| 归属 | 内核 | 外设 |

📄 [原文基本定时器01: 基本定时器TIM6与TIM7详解](../raw/嵌入式开发/定时器/基本定时器/01_基本定时器TIM6与TIM7详解.md)

### 3.2 时基单元三大寄存器

| 寄存器 | 全称 | 位宽 | 功能 |
|--------|------|:----:|------|
| **PSC** | Prescaler | 16位 | 对输入时钟进行分频 |
| **CNT** | Counter | 16位 | 从 0 向上递增计数 |
| **ARR** | Auto-Reload Register | 16位 | 设定计数上限 |

**PSC 预分频器**：
- 分频值 = PSC + 1（PSC 为寄存器值）
- PSC = 0 → 1 分频（不分频）
- PSC = 7199 → 7200 分频
- 范围：1 分频 ~ 65536 分频

**CNT 计数器**：
- 每个计数器时钟上升沿自动加 1
- 从 0 开始，达到 ARR 后下一个时钟沿溢出

**ARR 自动重装载**：
- 溢出条件：CNT 超过 ARR
- 溢出行为：CNT 归零，产生更新事件（UEV）
- 从 0 计到溢出共经过 ARR + 1 个计数周期

**ARR 预加载机制（影子寄存器）**：

| ARPE 值 | 模式 | ARR 更新时机 | 效果 |
|:-------:|------|-------------|------|
| 0 | 无预加载 | 写入后立即生效 | 即时生效 |
| 1 | 有预加载 | 等待更新事件后生效 | 延迟生效 |

📄 [原文基本定时器01: 基本定时器TIM6与TIM7详解](../raw/嵌入式开发/定时器/基本定时器/01_基本定时器TIM6与TIM7详解.md)

### 3.3 定时时间计算

**公式**：
```
定时时间 = (PSC + 1) × (ARR + 1) / 时钟频率
```

**常用参数速查**：

| 定时时间 | PSC | 分频值 | ARR | 计数次数 |
|----------|:---:|:------:|:---:|:--------:|
| 100 μs | 71 | 72 | 999 | 1000 |
| 1 ms | 71 | 72 | 999 | 1000 |
| 10 ms | 7199 | 7200 | 99 | 100 |
| 100 ms | 7199 | 7200 | 999 | 1000 |
| **1 s** | **7199** | **7200** | **9999** | **10000** |

> PSC 和 ARR 都是 16 位寄存器，最大值 65535。

📄 [原文基本定时器01: 基本定时器TIM6与TIM7详解](../raw/嵌入式开发/定时器/基本定时器/01_基本定时器TIM6与TIM7详解.md)

### 3.4 寄存器详解

| 寄存器 | 功能 | 关键位 |
|--------|------|--------|
| **CR1** | 控制寄存器 1 | CEN（使能）、ARPE（预装载） |
| **DIER** | 中断使能 | UIE（更新中断使能） |
| **SR** | 状态寄存器 | UIF（更新中断标志，软件写0清除） |
| **EGR** | 事件产生 | UG（写1产生更新事件） |
| **PSC** | 预分频器 | 实际分频 = PSC + 1 |
| **ARR** | 自动重装载 | 计数上限 |
| **CNT** | 计数器 | 当前计数值 |

**关键操作**：
- UIF 标志位必须手动清除（软件写 0），否则中断反复触发
- 写 UG = 1 立即产生更新事件，刷新所有预加载寄存器
- PSC 没有 ARPE 控制位，默认预装载模式

📄 [原文基本定时器02: 基本定时器寄存器详解](../raw/嵌入式开发/定时器/基本定时器/02_基本定时器寄存器详解.md)

### 3.5 初始化配置流程（寄存器方式）

```
① 开启 TIM6 时钟
   RCC->APB1ENR |= RCC_APB1ENR_TIM6EN;

② 配置预分频器 PSC
   TIM6->PSC = 7199;           // 7200 分频 → 10kHz

③ 配置自动重装载值 ARR
   TIM6->ARR = 9999;           // 10000 次计数 → 1s

④ 使能更新中断
   TIM6->DIER |= TIM_DIER_UIE; // UIE = 1

⑤ 配置 NVIC
   NVIC_SetPriority(TIM6_IRQn, 1);
   NVIC_EnableIRQ(TIM6_IRQn);

⑥ 使能计数器（最后）
   TIM6->CR1 |= TIM_CR1_CEN;   // CEN = 1
```

**中断服务函数**：
```c
void TIM6_IRQHandler(void)
{
    if (TIM6->SR & TIM_SR_UIF)
    {
        TIM6->SR &= ~TIM_SR_UIF;   // 清除标志位
        // 用户操作
    }
}
```

**注意事项**：
- TIM6 连接 APB1，不要写 `RCC_APB2ENR`
- 必须手动配置 NVIC（与 SysTick 不同）
- 必须手动清除 UIF 标志
- CEN 最后开启

📄 [原文基本定时器03: TIM6寄存器方式LED闪烁](../raw/嵌入式开发/定时器/基本定时器/03_TIM6寄存器方式LED闪烁.md)

### 3.6 HAL 库实现

**CubeMX 配置**：
- Timers → TIM6 → 勾选 Activate
- Prescaler: 7199
- Counter Period: 9999
- NVIC → TIM6 global interrupt → 勾选 Enable

**关键问题**：CubeMX 生成的代码**不会自动开启定时器**，需手动调用：
```c
HAL_TIM_Base_Start_IT(&htim6);  // 启动定时器 + 使能中断
```

**HAL 库中断处理流程**：
```
TIM6_IRQHandler()
  → HAL_TIM_IRQHandler(&htim6)
    → HAL_TIM_PeriodElapsedCallback(&htim6)  // 用户重写
```

**回调函数特点**：
- 所有定时器共享同一个回调函数
- 必须通过 `htim->Instance` 判断是哪个定时器
- 内部已自动清除 UIF 标志位

```c
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim)
{
    if (htim->Instance == TIM6)
    {
        HAL_GPIO_TogglePin(GPIOA, GPIO_PIN_1);
    }
}
```

## 四、通用定时器

### 4.1 基本特性

| 特性 | 说明 |
|------|------|
| 编号 | TIM2, TIM3, TIM4, TIM5 |
| 数量 | 4 个 |
| 位宽 | 16 位 |
| 连接总线 | APB1 |
| 定时器时钟 | 72 MHz |
| 功能定位 | 标准版（覆盖基本定时器 + 扩展功能） |
| 通道数 | 每个定时器 **4 个**通道 |

**相比基本定时器新增功能**：
- 向下计数、向上/向下交替计数
- 外部时钟源（ETR、通道输入、内部触发）
- 输入捕获、输出比较、PWM 输出
- 编码器接口、霍尔传感器接口

📄 [原文通用定时器01: 通用定时器时钟源详解](../raw/嵌入式开发/定时器/通用定时器/01_通用定时器时钟源详解.md)

### 4.2 时钟源（4 种）

| 时钟源 | 缩写 | 来源 | 使用频率 |
|--------|------|------|:--------:|
| **内部时钟** | CK_INT | APB1 × 2 = 72MHz | **最常用** |
| **外部时钟源模式 1** | TI1FP1/TI2FP2 | 通道 1/2 输入 | 较少 |
| **外部时钟源模式 2** | ETRF | 外部触发引脚 | 较少 |
| **内部触发** | ITR0~3 | 其他定时器输出 | 级联时 |

**内部时钟（默认）**：大部分情况直接使用，无需额外配置。

**外部时钟源模式 1**：
- 只有通道 1 和通道 2 可以作为时钟源
- 信号经过滤波和边沿检测处理

**外部时钟源模式 2**：
- 专用外部触发引脚（如 TIM3 的 ETR = PD2）
- 有独立的极性选择和预分频

**内部触发（ITR）**：
- 不需要外部引脚，芯片内部连线
- 用于定时器级联

**定时器级联**：
- 用一个定时器的输出控制另一个定时器的计数
- 目的：突破 16 位计数范围限制，实现更长定时

📄 [原文通用定时器01: 通用定时器时钟源详解](../raw/嵌入式开发/定时器/通用定时器/01_通用定时器时钟源详解.md)

### 4.3 三种计数模式

| 模式 | 计数方向 | 溢出条件 | 更新事件频率 |
|------|----------|----------|:------------:|
| **向上计数** | 0 → ARR | CNT 超过 ARR | 每 ARR+1 次 |
| **向下计数** | ARR → 0 | CNT 减到 0 以下 | 每 ARR+1 次 |
| **中央对齐** | 0 → ARR-1 → 0 | 到 ARR-1 或 1 | 每 2×ARR 次 |

**CR1 寄存器相关位**：

| 位 | 名称 | 功能 |
|:--:|------|------|
| 0 | CEN | 计数器使能 |
| 4 | DIR | 计数方向（0=向上，1=向下） |
| 6:5 | CMS | 中央对齐模式选择 |
| 7 | ARPE | 自动重装载预装载使能 |

**CMS 和 DIR 的关系**：
- CMS = 00 时：由 DIR 决定方向（向上或向下）
- CMS ≠ 00 时：DIR 无效，硬件自动控制方向（中央对齐）

**向上计数（最常用）**：
```
CNT: 0 → 1 → 2 → ... → ARR → [溢出] → 0 → 1 → ...
```

**向下计数**：
```
CNT: ARR → ARR-1 → ... → 1 → 0 → [溢出] → ARR → ...
```

**中央对齐**：
```
CNT: 0 → 1 → ... → ARR-1 → [向上溢出] → ARR → ARR-1 → ... → 1 → [向下溢出] → 0 → ...
一个完整周期 = 2 × ARR 个计数周期
```

**应用场景**：

| 模式 | 典型应用 |
|------|----------|
| 向上计数 | 定时中断、PWM 输出、输出比较、输入捕获 |
| 向下计数 | SysTick 类似功能、特定定时需求 |
| 中央对齐 | 电机控制（FOC）、对称 PWM 波形 |

## 五、PWM 与输出比较

### 5.1 PWM 原理

**PWM（Pulse Width Modulation）**：利用数字信号控制模拟电路的技术，通过调节方波的高电平占比改变输出功率。

**核心原理 — 面积等效法则**：
- 两个形状不同的信号，如果"面积"相同，对于惯性负载效果相同
- 等效于一个恒定的中间电压值

**PWM 三大关键参数**：

| 参数 | 符号 | 说明 | 影响 |
|------|:----:|------|------|
| 周期 | T | 一个完整波形的重复时间 | 决定频率 |
| 频率 | f | f = 1/T | 人眼是否看出闪烁 |
| 占空比 | D | 高电平时间 / 周期 | **决定等效输出功率** |

**占空比与亮度**：
- D = 0% → LED 熄灭
- D = 25% → LED 较暗
- D = 50% → LED 半亮
- D = 75% → LED 较亮
- D = 100% → LED 最亮

**PWM 与 DAC 的区别**：

| 对比项 | PWM | DAC |
|--------|-----|-----|
| 输出信号 | 数字方波 | 模拟电压 |
| 原理 | 面积等效 | 真正的数模转换 |
| 成本 | 低（只需一个 GPIO） | 较高 |
| 适用场景 | 电机、LED、蜂鸣器 | 音频、精密模拟输出 |

📄 [原文通用定时器03: PWM原理详解](../raw/嵌入式开发/定时器/通用定时器/03_PWM原理详解.md)

### 5.2 输出比较与 PWM 波形生成

**输出比较功能**：通过 CNT 与 CCR 比较，控制输出引脚电平。

**核心寄存器**：

| 寄存器 | 作用 | 可调性 |
|--------|------|:------:|
| ARR | 决定计数上限（PWM 周期/频率） | 固定 |
| CCR | 比较基准值（PWM 占空比） | **可调** |
| CNT | 当前计数值（自动递增） | 只读 |

**八种输出比较模式**：

| OCM[2:0] | 模式名称 | 输出行为 |
|:--------:|----------|----------|
| 000 | 冻结 | 输出不受比较影响 |
| 001 | 匹配时强制输出高 | CNT = CCR 时输出高 |
| 010 | 匹配时强制输出低 | CNT = CCR 时输出低 |
| 011 | 匹配时翻转 | 每次相等时翻转 |
| 100 | 强制输出低 | 无条件输出低 |
| 101 | 强制输出高 | 无条件输出高 |
| **110** | **PWM 模式 1** | **CNT < CCR → 高；CNT ≥ CCR → 低** |
| 111 | PWM 模式 2 | CNT < CCR → 低；CNT ≥ CCR → 高 |

**PWM 模式 1（最常用）**：
- 占空比 = CCR / (ARR + 1) × 100%
- CCR 值直接对应高电平占比，非常直观

**PWM 关键参数与定时器参数的关系**：

| PWM 参数 | 定时器参数 | 计算公式 |
|----------|-----------|----------|
| PWM 频率 | ARR + PSC + 时钟频率 | f = 时钟频率 / ((PSC+1) × (ARR+1)) |
| PWM 周期 | ARR + PSC + 时钟频率 | T = (PSC+1) × (ARR+1) / 时钟频率 |
| 占空比 | CCR 和 ARR | D = CCR / (ARR + 1) × 100% |
| 分辨率 | ARR | 1 / (ARR + 1) |

**GPIO 配置**：复用推挽输出（Alternate Function Push-Pull），High Speed。

### 5.3 PWM 输出寄存器配置

**CCMR1 寄存器（核心）**：
- 16 位分两半：低 8 位控制通道 1，高 8 位控制通道 2
- CCS = 00 → 输出比较模式
- CCS ≠ 00 → 输入捕获模式

**通道 2 输出模式关键位**：

| 位 | 名称 | 功能 | 配置 |
|:--:|------|------|:----:|
| 9:8 | CC2S | 通道方向选择 | **00** = 输出 |
| 12:10 | OC2M | 输出比较模式 | **110** = PWM 模式 1 |
| 13 | OC2PE | 预装载使能 | 0 或 1 |

**CCER 寄存器**：

| 位 | 名称 | 功能 | 配置 |
|:--:|------|------|:----:|
| 4 | CC2E | 输出使能 | **1** = 使能 |
| 5 | CC2P | 极性 | 0 = 高电平有效 |

**CCR 预装载**：
- OC2PE = 0：写入后立即生效
- OC2PE = 1：等到更新事件后才生效

### 5.4 PWM 呼吸灯实现（寄存器方式）

**配置流程**：
```
① 开启时钟
   ├── RCC->APB1ENR |= TIM5EN
   └── RCC->APB2ENR |= IOPAEN    ← 必须开启 GPIO 时钟

② 配置 GPIO（PA1 复用推挽输出）
   → CNF = 10（复用推挽），MODE = 11（50MHz）

③ 配置 PSC = 7199（7200 分频 → 10kHz）

④ 配置 ARR = 99（100 级分辨率）

⑤ 配置 CCR2 = 50（初始占空比）

⑥ 配置 CCMR1
   ├── CC2S = 00（输出模式）
   └── OC2M = 110（PWM 模式 1）

⑦ 配置 CCER
   └── CC2E = 1（使能输出）

⑧ 使能定时器（CR1.CEN = 1）
```

**呼吸灯控制逻辑**：
```c
uint8_t duty_cycle = 0;
uint8_t dir = 0;  // 0=增大, 1=减小

while (1)
{
    if (dir == 0) {
        duty_cycle += 1;
        if (duty_cycle >= 99) dir = 1;
    } else {
        duty_cycle -= 1;
        if (duty_cycle <= 1) dir = 0;
    }
    TIM5->CCR2 = duty_cycle;
    Delay_ms(20);
}
```

**GPIO 时钟必须开启的原因**：
- 复用信号通过多路选择器（由 TIM5 时钟驱动）
- 但输出控制电路和 MOS 管驱动器由 GPIO 模块时钟驱动
- 两者都需要时钟

- Channel 2: PWM Generation CH2

- PWM Mode 1
- CH Polarity: High

## 六、输入捕获

### 6.1 功能概述

| 项目 | 说明 |
|------|------|
| 功能 | 捕获输入通道上信号的上升沿或下降沿 |
| 核心操作 | 信号边沿到来时，**记录当前 CNT 值到 CCR** |
| 本质 | 对计数器做"快照" |
| 主要应用 | 测量信号周期/频率、高电平宽度、占空比 |

**能测量的参数**：

| 测量项 | 方法 |
|--------|------|
| 周期 T | 连续两个上升沿的 CCR 差值 |
| 频率 f | f = 1/T |
| 高电平宽度 | 上升沿到下降沿的 CCR 差值 |
| 占空比 | 高电平宽度 / 周期 |

📄 [原文通用定时器08: 输入捕获功能详解](../raw/嵌入式开发/定时器/通用定时器/08_输入捕获功能详解.md)

### 6.2 信号处理流程

```
① 外部信号通过 CHx 引脚输入 → TIx（原始输入信号）
② 输入滤波器 → 滤除毛刺噪声 → TIxF
③ 边沿检测器 → 选择上升沿或下降沿 → TIxFP
④ 多路选择器 → 选择信号来源 → ICxPS
⑤ 输入预分频器 → 对捕获事件分频
⑥ 捕获事件 → 将 CNT 值锁存到 CCR
```

**信号命名规则**：

| 名称 | 含义 |
|------|------|
| TI1 | 通道 1 原始输入信号 |
| TI1F | 经过滤波后的信号 |
| TI1FP1 | 滤波 + 极性处理后，直通映射到 CCR1 |
| TI1FP2 | 滤波 + 极性处理后，交叉映射到 CCR2 |
| IC1 | 多路选择器输出 |

### 6.3 输入捕获 vs 输出比较

| 对比项 | 输入捕获 | 输出比较 |
|--------|----------|----------|
| 信号方向 | **外部 → 内部** | **内部 → 外部** |
| CCR 用途 | **保存** CNT 快照值 | 作为与 CNT **比较**的基准 |
| 引脚配置 | 复用浮空/上拉**输入** | 复用推挽**输出** |
| CCMR.CCS | **01**（输入） | **00**（输出） |
| CCxP 含义 | 选择捕获边沿 | 选择输出有效电平 |
| 典型应用 | 测量信号周期/频率 | 产生 PWM 波形 |

### 6.4 寄存器配置

**CCMR1 输入模式关键位**：

| 位 | 名称 | 功能 | 配置 |
|:--:|------|------|:----:|
| 1:0 | CC1S | 通道方向 | **01** = 输入（映射到 TI1） |
| 3:2 | IC1PSC | 输入预分频 | 00 = 不分频 |
| 7:4 | IC1F | 输入滤波 | 0000 = 不滤波 |

**CCER 寄存器**：

| 位 | 名称 | 功能 | 配置 |
|:--:|------|------|:----:|
| 0 | CC1E | 捕获使能 | **1** = 使能 |
| 1 | CC1P | 边沿选择 | **0** = 上升沿，**1** = 下降沿 |

**DIER 寄存器**：
- CC1IE = 1：使能捕获中断

**SR 寄存器**：
- CC1IF：捕获中断标志（硬件置位，读 CCR 或软件写 0 清除）

**定时器参数**：
- PSC = 71（72 分频 → 1MHz → 1μs/次）
- ARR = 65535（最大值，避免溢出）

**测量范围**：
- 最小可测周期：1μs（信号频率上限 1MHz）
- 最大可测周期：65535μs ≈ 65.5ms（信号频率下限 15.26Hz）

### 6.5 配置流程（寄存器方式）

```
① 开启时钟
   ├── RCC->APB1ENR |= TIM4EN
   └── RCC->APB2ENR |= IOPBEN

② 配置 GPIO（PB6 复用浮空输入）
   → CNF = 10（复用功能），MODE = 00（输入）

③ 配置 PSC = 71（72 分频 → 1MHz）

④ 配置 ARR = 65535（最大值）

⑤ 配置 CCMR1（输入模式）
   ├── CC1S = 01（输入，映射到 TI1）
   ├── IC1PSC = 00（不分频）
   └── IC1F = 0000（不滤波）

⑥ 配置 CCER
   ├── CC1P = 0（上升沿捕获）
   └── CC1E = 1（使能捕获）

⑦ 配置 DIER
   └── CC1IE = 1（使能捕获中断）

⑧ 配置 NVIC

⑨ 使能定时器（CR1.CEN = 1）
```

**中断服务函数**：
```c
void TIM4_IRQHandler(void)
{
    if (TIM4->SR & TIM_SR_CC1IF)
    {
        uint32_t current_capture = TIM4->CCR1;  // 读取捕获值

        if (last_capture != 0)
        {
            if (current_capture >= last_capture)
                capture_value = current_capture - last_capture;
            else
                capture_value = 65536 - last_capture + current_capture;  // 溢出处理
            capture_flag = 1;
        }
        last_capture = current_capture;
        TIM4->SR &= ~TIM_SR_CC1IF;  // 清除标志
    }
}
```

**溢出处理**：
- 当信号周期 > 65535μs 时，CNT 溢出后再捕获
- 处理公式：周期 = 65536 - last_capture + current_capture

## 七、输入捕获测量周期与频率

### 7.1 中断方式实现

**核心思路**：
- 使用输入捕获检测信号上升沿
- 在中断中记录 CNT 值并清零
- 通过两次上升沿的 CCR 差值计算周期

**关键变量**：
```c
volatile uint16_t cycle = 0;    // 保存周期值（μs）
volatile uint8_t  count = 0;    // 上升沿次数计数
```

**中断服务函数**：
```c
void TIM4_IRQHandler(void)
{
    if (TIM4->SR & TIM_SR_CC1IF)
    {
        TIM4->SR &= ~TIM_SR_CC1IF;  // 清除中断标志
        count++;

        if (count == 1)
            TIM4->CNT = 0;           // 第一次上升沿：清零计数器
        else if (count == 2)
        {
            cycle = TIM4->CCR1;      // 第二次上升沿：读取周期
            count = 0;
        }
    }
}
```

**单位换算**：
- 周期：cycle / 1000.0 → ms
- 频率：1000000.0 / cycle → Hz

📄 [原文03: 输入捕获中断方式实现](../raw/嵌入式开发/定时器/定时器学习/03_输入捕获中断方式实现.md)

### 7.2 事件先于中断的原理

**核心概念**：
- **事件**：硬件瞬间完成（1 个时钟周期）
- **中断**：软件延迟执行（若干时钟周期）

**关键时序**：
```
上升沿到来
    │
    ├─[0]────→ 硬件事件：CCR1 = CNT（快照）    ← 瞬间完成
    │
    ├─[1~N]──→ 中断处理：CNT = 0（清零）       ← 延迟 N 个周期
    │
    └─[N+1]──→ 中断返回
```

**优化方案**：
- 利用"事件先于中断"的硬件保证
- 每次上升沿只需：清除标志 + 清零 CNT
- 无需全局变量，无需状态机

```c
void TIM4_IRQHandler(void)
{
    if (TIM4->SR & TIM_SR_CC1IF)
    {
        TIM4->SR &= ~TIM_SR_CC1IF;  // 清除标志
        TIM4->CNT = 0;               // 清零计数器
    }
}

double TIM4_GetPWMPeriod(void)
{
    return (double)TIM4->CCR1 / 1000.0;
}
```

```c
// 读取结果
double TIM4_GetPWMPeriod(void)
{
    return (double)__HAL_TIM_GET_COMPARE(&htim4, TIM_CHANNEL_1) / 1000.0;
}
```

## 八、触发输入与从模式

### 8.1 触发信号

| 信号 | 全称 | 方向 | 功能 |
|------|------|------|------|
| **TRGI** | Trigger In | 输入 | 外部信号控制本定时器行为 |
| **TRGO** | Trigger Out | 输出 | 本定时器信号输出控制其他外设 |

**触发输入源（SMCR.TS，8 种）**：

| TS[2:0] | 来源 | 类别 |
|:-------:|------|------|
| 000~011 | ITR0~3 | 内部触发（其他定时器） |
| 100 | TI1F_ED | 通道 1 双边沿检测 |
| **101** | **TI1FP1** | **通道 1 直通信号** |

### 8.2 从模式（SMCR.SMS，8 种）

| SMS[2:0] | 模式 | 功能 | 计数器行为 |
|:--------:|------|------|-----------|
| **000** | **关闭** | 不使用从模式 | 内部时钟正常计数 |

| **100** | **复位模式** | TRGI 上升沿 → CNT 清零 | 复位 |
| 101 | 门控模式 | TRGI 高=使能，低=停止 | 开关控制 |
| 110 | 触发模式 | TRGI 上升沿 → 启动计数 | 仅启动 |
| **111** | **外部时钟** | TRGI 上升沿 → CNT 加 1 | 替换时钟源 |

### 8.3 复位模式的优势

| 对比项 | 中断清零 | 硬件复位（从模式） |
|--------|:--------:|:-----------------:|
| CNT 清零方式 | 软件（中断服务函数） | **硬件（TRGI 信号）** |
| 响应延迟 | 若干时钟周期 | **瞬间（1 个周期）** |
| 需要中断 | ✅ | **❌** |
| 需要全局变量 | ✅ | **❌** |
| 精度 | 受中断延迟影响 | **更高** |

## 九、PWM 输入模式（测周期+占空比）

### 9.1 核心原理

**需求**：同时测量 PWM 方波的周期 T 和高电平时间 t

**方案**：
- 同一信号从 CH1 输入，分配到两个通道
- 通道 1（直通）：上升沿检测 → CCR1 = 周期 T
- 通道 2（交叉）：下降沿检测 → CCR2 = 高电平时间 t
- 从模式复位：TI1FP1 上升沿 → CNT 自动清零

**信号通路**：
```
CH1 引脚输入 PWM
    │
    ├─→ 上升沿检测 → TI1FP1
    │       │
    │       ├─→ 直连(CCR1) → 测周期 T
    │       │
    │       └─→ 触发输入(TRGI) → 复位模式 → CNT 清零
    │
    └─→ 下降沿检测 → TI2FP2
            │
            └─→ 交叉(CCR2) → 测高电平时间 t
```

### 9.2 寄存器配置

| 配置项 | 通道 1 | 通道 2 |
|--------|:------:|:------:|
| CCMR.CCxS | **01**（直通） | **10**（交叉） |
| CCER.CCxP | **0**（上升沿） | **1**（下降沿） |
| CCER.CCxE | **1** | **1** |

**从模式配置**：
- SMCR.TS = 101（TI1FP1 作为触发源）
- SMCR.SMS = 100（复位模式）

### 9.3 计算公式

```
PSC = 71 → 1MHz → 1μs/次

CCR1 = 周期 T（μs）
CCR2 = 高电平时间 t（μs）

周期   = CCR1 / 1000.0           （ms）
频率   = 1000000.0 / CCR1         （Hz）
占空比 = CCR2 / CCR1 × 100%      （%）
```

### 9.4 时序分析

```
PWM 信号：  ┌──────┐                ┌──────┐
            │      │                │      │
       ─────┘      └────────────────┘      └──────
            ↑      ↑                ↑      ↑
            ①      ②                ③      ④

① 上升沿：CCR1=上次周期 + CNT=0（复位）
② 下降沿：CCR2=2（高电平时间）
③ 上升沿：CCR1=4（周期）+ CNT=0（复位）
④ 下降沿：CCR2=2（高电平时间）

主循环读取：
  周期 = CCR1 = 4
  高电平 = CCR2 = 2
  占空比 = 2/4 = 50%
```

### 9.5 与中断方式对比

| 对比项 | 中断状态机 | 中断优化 | **PWM 输入模式** |
|--------|:----------:|:--------:|:----------------:|
| 通道数 | 1 个 | 1 个 | **2 个** |
| 从模式 | 不使用 | 不使用 | **复位模式** |
| 中断 | ✅ | ✅ | **❌** |
| 全局变量 | ✅ | ❌ | **❌** |
| 能测占空比 | ❌ | ❌ | **✅** |
| 精度 | 一般 | 较好 | **最高** |

## 十、高级定时器（TIM1/TIM8）

### 10.1 基本特性

| 特性 | 说明 |
|------|------|
| 编号 | TIM1、TIM8 |
| 继承 | 具有通用定时器的**所有功能** |
| 总线 | APB2（高速外设总线） |
| 时钟 | 72MHz |

**新增三大功能**：

| 功能 | 说明 | 寄存器 |
|------|------|--------|
| **重复计数器（RCR）** | 控制更新事件产生频率 | RCR |
| **互补输出 + 死区时间** | CH1/CH1N 互补信号，带可编程死区 | BDTR.DTG |
| **刹车输入（Break）** | 外部信号强制关断所有输出 | BDTR.BKE |

### 10.2 重复计数器（RCR）

**与通用定时器的区别**：
```
通用定时器（无 RCR）：
  CNT 溢出 → 直接产生更新事件

高级定时器（有 RCR）：
  CNT 溢出 → RCR 减 1（不产生更新事件）
  RCR 减到 0 → 再次溢出时 → 产生更新事件
```

**核心公式**：
```
PWM 周期输出个数 = RCR + 1
闪烁次数 = RCR + 1
```

**RCR 寄存器**：
- 8 位，范围 0~255
- 实际周期数 = RCR + 1（最大 256）

| RCR 值 | PWM 周期数 | LED 闪烁次数 |
|:------:|:----------:|:------------:|
| 0 | 1 | 1 |
| 1 | 2 | 2 |
| 4 | 5 | 5 |
| 9 | 10 | 10 |
| 255 | 256 | 256 |

### 10.3 互补输出与死区时间

**互补输出**：
- 通道 1~3 可输出两路互补信号：CHx + CHxN
- 频率相同，相位相差 180°（完全反相）

**死区时间的必要性**：
```
问题：MOSFET 开通快、关断慢

理想情况：CH1 高→低 时 CH1N 低→高
实际情况：CH1 还没完全关断时 CH1N 已经开通
          → 电源短路！→ 烧毁！

解决方案：插入死区时间
  → 由高变低（关断）：立即执行
  → 由低变高（开通）：延迟一段时间再执行
  → 保证任何时刻都不会有两个管子同时导通
```

**死区时间配置**：
- 寄存器：BDTR
- 位域：DTG[7:0]
- 范围：可编程，从几 ns 到几百 μs

### 10.4 刹车输入（Break）

**功能**：
- 外部信号 → BKE（刹车使能）→ BKP（极性检测）
- 检测到指定边沿 → MOE 自动清零 → 所有输出立即关断

**应用场景**：
- 电机过流保护
- 紧急停止按钮
- 电源异常保护

### 10.5 MOE（主输出使能）

**三层使能**：
```
① CEN（CR1）    → 定时器总开关
② CCxE（CCER）  → 通道使能
③ MOE（BDTR）   → 主输出使能（高级定时器特有）

三层都必须开启，缺一不可
```

**MOE 位**：
- BDTR 寄存器第 15 位
- MOE = 0：所有输出强制无效（安全状态）
- MOE = 1：正常输出

### 10.6 空闲状态（OISx）

**作用**：
- MOE = 0（主输出关闭）时输出引脚的默认电平

**配置**：
- 寄存器：CR2
- 位：OISx
- OISx = 0（Reset）：空闲时输出低电平
- OISx = 1（Set）：空闲时输出高电平

**应用场景**：
- LED 电路（低电平点亮）：希望关闭后 LED 不亮 → OISx = 1

## 十一、高级定时器有限次数 PWM 输出

### 11.1 实验需求

| 项目 | 说明 |
|------|------|
| 定时器 | TIM1（高级定时器） |
| 功能 | 输出 5 个周期的 PWM 方波后停止 |
| PWM 频率 | 2Hz（周期 0.5s） |
| 占空比 | 50% |
| 总时长 | 2.5 秒 |

### 11.2 关键参数计算

```
PSC = 7199 → 7200 分频 → 10kHz
ARR = 4999 → 5000 次计数 → 0.5s
CCR1 = 2500 → 占空比 50%
RCR = 4 → 5 次溢出后停止

总时长 = 5 × 0.5 = 2.5s
```

### 11.3 影子寄存器问题

**问题现象**：
- 启动后 LED 不闪烁，串口显示一启动就进入中断

**根本原因**：
- PSC 和 RCR 有影子寄存器，无控制位
- 初始化时写入的是预装载值，影子值未刷新
- PSC 影子值 = 0（72MHz 直接计数）
- RCR 影子值 = 0（不重复，溢出即更新）
- 极短时间内完成 5000 次计数 + 更新事件

**解决方案（URS + UG）**：
```c
// 在初始化末尾、使能中断之后、开启定时器之前
TIM1->CR1 |= TIM_CR1_URS;  // URS = 1（UG 不产生中断）
TIM1->EGR |= TIM_EGR_UG;   // 手动更新（刷新影子寄存器）
```

**URS 位作用**：
- URS = 0（默认）：计数器溢出和 UG 都会产生更新中断
- URS = 1：只有计数器溢出产生更新中断，UG 不产生中断

### 11.4 配置要点

**寄存器方式**：
```c
// 时基配置
TIM1->PSC = 7199;
TIM1->ARR = 4999;
TIM1->RCR = 4;

// 输出通道配置
TIM1->CCMR1 |= (0x6 << 4);  // PWM 模式 1
TIM1->CCR1 = 2500;           // 50% 占空比
TIM1->CCER |= TIM_CCER_CC1E; // 通道使能
TIM1->BDTR |= TIM_BDTR_MOE;  // 主输出使能

// 中断配置
TIM1->DIER |= TIM_DIER_UIE;
NVIC_EnableIRQ(TIM1_UP_IRQn);  // 注意：是 TIM1_UP_IRQn

// 刷新影子寄存器
TIM1->CR1 |= TIM_CR1_URS;
TIM1->EGR |= TIM_EGR_UG;
TIM1->SR &= ~TIM_SR_UIF;  // 清除标志
```

**HAL 库方式**：
```c
// CubeMX 配置
// Prescaler: 7199
// Counter Period: 4999
// Repetition Counter: 4
// Pulse: 2500
// CH Polarity: Low（低电平有效）
// OC Idle State: Set（空闲高电平）

// 启动
HAL_TIM_PWM_Start(&htim1, TIM_CHANNEL_1);
__HAL_TIM_CLEAR_FLAG(&htim1, TIM_FLAG_UPDATE);
__HAL_TIM_ENABLE_IT(&htim1, TIM_IT_UPDATE);

// 中断服务函数
void TIM1_UP_IRQHandler(void)
{
    HAL_TIM_PWM_Stop(&htim1, TIM_CHANNEL_1);
}
```

### 11.5 空闲状态配置

**问题**：停止定时器后 LED 保持亮（低电平）

**解决**：
- OC Idle State = Set（空闲高电平）
- CH Polarity = Low（低电平有效）

**效果**：
- PWM 输出时：低电平点亮 LED
- 定时器停止后：空闲高电平 → LED 熄灭

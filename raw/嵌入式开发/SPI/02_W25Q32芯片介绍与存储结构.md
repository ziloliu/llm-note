# SPI 通信 — W25Q32 Flash 存储器 笔记

---

## 一、W25Q32 芯片概述

| 项目 | 说明 |
|------|------|
| 芯片型号 | **W25Q32** |
| 存储类型 | **NOR Flash**（基于或非门） |
| 存储容量 | 32 Mbit = **4 MB** |
| 通信协议 | **SPI** |
| 引脚数量 | **8 个** |
| 支持模式 | SPI **Mode 0** 和 Mode 3 |
| 数据位序 | **高位优先**（MSB First） |
| 页大小 | **256 字节** |
| 擦除最小单位 | **段（Sector）= 4 KB** |

### 1.1 与 EEPROM（M24C02）对比

| 对比项 | M24C02 (EEPROM) | W25Q32 (NOR Flash) |
|--------|:---------------:|:------------------:|
| 存储类型 | EEPROM | NOR Flash |
| 容量 | 256 字节 | **4 MB** |
| 通信协议 | I²C | **SPI** |
| 速度 | 较慢 | **较快** |
| 芯片面积 | 较小 | 较大 |

## 二、W25Q32 引脚定义与接线

### 2.1 引脚定义

| 引脚号 | 名称 | 功能 | 连接 |
|:------:|------|------|------|
| 1 | **CS** | 片选（低电平有效） | **PC13**（GPIO） |
| 2 | **DO (IO1)** | 数据输出（MISO） | PA6（SPI1_MISO） |
| 3 | **WP (IO2)** | 写保护（低有效） | **3.3V**（禁用） |
| 4 | **GND** | 接地 | GND |
| 5 | **DI (IO0)** | 数据输入（MOSI） | PA7（SPI1_MOSI） |
| 6 | **CLK** | 时钟 | PA5（SPI1_SCK） |
| 7 | **HOLD (IO3)** | 暂停/复位（低有效） | **3.3V**（禁用） |
| 8 | **VCC** | 电源 | **3.3V** |

### 2.2 原理图连接

```
STM32F103                    W25Q32
─────────                    ──────
PA5 (SPI1_SCK)  ──────────── CLK   (Pin 6)
PA7 (SPI1_MOSI) ──────────── DI    (Pin 5)
PA6 (SPI1_MISO) ──────────── DO    (Pin 2)
PC13 (GPIO)     ──────────── CS    (Pin 1)  ← 片选信号

                 3.3V ──────── WP    (Pin 3)  ← 写保护禁用
                 3.3V ──────── HOLD  (Pin 7)  ← 暂停功能禁用
                 3.3V ──────── VCC   (Pin 8)
                 GND  ──────── GND   (Pin 4)

注意：
  → CS 连接到 PC13（GPIO），不连到 PA4（SPI1_NSS）
  → 因为 STM32 是主机，NSS 引脚不应被选中为从机
  → 用 GPIO 模拟片选，方便一主多从扩展
```

### 2.3 为什么不连接 NSS（PA4）

```
PA4 是 SPI1 的 NSS 引脚：
  → 如果连到 W25Q32 的 CS → 意味着 STM32 被选为从机（错误！）
  → STM32 是主机，NSS 应拉高（不被选中）
  → 片选从机用 GPIO 引脚即可

一主多从场景：
  → 每个从机的 CS 连接独立的 GPIO 引脚
  → 片选时将对应 GPIO 拉低
  → PA4 的 NSS 直连功能：广播模式（同时选中所有从机）
```

## 三、STM32 与 W25Q32 的 SPI 连接

```
引脚复用关系（SPI1，挂载在 APB2）：
  PA5 → SPI1_SCK  → W25Q32 CLK
  PA6 → SPI1_MISO → W25Q32 DO（从机输出）
  PA7 → SPI1_MOSI → W25Q32 DI（从机输入）
  PC13 → GPIO     → W25Q32 CS（片选）

软件模拟 SPI 时：
  → 不需要开启 SPI1 硬件模块
  → 直接操作 PA5/PA6/PA7/PC13 的 GPIO 即可
```

## 四、W25Q32 存储空间组织

### 4.1 层级结构

```
4 MB 总存储空间
├── Block 0   (64 KB) ─┐
├── Block 1   (64 KB)  │
├── Block 2   (64 KB)  │ 共 64 个 Block
│   ...                │
├── Block 63  (64 KB) ─┘

每个 Block (64 KB)
├── Sector 0  (4 KB)  ─┐
├── Sector 1  (4 KB)   │
├── Sector 2  (4 KB)   │ 共 16 个 Sector
│   ...                │
├── Sector 15 (4 KB)  ─┘

每个 Sector (4 KB)
├── Page 0    (256 B) ─┐
├── Page 1    (256 B)  │
├── Page 2    (256 B)  │ 共 16 个 Page
│   ...               │
├── Page 15   (256 B) ─┘

每个 Page (256 B)
└── 256 个字节地址
```

### 4.2 数量关系

| 层级 | 数量 | 大小 |
|------|:----:|:----:|
| 整个芯片 | 1 | **4 MB** (2^22) |
| Block | **64** (2^6) | 64 KB (2^16) |
| Sector | **16** × 64 = 1024 (2^10) | 4 KB (2^12) |
| Page | **16** × 1024 = 16384 (2^14) | 256 B (2^8) |

### 4.3 24 位地址划分

```
24 位地址（3 字节，6 位十六进制）：

  [A23 A22 A21 A20 A19 A18 A17 A16] [A15 A14 A13 A12] [A11 A10 A9 A8] [A7 A6 A5 A4 A3 A2 A1 A0]
  │←──────── 第 1 字节 ────────→│  │←─ 高4位 ─→│    │←─ 低4位 ─→│  │←──── 第 3 字节 ────→│
  │         Block 号              │   Sector 号    │    Page 号     │     Page 内地址         │
  │      (实际有效位: 0~63)        │   (0~15)       │   (0~15)      │      (0~255)           │

示例地址 0x012345：
  第 1 字节 0x01 = Block 1
  第 2 字节 0x23 → 高4位 0x2 = Sector 2, 低4位 0x3 = Page 3
  第 3 字节 0x45 = Page 内偏移 0x45 = 69

地址范围：
  Block 0:  0x000000 ~ 0x00FFFF
  Block 1:  0x010000 ~ 0x01FFFF
  ...
  Block 63: 0x3F0000 ~ 0x3FFFFF
```

## 五、Flash 读写特性

### 5.1 写入特性

| 特性 | 说明 |
|------|------|
| 写入前 | **必须先擦除**（只能 1→0，不能 0→1） |
| 写使能 | 每次写入前必须发送**写使能指令** |
| 写入最小单位 | **页（256 字节）** |
| 超出页范围 | 地址**回卷**，覆盖同页开头数据 |
| 写入后 | 进入 **Busy 状态**，需等待完成 |
| 多字节写入 | 地址连续递增，自动换行回卷 |

### 5.2 擦除特性

| 特性 | 说明 |
|------|------|
| 擦除后 | 所有位变为 **1**（0xFF） |
| 擦除最小单位 | **段（Sector = 4 KB）** |
| 擦除选项 | 段擦除(4KB) / 块擦除(64KB) / 整片擦除 |
| 擦除后 | 进入 Busy 状态，需等待完成 |

### 5.3 读取特性

| 特性 | 说明 |
|------|------|
| 读取前 | **无需使能**，直接读取 |
| 地址限制 | **无页限制**，可连续读取任意长度 |
| 状态 | 读取**不进入 Busy 状态** |
| 限制 | **Busy 状态下不能读取** |

### 5.4 写入翻转规则

```
Flash 存储位状态变化：
  1 → 0 ✅（可以直接写入）
  0 → 1 ❌（必须先擦除！）

写入流程：
  擦除（全变 1）→ 写入（1 变 0）

例：
  擦除后：1111 1111
  写入 0xA5 = 1010 0101：1111 1111 → 1010 0101 ✅
  再写 0x5A = 0101 1010：1010 0101 → 0000 0000 ❌（0 不能变 1！）
```

### 5.5 页写回卷

```
页大小：256 字节（地址 0x00 ~ 0xFF）

写入从页中间开始，超出范围会回卷：
  起始地址：0x80（页中间）
  写入 256 字节：
    0x80 ~ 0xFF（后半部分）→ 正常写入
    0x00 ~ 0x7F（前半部分）→ 回卷覆盖！

避免回卷：写入字节数 ≤ (256 - 起始偏移)
  例：从 0x80 开始，最多写 128 字节
```

## 六、W25Q32 指令集

### 6.1 常用指令

| 指令名称 | 指令码 | 功能 | 后续数据 |
|---------|:------:|------|---------|
| **Write Enable** | **0x06** | 写使能 | 无 |
| **Page Program** | **0x02** | 页写入 | 3 字节地址 + 数据 |
| **Read Data** | **0x03** | 读取数据 | 3 字节地址 + 读出数据 |
| **Sector Erase** | **0x20** | 段擦除（4KB） | 3 字节地址 |
| **Read Status Register** | **0x05** | 读状态寄存器 | 读出 1 字节状态 |
| **JEDEC ID** | **0x9F** | 读取芯片 ID | 读出 3 字节 |

### 6.2 写使能（0x06）

```
主机发送：0x06（1 字节）
Flash 响应：开启写使能
后续：可以执行写入/擦除操作
```

### 6.3 页写入（0x02）

```
时序：
  ① 主机发送：0x02（指令码，1 字节）
  ② 主机发送：地址（3 字节，24 位）
  ③ 主机发送：数据（1~256 字节）
  
注意：
  → 写入前必须先发送写使能（0x06）
  → 数据长度不超过 256 字节（一页）
  → 超出页范围会回卷覆盖
  → 写入后进入 Busy 状态，需等待
```

### 6.4 读取数据（0x03）

```
时序：
  ① 主机发送：0x03（指令码，1 字节）
  ② 主机发送：地址（3 字节，24 位）
  ③ Flash 返回：数据（任意长度，连续读取）
  
特点：
  → 无页限制，可连续读取
  → 不进入 Busy 状态
  → 但 Busy 状态下不能读取
```

### 6.5 段擦除（0x20）

```
时序：
  ① 主机发送：0x20（指令码，1 字节）
  ② 主机发送：地址（3 字节，24 位，核心是段号）

效果：
  → 指定地址所在的整个 4KB 段被擦除
  → 段内所有字节变为 0xFF
  → 擦除后进入 Busy 状态
```

### 6.6 读状态寄存器（0x05）

```
时序：
  ① 主机发送：0x05（指令码，1 字节）
  ② Flash 返回：状态寄存器（1 字节）

关键位：
  Bit 0 = BUSY
    1 = 正在写入/擦除周期（忙）
    0 = 空闲，可以操作

用途：
  → 写入/擦除后，循环读取状态
  → 等待 BUSY = 0 后再进行下一步操作
```

### 6.7 JEDEC ID（0x9F）

```
时序：
  ① 主机发送：0x9F（指令码，1 字节）
  ② Flash 返回：3 字节
     Byte 1：制造商 ID（Manufacturer ID）
     Byte 2：存储类型（Memory Type）
     Byte 3：容量（Capacity）

用途：
  → 验证 SPI 通信是否正常
  → 确认识别到正确的 Flash 芯片
```

## 七、完整读写流程

### 7.1 写入流程

```
① 片选使能：CS 拉低
② 发送写使能指令：0x06
③ 片选失能：CS 拉高

④ 片选使能：CS 拉低
⑤ 发送页写指令：0x02
⑥ 发送 3 字节地址
⑦ 发送写入数据（1~256 字节）
⑧ 片选失能：CS 拉高

⑨ 等待写入完成：
   a. 片选使能：CS 拉低
   b. 发送读状态指令：0x05
   c. 循环读取状态字节，检查 BUSY 位
   d. BUSY = 0 时退出循环
   e. 片选失能：CS 拉高
```

### 7.2 读取流程

```
① 片选使能：CS 拉低
② 发送读数据指令：0x03
③ 发送 3 字节地址
④ 读取数据（任意长度，连续读取）
⑤ 片选失能：CS 拉高
```

### 7.3 擦除流程

```
① 片选使能：CS 拉低
② 发送写使能指令：0x06
③ 片选失能：CS 拉高

④ 片选使能：CS 拉低
⑤ 发送段擦除指令：0x20
⑥ 发送 3 字节地址（核心是段号）
⑦ 片选失能：CS 拉高

⑧ 等待擦除完成（同写入等待流程）
```

## 八、软件模拟 SPI 方案

```
不使用 SPI1 硬件模块，直接操作 GPIO：

PA5 → 输出（模拟 SCK）   → W25Q32 CLK
PA6 → 输入（模拟 MISO）  ← W25Q32 DO
PA7 → 输出（模拟 MOSI）  → W25Q32 DI
PC13 → 输出（模拟 CS）   → W25Q32 CS

优势：
  → 不受 SPI 硬件引脚限制
  → 可用任意 GPIO 引脚
  → 便于理解底层原理
  → 便于调试

劣势：
  → 速度较慢（软件翻转 GPIO）
  → CPU 占用高
```

## 九、关键概念速查

| 概念 | 说明 |
|------|------|
| W25Q32 | NOR Flash 存储器，4MB，SPI 接口 |
| Block | 块，64KB，共 64 个 |
| Sector | 段，4KB，共 1024 个（擦除最小单位） |
| Page | 页，256B，共 16384 个（写入最小单位） |
| Page Program | 页写指令（0x02），每次写一页 |
| Sector Erase | 段擦除指令（0x20），每次擦 4KB |
| Write Enable | 写使能指令（0x06），写入前必须发送 |
| BUSY | 状态寄存器 Bit 0，忙标志（1=忙，0=空闲） |
| JEDEC ID | 芯片识别指令（0x9F），返回 3 字节 ID |
| CS/SS/NSS | 片选信号（低电平有效） |
| 片选方式 | GPIO 模拟（PC13），不使用 SPI1_NSS（PA4） |
| 页回卷 | 写入超出 256 字节时，地址回卷覆盖开头 |
| 写入翻转 | 只能 1→0，不能 0→1（需先擦除） |

## 十、地址计算速查表

```
完整地址 = Block 号 × 64K + Sector 号 × 4K + Page 号 × 256 + 页内偏移

示例：
  Block 2, Sector 3, Page 5, 偏移 0x10
  = 2 × 0x10000 + 3 × 0x1000 + 5 × 0x100 + 0x10
  = 0x20000 + 0x3000 + 0x500 + 0x10
  = 0x23510

地址范围：
  整个芯片：0x000000 ~ 0x3FFFFF
  Block 0：  0x000000 ~ 0x00FFFF
  Block 63： 0x3F0000 ~ 0x3FFFFF
  Sector 0 of Block 0：0x000000 ~ 0x000FFF
  Page 0 of Sector 0：0x000000 ~ 0x0000FF
```

## 十一、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 写入后数据不对 | 未先擦除（0 不能变 1） | 先执行段擦除 |
| 写入无效 | 未发送写使能（0x06） | 写入前先发 0x06 |
| 数据覆盖 | 超出 256 字节页范围 | 分页写入 |
| 操作失败 | Busy 状态下操作 | 读状态寄存器等 BUSY=0 |
| 读取数据全 0xFF | 地址错误或芯片未连接 | 检查接线和地址 |
| 通信不正常 | SPI Mode 不匹配 | 使用 Mode 0 或 Mode 3 |
| CS 不生效 | CS 接到 PA4（NSS） | CS 接到独立 GPIO |

## 一、SPI Mode 0 数据交换时序

### 1.1 时序图（看图说话）

```
        ┌─────────────────────────────────────────────────────────────────┐
NSS(CS) ┘                           传输过程                            └─
        空闲(高)                    拉低使能                            拉高结束

              ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐
SCK     ──────┘   └───┘   └───┘   └───┘   └───┘   └───┘   └───┘   └───┘
              ↑       ↑       ↑       ↑       ↑       ↑       ↑       ↑
           上升沿  上升沿  上升沿  上升沿  上升沿  上升沿  上升沿  上升沿
           采样    采样    采样    采样    采样    采样    采样    采样

MOSI    ──┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──
    (输出) │ B7   │ B6   │ B5   │ B4   │ B3   │ B2   │ B1   │ B0
          (最高位先行)

MISO    ──┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──
    (输入) │ B7   │ B6   │ B5   │ B4   │ B3   │ B2   │ B1   │ B0
```

### 1.2 每个时钟周期的操作步骤

```
① SCK 低电平：准备数据（MOSI 输出，MISO 从设备已准备好）
② SCK 上升沿：数据有效，双方同时采样（读取数据）
③ SCK 高电平：保持（延迟半个周期）
④ SCK 下降沿：无特殊操作
⑤ SCK 低电平：准备下一位数据

每次循环 = 交换 1 位数据
8 次循环 = 交换 1 字节数据
```

## 三、SPI.h — 头文件实现

### 3.1 宏定义（引脚操作）

```c
#ifndef __SPI_H
#define __SPI_H

#include "stm32f10x.h"
#include "delay.h"

/* ========================================
 *  引脚接线定义
 *  CS   → PC13（GPIO 输出，片选）
 *  SCK  → PA5 （GPIO 输出，时钟）
 *  MOSI → PA7 （GPIO 输出，主出从入）
 *  MISO → PA6 （GPIO 输入，主入从出）
 * ======================================== */

/* 片选信号（PC13） */
#define CS_HIGH()    GPIOC->ODR |= GPIO_ODR_ODR13
#define CS_LOW()     GPIOC->ODR &= ~GPIO_ODR_ODR13

/* 时钟信号（PA5） */
#define SCK_HIGH()   GPIOA->ODR |= GPIO_ODR_ODR5
#define SCK_LOW()    GPIOA->ODR &= ~GPIO_ODR_ODR5

/* 主出从入（PA7） */
#define MOSI_HIGH()  GPIOA->ODR |= GPIO_ODR_ODR7
#define MOSI_LOW()   GPIOA->ODR &= ~GPIO_ODR_ODR7

/* 主入从出（PA6）— 读取输入 */
#define MISO_READ()  (GPIOA->IDR & GPIO_IDR_IDR6)

/* SPI 延迟（半个时钟周期） */
#define SPI_DELAY()  Delay_us(5)

/* 函数声明 */
void SPI_Init(void);
void SPI_Start(void);
void SPI_Stop(void);
uint8_t SPI_SwapByte(uint8_t byte);

#endif
```

### 3.2 引脚接线速查

| 信号 | STM32 引脚 | 方向 | 操作 |
|------|:----------:|:----:|------|
| CS | **PC13** | 输出 | ODR 置位/清零 |
| SCK | **PA5** | 输出 | ODR 置位/清零 |
| MOSI | **PA7** | 输出 | ODR 置位/清零 |
| MISO | **PA6** | **输入** | **IDR 读取** |

## 四、SPI.c — 底层通信实现

### 4.1 初始化函数

```c
#include "SPI.h"

void SPI_Init(void)
{
    /* 1. 开启 GPIO 时钟 */
    RCC->APB2ENR |= RCC_APB2ENR_IOPAEN;  // GPIOA 时钟
    RCC->APB2ENR |= RCC_APB2ENR_IOPCEN;  // GPIOC 时钟

    /* 2. 配置 GPIO 工作模式 */

    /* PC13 → 通用推挽输出（片选 CS） */
    GPIOC->CRH |= GPIO_CRH_MODE13;       // MODE = 11（50MHz 输出）
    GPIOC->CRH &= ~GPIO_CRH_CNF13;       // CNF = 00（推挽输出）

    /* PA5 → 通用推挽输出（时钟 SCK） */
    GPIOA->CRL |= GPIO_CRL_MODE5;        // MODE = 11
    GPIOA->CRL &= ~GPIO_CRL_CNF5;        // CNF = 00

    /* PA7 → 通用推挽输出（数据 MOSI） */
    GPIOA->CRL |= GPIO_CRL_MODE7;        // MODE = 11
    GPIOA->CRL &= ~GPIO_CRL_CNF7;        // CNF = 00

    /* PA6 → 浮空输入（数据 MISO） */
    GPIOA->CRL &= ~GPIO_CRL_MODE6;       // MODE = 00（输入）
    GPIOA->CRL = (GPIOA->CRL & ~GPIO_CRL_CNF6) | GPIO_CRL_CNF6_0;
                                          // CNF = 01（浮空输入）

    /* 3. 设置 SPI Mode 0 初始状态 */
    SCK_LOW();                            // 时钟空闲为低电平
    CS_HIGH();                            // 片选默认不使能
    SPI_DELAY();                          // 保持延迟
}
```

### 4.2 片选控制函数

```c
void SPI_Start(void)
{
    CS_LOW();             // 拉低片选 → 选中从设备 → 开始传输
}

void SPI_Stop(void)
{
    CS_HIGH();            // 拉高片选 → 取消选中 → 结束传输
}
```

### 4.3 数据交换函数（核心！）

```c
/**
 * @brief  交换一个字节（全双工：同时发送和接收）
 * @param  byte: 要发送的字节
 * @retval 接收到的字节
 *
 * 时序依据：SPI Mode 0 时序图
 *   ① SCK 低电平：准备发送数据到 MOSI
 *   ② SCK 上升沿：双方同时采样（读取数据）
 *   ③ SCK 高电平：保持
 *   ④ SCK 下降沿：为下一位做准备
 */
uint8_t SPI_SwapByte(uint8_t byte)
{
    uint8_t rByte = 0x00;  // 保存接收到的字节

    for (int i = 0; i < 8; i++)
    {
        /* === 第一步：准备发送数据（SCK 低电平时）=== */
        // 判断当前字节的最高位，输出到 MOSI
        if (byte & 0x80)         // 最高位为 1
        {
            MOSI_HIGH();         // MOSI 拉高
        }
        else                      // 最高位为 0
        {
            MOSI_LOW();          // MOSI 拉低
        }
        byte <<= 1;              // 左移，准备下一位

        /* === 第二步：拉高时钟（上升沿，双方采样）=== */
        SCK_HIGH();
        SPI_DELAY();

        /* === 第三步：在 MISO 上采样从设备数据 === */
        rByte <<= 1;             // 先左移（给最低位腾位置）
        if (MISO_READ())         // 读取 MISO 引脚
        {
            rByte |= 0x01;      // 最高位置 1（写入最低位）
        }
        // else: 最低位已经是 0，无需操作

        /* === 第四步：拉低时钟（为下一次传输做准备）=== */
        SCK_LOW();
        SPI_DELAY();
    }

    return rByte;                 // 返回接收到的字节
}
```

### 4.4 数据交换步骤详解

```
循环 8 次，每次交换 1 位：

┌─────────────────────────────────────────────────────────────────┐
│  步骤① 准备发送（SCK 低电平时）                                   │
│    → 取 byte 的最高位（byte & 0x80）                              │
│    → 最高位=1：MOSI_HIGH()                                       │
│    → 最高位=0：MOSI_LOW()                                        │
│    → byte <<= 1（左移，为下一位做准备）                            │
├─────────────────────────────────────────────────────────────────┤
│  步骤② 拉高时钟（上升沿，采样时刻）                                │
│    → SCK_HIGH()                                                  │
│    → SPI_DELAY()（保持半个周期）                                   │
│    → 此时：主机发送的数据被从机读取                                  │
│    → 此时：从机发送的数据被主机读取                                  │
├─────────────────────────────────────────────────────────────────┤
│  步骤③ 读取接收数据                                               │
│    → rByte <<= 1（先左移，给最低位腾位置）                          │
│    → if (MISO_READ()) rByte |= 0x01                              │
│    → 将读到的位写入 rByte 的最低位                                 │
├─────────────────────────────────────────────────────────────────┤
│  步骤④ 拉低时钟（为下一次传输做准备）                               │
│    → SCK_LOW()                                                    │
│    → SPI_DELAY()（保持半个周期）                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 4.5 接收数据的移位技巧

```
问题：为什么先左移再写入最低位？

错误做法（先写入再左移）：
  第 1 次：rByte = 0000 0001  → 左移 → 0000 0010（最后一位被移出！）
  第 2 次：rByte = 0000 001x  → 左移 → 0000 0x_0（每次丢失数据）
  ...最终少一位

正确做法（先左移再写入）：
  第 1 次：rByte = 0000 0000  → 左移 → 0000 0000  → 写入 → 0000 000x
  第 2 次：rByte = 0000 000x  → 左移 → 0000 00x0  → 写入 → 0000 00xy
  ...
  第 8 次：rByte = 0xxxxxxx  → 左移 → xxxxxxx0  → 写入 → xxxxxxxx
  → 第 8 次写入后不再左移，数据完整！

rByte 初始值 = 0x00：
  → 第一次左移无影响（0 左移还是 0）
  → 前 7 次左移，第 8 次写入后直接返回
```

## 五、完整调用流程

```
主设备（STM32）操作流程：

① SPI_Init()                     → 初始化 GPIO 和 SPI 引脚
② SPI_Start()                    → CS 拉低，选中从设备
③ SPI_SwapByte(指令码)            → 发送指令（如 0x03 读数据）
④ SPI_SwapByte(地址高字节)         → 发送 24 位地址（3 字节）
⑤ SPI_SwapByte(地址中字节)
⑥ SPI_SwapByte(地址低字节)
⑦ data = SPI_SwapByte(0xFF)      → 接收数据（发 0xFF 是占位，只关心接收）
⑧ SPI_Stop()                     → CS 拉高，结束传输

注意：
  → 每次 SwapByte 都是全双工（同时收发）
  → 发送时：关心发送的数据，忽略返回值
  → 接收时：发送任意数据（如 0xFF），关心返回值
```

## 六、软件模拟 SPI vs 硬件 SPI

| 对比项 | 软件模拟 | 硬件 SPI |
|--------|---------|---------|
| 速度 | 较慢（GPIO 翻转） | 较快（硬件自动） |
| CPU 占用 | 高（CPU 逐位控制） | 低（配合 DMA） |
| 引脚灵活性 | **任意 GPIO** | 固定复用引脚 |
| 实现难度 | 简单（看时序图） | 需配置寄存器 |
| 学习价值 | **高（理解底层）** | 低（黑盒使用） |
| 调试方便 | **方便** | 较困难 |

| PA6 | CRL | MODE6 = 00 | CNF6 = 01 | 浮空输入 |

### 7.2 浮空输入配置（PA6）

```c
// 清零 MODE6 和 CNF6
GPIOA->CRL &= ~GPIO_CRL_MODE6;   // MODE = 00
GPIOA->CRL &= ~GPIO_CRL_CNF6;    // CNF 先清零
GPIOA->CRL |= GPIO_CRL_CNF6_0;   // CNF = 01（浮空输入）

// 或合并写法：
GPIOA->CRL = (GPIOA->CRL & ~(GPIO_CRL_MODE6 | GPIO_CRL_CNF6))
             | GPIO_CRL_CNF6_0;
```

### 7.3 最高位判断技巧

```c
// 判断最高位
if (byte & 0x80)    // 0x80 = 1000 0000，取最高位
{
    MOSI_HIGH();
}
else
{
    MOSI_LOW();
}
byte <<= 1;         // 左移，下次判断新的最高位
```

## 九、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 数据错位 | 移位顺序错误 | 先左移再写入最低位 |
| 最高位丢失 | 先写入再左移 | 改为先左移再写入 |
| 从机无响应 | CS 未拉低 | 调用 SPI_Start() |
| 数据全部为 0 | SPI Mode 不匹配 | 确认使用 Mode 0 |
| 通信不稳定 | 延迟太短 | 增大 SPI_DELAY 值 |
| MISO 读取异常 | PA6 配置错误 | 确认浮空输入（CNF=01） |
| 全是 0xFF | 地址错误或芯片未连接 | 检查接线和地址 |

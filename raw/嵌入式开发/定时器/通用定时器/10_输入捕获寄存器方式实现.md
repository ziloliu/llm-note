## 四、GPIO 配置

### 4.1 PB6 配置

```
复用浮空输入：
  CNF = 10（复用功能）
  MODE = 00（输入）

GPIOB->CRL 配置：
  MODE6[1:0] = 00（输入模式）
  CNF6[1:0]  = 10（复用功能）

⚠️ 与之前 PWM 输出（PA1）的区别：
  PA1 输出：CNF=10, MODE=11（复用推挽输出，50MHz）
  PB6 输入：CNF=10, MODE=00（复用浮空输入）
```

### 4.2 GPIO 模式对比

| 场景 | CNF | MODE | 说明 |
|------|:---:|:----:|------|
| 模拟输入 | 00 | 00 | ADC 用 |
| **浮空输入** | **01** | 00 | 通用浮空输入 |
| 上拉/下拉输入 | 10 | 00 | 通用上拉/下拉 |
| **复用浮空输入** | **10** | 00 | **本实验**，定时器捕获 |
| 复用推挽输出 | 10 | 11 | PWM 输出时用 |
| 通用推挽输出 | 00 | 11 | LED 控制 |

## 六、各步骤代码

### 6.1 步骤①：CR2.TIS — TI1 输入选择

```c
// CR2.TIS = 0：TI1 直通（通道 1 输入直接连到 TI1）
// CR2.TIS = 1：TI1/TI2/TI3 异或后连接（霍尔传感器用）
TIM4->CR2 &= ~TIM_CR2_TI1S;    // TIS = 0
```

> CR2 之前很少用到，这里第一次出现。
> 只有通道 1 有多路选择器（直通 or 异或）。

### 6.2 步骤②：CCMR1.IC1F — 输入滤波器

```c
// 信号质量好，不滤波
TIM4->CCMR1 &= ~TIM_CCMR1_IC1F;    // IC1F = 0000
```

### 6.3 步骤③：CCER.CC1P — 边沿极性

```c
// 上升沿捕获
TIM4->CCER &= ~TIM_CCER_CC1P;    // CC1P = 0
```

### 6.4 步骤④：CCMR1.CC1S — 输入映射（关键）

```c
// CC1S = 01：IC1 映射到 TI1（直通）
TIM4->CCMR1 |=  TIM_CCMR1_CC1S_0;    // bit0 = 1
TIM4->CCMR1 &= ~TIM_CCMR1_CC1S_1;    // bit1 = 0
```

### 6.5 步骤⑤：CCMR1.IC1PSC — 输入预分频

```c
// 不分频
TIM4->CCMR1 &= ~TIM_CCMR1_IC1PSC;    // IC1PSC = 00
```

### 6.6 步骤⑥：CCER.CC1E — 捕获使能

```c
TIM4->CCER |= TIM_CCER_CC1E;    // CC1E = 1
```

### 6.7 步骤⑦：DIER.CC1IE — 中断使能

```c
TIM4->DIER |= TIM_DIER_CC1IE;    // CC1IE = 1
```

## 七、完整初始化代码

```c
volatile uint32_t capture_value = 0;
volatile uint8_t  capture_flag  = 0;
static uint32_t   last_capture  = 0;

void TIM4_Init(void)
{
    /* ========== 时基部分 ========== */

    /* ① 开启时钟 */
    RCC->APB1ENR |= RCC_APB1ENR_TIM4EN;     // TIM4 时钟
    RCC->APB2ENR |= RCC_APB2ENR_IOPBEN;     // GPIOB 时钟

    /* ② GPIO：PB6 复用浮空输入 */
    GPIOB->CRL &= ~GPIO_CRL_MODE6;          // MODE = 00（输入）
    GPIOB->CRL |=  GPIO_CRL_CNF6_1;         // CNF = 10（复用功能）
    GPIOB->CRL &= ~GPIO_CRL_CNF6_0;         //

    /* ③ PSC：72 分频 → 1MHz（1μs/次） */
    TIM4->PSC = 71;

    /* ④ ARR：最大值，避免溢出 */
    TIM4->ARR = 65535;

    /* ========== 输入通道部分（看图说话） ========== */

    /* ① CR2.TIS = 0：TI1 直通 */
    TIM4->CR2 &= ~TIM_CR2_TI1S;

    /* ② CCMR1.IC1F = 0000：不滤波 */
    TIM4->CCMR1 &= ~TIM_CCMR1_IC1F;

    /* ③ CCER.CC1P = 0：上升沿捕获 */
    TIM4->CCER &= ~TIM_CCER_CC1P;

    /* ④ CCMR1.CC1S = 01：IC1 映射到 TI1（直通） */
    TIM4->CCMR1 |=  TIM_CCMR1_CC1S_0;
    TIM4->CCMR1 &= ~TIM_CCMR1_CC1S_1;

    /* ⑤ CCMR1.IC1PSC = 00：不分频 */
    TIM4->CCMR1 &= ~TIM_CCMR1_IC1PSC;

    /* ⑥ CCER.CC1E = 1：使能捕获 */
    TIM4->CCER |= TIM_CCER_CC1E;

    /* ⑦ DIER.CC1IE = 1：使能捕获中断 */
    TIM4->DIER |= TIM_DIER_CC1IE;

    /* NVIC 配置 */
    NVIC_SetPriorityGrouping(3);
    NVIC_SetPriority(TIM4_IRQn, 2);
    NVIC_EnableIRQ(TIM4_IRQn);
}
```

## 八、两种配置思路

### 8.1 按信号路径顺序（看图说话，推荐）

```
沿着信号通路从左到右：
  CR2.TIS → CCMR1.IC1F → CCER.CC1P → CCMR1.CC1S → CCMR1.IC1PSC → CCER.CC1E → DIER.CC1IE

优点：逻辑清晰，不容易遗漏
```

### 8.2 按寄存器分组

```
CR2：  TIS（一般不用配，默认 0）
CCMR1：CC1S（必须）+ IC1F（可选）+ IC1PSC（可选）
CCER： CC1P（必须）+ CC1E（必须）
DIER： CC1IE（用中断时必须）
NVIC： 优先级 + 使能

优点：适合对寄存器已经非常熟悉的开发者
```

### 8.3 CCMR1 中三个配置项的必要性

| 配置项 | 位 | 必须？ | 说明 |
|--------|:--:|:------:|------|
| **CC1S**（输入/输出 + 映射） | bit[1:0] | **必须** | 决定输入还是输出，以及映射关系 |
| IC1F（滤波器） | bit[7:4] | 可选 | 信号质量好时可不配 |
| IC1PSC（预分频） | bit[3:2] | 可选 | 频率合适时可不配 |

> 实际上最小配置只需要 CCMR1（CC1S）+ CCER（CC1P + CC1E）三个地方。

## 十、中断服务函数

```c
void TIM4_IRQHandler(void)
{
    if (TIM4->SR & TIM_SR_CC1IF)
    {
        uint32_t current = TIM4->CCR1;    // 读取捕获值

        if (last_capture != 0)
        {
            if (current >= last_capture)
                capture_value = current - last_capture;
            else
                capture_value = 65536 - last_capture + current;

            capture_flag = 1;
        }

        last_capture = current;
        // 读 CCR1 时硬件自动清除 CC1IF 标志
    }
}
```

> cycle 最大值为 65535（ARR = 65535），不会溢出。

## 二、输入捕获中断服务函数

### 2.1 完整代码

```c
void TIM4_IRQHandler(void)
{
    // 判断是否是通道 1 的捕获中断
    if (TIM4->SR & TIM_SR_CC1IF)
    {
        // ① 清除中断标志位
        TIM4->SR &= ~TIM_SR_CC1IF;

        // ② 上升沿个数加一
        count++;

        // ③ 判断是第几次上升沿
        if (count == 1)
        {
            // 第一次上升沿：清零计数器，从零开始计数
            TIM4->CNT = 0;
        }
        else if (count == 2)
        {
            // 第二次上升沿：读取 CCR1 中的值作为周期
            cycle = TIM4->CCR1;

            // 清零 count，准备下一次测量
            count = 0;
        }
    }
}
```

### 2.2 中断处理流程

```
第 1 个上升沿到来
    ↓
进入中断 → 清除标志位 → count = 1
    ↓
count == 1 → CNT = 0（从零开始计数）
    ↓
    ··· 计数器递增 ···
    ↓
第 2 个上升沿到来
    ↓
进入中断 → 清除标志位 → count = 2
    ↓
count == 2 → cycle = CCR1（捕获周期值）
           → count = 0（准备下一次测量）
    ↓
    循环往复，持续更新 cycle 值
```

### 2.3 关键操作说明

| 步骤 | 操作 | 寄存器 | 说明 |
|:----:|------|--------|------|
| ① | 清除中断标志 | `SR &= ~CC1IF` | **必须手动清除**，否则反复进入中断 |
| ② | 计数加一 | `count++` | 记录上升沿次数 |
| ③第1次 | 清零计数器 | `CNT = 0` | 第一个上升沿到来时，计数从零开始 |
| ③第2次 | 读取捕获值 | `cycle = CCR1` | 第二个上升沿到来时，读取 CCR1 = 周期（μs） |
| ③第2次 | 重置计数 | `count = 0` | 准备下一次测量 |

### 2.4 为什么清零标志位必须在最前面

```
不清除标志位的后果：
  → 中断返回后，SR.CC1IF 仍为 1
  → 硬件再次触发中断
  → 反复进出中断，程序卡死

正确做法：
  → 进入中断后第一件事：清除标志位
  → 寄存器方式没有 HAL 库自动清除，必须手动操作
```

## 三、获取周期和频率函数

### 3.1 获取周期

```c
/**
 * @brief  获取输入信号的周期
 * @retval 周期值（单位：ms）
 */
double TIM4_GetPWMPeriod(void)
{
    // cycle 单位为 μs，转换为 ms
    return (double)cycle / 1000.0;
}
```

```
cycle 单位：μs（微秒）
返回值单位：ms（毫秒）
转换：周期(ms) = cycle(μs) / 1000.0
```

### 3.2 获取频率

```c
/**
 * @brief  获取输入信号的频率
 * @retval 频率值（单位：Hz）
 */
double TIM4_GetPWMFrequency(void)
{
    if (cycle == 0)
        return 0;
    // f = 1,000,000 / cycle（μs）
    return 1000000.0 / (double)cycle;
}
```

```
cycle 单位：μs（微秒）
频率单位：Hz（赫兹）
计算：f = 1,000,000 / cycle

推导过程：
  周期 T = cycle μs = cycle × 10⁻⁶ s
  频率 f = 1/T = 1/(cycle × 10⁻⁶) = 10⁶/cycle = 1,000,000/cycle Hz

⚠️ 不能用 1000.0/cycle（得到的是 kHz 不是 Hz）
```

### 3.3 单位换算汇总

| 量 | 原始值 | 单位 | 换算公式 | 目标单位 |
|----|--------|------|----------|----------|
| 周期 | cycle | μs | cycle / 1000.0 | **ms** |
| 频率 | cycle | μs | 1000000.0 / cycle | **Hz** |

### 3.4 常见单位错误

```
❌ 错误写法：
  频率 = 1000.0 / cycle
  → 这算出来的是 kHz，不是 Hz

✅ 正确写法：
  频率 = 1000000.0 / cycle
  → cycle 是微秒数，一百万除以微秒数 = 赫兹数

示例：
  cycle = 10000（μs）= 10ms
  频率 = 1000000.0 / 10000 = 100 Hz ✅
  频率 = 1000.0 / 10000 = 0.1 kHz ❌（数值虽对，但单位不是 Hz）
```

## 四、主函数实现

### 4.1 完整代码

## 十、输入捕获测量精度分析

### 10.1 精度来源

| 因素 | 影响 | 说明 |
|------|------|------|
| 计数器时钟 | **1μs 精度** | PSC=71，72MHz/72=1MHz |
| 中断延迟 | ~几 μs | 中断响应和处理需要时间 |
| 浮点运算 | 小数位精度损失 | double 精度有限 |
| 时钟同步 | ±1 计数 | CNT 清零和上升沿不完全同步 |

### 10.2 提高精度的方法

```
① 减小 PSC → 提高计数频率
   PSC=71  → 1MHz   → 1μs 精度
   PSC=7   → 10MHz  → 0.1μs 精度（但最大可测周期缩小）
   PSC=0   → 72MHz  → ~14ns 精度（但溢出范围仅 ~932μs）

② 测量多个周期取平均
   → 测 N 个周期后除以 N

③ 使用硬件捕获（本实验已使用）
   → 硬件自动锁存 CNT，精度优于软件轮询
```

## 十二、代码汇总

### 12.1 tim4.c 完整代码

```c
#include "tim4.h"

volatile uint16_t cycle = 0;
volatile uint8_t  count = 0;

void TIM4_Init(void)
{
    /* 时钟开启 */
    RCC->APB1ENR |= RCC_APB1ENR_TIM4EN;
    RCC->APB2ENR |= RCC_APB2ENR_IOPBEN;

    /* PB6 复用浮空输入 */
    GPIOB->CRL &= ~GPIO_CRL_MODE6;
    GPIOB->CRL |=  GPIO_CRL_CNF6_1;
    GPIOB->CRL &= ~GPIO_CRL_CNF6_0;

    /* 时基 */
    TIM4->PSC = 71;
    TIM4->ARR = 65535;

    /* 输入通道（看图说话） */
    TIM4->CR2   &= ~TIM_CR2_TI1S;           // ① TI1 直通
    TIM4->CCMR1 &= ~TIM_CCMR1_IC1F;          // ② 不滤波
    TIM4->CCER  &= ~TIM_CCER_CC1P;            // ③ 上升沿
    TIM4->CCMR1 |=  TIM_CCMR1_CC1S_0;        // ④ CC1S=01
    TIM4->CCMR1 &= ~TIM_CCMR1_CC1S_1;
    TIM4->CCMR1 &= ~TIM_CCMR1_IC1PSC;        // ⑤ 不分频
    TIM4->CCER  |=  TIM_CCER_CC1E;            // ⑥ 使能捕获
    TIM4->DIER  |=  TIM_DIER_CC1IE;           // ⑦ 使能中断

    /* NVIC */
    NVIC_SetPriorityGrouping(3);
    NVIC_SetPriority(TIM4_IRQn, 2);
    NVIC_EnableIRQ(TIM4_IRQn);
}

void TIM4_Start(void)
{
    TIM4->CR1 |= TIM_CR1_CEN;
}

void TIM4_Stop(void)
{
    TIM4->CR1 &= ~TIM_CR1_CEN;
}

double TIM4_GetPWMPeriod(void)
{
    return (double)cycle / 1000.0;           // μs → ms
}

double TIM4_GetPWMFrequency(void)
{
    if (cycle == 0)
        return 0;
    return 1000000.0 / (double)cycle;        // μs → Hz
}

void TIM4_IRQHandler(void)
{
    if (TIM4->SR & TIM_SR_CC1IF)
    {
        TIM4->SR &= ~TIM_SR_CC1IF;          // 清除标志
        count++;

        if (count == 1)
        {
            TIM4->CNT = 0;                   // 第1上升沿：清零计数
        }
        else if (count == 2)
        {
            cycle = TIM4->CCR1;              // 第2上升沿：读取周期
            count = 0;                        // 重置计数
        }
    }
}
```

## 十三、进阶思考：能否直接读 CCR 而不用中断？

```
可以，但有两个问题：

① 不知道何时读取
  → 没有中断，不知道上升沿何时到来
  → 必须在主循环中不断轮询 SR.CC1IF
  → 效率低，且可能错过

② 需要两次读取
  → 第1次上升沿后读 CCR1（但此时值无意义）
  → 第2次上升沿后再读 CCR1 才是周期
  → 轮询方式难以精确控制时机

→ 中断方式更可靠、更高效

## 二、直接用 CCR1 出错的原因

### 2.1 问题现象

```c
// 直接返回 CCR1 的值
double TIM4_GetPWMPeriod(void)
{
    return (double)TIM4->CCR1 / 1000.0;
}

// 运行结果：周期在 10ms 和 20ms 之间交替变化！
```

### 2.2 根本原因：CNT 未及时清零

```
PWM 周期 = 10ms，计数频率 = 1μs/次

时间线：

第 1 个上升沿：
  count: 0 → 1
  中断处理：CNT = 0（清零）
  开始计数：0, 1, 2, ... 9999

第 2 个上升沿：
  ↓ 硬件事件：CCR1 = CNT = 10000  ← 捕获正确值
  ↓ 中断处理：count: 1 → 2 → CCR1 读取 → count = 0
  ↓ 中断处理：CNT = 0（清零）

第 3 个上升沿：
  ↓ 关键问题：从第2次清零到第3次上升沿，CNT 又从 0 计到了 10000
  ↓ 但是！第2次中断清零是在第2次上升沿之后才执行的
  ↓ 在清零之前，CNT 继续从 10000 往上计数（不清零！）
  ↓ 到第3个上升沿时，CNT = 20000
  ↓ 硬件事件：CCR1 = CNT = 20000  ← 捕获到错误值 20000！
  ↓ 中断处理：count: 0 → 1 → CNT = 0（清零）

结果：CCR1 在 10000 和 20000 之间交替
     → 周期在 10ms 和 20ms 之间变化
```

### 2.3 时序图

```
信号:  ┌────┐         ┌────┐         ┌────┐         ┌────┐
       │    │         │    │         │    │         │    │
  ─────┘    └─────────┘    └─────────┘    └─────────┘    └───
       ↑         ↑              ↑         ↑
      ①上升沿   ②上升沿       ③上升沿   ④上升沿

原始方案(count)：
  ①：count=1, CNT=0（清零）
  ②：CCR1=10000（捕获）, count=2→0, CNT=0（清零）
  ③：CCR1=10000（捕获）, count=2→0, CNT=0（清零）← 正确 ✅

直接用 CCR1 方案(count 简化)：
  ①：count=1, CNT=0（清零）
  ②：CCR1=10000（捕获）, count=2→0
  ③：count=1, 但此时 CNT=20000！
      CCR1=20000（捕获到 20000！）← 错误 ❌
      然后才 CNT=0（清零）
```

## 三、优化方案

### 3.1 核心思想

```
不需要 count 变量！
不需要 cycle 变量！

每次上升沿到来时：
  ① 硬件自动将 CNT 值捕获到 CCR1（此时 CCR1 = 周期值，如 10000）
  ② 进入中断：只需将 CNT 清零即可

为什么？
  → 硬件捕获在先（事件响应，瞬间完成）
  → 中断处理在后（软件响应，延迟若干时钟周期）
  → CCR1 已经捕获到正确值后再清零 CNT
  → 下一个周期 CNT 从 0 开始，CCR1 保持上次值不变
```

### 3.2 优化后代码

```c
// 不需要任何全局变量！

void TIM4_IRQHandler(void)
{
    if (TIM4->SR & TIM_SR_CC1IF)
    {
        TIM4->SR &= ~TIM_SR_CC1IF;    // 清除中断标志
        TIM4->CNT = 0;                  // 清零计数器
    }
}

double TIM4_GetPWMPeriod(void)
{
    return (double)TIM4->CCR1 / 1000.0;   // 直接读 CCR1
}

double TIM4_GetPWMFrequency(void)
{
    if (TIM4->CCR1 == 0)
        return 0;
    return 1000000.0 / (double)TIM4->CCR1;
}
```

### 3.3 优化后执行流程

```
信号:  ┌────┐         ┌────┐         ┌────┐
       │    │         │    │         │    │
  ─────┘    └─────────┘    └─────────┘    └─────
       ↑         ↑              ↑
      ①上升沿   ②上升沿       ③上升沿

① 上升沿：
  硬件：CCR1 = CNT（捕获，此时无意义）
  中断：CNT = 0（清零）

② 上升沿：
  硬件：CCR1 = CNT = 10000（捕获正确周期）← 硬件先完成
  中断：CNT = 0（清零）                     ← 软件后执行
  → CCR1 保持 10000

③ 上升沿：
  硬件：CCR1 = CNT = 10000（再次捕获，值不变）✅
  中断：CNT = 0（清零）
  → CCR1 保持 10000

主循环读取：CCR1 = 10000 → 周期 = 10ms，频率 = 100Hz ✅
```

## 四、事件 vs 中断 — 核心机制

### 4.1 两条信号路径

```
输入信号 → 滤波 → 边沿检测 → 产生标志
                                │
                ┌───────────────┼───────────────┐
                ↓                               ↓
        ┌──────────────┐                ┌──────────────┐
        │ 事件路径       │                │ 中断路径       │
        │ (硬件处理)     │                │ (软件处理)     │
        │              │                │              │
        │ 事件屏蔽寄存器 │                │ 中断屏蔽寄存器 │
        │   & 事件标志  │                │   & 中断标志   │
        │      ↓       │                │      ↓       │
        │ 硬件电路直接   │                │ → NVIC       │
        │ 执行对应操作   │                │ → CPU        │
        │              │                │ → 中断向量表   │
        │ CCR1 = CNT   │                │ → 中断服务函数 │
        └──────────────┘                └──────────────┘
                ↓                               ↓
          瞬间完成                         若干时钟周期后
         (硬件自动)                        (软件执行)
```

### 4.2 关键区别

| 对比项 | 事件处理 | 中断处理 |
|--------|:--------:|:--------:|
| 处理方式 | **硬件**自动完成 | **软件**（CPU 执行指令） |
| 执行速度 | **瞬间**（1 个时钟周期） | **延迟**（若干时钟周期） |
| 响应对象 | 直接操作寄存器（如 CCR1 = CNT） | 调用中断服务函数 |
| 是否可控 | **不可控**（由硬件电路决定） | **可控**（我们编写代码） |
| 编程方式 | 通过事件屏蔽寄存器使能/禁用 | 通过中断使能寄存器 + NVIC |

### 4.3 时序关系

```
上升沿到来
    │
    ├──→ 硬件事件：CCR1 = CNT（瞬间完成）
    │         ↑
    │    第 1 个时钟周期就完成了
    │
    └──→ 中断请求：标志位置位
              │
              ├── NVIC 仲裁
              ├── CPU 响应
              ├── 保存现场
              ├── 跳转中断服务函数
              ├── 执行代码：CNT = 0
              └── 恢复现场
                   ↑
              可能需要 12~几十个时钟周期
```

### 4.4 为什么这个顺序至关重要

```
正确的顺序（本实验）：
  先：硬件捕获 CCR1 = CNT = 10000（保存正确值）
  后：软件清零 CNT = 0（准备下一次计数）
  → CCR1 保持 10000 ✅

假设反过来（如果可能）：
  先：软件清零 CNT = 0
  后：硬件捕获 CCR1 = CNT = 0（捕获到错误值！）
  → CCR1 = 0 ❌

结论：事件处理（硬件）永远快于中断处理（软件）
     这是硬件设计保证的，不会改变
```

void TIM4_Init(void)
{
    /* 时钟开启 */
    RCC->APB1ENR |= RCC_APB1ENR_TIM4EN;
    RCC->APB2ENR |= RCC_APB2ENR_IOPBEN;

    /* PB6 复用浮空输入 */
    GPIOB->CRL &= ~GPIO_CRL_MODE6;
    GPIOB->CRL |=  GPIO_CRL_CNF6_1;
    GPIOB->CRL &= ~GPIO_CRL_CNF6_0;

    /* 时基 */
    TIM4->PSC = 71;           // 72分频 → 1MHz → 1μs/次
    TIM4->ARR = 65535;        // 最大值

    /* 输入通道（看图说话） */
    TIM4->CR2   &= ~TIM_CR2_TI1S;        // ① TI1 直通
    TIM4->CCMR1 &= ~TIM_CCMR1_IC1F;      // ② 不滤波
    TIM4->CCER  &= ~TIM_CCER_CC1P;        // ③ 上升沿
    TIM4->CCMR1 |=  TIM_CCMR1_CC1S_0;    // ④ CC1S=01
    TIM4->CCMR1 &= ~TIM_CCMR1_CC1S_1;
    TIM4->CCMR1 &= ~TIM_CCMR1_IC1PSC;    // ⑤ 不分频
    TIM4->CCER  |=  TIM_CCER_CC1E;        // ⑥ 使能捕获
    TIM4->DIER  |=  TIM_DIER_CC1IE;       // ⑦ 使能中断

    /* NVIC */
    NVIC_SetPriorityGrouping(3);
    NVIC_SetPriority(TIM4_IRQn, 2);
    NVIC_EnableIRQ(TIM4_IRQn);
}

void TIM4_Start(void)
{
    TIM4->CR1 |= TIM_CR1_CEN;
}

void TIM4_Stop(void)
{
    TIM4->CR1 &= ~TIM_CR1_CEN;
}

double TIM4_GetPWMPeriod(void)
{
    return (double)TIM4->CCR1 / 1000.0;
}

double TIM4_GetPWMFrequency(void)
{
    if (TIM4->CCR1 == 0)
        return 0;
    return 1000000.0 / (double)TIM4->CCR1;
}

/* 中断服务函数 — 极简版 */
void TIM4_IRQHandler(void)
{
    if (TIM4->SR & TIM_SR_CC1IF)
    {
        TIM4->SR &= ~TIM_SR_CC1IF;    // 清除标志
        TIM4->CNT = 0;                  // 清零计数器
    }
}

## 七、为什么原始方案中 cycle 是必要的

```
原始方案：中断中先判断 count 再决定操作

  count=1 时：清零 CNT（此时 CCR1 可能是上次的两倍值 20000）
  count=2 时：读取 CCR1（此时 CCR1 是正确的 10000）

  如果直接读 CCR1 不用 cycle：
  → 在 count=1 的时刻读到的 CCR1 = 20000（错误值！）
  → 因为 CNT 没有被及时清零

  cycle 的作用：
  → 只在 count=2 时才把 CCR1 写入 cycle
  → 确保只保存正确的值（10000），过滤掉错误值（20000）
```

## 八、核心要点总结

| 要点 | 说明 |
|------|------|
| **事件处理先于中断处理** | 硬件事件瞬间完成，软件中断延迟执行 |
| **CCR1 捕获先于 CNT 清零** | 这是硬件设计保证的时序关系 |
| **每次上升沿只需清零 CNT** | 不需要计数、不需要状态机 |
| **CCR1 在下次捕获前保持不变** | 可以直接读取，无需全局变量缓存 |
| **优化后代码极简** | 中断中只有两行：清标志 + 清 CNT |

---

## 九、优化思路总结

```
原始思路（复杂）：
  上升沿 → count++ → 判断第几次 → 第1次清零 / 第2次读取保存

优化思路（简洁）：
  上升沿 → 硬件自动捕获 CCR1 = CNT → 中断中清零 CNT
  → 利用"事件先于中断"的硬件特性
  → 无需状态机，无需全局变量
```

> 核心理解：**硬件事件处理（瞬间） vs 软件中断处理（延迟）的先后关系**，是正确实现输入捕获的关键。

# 嵌入式单片机开发学习路径 —— 系统整理笔记

> 整理自B站视频 BV1W44y1k729，原作者：科研成果

---

## 总体框架

嵌入式按处理器性能分为两大类：

| 类别 | 芯片举例 | 能否运行Linux | 典型产品 |
|------|----------|---------------|----------|
| 低性能（单片机） | 51单片机、STM32、RISC-V | 不能 | 手环、插座、电动牙刷、小型玩具 |
| 高性能 | 手机处理器 | 能运行Linux/Android | 智能音箱、自动售货机、摄像头、机器人 |

本路径聚焦 **低性能单片机** 方向，分四个阶段：**菜鸟期 → 入门期 → 进阶期 → 老司机期**。

---

## 一、菜鸟期（约 2 周）

### 1. 技术概念入门

- **单片机是什么**：一种集成电路芯片，包含MCU、RAM、ROM、中断控制器等
- 位数概念：8位、16位、32位单片机的区别
- **电路板**：了解原理图（schematic）和PCB的概念
- **固件**：烧录到单片机中的程序，本质是一系列计算机指令

### 2. 工具链了解

- 程序烧录到硬件的三个步骤：**编译 → 下载 → 运行**
- 常用软件：**Keil（MDK-ARM）**、**IAR**、**STM32CubeIDE**

### 3. 第一个程序：点灯（LED点灯）

- 点灯 = 嵌入式的 "Hello World"
- 点亮LED说明：电源正常、处理器正常、引脚正常、烧录流程正常

### 4. C语言基础（菜鸟阶段）

> C语言基础达到菜鸟期即可上手单片机，最好是入门期基础打牢

需掌握：
- 基础概念：编译器、栈、堆
- 基础语法：**常量、变量、数据类型、表达式、函数、分支/循环、数组、结构体、指针、C标准库**

### 5. 单片机基础

**① 选型**：选择学习资料丰富的MCU → **推荐直接学STM32**

**② 程序触发方式**（三种核心机制）：
- **轮询**（Polling）
- **定时器**（Timer）
- **中断**（Interrupt）

**③ 外设学习**：
- 输出型：LED、蜂鸣器
- 输入型：按键
- 基础外设：GPIO引脚控制
- 通信接口：**UART、SPI、I²C**

**④ 通信协议**：
- 掌握通信协议设计（帧头 + 长度 + 数据 + 帧尾）
- 二进制数据的编解码处理

**⑤ 基础传感器使用**：
- 温湿度传感器、红外传感器、超声波传感器等

**⑥ 调试技能**：
- Keil / IAR 的调试工具
- 查看内存、变量值来排查问题

**⑦ 菜鸟期实践项目**：
- 测距仪（红外/超声波传感器）
- 传感器数据采集 + OLED/LCD屏幕显示

## 二、入门期（工作 1~3 年）

### 1. 单片机进阶编程

- **编码规范**：注释、命名、代码可读性
- **数据结构**：栈、队列、链表
- **基础算法**：排序和查找算法
- **状态机编程**：处理更复杂的程序逻辑（轮询/定时器/中断 → 状态机）
- **实时操作系统（RTOS）**：会用任务、锁、信号量、消息队列

### 2. 网络编程

- 协议：**TCP/IP、UDP、MQTT、CoAP**
- 无线模块：WiFi、蓝牙、ZigBee、2G/4G/5G、NB-IoT
- 控制方式：**串口透传 / AT指令**
- 典型架构：MCU + 无线模块

### 3. 低功耗编程

- 单片机低功耗模式：睡眠、停机、待机
- 功耗测量：电流、功耗指标及测试方法

### 4. 电路板知识

- 能看懂**简单的原理图和PCB**

### 5. 入门期实践项目

- 无线设备连接云平台（阿里云/腾讯云/百度云）
- 联网温湿度采集系统（手机APP查看数据）
- 智能插座 / 智能灯（APP远程控制）

## 三、进阶期（工作 5~10 年）

分为 **通用内功**（跨行业通用）和 **独门绝技**（细分领域专精）两部分。

### A. 通用内功

#### ① 实时操作系统（RTOS）深入

- 学习 **FreeRTOS、RT-Thread、μC/OS**
- 进阶要求：不只会用，要理解**任务、锁、信号量、消息队列的底层实现**

#### ② 网络协议栈源码学习

- 深入 TCP/IP、UDP、MQTT、HTTP 协议栈的源码实现
- 参考开源协议栈进行学习

#### ③ 无线模块二次开发

- 不再通过MCU用AT指令控制模块，而是**直接在模块上开发**
- 典型模块：蓝牙、WiFi、蜂窝模组
- 目的：减少一颗主控芯片，降低量产成本

#### ④ 代码性能优化

- 了解常用的性能优化手段

#### ⑤ 代码可移植性

- 按规范编写代码，便于移植到不同硬件平台
- 产品升级换代时可快速迁移

#### ⑥ 编译与链接脚本

- C语言代码如何编译成机器指令
- 机器指令如何在芯片中组织和存储
- 加载地址、代码段/数据段地址的指定

#### ⑦ 其他编程语言

- **Python、Rust、Shell**（用于处理二进制固件等工具性工作）

#### ⑧ 底层硬件知识

| 层级 | 学习内容 |
|------|----------|
| 芯片手册 | 学会阅读Datasheet，了解芯片特性和使用方法 |
| ARM指令集 | Cortex-M系列，37个寄存器，常见汇编指令（用于性能优化场景） |
| 电子学 | 模电/数电：电压、电流、电阻、电容、半导体、门电路、CPU/存储器设计 |

### B. 独门绝技（细分领域）

| 领域 | 内容 |
|------|------|
| **通信协议栈开发** | WiFi/4G/5G的底层MAC层、端口等各层协议开发 |
| **算法** | 预测、姿态、惯导、PID、SLAM算法（门槛高，一般需研究生学历） |
| **安全** | 芯片安全、网络安全、数据加密存储、固件签名 |
| **FPGA设计** | Verilog/HDL语言，组合逻辑/时序逻辑电路设计 → 更深层：CPU设计（流水线、取值、译码、总线、中断） |

## 四、老司机期（资深/管理层）

### 四个维度

```
老司机 = 技术（横向扩展）+ 管理能力 + 行业认知 + 公司认知
```

### 1. 技术横向扩展

- 上位机 / APP 开发
- 测试工具、流程、方法
- MATLAB仿真
- PCB版图设计
- 工业设计 / 结构设计

### 2. 管理能力

- 团队管理、项目进度管控
- 需求分析、竞品分析
- 供应链管理、成本控制
- 带领团队实现产品目标

### 3. 行业认知

> **成为某个行业的专家**（重要三遍）

- 了解行业市场空间、是否处于上升期
- 懂行业知识（如做血糖仪要懂血糖医学知识）
- 了解产品设计、生产流程、客户群体、销售渠道、行业潜规则
- 常见行业方向：消费电子、智能家居、可穿戴、汽车电子、医疗电子、工业控制、航天/机器人

### 4. 公司认知

电子行业上下游产业链：

```
芯片原厂 → 模组厂/传感器厂/云平台/OS/运营商 → 终端硬件厂商 → 集成商/渠道商
```

- 了解所在行业的头部、腰部、尾部公司
- 了解商业模式、经营状态、同行在做什么及业务逻辑
- 这些经验只能在多年从业中积累，书本上学不到

| 阶段 | 工作年限 | 核心关键词 |
|------|----------|------------|
| 菜鸟期 | 0（入门学习） | C语言基础、STM32、点灯、GPIO、UART/SPI/I²C、基础传感器 |

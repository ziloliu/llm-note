## 一、ADC 基本概念

### 1.1 术语定义

| 缩写 | 全称 | 含义 |
|------|------|------|
| **ADC** | Analog to Digital Converter | 模数转换器（硬件）/ 模数转换（功能） |
| **DAC** | Digital to Analog Converter | 数模转换器（硬件）/ 数模转换（功能） |

**ADC 的作用**：将外部模拟信号（连续变化）转换为数字信号（离散的 0 和 1）

```
传感器采集 → ADC 转换 → 数字处理
模拟物理量 → 0 和 1 信号 → CPU 处理

例：温度传感器 → 电压值 2.65V → 数字量 0xCCC（12位）
```

### 1.2 模拟信号 vs 数字信号

| 特性 | 模拟信号 | 数字信号 |
|------|---------|---------|
| 波形 | **连续** | **离散** |
| 取值 | 无限精确 | 有限个离散值 |
| 表示 | 电压、电流连续变化 | 0 和 1（高低电平） |
| 典型 | 传感器输出 | 单片机内部数据 |

## 二、ADC 工作原理

### 2.1 两个阶段

```
原始模拟信号（连续）
    ↓
① 采样阶段（Sampling）：时间维度，每隔 Δt 取一个瞬时值
    ↓
② 量化阶段（Quantization）：幅度维度，将瞬时值转换为二进制代码
    ↓
数字信号（离散的二进制代码）
```

### 2.2 香农采样定理

```
采样频率 fs 必须满足：fs ≥ 2 × f_max

其中：
  fs    = 采样频率（每秒采样次数）
  f_max = 信号中最高频率分量

只有满足此条件，才能从采样信号完美恢复原始信号
```

## 三、ADC 主要参数

| 参数 | 含义 | STM32 典型值 |
|------|------|:------------:|
| **参考电压** | 确定模拟信号幅度范围 | VREF+ = 3.3V, VREF- = 0V |
| **通道数** | 能同时采集的模拟信号路数 | 最多 16 个外部通道 |
| **采样率** | 每秒采样次数 | 最高 1 MSPS |
| **分辨率** | 数字信号位数 | **12 位** |
| **转换时间** | 完成一次转换的时间 | 采样时间 + 12.5 个时钟周期 |

### 3.1 分辨率

```
分辨率 = ADC 输出数字量的位数

| 位数 | 离散级别 | 最小变化量（3.3V 参考） |
|:----:|:-------:|:----------------------:|
| 8 位 | 256 | 3.3V/256 ≈ 12.9mV |
| 10 位 | 1024 | 3.3V/1024 ≈ 3.2mV |
| 12 位 | 4096 | 3.3V/4096 ≈ 0.8mV |

STM32F103 的 ADC 为 12 位分辨率：
  → 2^12 = 4096 个离散级别
  → 最小分辨率 = 3.3V / 4096 ≈ 0.806mV
```

### 3.2 转换公式

```
数字量→电压：V = D × VREF / 2^n
电压→数字量：D = V × 2^n / VREF

STM32F103（12 位 ADC，VREF = 3.3V）：
  D = V × 4096 / 3.3
  V = D × 3.3 / 4096
```

## 五、逐次逼近型 ADC 工作原理

### 5.1 核心类比：天平称重

```
逐次逼近 = 天平称重

天平称重：             ADC 转换：
  左盘：待测物体          正端：待测电压 Ux
  右盘：砝码             负端：DAC 输出电压
  操作：逐个加减砝码       操作：逐位判断 0/1
  结果：平衡时读取砝码值    结果：逐位确定二进制数

原则：先大后小，逐个尝试，大的保留，小的调整
```

### 5.2 ADC 砝码（位权）

| 位 | 位权（砝码重量） | 说明 |
|:--:|:--------------:|------|
| Bit 11 | **2048** | 最高位，最先比较 |
| Bit 10 | **1024** | |
| Bit 9 | **512** | |
| ... | ... | 每次砍一半 |
| Bit 1 | **2** | |
| Bit 0 | **1** | 最低位 |

### 5.3 转换过程（12 位）

```
① 从最高位 Bit11 开始，将当前位 置 1
② DAC 将寄存器值转换为模拟电压
③ 比较器比较待测电压与 DAC 输出：
   → Ux > DAC：砝码偏小，保留当前位 = 1
   → Ux < DAC：砝码偏大，清除当前位 = 0
④ 移到下一位，重复 ①②③
⑤ 所有位判断完毕，寄存器中的值即为转换结果

时间复杂度：12 位需要 12 次比较
```

### 5.4 逐次逼近型 ADC 特点

| 特性 | 说明 |
|------|------|
| 转换速度 | **中等**（n 位需 n 个时钟周期） |
| 转换精度 | **中等**（取决于位数） |
| 电路复杂度 | **中等**（1 个 DAC + 1 个比较器） |
| 集成度 | **高**（适合片上集成） |

## 七、通道分组 — 歌单类比

### 7.1 两个通道组

| 通道组 | 英文名 | 最大通道数 | 数据寄存器 | 特点 |
|--------|--------|:----------:|:----------:|------|
| **规则通道组** | Regular | **16** | **1 个 DR** | 常规使用 |
| **注入通道组** | Injection | **4** | **4 个 JDR** | 可插队 |

### 7.2 规则组 vs 注入组

| 对比项 | 规则通道组 | 注入通道组 |
|--------|:----------:|:----------:|
| 最大通道数 | 16 | 4 |
| 数据寄存器 | 1 个 DR（可能覆盖） | 4 个 JDR（各自独立） |
| 可否插队 | ❌ | ✅ 可打断规则组 |
| 一般使用 | ✅ 主要使用 | 较少使用 |

### 7.3 数据覆盖问题（规则组）

```
问题：多个通道共用 1 个 DR → 后续转换覆盖前面结果

解决方案：
① 标志位/中断 + 及时读取
② DMA 自动传输（推荐）→ 转换完成自动触发 DMA 请求

DMA 通道对应：
  ADC1 → DMA1 Channel 1 ✅
  ADC2 → 无 DMA 支持     ❌
  ADC3 → DMA2 Channel 5 ✅
```

## 八、转换方式（SCAN + CONT）

### 8.1 两个关键控制位

| 寄存器 | 位 | 含义 |
|--------|:--:|------|
| CR1 | **SCAN** | 扫描模式（多通道顺序播放） |
| CR2 | **CONT** | 连续转换（循环播放） |

### 8.2 四种组合模式

| SCAN | CONT | 模式 | 歌单类比 | 行为 |
|:----:|:----:|------|---------|------|
| 0 | 0 | **单次单通道** | 单曲播放一次 | 只转换第一个通道，转换完停止 |
| 1 | 0 | **单次扫描** | 顺序播放一次 | 所有通道依次转换一次，转换完停止 |
| 0 | 1 | **连续单通道** | **单曲循环** | 只转换第一个通道，不停重复 |
| 1 | 1 | **连续扫描** | **列表循环** | 所有通道依次转换，不停循环 |

## 九、事件、中断与 DMA

### 9.1 三类事件

| 事件 | 标志位 | 含义 | 中断使能位 |
|------|:------:|------|:----------:|
| **转换结束** | **EOC** | End Of Conversion | EOCIE |
| **注入转换结束** | **JEOC** | End Of Injected Conversion | JEOCIE |
| **模拟看门狗** | **AWD** | Analog WatchDog | AWDIE |

### 9.2 转换时间

```
总转换时间 = 采样时间 + 12.5 个 ADC 时钟周期

采样时间可配置：1.5, 7.5, 13.5, 28.5, 41.5, 55.5, 71.5, 239.5 周期

例：ADC 时钟 = 12MHz，采样时间 = 1.5 周期
  总周期数 = 1.5 + 12.5 = 14 个周期
  总时间 = 14 / 12MHz ≈ 1.17μs
```

## 十三、独立模式单通道采集实现

### 13.1 实验概述

| 项目 | 说明 |
|------|------|
| ADC 模块 | ADC1 |
| 通道 | 通道 10（ADC123_IN10） |
| 输入引脚 | PC0 |
| 工作模式 | 独立模式 + 单通道 + 连续转换 |
| 触发方式 | 软件触发（SWSTART） |
| 数据对齐 | 右对齐（默认） |

### 13.2 寄存器方式配置流程

```
① 开启 ADC1 时钟
   RCC->APB2ENR |= RCC_APB2ENR_ADC1EN;

② ADC 时钟分频（72MHz / 6 = 12MHz）
   RCC->CFGR ADCPRE = 10（6 分频）

③ GPIO 配置（PC0 模拟输入）
   GPIOC->CRL &= ~(0xF << 0);  // 4位全零 = 模拟输入

④ CR1：禁用扫描模式
   ADC1->CR1 &= ~ADC_CR1_SCAN;

⑤ CR2：开启连续转换、右对齐
   ADC1->CR2 |= ADC_CR2_CONT;       // CONT = 1
   ADC1->CR2 &= ~ADC_CR2_ALIGN;     // ALIGN = 0

⑥ SMPR1：配置通道 10 采样时间
   ADC1->SMPR1 |= (0x1 << 0);      // SMP10 = 001（7.5 周期）

⑦ SQR1：配置序列长度 L = 0（1 个通道）

⑧ SQR3：配置 SQ1 = 通道 10
   ADC1->SQR3 |= (10 << 0);

⑨ CR2：配置软件触发
   ADC1->CR2 |= ADC_CR2_EXTTRIG;    // EXTTRIG = 1
   ADC1->CR2 |= ADC_CR2_EXTSEL;     // EXTSEL[2:0] = 111
```

### 13.3 启动转换

```c
void ADC1_StartConvert(void)
{
    // ① 上电唤醒（第一次写 ADON = 1）
    ADC1->CR2 |= ADC_CR2_ADON;

    // ② 执行校准
    ADC1->CR2 |= ADC_CR2_CAL;

    // ③ 等待校准完成（硬件自动清零 CAL）
    while (ADC1->CR2 & ADC_CR2_CAL);

    // ④ 软件触发启动转换
    ADC1->CR2 |= ADC_CR2_SWSTART;
}
```

### 13.4 读取转换结果

```c
double ADC1_ReadV(void)
{
    // 等待转换完成（EOC = 1）
    while (!(ADC1->SR & ADC_SR_EOC));

    // 读取数据寄存器（读 DR 自动清除 EOC）
    return ADC1->DR * 3.3 / 4095;   // 转换为电压
}
```

### 13.5 SWSTART vs EOC 重要区分

```
SWSTART 位：
  写 1 → 开始转换 → 硬件立即清除
  SWSTART = 0 只表示"已经开始"，不表示"已经完成"

EOC 位：
  转换完成 → 硬件置 1
  读 DR → 硬件清除
  EOC = 1 才表示转换完成

时序：
  写 SWSTART = 1
    ↓ （硬件立即清零 SWSTART）
    ↓ （进行 12 位逐次逼近转换）
    ↓ （转换完成）
  EOC = 1
    ↓ （读取 DR）
  EOC = 0
```

### 13.6 HAL 库方式实现

**CubeMX 配置**：
- Mode: Independent mode
- Data Alignment: Right alignment
- Scan Conversion Mode: Disable
- Continuous Conversion Mode: Enable
- Number Of Conversion: 1
- External Trigger: Software trigger
- Rank 1: Channel 10
- Sampling Time: 7.5 Cycles

**⚠️ 关键：CubeMX 不会自动配置 ADC 分频**，需手动在 Clock Configuration 中设置 ADC Prescaler = /6

**关键函数**：

| 函数 | 功能 |
|------|------|
| `HAL_ADC_Start()` | 启动 ADC 转换（轮询） |
| `HAL_ADCEx_Calibration_Start()` | **ADC 校准**（必须） |
| `HAL_ADC_GetValue()` | 获取转换结果（读 DR） |
| `HAL_ADC_Start_IT()` | 启动 ADC 转换（中断） |
| `HAL_ADC_Start_DMA()` | 启动 ADC 转换（DMA） |

**主函数示例**：
```c
// ⚠️ 校准（必须！否则最大值只能到 3.25V）
HAL_ADCEx_Calibration_Start(&hadc1);

// 启动 ADC 转换
HAL_ADC_Start(&hadc1);

while (1)
{
    double v = HAL_ADC_GetValue(&hadc1) * 3.3 / 4095;
    printf("V = %.2f V\n", v);
    HAL_Delay(1000);
}
```

### 13.7 校准的重要性

```
问题：未校准时最大值只能到 3.25V，到不了 3.3V

原因：ADC 转换器存在固有偏差（类似天平没调零）

校准过程：
  ① HAL_ADCEx_Calibration_Start(&hadc1)
  ② 硬件自动执行校准（类似调零）
  ③ 校准完成，偏差被消除

正确顺序：
  ① HAL_ADCEx_Calibration_Start()  ← 先校准
  ② HAL_ADC_Start()                ← 再启动
```

### 13.8 寄存器 vs HAL 库对比

| 对比项 | 寄存器方式 | HAL 库方式 |
|--------|-----------|-----------|
| 时钟配置 | 手动 RCC->CFGR | CubeMX 图形化 |
| GPIO 配置 | 手动 CRL/CNF | CubeMX 自动生成 |
| 校准 | 手动 CAL 位 + 等待 | `HAL_ADCEx_Calibration_Start()` |
| 启动 | 手动 SWSTART 位 | `HAL_ADC_Start()` |
| 读取 | 手动读 SR + DR | `HAL_ADC_GetValue()` |
| 代码量 | 较多 | **极少** |

### 13.9 GPIO 时钟可省略的原因

```
GPIO 模拟输入模式下的内部电路：

  引脚 ────────────────────→ ADC 输入
          （一根导线直连）

  不需要的电路：
    ❌ 施密特触发器（数字输入用）
    ❌ 上拉/下拉电阻
    ❌ 输出驱动器

  这些电路都关闭 → 不需要时钟驱动
  → GPIO 时钟可以不开启
  → 但开启也不影响功能
```

## 十四、寄存器配置详解（单通道）

> 本章是上一章的**寄存器配置原理补充**，详细解释了每位寄存器的作用和配置决策。

### 14.1 寄存器存储器映像

| 寄存器 | 地址偏移 | 说明 |
|--------|:--------:|------|
| SR | 0x00 | 状态寄存器（EOC、JEOC、AWD） |
| CR1 | 0x04 | 控制寄存器 1（SCAN/中断/看门狗） |
| CR2 | 0x08 | 控制寄存器 2（CONT/ALIGN/ADON/CAL/触发） |
| SMPR1 | 0x0C | 采样时间 1（通道 10~17） |
| SMPR2 | 0x10 | 采样时间 2（通道 0~9） |
| SQR1~3 | 0x2C~0x34 | 规则序列寄存器 |
| DR | 0x4C | 规则通道数据寄存器（16 位） |

### 14.2 CR1 关键位

| 位 | 名称 | 功能 |
|:--:|------|------|
| 8 | **SCAN** | 扫描模式（0=单曲播放，1=顺序播放） |
| 25 | JAUTO | 规则组完成后自动转换注入组 |
| 11 | DISCEN | 间断模式（分组转换） |

### 14.3 CR2 关键位（核心寄存器）

| 位 | 名称 | 功能 |
|:--:|------|------|
| 0 | **ADON** | ADC 开关（0→1 上电，已为 1 再写 1 启动） |
| 1 | **CONT** | 连续转换模式 |
| 2 | **CAL** | 校准启动（写 1 开始，硬件自动清零） |
| 11 | **ALIGN** | 数据对齐（0=右对齐，1=左对齐） |
| 17 | **EXTTRIG** | 外部触发使能（打开"与门"） |
| 19:17 | **EXTSEL** | 触发源选择（111=SWSTART） |
| 22 | **SWSTART** | 规则组软件启动 |

### 14.4 ADON 双重作用

```
第一次写 1（ADON 从 0→1）：从断电模式唤醒，不启动转换
再次写 1（ADON 已为 1）：启动规则通道转换

⚠️ ADON 方式只能启动规则组，注入组需要 SWSTART
```

### 14.5 SQR 规则序列寄存器详解

```
3 个寄存器共 96 位，存放最多 16 个通道号（每通道 5 位）+ 4 位长度：

SQR3[4:0]   = SQ1（第 1 个转换的通道）
SQR3[9:5]   = SQ2
...
SQR2[4:0]   = SQ7
...
SQR1[4:0]   = SQ13
SQR1[19:15] = SQ16
SQR1[23:20] = L（序列长度：0~15 → 实际 1~16 个通道）

每个通道号占 5 位（因为 0~17 需要 5 位，4 位不够）
```

### 14.6 触发配置三步走

```
① CR2.EXTTRIG = 1    → 打开"与门"
② CR2.EXTSEL = 111   → 选择 SWSTART（8 选 1 选最后一项）
③ CR2.SWSTART = 1    → 触发启动（硬件立即清零）
```

| EXTSEL | 触发源 |
|:------:|--------|
| 000~110 | 定时器/EXTI 硬件触发 |
| **111** | **SWSTART（软件触发）** |

### 14.7 SQR 与 SMPR 配合示例

```
配置流程（从 SQR3 开始倒序配置）：
  SQR1[23:20] = L（长度-1）
  SQR3[4:0]   = SQ1 = 10（通道 10）
  SMPR1[2:0]  = SMP10 = 001（7.5 周期）

总转换时间 = 采样时间 + 12.5 个 ADC 时钟周期
例：12MHz, SMP=001 → (7.5+12.5)/12MHz ≈ 1.67μs
```

## 十五、功能框图复习总结

### 15.1 ADC 模块三大部分回顾

```
上部：事件/中断/DMA
  → EOC/JEOC/AWD 事件 → 中断使能 → NVIC
  → DMA 请求（ADC1 DMA1_CH1, ADC3 DMA2_CH5）

中部：核心转换
  → 18 个输入通道 → 规则/注入通道组 → SAR 转换器 → 数据寄存器

下部：触发源
  → 软件触发（SWSTART / ADON 再写 1）
  → 硬件触发（定时器/EXTI）
```

### 15.2 关键区分

| 概念 | 说明 |
|------|------|
| **EOC 是组标志** | 整组转换完毕才置 1，不是每个通道单独置 1 |
| **ADON** | 简单，只能启动规则组 |
| **SWSTART** | 通用，规则组+注入组都能启动 |
| **间断模式** | 每次触发转换 N 个通道，支持断点续传 |
| **自动注入** | 规则组完成自动追加注入组 |
| **ADC3** | 触发电路独立定义，与 ADC1/ADC2 不同 |

### 15.3 C 语言运算符优先级陷阱

```c
// ❌ 错误：== 优先级 高于 &
while (ADC1->SR & ADC_SR_EOC == 0);
// 实际执行：ADC_SR_EOC == 0 → 假 → 0 → 与 SR 按位与 → 恒为 0 → 不等待

// ✅ 正确：加括号强制先按位与
while ((ADC1->SR & ADC_SR_EOC) == 0);
```

## 十六、独立模式多通道采集（DMA 方式）

### 16.1 为什么多通道必须用 DMA

```
单通道时：每转换一次 → EOC 置 1 → 及时读取 DR，不会覆盖

多通道时（扫描模式）：
  → 整组转换完成后才置 EOC
  → 中间各通道结果依次写入 DR，互相覆盖
  → 来不及逐通道读取 → 必须 DMA 自动搬运

DMA 通道：ADC1 → DMA1 Channel 1（硬件固定）
```

### 16.2 多通道关键配置差异

| 配置项 | 单通道 | 多通道 |
|--------|:------:|:------:|
| CR1.SCAN | 0 | **1**（扫描模式） |
| SQR1.L | 0 | **1**（2 个通道） |
| SQR3.SQ2 | — | **12**（新增） |
| SMPR1.SMP12 | — | **001**（新增） |
| CR2.DMA | 0 | **1**（开启 DMA） |

### 16.3 DMA CCR 关键配置

| 位 | 配置 | 说明 |
|:--:|:----:|------|
| DIR | 0 | 外设→存储器 |
| PSIZE/MSIZE | 01 | 16 位（匹配 DR 宽度） |
| PINC | 0 | DR 地址固定，不自增 |
| MINC | 1 | 数组地址自增 |
| CIRC | 1 | 循环模式（配合 CONT=1） |

### 16.4 启动顺序（关键！）

```
✅ 正确顺序：
  ① ADC 上电（CR2.ADON = 1）
  ② DMA 通道使能（CCR.EN = 1）
  ③ 校准
  ④ 启动转换（ADON 再次写 1 或 SWSTART）

❌ 错误：先 DMA 使能 → 再 ADC 上电
  → ADC 断电时 DR 无有效信号 → DMA 通道建立失败
  → 数据始终为 0
```

### 16.5 HAL 库方式：多通道 DMA

**CubeMX 关键配置**：
- Scan Conversion Mode: Enable（多通道勾选后自动开启）
- Continuous Conversion Mode: Enable
- Number Of Conversion: 2
- DMA Settings: ADC1, Circular mode, Half Word
- ⚠️ **必须关闭 DMA 中断**（取消 "Force DMA channels interrupt"）

```c
uint16_t data[2] = {0, 0};

// 校准后启动 DMA 方式
HAL_ADCEx_Calibration_Start(&hadc1);
HAL_ADC_Start_DMA(&hadc1, (uint32_t *)data, 2);

while (1)
{
    printf("Ch10: %.2f V\tCh12: %.2f V\n",
           data[0] * 3.3 / 4095,
           data[1] * 3.3 / 4095);
    HAL_Delay(1000);
}
```

**关键注意**：
- `HAL_ADC_Start_DMA` 参数需 `(uint32_t *)` 强转（即使 data 是 uint16_t 数组）
- 浮空引脚电压不稳定（约 1.64V，受周围环境干扰）

# 硬件 I²C 寄存器实现 — 数据收发完整代码 笔记

---

## 一、所有函数汇总（i2c.c 完整实现）

### 1.1 I2C_Init() — 初始化

```c
void I2C_Init(void)
{
    // ① 开启时钟
    RCC->APB2ENR |= RCC_APB2ENR_IOPBEN;
    RCC->APB1ENR |= RCC_APB1ENR_I2C2EN;

    // ② GPIO：PB10/PB11 复用开漏输出 50MHz
    GPIOB->CRH |= (GPIO_CRH_CNF10 | GPIO_CRH_CNF11
                  | GPIO_CRH_MODE10 | GPIO_CRH_MODE11);

    // ③ CR2：输入时钟 36MHz
    I2C2->CR2 = 36;

    // ④ CCR：标准模式，分频 180
    I2C2->CCR = 180;

    // ⑤ TRISE：最大上升时间 37
    I2C2->TRISE = 37;

    // ⑥ CR1：I²C 模式 + ACK 使能
    I2C2->CR1 &= ~I2C_CR1_SMBTYPE;
    I2C2->CR1 |= I2C_CR1_ACK;

    // ⑦ CR1：外设使能（最后）
    I2C2->CR1 |= I2C_CR1_PE;
}
```

---

### 1.2 I2C_Start() — 产生起始信号

```c
uint8_t I2C_Start(void)
{
    uint16_t timeout = 0xFFFF;

    // ① 请求产生起始信号
    I2C2->CR1 |= I2C_CR1_START;

    // ② 等待 SB = 1（起始信号已发出）
    while (!(I2C2->SR1 & I2C_SR1_SB) && timeout)
    {
        timeout--;
    }

    // ③ 返回结果
    return timeout ? OK : TIMEOUT;
}
```

| 操作 | 寄存器 | 位 |
|------|--------|:--:|
| 请求起始 | CR1 | START = 1 |
| 等待起始发出 | SR1 | SB = 1 |
| 硬件清除 | CR1 | START 自动清零 |

---

### 1.3 I2C_Stop() — 设置停止信号

```c
void I2C_Stop(void)
{
    I2C2->CR1 |= I2C_CR1_STOP;   // 配置项：字节传输结束后发出停止
}
```

| 操作 | 寄存器 | 位 | 说明 |
|------|--------|:--:|------|
| 设置停止 | CR1 | STOP = 1 | 不立即生效，延迟到当前字节传输结束后 |

---

### 1.4 I2C_EnableAck() / I2C_DisableAck()

```c
void I2C_EnableAck(void)
{
    I2C2->CR1 |= I2C_CR1_ACK;      // 收到字节后自动返回 ACK
}

void I2C_DisableAck(void)
{
    I2C2->CR1 &= ~I2C_CR1_ACK;     // 收到字节后不返回（NACK）
}
```

| 操作 | CR1.ACK | 收到字节后行为 |
|------|:-------:|---------------|
| EnableAck | 1 | 硬件自动拉低 SDA（ACK） |
| DisableAck | 0 | 硬件不操作 SDA（NACK） |

---

### 1.5 I2C_SendAddr() — 发送设备地址并等待应答

```c
uint8_t I2C_SendAddr(uint8_t addr)
{
    uint16_t timeout = 0xFFFF;

    // ① 直接写入 DR（起始后 DR 为空，无需等 TXE）
    I2C2->DR = addr;

    // ② 等待 ADDR = 1（地址已发送，收到 ACK）
    while (!(I2C2->SR1 & I2C_SR1_ADDR) && timeout)
    {
        timeout--;
    }

    // ③ 清除 ADDR 标志（读 SR1 后读 SR2）
    if (timeout)
    {
        (void)I2C2->SR2;    // 读 SR2 清除 ADDR
        return OK;
    }
    else
    {
        return TIMEOUT;
    }
}
```

**关键点：**

| 项目 | 说明 |
|------|------|
| 不等 TXE | 起始信号后 DR 肯定为空 |
| 等待标志 | **ADDR**（非 BTF） |
| 清除方式 | 读 SR1 + **读 SR2** |
| 为何要清除 | 读取操作需发送两次地址（假写+真读），不清除则第二次跳过等待 |

---

### 1.6 I2C_SendByte() — 发送一个字节并等待应答

```c
uint8_t I2C_SendByte(uint8_t byte)
{
    uint16_t timeout = 0xFFFF;

    // ① 等待 TXE = 1（发送缓冲区空）
    while (!(I2C2->SR1 & I2C_SR1_TXE) && timeout)
    {
        timeout--;
    }

    // ② 写入数据到 DR
    I2C2->DR = byte;

    // ③ 重新赋值 timeout，等待 BTF = 1（字节传输完成）
    timeout = 0xFFFF;
    while (!(I2C2->SR1 & I2C_SR1_BTF) && timeout)
    {
        timeout--;
    }

    // ④ 返回结果
    return timeout ? OK : TIMEOUT;
}
```

**关键点：**

| 项目 | 说明 |
|------|------|
| 先等 TXE | 上一个字节可能还在发送 |
| 等待标志 | **BTF**（非 ADDR） |
| 清除方式 | 无需手动，下次操作 DR 或产生 START/STOP 自动清除 |
| timeout 重赋值 | 第二个等待需要新的超时计数 |

---

### 1.7 I2C_ReceiveByte() — 接收一个字节

```c
uint8_t I2C_ReceiveByte(void)
{
    uint16_t timeout = 0xFFFF;

    // ① 等待 RXNE = 1（接收缓冲区非空）
    while (!(I2C2->SR1 & I2C_SR1_RXNE) && timeout)
    {
        timeout--;
    }

    // ② 返回数据（或超时标志）
    return timeout ? I2C2->DR : TIMEOUT;
}
```

**关键点：**

| 项目 | 说明 |
|------|------|
| 等待标志 | **RXNE**（收到数据） |
| ACK 处理 | **无需手动**，由 CR1.ACK 配置位决定硬件自动行为 |
| 只需一次等待 | 比 SendByte（两次等待）更简单 |

## 二、所有函数操作流程图

```
I2C_Start():
  CR1.START=1 → [等 SB=1] → 返回 OK/TIMEOUT

I2C_Stop():
  CR1.STOP=1（延迟生效）

I2C_EnableAck():
  CR1.ACK=1

I2C_DisableAck():
  CR1.ACK=0

I2C_SendAddr(addr):
  DR=addr → [等 ADDR=1] → 读SR2清除 → 返回 OK/TIMEOUT

I2C_SendByte(byte):
  [等 TXE=1] → DR=byte → [等 BTF=1] → 返回 OK/TIMEOUT

I2C_ReceiveByte():
  [等 RXNE=1] → 返回 DR/TIMEOUT
```

---

## 三、SR1 标志位操作对照表

| 标志位 | 含义 | 使用函数 | 清除方式 |
|:------:|------|----------|----------|
| **SB** | 起始已发出 | I2C_Start | 读 SR1 + 写 DR |
| **ADDR** | 地址已发送 | I2C_SendAddr | 读 SR1 + **读 SR2** |
| **TXE** | 发送缓冲区空 | I2C_SendByte | 写 DR |
| **BTF** | 字节传输完成 | I2C_SendByte | 读/写 DR 或 START/STOP |
| **RXNE** | 接收缓冲区非空 | I2C_ReceiveByte | 读 DR |

## 四、软硬件 I²C 函数接口对比

| 操作 | 软件模拟 | 硬件 I²C | 变化 |
|------|----------|----------|------|
| 初始化 | `I2C_Init()` | `I2C_Init()` | GPIO→复用开漏 + 寄存器配置 |
| 起始 | `void I2C_Start()` | `uint8_t I2C_Start()` | 增加返回值 |
| 停止 | `void I2C_Stop()` | `void I2C_Stop()` | 不变 |
| 发 ACK | `I2C_SendAck()` | `I2C_EnableAck()` | 改名（配置项） |
| 发 NACK | `I2C_SendNack()` | `I2C_DisableAck()` | 改名（配置项） |
| 等 ACK | `I2C_WaitAck()` | **删除** | 融入 SendAddr/SendByte |
| 发字节 | `void I2C_SendByte()` | `uint8_t I2C_SendByte()` | 增加返回值 |
| 收字节 | `uint8_t I2C_ReceiveByte()` | `uint8_t I2C_ReceiveByte()` | 不变 |
| — | — | `uint8_t I2C_SendAddr()` | **新增** |

## 五、接口层（m24c02.c）调用适配

硬件 I²C 的接口层与软件模拟基本一致，主要区别：

| 差异点 | 说明 |
|--------|------|
| `I2C_SendByte(0xA0)` + `I2C_WaitAck()` | → `I2C_SendAddr(0xA0)`（合并为一个函数） |
| `I2C_SendByte(data)` + `I2C_WaitAck()` | → `I2C_SendByte(data)`（内部已等待 BTF） |
| `I2C_SendAck()` | → `I2C_EnableAck()` |
| `I2C_SendNack()` | → `I2C_DisableAck()` |
| 可检查返回值 | Start / SendAddr / SendByte 均返回 OK/TIMEOUT |

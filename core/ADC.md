## 一、ADC 基本概念

### 1.1 术语定义

| 缩写 | 全称 | 含义 |
|------|------|------|
| **ADC** | Analog to Digital Converter | 模数转换器（硬件）/ 模数转换（功能） |
| **DAC** | Digital to Analog Converter | 数模转换器（硬件）/ 数模转换（功能） |

**ADC 的作用**：将外部模拟信号（连续变化）转换为数字信号（离散的 0 和 1）

```
传感器采集 → ADC 转换 → 数字处理
模拟物理量 → 0 和 1 信号 → CPU 处理

例：温度传感器 → 电压值 2.65V → 数字量 0xCCC（12位）
```

### 1.2 模拟信号 vs 数字信号

| 特性 | 模拟信号 | 数字信号 |
|------|---------|---------|
| 波形 | **连续** | **离散** |
| 取值 | 无限精确 | 有限个离散值 |
| 表示 | 电压、电流连续变化 | 0 和 1（高低电平） |
| 典型 | 传感器输出 | 单片机内部数据 |

## 二、ADC 工作原理

### 2.1 两个阶段

```
原始模拟信号（连续）
    ↓
① 采样阶段（Sampling）：时间维度，每隔 Δt 取一个瞬时值
    ↓
② 量化阶段（Quantization）：幅度维度，将瞬时值转换为二进制代码
    ↓
数字信号（离散的二进制代码）
```

### 2.2 香农采样定理

```
采样频率 fs 必须满足：fs ≥ 2 × f_max

其中：
  fs    = 采样频率（每秒采样次数）
  f_max = 信号中最高频率分量

只有满足此条件，才能从采样信号完美恢复原始信号
```

## 三、ADC 主要参数

| 参数 | 含义 | STM32 典型值 |
|------|------|:------------:|
| **参考电压** | 确定模拟信号幅度范围 | VREF+ = 3.3V, VREF- = 0V |
| **通道数** | 能同时采集的模拟信号路数 | 最多 16 个外部通道 |
| **采样率** | 每秒采样次数 | 最高 1 MSPS |
| **分辨率** | 数字信号位数 | **12 位** |
| **转换时间** | 完成一次转换的时间 | 采样时间 + 12.5 个时钟周期 |

### 3.1 分辨率

```
分辨率 = ADC 输出数字量的位数

| 位数 | 离散级别 | 最小变化量（3.3V 参考） |
|:----:|:-------:|:----------------------:|
| 8 位 | 256 | 3.3V/256 ≈ 12.9mV |
| 10 位 | 1024 | 3.3V/1024 ≈ 3.2mV |
| 12 位 | 4096 | 3.3V/4096 ≈ 0.8mV |

STM32F103 的 ADC 为 12 位分辨率：
  → 2^12 = 4096 个离散级别
  → 最小分辨率 = 3.3V / 4096 ≈ 0.806mV
```

### 3.2 转换公式

```
数字量→电压：V = D × VREF / 2^n
电压→数字量：D = V × 2^n / VREF

STM32F103（12 位 ADC，VREF = 3.3V）：
  D = V × 4096 / 3.3
  V = D × 3.3 / 4096
```

## 四、STM32 ADC 模块特性

| 特性 | 说明 |
|------|------|
| ADC 数量 | **3 个**（ADC1、ADC2、ADC3） |
| 分辨率 | **12 位**（三个 ADC 相同） |
| 外部通道 | 每个 ADC 最多 **16 个**外部输入通道 |
| 内部信号源 | 每个 ADC **2 个**内部信号源 |
| 工作原理 | **逐次逼近型**（SAR） |

### 4.1 ADC 信号源

| 类型 | 通道号 | 数量 | 来源 |
|------|:------:|:----:|------|
| **外部通道** | 0 ~ 15 | **16 个** | GPIO 引脚复用 |
| **内部温度传感器** | **16** | 1 个 | 芯片内部 |
| **内部参考电压** | **17** | 1 个 | 芯片内部 |

### 4.2 ADC 时钟配置

```
⚠️ ADC 输入时钟不得超过 14MHz

APB2 (72MHz) → ADC 预分频器（RCC->CFGR 中 ADCPRE 位）→ ADC 时钟

| ADCPRE[1:0] | 分频系数 | ADC 时钟 | 是否符合要求 |
|:-----------:|:--------:|:--------:|:----------:|
| 10 | /6 | 12 MHz | ✅ 推荐 |
| 11 | /8 | 9 MHz | ✅ |
```

## 五、逐次逼近型 ADC 工作原理

### 5.1 核心类比：天平称重

```
逐次逼近 = 天平称重

天平称重：             ADC 转换：
  左盘：待测物体          正端：待测电压 Ux
  右盘：砝码             负端：DAC 输出电压
  操作：逐个加减砝码       操作：逐位判断 0/1
  结果：平衡时读取砝码值    结果：逐位确定二进制数

原则：先大后小，逐个尝试，大的保留，小的调整
```

### 5.2 ADC 砝码（位权）

| 位 | 位权（砝码重量） | 说明 |
|:--:|:--------------:|------|
| Bit 11 | **2048** | 最高位，最先比较 |
| Bit 10 | **1024** | |
| Bit 9 | **512** | |
| ... | ... | 每次砍一半 |
| Bit 1 | **2** | |
| Bit 0 | **1** | 最低位 |

### 5.3 转换过程（12 位）

```
① 从最高位 Bit11 开始，将当前位 置 1
② DAC 将寄存器值转换为模拟电压
③ 比较器比较待测电压与 DAC 输出：
   → Ux > DAC：砝码偏小，保留当前位 = 1
   → Ux < DAC：砝码偏大，清除当前位 = 0
④ 移到下一位，重复 ①②③
⑤ 所有位判断完毕，寄存器中的值即为转换结果

时间复杂度：12 位需要 12 次比较
```

### 5.4 逐次逼近型 ADC 特点

| 特性 | 说明 |
|------|------|
| 转换速度 | **中等**（n 位需 n 个时钟周期） |
| 转换精度 | **中等**（取决于位数） |
| 电路复杂度 | **中等**（1 个 DAC + 1 个比较器） |
| 集成度 | **高**（适合片上集成） |

## 六、ADC 功能框图

### 6.1 总体结构

```
┌─────────────────────────────────────────────────────────┐
│  ③ 上部：事件/中断/DMA 请求                               │
│     EOC、JEOC、AWD 标志 → 中断 → NVIC                     │
│     DMA 请求（ADC1、ADC3）                                │
├─────────────────────────────────────────────────────────┤
│  ① 中部：核心转换部分（重点）                              │
│     GPIO/内部信号 → 多路选择 → 通道分组                    │
│     → 逐次逼近 ADC 转换 → 数据寄存器 → 数据总线           │
├─────────────────────────────────────────────────────────┤
│  ② 下部：触发源选择（一般不关注）                          │
│     定时器/EXTI/软件 → 控制转换何时开始                    │
└─────────────────────────────────────────────────────────┘
```

| **VREF+** | 参考电压正极 | **3.3V** | 测量上限 |
| **VREF-** | 参考电压负极 | **GND** | 测量下限 |

## 七、通道分组 — 歌单类比

### 7.1 两个通道组

| 通道组 | 英文名 | 最大通道数 | 数据寄存器 | 特点 |
|--------|--------|:----------:|:----------:|------|
| **规则通道组** | Regular | **16** | **1 个 DR** | 常规使用 |
| **注入通道组** | Injection | **4** | **4 个 JDR** | 可插队 |

### 7.2 规则组 vs 注入组

| 对比项 | 规则通道组 | 注入通道组 |
|--------|:----------:|:----------:|
| 最大通道数 | 16 | 4 |
| 数据寄存器 | 1 个 DR（可能覆盖） | 4 个 JDR（各自独立） |
| 可否插队 | ❌ | ✅ 可打断规则组 |
| 一般使用 | ✅ 主要使用 | 较少使用 |

### 7.3 数据覆盖问题（规则组）

```
问题：多个通道共用 1 个 DR → 后续转换覆盖前面结果

解决方案：
① 标志位/中断 + 及时读取
② DMA 自动传输（推荐）→ 转换完成自动触发 DMA 请求

DMA 通道对应：
  ADC1 → DMA1 Channel 1 ✅
  ADC2 → 无 DMA 支持     ❌
  ADC3 → DMA2 Channel 5 ✅
```

## 八、转换方式（SCAN + CONT）

### 8.1 两个关键控制位

| 寄存器 | 位 | 含义 |
|--------|:--:|------|
| CR1 | **SCAN** | 扫描模式（多通道顺序播放） |
| CR2 | **CONT** | 连续转换（循环播放） |

### 8.2 四种组合模式

| SCAN | CONT | 模式 | 歌单类比 | 行为 |
|:----:|:----:|------|---------|------|
| 0 | 0 | **单次单通道** | 单曲播放一次 | 只转换第一个通道，转换完停止 |
| 1 | 0 | **单次扫描** | 顺序播放一次 | 所有通道依次转换一次，转换完停止 |
| 0 | 1 | **连续单通道** | **单曲循环** | 只转换第一个通道，不停重复 |
| 1 | 1 | **连续扫描** | **列表循环** | 所有通道依次转换，不停循环 |

## 九、事件、中断与 DMA

### 9.1 三类事件

| 事件 | 标志位 | 含义 | 中断使能位 |
|------|:------:|------|:----------:|
| **转换结束** | **EOC** | End Of Conversion | EOCIE |
| **注入转换结束** | **JEOC** | End Of Injected Conversion | JEOCIE |
| **模拟看门狗** | **AWD** | Analog WatchDog | AWDIE |

### 9.2 转换时间

```
总转换时间 = 采样时间 + 12.5 个 ADC 时钟周期

采样时间可配置：1.5, 7.5, 13.5, 28.5, 41.5, 55.5, 71.5, 239.5 周期

例：ADC 时钟 = 12MHz，采样时间 = 1.5 周期
  总周期数 = 1.5 + 12.5 = 14 个周期
  总时间 = 14 / 12MHz ≈ 1.17μs
```

## 十二、关键公式速查

```
分辨率：    离散级别数 = 2^n
最小变化量：ΔV = VREF / 2^n

数字量→电压：V = D × VREF / 2^n
电压→数字量：D = V × 2^n / VREF

采样定理：  fs ≥ 2 × f_max

转换时间：  T = 采样时间 + 12.5 个 ADC 时钟周期

STM32F103（12 位 ADC，VREF = 3.3V）：
  离散级别 = 4096
  最小分辨率 ≈ 0.806mV
  D = V × 4096 / 3.3
  V = D × 3.3 / 4096
```

# STM32 定时器全面总结 笔记

---

## 一、STM32 定时器分类

### 1.1 两大类

```
┌─────────────────────────────────────────────────┐
│                 STM32 定时器                      │
│                                                  │
│   ┌──────────────┐      ┌───────────────────┐   │
│   │  系统定时器    │      │   STM32 外设定时器  │   │
│   │  (SysTick)    │      │                   │   │
│   │  CM3 内核     │      │  ┌──────┐         │   │
│   │  24位向下递减  │      │  │基本  │ TIM6/7  │   │
│   │  功能单一     │      │  ├──────┤         │   │
│   │  定时/延时    │      │  │通用  │ TIM2~5  │   │
│   └──────────────┘      │  ├──────┤         │   │
│                          │  │高级  │ TIM1/8  │   │
│                          │  └──────┘         │   │
│                          └───────────────────┘   │
└─────────────────────────────────────────────────┘
```

### 1.2 功能对比

| 特性 | SysTick | 基本 (TIM6/7) | 通用 (TIM2~5) | 高级 (TIM1/8) |
|------|:-------:|:-------------:|:-------------:|:-------------:|
| 计数方向 | 向下 | 向上 | 向上/向下/中央对齐 | 向上/向下/中央对齐 |
| 计数位宽 | 24位 | 16位 | 16位 | 16位 |
| 通道数 | 0 | 0 | 4 | 4 |
| 输入捕获 | ❌ | ❌ | ✅ | ✅ |
| 输出比较/PWM | ❌ | ❌ | ✅ | ✅ |
| 互补输出 | ❌ | ❌ | ❌ | ✅ |
| RCR（重复计数器） | ❌ | ❌ | ❌ | ✅ |
| 刹车/死区 | ❌ | ❌ | ❌ | ✅ |
| 触发控制器/从模式 | ❌ | 仅触发输出 | ✅ | ✅ |
| 连接总线 | 内核 | APB1 | APB1 | APB2 |
| 最终时钟频率 | 72MHz | 72MHz | 72MHz | 72MHz |

### 1.3 时钟来源

```
高级定时器 TIM1/8 → APB2 高速总线 → 72MHz
通用定时器 TIM2~5 → APB1 低速总线 → 36MHz × 2 = 72MHz
基本定时器 TIM6/7 → APB1 低速总线 → 36MHz × 2 = 72MHz

最终所有定时器内部时钟都是 72MHz
原因：APB1 经过 2 分频后，定时器自动 ×2 补偿
```

## 二、系统滴答定时器 (SysTick)

### 2.1 特点

| 特性 | 说明 |
|------|------|
| 归属 | CM3 内核 |
| 计数方向 | **向下递减** |
| 位宽 | 24位 |
| 功能 | 定时、延时 |

### 2.2 关键寄存器

| 寄存器 | 功能 | 说明 |
|--------|------|------|
| **CTRL** | 控制和状态 | 使能、时钟源、中断使能、计数标志 |
| **LOAD** | 重装载值 | 设定初始计数值 |
| **VAL** | 当前值 | 读取当前计数值，写入则清零 |
| CALIB | 校准值 | 只读，出厂写入 |

### 2.3 CTRL 关键位

| 位 | 名称 | 功能 |
|:--:|------|------|
| 0 | ENABLE | 使能计数器 |
| 1 | TICKINT | 中断使能（数到 0 时产生中断） |
| 2 | CLKSOURCE | 0=外部时钟/8(9MHz)，1=内核时钟(72MHz) |
| 16 | COUNTFLAG | 计数到 0 时置 1 |

### 2.4 应用

```
① 轮询方式：while 循环判断 COUNTFLAG → 包装延时函数
② 中断方式：TICKINT=1 → 每次数到 0 进中断 → 全局变量累加
③ 操作系统心跳：维持 RTOS 的时间片调度
```

## 三、基本定时器 (TIM6/7)

### 3.1 框图核心

```
内部时钟 (72MHz)
    │
    ↓
┌─────┐      ┌─────┐      ┌─────┐
│ PSC │ ──→  │ CNT │ ──→  │ ARR │
│预分频│      │计数器│      │重装载│
└─────┘      └──┬──┘      └─────┘
                │
                ↓ 溢出
           更新事件 (UEV)
                │
        ┌───────┼───────┐
        ↓               ↓
   影子寄存器刷新     更新中断(UIE)
   (PSC/ARR 生效)
```

### 3.2 关键寄存器

| 寄存器 | 功能 | 影子寄存器 |
|--------|------|:----------:|
| **PSC** | 预分频 | ✅（无使能位，更新事件时刷新） |
| **ARR** | 自动重装载 | ✅（ARPE 位控制） |
| **CNT** | 计数器 | 无 |
| **CR1** | 控制（CEN 使能、ARPE、UDIS） | — |
| **DIER** | 中断使能（UIE） | — |
| **SR** | 状态（UIF 标志） | — |
| **EGR** | 事件生成（UG 手动产生更新事件） | — |

### 3.3 计数时序

```
启动计数器：
  ① 以 72MHz 快速计数（PSC 尚未生效）
  ② 数到 ARR 值 → 溢出 → 产生更新事件
  ③ 更新事件将 PSC/ARR 预装载值刷入影子寄存器
  ④ 之后以正确的分频频率继续计数

启动瞬间就产生更新事件（几乎瞬间完成）
→ 如果开启了 UIE，会立即进入一次中断
```

### 3.4 预装载机制

```
ARR 的预装载（ARPE 位控制）：
  ARPE = 0：写 ARR 立即生效（直通影子寄存器）
  ARPE = 1：写 ARR 先存预装载 → 更新事件时刷新到影子

PSC 的预装载（无使能位）：
  永远通过预装载 → 更新事件时刷新到影子寄存器
```

### 3.5 事件 vs 中断

| 对比项 | 事件 | 中断 |
|--------|------|------|
| 处理方式 | **硬件**直接完成 | **软件**（CPU 执行指令） |
| 响应速度 | 瞬间（1 个时钟周期） | 延迟（若干时钟周期） |
| 编程可控性 | 不可控（硬件自动） | 可控（编写中断函数） |
| 使能 | 事件屏蔽寄存器 | 中断使能寄存器 + NVIC |

### 3.6 应用

```
→ LED 定时闪烁
→ 精确延时函数
→ 周期性任务调度
→ 与通用定时器类似但功能更简单
```

## 四、通用定时器 (TIM2~5)

### 4.1 功能概览

```
通用定时器 = 基本定时器功能
           + 输入捕获
           + 输出比较/PWM
           + 触发控制器/从模式
           + 编码器接口
           + 多种时钟源
```

```
┌──────────────────────────────────────────────────────┐
│                    通用定时器                          │
│                                                      │
│  时钟源：                                             │
│    内部时钟 (72MHz)                                   │
│    外部引脚 (ETR)                                     │
│    通道输入 (TI1/TI2)                                │
│    其他定时器 (ITR0~3)                                │
│         │                                            │
│         ↓                                            │
│  ┌─────────────┐    ┌─────┐    ┌─────┐              │
│  │ 触发控制器    │───→│ PSC │───→│ CNT │───→ ARR      │
│  │ TRGI/TRGO   │    └─────┘    └──┬──┘              │
│  │ SMS/TS      │                  │                  │
│  └─────────────┘                  │                  │
│         ↕                         ↓                  │
│  ┌─────────────────────────────────────────────┐     │
│  │          4 个输入/输出通道                     │     │
│  │                                             │     │
│  │  CH1  CH2  CH3  CH4                        │     │

│  │  (CCMR1/2 配置)    (CCER 配置)              │     │
│  └─────────────────────────────────────────────┘     │
└──────────────────────────────────────────────────────┘
```

### 4.3 时钟源

| 来源 | 信号 | 说明 |
|------|------|------|
| 内部时钟 | CK_INT | 72MHz |
| 外部引脚 | ETR | 专用外部触发引脚 |
| 通道输入 | TI1/TI2 | CH1/CH2 引脚输入 |
| 内部级联 | ITR0~3 | 其他定时器的 TRGO（内部固定连线） |

### 4.4 输出比较 / PWM 输出

#### 核心思想

```
CCR（比较值）与 CNT（计数器值）比较
→ 根据比较结果控制输出引脚电平
→ 产生 PWM 方波
```

#### 八种输出模式

| OCM[2:0] | 模式 | 功能 | PWM 能力 |
|:--------:|------|------|:--------:|
| 000 | 冻结 | 输出不受比较影响 | ❌ |
| 001 | 匹配强制高 | CNT=CCR 后输出高 | ❌ |
| 010 | 匹配强制低 | CNT=CCR 后输出低 | ❌ |
| 011 | 匹配翻转 | CNT=CCR 时翻转 | ⚠️ 50% |
| 100 | 强制低 | 无条件输出低 | ❌ |
| 101 | 强制高 | 无条件输出高 | ❌ |
| **110** | **PWM 模式 1** | **CNT<CCR→有效电平** | **✅** |
| **111** | **PWM 模式 2** | **CNT<CCR→无效电平** | **✅** |

#### PWM 模式 1 详解

```
PWM 模式 1：CNT < CCR → 输出有效电平

极性 CCxP = 0（高电平有效）时：
  CNT < CCR → 高电平
  CNT ≥ CCR → 低电平

占空比 = CCR / (ARR+1) × 100%
PWM 频率 = 时钟频率 / ((PSC+1) × (ARR+1))
```

#### 关键寄存器

| 寄存器 | 位 | 功能 |
|--------|:--:|------|
| CCMR1/2 | OCxM[2:0] | 输出模式选择 |
| CCMR1/2 | CCxS = 00 | 选择输出模式 |
| CCER | CCxP | 极性（高/低电平有效） |
| CCER | CCxE | 输出使能 |
| CCRx | — | 比较值（占空比） |
| CR1 | CEN | 定时器使能 |
| BDTR | MOE | **高级定时器**主输出使能 |

#### 应用

```
→ LED 呼吸灯：循环改变 CCR 值
→ 电机速度控制：改变占空比
→ DAC 模拟：PWM + 滤波
→ 信号发生器
```

### 4.5 输入捕获

#### 核心思想

```
信号边沿到来时 → 硬件自动将 CNT 值锁存到 CCR
→ "快照"机制：边沿是快门，CNT 是被拍摄对象，CCR 是照片
```

#### 信号通路

```
CHx 引脚 → TIx → 滤波(ICxF) → 边沿检测(CCxP) → TIxFPx
                                                    │
                          ┌─────────────────────────┤
                          ↓                         ↓
                    多路选择(CCxS)            触发控制器
                    ┌─────┴─────┐             (TS/SMS)
                    ↓           ↓
                 直通(01)    交叉(10)
                    ↓           ↓
              输入预分频(ICxPSC)
                    ↓
              CCRx（捕获事件锁存 CNT 值）
```

#### 关键寄存器

| 寄存器 | 位 | 功能 |
|--------|:--:|------|
| CCMR1/2 | CCxS[1:0] | 01=直通，10=交叉 |
| CCMR1/2 | ICxF[3:0] | 输入滤波 |
| CCMR1/2 | ICxPSC[1:0] | 输入预分频 |
| CCER | CCxP | 极性（0=上升沿，1=下降沿） |
| CCER | CCxE | 捕获使能 |
| CCRx | — | 存储捕获值 |

#### 应用

```
→ 测量信号周期和频率
→ 测量脉冲宽度
→ 测量 PWM 占空比（PWM 输入模式）
→ 超声波测距
```

### 4.6 触发输入与从模式

#### 触发输入源（SMCR.TS，8 种选择）

| TS[2:0] | 来源 | 说明 |
|:-------:|------|------|
| 000~011 | ITR0~3 | 其他定时器（内部固定连线） |
| 100 | TI1F_ED | 通道 1 **双边沿**检测 |
| **101** | **TI1FP1** | 通道 1 滤波+极性后信号 |
| 110 | TI2FP2 | 通道 2 滤波+极性后信号 |
| 111 | ETRF | 外部触发引脚 |

#### 从模式（SMCR.SMS，8 种模式）

| SMS[2:0] | 模式 | 功能 |
|:--------:|------|------|
| **000** | **关闭** | 默认，内部时钟 |
| 001~011 | 编码器模式 | 旋转编码器接口 |
| **100** | **复位模式** | TRGI 上升沿 → CNT 清零 |
| 101 | 门控模式 | TRGI 高=使能，低=停止 |
| 110 | 触发模式 | TRGI 上升沿 → 启动计数 |
| **111** | **外部时钟** | TRGI 上升沿 → CNT 加 1 |

#### 主从模式

```
主模式：本定时器 TRGO 输出 → 驱动其他定时器
从模式：其他信号通过 TRGI → 控制本定时器行为

级联关系由内部连线固定（查表确定 ITR 对应关系）
```

### 4.7 PWM 输入模式（同时测周期和占空比）

#### 原理

```
同一信号从 CH1 输入，分配到两个通道：

通道 1（直通 CC1S=01）：
  上升沿检测（CC1P=0）→ CCR1 = 周期 T

通道 2（交叉 CC2S=10）：
  下降沿检测（CC2P=1）→ CCR2 = 高电平时间 t

从模式复位（SMCR: TS=101, SMS=100）：
  TI1FP1 上升沿 → CNT 自动清零

占空比 = CCR2 / CCR1 × 100%
频率 = 1,000,000 / CCR1 Hz
```

#### 时序

```
PWM 信号：  ┌─────┐              ┌─────┐
            │     │              │     │
       ─────┘     └──────────────┘     └─────
            ↑     ↑              ↑
           ①    ②             ③

① 上升沿：CCR1 捕获(上次周期) + CNT 复位 = 0
② 下降沿：CCR2 捕获(CNT=高电平时间)
③ 上升沿：CCR1 捕获(本次周期) + CNT 复位 = 0
```

## 五、高级定时器 (TIM1/8)

### 5.1 新增功能

| 功能 | 说明 | 寄存器 |
|------|------|--------|
| **RCR（重复计数器）** | 控制更新事件产生频率 | RCR |
| **互补输出** | CHx/CHxN 互补信号 | — |
| **死区时间** | 防止互补输出同时导通 | BDTR.DTG |
| **刹车输入（Break）** | 外部信号强制关断输出 | BDTR.BKE/BKP |
| **MOE（主输出使能）** | 总开关，控制所有输出 | BDTR.MOE |
| **空闲状态** | MOE 关闭时输出引脚默认电平 | CR2.OISx |
| **专用中断入口** | TIM1_UP_IRQHandler | — |

### 5.2 RCR（重复计数器）

```
高级定时器特有，8 位寄存器

工作原理：
  CNT 溢出 → RCR 减 1（不产生更新事件）
  RCR 减到 0 → 再次溢出时 → 产生更新事件

更新事件频率 = 溢出频率 / (RCR + 1)
闪烁次数 = RCR + 1

例：RCR=4 → 5 次溢出后产生一次更新事件 → 闪 5 次
```

### 5.3 互补输出与死区时间

```
互补输出：CH1 高时 CH1N 低，CH1 低时 CH1N 高
死区时间：切换时插入延迟，防止同时导通（H 桥短路）

应用场景：电机驱动（H 桥电路）
```

### 5.4 刹车输入

```
外部信号 → Break 输入 → 检测极性
  → 立刻关断所有输出（MOE 清零）
  → 输出空闲电平（由 OISx 位定义）

应用场景：紧急停止（如电机过流保护）
```

### 5.5 MOE 与空闲状态

```
MOE = 1：主输出使能，正常输出 PWM
MOE = 0：主输出关闭，输出空闲电平

空闲状态由 CR2.OISx 位控制：
  OISx = 0（Reset）：空闲时输出低电平
  OISx = 1（Set）：空闲时输出高电平

本实验：定时器停止后需要 LED 熄灭
  → 空闲状态设为高电平（Set）
  → 极性设为低电平有效
```

## 七、CCMR1/2 详解

### 7.1 结构

```
CCMR1（16位）：
  低 8 位 [7:0]  → 通道 1 配置
  高 8 位 [15:8] → 通道 2 配置

CCMR2（16位）：
  低 8 位 → 通道 3
  高 8 位 → 通道 4

每个 8 位根据 CCxS 有两种定义：
  CCxS = 00 → 输出模式（OC: OCxM、OCxPE、OCxFE）
  CCxS ≠ 00 → 输入模式（IC: ICxF、ICxPSC）
```

### 7.2 输出模式位

| 位 | 功能 |
|:--:|------|
| CCxS[1:0] | **00** = 输出 |
| OCxM[2:0] | 输出模式（000~111 八种） |
| OCxPE | 输出预装载使能 |
| OCxFE | 输出快速使能 |

### 7.3 输入模式位

| 位 | 功能 |
|:--:|------|
| CCxS[1:0] | **01**=直通，**10**=交叉，11=TRC |
| ICxPSC[1:0] | 输入预分频（00/01/10/11） |
| ICxF[3:0] | 输入滤波器 |

---

## 八、CCER 详解

| 位 | 输出模式含义 | 输入模式含义 |
|:--:|------------|------------|
| CCxP | 有效电平（0=高有效，1=低有效） | 捕获边沿（0=上升沿，1=下降沿） |
| CCxE | 输出使能 | 捕获使能 |

---

## 九、SMCR 详解

| 位 | 功能 |
|:--:|------|
| SMS[2:0] | 从模式选择（000=关闭，100=复位，111=外部时钟） |
| TS[2:0] | 触发源选择（8 种来源） |
| MSM | 主/从模式 |

## 十三、核心概念速查

| 概念 | 说明 |
|------|------|
| **影子寄存器** | 真正工作的寄存器，由预装载寄存器在更新事件时刷新 |
| **预装载寄存器** | CPU 写入的缓冲区 |
| **更新事件** | CNT 溢出时产生，刷新影子寄存器 |
| **更新中断** | 更新事件 + UIE 使能 → 进入中断服务函数 |
| **UG** | 手动产生更新事件（EGR 寄存器） |
| **PWM 模式 1** | CNT < CCR → 有效电平 |
| **有效电平** | 由 CCxP 极性决定（默认高电平有效） |
| **直通模式** | CCxS=01，本通道信号给到本通道 CCR |
| **交叉模式** | CCxS=10，其他通道信号给到本通道 CCR |
| **从模式复位** | TRGI 上升沿 → CNT 自动清零 |
| **RCR** | 高级定时器重复计数器，控制更新事件频率 |
| **MOE** | 高级定时器主输出使能（总开关） |
| **空闲状态** | MOE 关闭时输出引脚的默认电平 |

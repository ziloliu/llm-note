# STM32 基本定时器寄存器详解与 LED 闪烁实现 笔记

---

## 一、实验需求

| 需求 | 说明 |
|------|------|
| 功能 | LED2（PA1）每隔 1 秒闪烁一次 |
| 实现方式 | 基本定时器 TIM6 **中断方式** |
| 定时时间 | 1 秒 |

## 二、涉及的寄存器总览

| 寄存器 | 全称 | 位宽 | 功能 |
|--------|------|:----:|------|
| **CR1** | Control Register 1 | 16位 | 控制寄存器（ARPE、CEN 等） |
| **CR2** | Control Register 2 | 16位 | 控制寄存器 2（MMS 主模式） |
| **DIER** | DMA/Interrupt Enable | 16位 | DMA 和中断使能 |
| **SR** | Status Register | 16位 | 状态寄存器（UIF 标志） |
| **EGR** | Event Generation | 16位 | 事件产生寄存器（UG） |
| **PSC** | Prescaler | 16位 | 预分频器 |
| **ARR** | Auto-Reload Register | 16位 | 自动重装载寄存器 |
| **CNT** | Counter | 16位 | 计数器 |

## 三、各寄存器关键位详解

### 3.1 CR1 — 控制寄存器 1

| 位 | 名称 | 功能 | 本实验 |
|:--:|------|------|:------:|
| 0 | **CEN** | 计数器使能 | **置 1**（最后开启） |
| 1 | UDIS | 禁止更新 | 0（不禁止） |
| 2 | URS | 更新请求源选择 | 0 |
| 3 | OPM | 单脉冲模式 | 0 |
| 7 | **ARPE** | 自动重装载预装载使能 | **0 或 1** |

#### CEN — 计数器使能（位 0）

```
CEN = 0：计数器关闭（复位默认）
CEN = 1：计数器开启，开始向上递增计数
```

#### ARPE — 预装载使能（位 7）

```
ARPE = 0：无预装载，写入 ARR 后立即生效（写预加载 → 立即刷到影子）
ARPE = 1：有预装载，写入 ARR 后等待更新事件才生效

本实验：0 即可（初始化时不需要动态修改 ARR）
```

### 3.2 DIER — DMA/中断使能寄存器

| 位 | 名称 | 功能 | 本实验 |
|:--:|------|------|:------:|
| 0 | **UIE** | 更新中断使能 | **置 1** |
| 8 | UDE | 更新 DMA 请求使能 | 0 |

```
UIE = 0：更新事件不产生中断
UIE = 1：更新事件产生中断 → 进入 TIM6_IRQHandler
```

### 3.3 SR — 状态寄存器

| 位 | 名称 | 功能 | 操作 |
|:--:|------|------|------|
| 0 | **UIF** | 更新中断标志 | 中断处理后**软件写 0 清除** |

```
UIF = 1：更新事件发生，硬件自动置位
UIF 清除方式：软件写 0（不是读操作清除！）

⚠️ 中断服务函数中必须手动清除 UIF，否则会反复进入中断
```

### 3.4 EGR — 事件产生寄存器

| 位 | 名称 | 功能 | 操作 |
|:--:|------|------|------|
| 0 | **UG** | 产生更新事件 | 软件写 1 触发 |

```
写 UG = 1 的效果：
  → 立即产生一个更新事件
  → 计数器归零（重新初始化）
  → 所有预加载寄存器的值刷新到影子寄存器（即时生效）
  → 包括 PSC 和 ARR 的影子寄存器都会被更新

用途：
  → 初始化后手动触发一次 UG，确保所有配置立即生效
  → 运行中需要立即改变 PSC/ARR 时使用
```

### 3.5 PSC — 预分频器

| 位 | 名称 | 功能 |
|:--:|------|------|
| 15:0 | PSC | 预分频值（16 位） |

```
实际分频系数 = PSC + 1

写入 PSC 后不会立即生效
→ 需要等待更新事件（溢出或手动 UG）才能刷新到影子寄存器
→ PSC 没有 ARPE 控制位，默认就是预装载模式
```

### 3.6 ARR — 自动重装载寄存器

| 位 | 名称 | 功能 |
|:--:|------|------|
| 15:0 | ARR | 自动重装载值（16 位） |

```
计数器从 0 递增到 ARR → 下一个时钟沿溢出 → 产生更新事件

ARR 底层有两个寄存器：
  预加载寄存器 ← CPU 直接写入
  影子寄存器   ← 真正参与比较的值（由 ARPE 控制更新时机）

溢出条件：CNT 超过 ARR
从 0 计到溢出共经过：ARR + 1 个计数周期
```

### 3.7 CNT — 计数器

| 位 | 名称 | 功能 |
|:--:|------|------|
| 15:0 | CNT | 当前计数值（16 位，只读/可写） |

```
每个计数器时钟上升沿自动加 1
从 0 开始向上递增
达到 ARR 后下一个时钟沿溢出归零
```

## 四、PSC 影子寄存器说明

```
功能框图中 PSC 下方有阴影 → 表示存在影子寄存器

结构：
  ┌─────────────┐
  │ 预加载寄存器  │ ← CPU 写入的值
  └──────┬──────┘
         │ 更新事件到来时刷新
         ↓
  ┌─────────────┐
  │ 影子寄存器    │ ← 真正参与分频的值
  └─────────────┘

PSC 没有 ARPE 控制位 → 默认只有更新事件才能刷新
  → 写入新 PSC 值后不会立即生效
  → 需要等到下次溢出或手动触发 UG 才生效

解决方法：写入 PSC 后手动触发 UG（EGR.UG = 1）
  → 立即产生更新事件 → PSC 影子寄存器立即刷新
```

## 五、定时参数计算

### 5.1 设计思路

```
目标：定时 1 秒

方案：PSC = 7199, ARR = 9999

验证：
  ① 计数器时钟频率 = 72MHz / (7199 + 1) = 72MHz / 7200 = 10,000 Hz
  ② 计数器周期 = 1 / 10,000 = 0.1 ms = 100 μs
  ③ 定时时间 = 100 μs × (9999 + 1) = 100 μs × 10,000 = 1,000,000 μs = 1 秒 ✅
```

### 5.2 为什么 PSC = 7199 而不是 72000

```
PSC 是 16 位寄存器，最大值 = 65535
  → 最大分频值 = 65535 + 1 = 65536

如果 PSC = 72000 - 1 = 71999：
  → 分频值 = 72000
  → 72MHz / 72000 = 1000 Hz（1kHz）
  → 每次计数 = 1ms
  → ARR 需要 = 999 → 1000 次 × 1ms = 1s

  ⚠️ 但 71999 < 65535 ❌ 超出 16 位范围！

解决：降低分频值
  PSC = 7199 → 分频值 = 7200
  → 72MHz / 7200 = 10000 Hz（10kHz）
  → 每次计数 = 100μs = 0.1ms
  → ARR = 9999 → 10000 次 × 0.1ms = 1s ✅
```

### 5.3 两种参数方案对比

| 方案 | PSC | 分频值 | 计数频率 | ARR | 计数次数 | 定时时间 |
|------|-----|:------:|:--------:|:---:|:--------:|:--------:|
| A | 71999 | 72000 | 1 kHz | 999 | 1000 | **1s** |
| B | 7199 | 7200 | 10 kHz | 9999 | 10000 | **1s** |
| C | 71 | 72 | 1 MHz | 999999 | — | 超出 16 位 ❌ |

> 方案 A 的 PSC = 71999 超出 16 位范围（65535），**不可用**。
> 方案 B **可行**，PSC 和 ARR 均在 16 位范围内。

## 六、初始化配置流程

```
① 开启 TIM6 时钟
   RCC->APB1ENR |= RCC_APB1ENR_TIM6EN;

② 配置预分频器 PSC
   TIM6->PSC = 7199;           // 7200 分频 → 10kHz

③ 配置自动重装载值 ARR
   TIM6->ARR = 9999;           // 10000 次计数 → 1s

④ 使能更新中断
   TIM6->DIER |= TIM_DIER_UIE; // UIE = 1

⑤ 使能计数器（最后）
   TIM6->CR1 |= TIM_CR1_CEN;   // CEN = 1
```

## 七、中断服务函数

### 7.1 中断服务函数名称

```c
// 在启动文件 startup_stm32f103xx.s 中查找

DCD     TIM6_IRQHandler    ; TIM6

// TIM6 和 TIM7 各有独立的中断服务函数
```

### 7.2 中断服务函数实现

```c
void TIM6_IRQHandler(void)
{
    // ① 检查 UIF 标志位
    if (TIM6->SR & TIM_SR_UIF)
    {
        // ② 清除 UIF 标志（软件写 0）
        TIM6->SR &= ~TIM_SR_UIF;

        // ③ 执行用户操作
        LED2_Toggle();
    }
}
```

### 7.3 UIF 清除方式

| 操作 | 说明 |
|------|------|
| 硬件置位 | 更新事件发生时 UIF = 1（自动） |
| 软件清除 | `TIM6->SR &= ~TIM_SR_UIF;`（手动写 0） |
| ⚠️ 不清除 | 中断会反复触发，程序卡死在中断中 |

## 八、NVIC 配置

```
基本定时器是外设中断，需要手动配置 NVIC（与 SysTick 不同）

NVIC 配置：
  ① 使能 TIM6 中断通道
  ② 设置抢占优先级和子优先级
```

### NVIC 配置代码（寄存器方式）

```c
// 在 stm32f10x.h 中查找中断号
// TIM6_IRQn = 43（不同型号可能不同）

NVIC_SetPriority(TIM6_IRQn, 1);    // 设置优先级
NVIC_EnableIRQ(TIM6_IRQn);          // 使能中断
```

### 9.2 basic_timer.h

```c

#define __BASIC_TIMER_H

#include "stm32f10x.h"

void BasicTimer_Init(void);

#endif
```

### 9.3 basic_timer.c

```c
#include "basic_timer.h"
#include "led.h"

void BasicTimer_Init(void)
{
    /* ① 开启 TIM6 时钟 */
    RCC->APB1ENR |= RCC_APB1ENR_TIM6EN;

    /* ② 配置预分频器：7200 分频 → 10kHz */
    TIM6->PSC = 7199;

    /* ③ 配置自动重装载值：10000 次计数 → 1s */
    TIM6->ARR = 9999;

    /* ④ 使能更新中断 */
    TIM6->DIER |= TIM_DIER_UIE;

    /* ⑤ 使能计数器（最后） */
    TIM6->CR1 |= TIM_CR1_CEN;

    /* ⑥ 配置 NVIC */
    NVIC_SetPriority(TIM6_IRQn, 1);
    NVIC_EnableIRQ(TIM6_IRQn);
}

/* 中断服务函数 */
void TIM6_IRQHandler(void)
{
    if (TIM6->SR & TIM_SR_UIF)
    {
        TIM6->SR &= ~TIM_SR_UIF;   // 清除标志位
        LED2_Toggle();               // 翻转 LED2
    }
}
```

### 9.4 main.c

```c
#include "stm32f10x.h"
#include "led.h"
#include "basic_timer.h"

int main(void)
{
    LED_Init();              // 初始化 LED
    BasicTimer_Init();       // 初始化 TIM6

    while (1)
    {
        // 空循环，LED 翻转由中断驱动
    }
}
```


## 十一、SysTick vs 基本定时器实现对比

| 对比项 | SysTick（中断方式） | TIM6（中断方式） |
|--------|:------------------:|:---------------:|
| 预分频器 | **无** | **有**（PSC） |
| 位宽 | 24 位 | 16 位 |
| 计数方向 | 向下 | **向上** |
| 直接定时 1s | 不行（超出 24 位） | **可以**（PSC + ARR 配合） |
| 需要软件计数器 | 需要（counter 计到 1000） | **不需要** |
| NVIC 配置 | 不需要（内核自动） | **需要手动配置** |
| 清除标志 | 硬件自动 | **软件手动清除 UIF** |
| 中断函数名 | SysTick_Handler | **TIM6_IRQHandler** |

## 十二、PSC 与 ARR 的选择策略

### 12.1 基本原则

```
① PSC + 1 和 ARR + 1 都不能超过 65536（16 位限制）
② (PSC + 1) × (ARR + 1) = 定时时间 × 时钟频率
③ 选择整数拆分，避免出现小数
```

### 12.2 常用定时参数速查

| 定时时间 | PSC | 分频值 | ARR | 计数次数 | 验证 |
|----------|:---:|:------:|:---:|:--------:|------|
| 100 μs | 71 | 72 | 9999 | 10000 | 72/72M × 10000 = 1ms ❌ |
| 100 μs | 71 | 72 | 999 | 1000 | 72/72M × 1000 = 100μs ✅ |
| 1 ms | 7199 | 7200 | 9 | 10 | 7200/72M × 10 = 1ms ✅ |
| 10 ms | 7199 | 7200 | 99 | 100 | 7200/72M × 100 = 10ms ✅ |
| 100 ms | 7199 | 7200 | 999 | 1000 | 7200/72M × 1000 = 100ms ✅ |
| **1 s** | **7199** | **7200** | **9999** | **10000** | **7200/72M × 10000 = 1s ✅** |

|--------|:--:|------|:----:|------|

# AT24C02 (M24C02) 接口层实现 — 读写字节 笔记

---

## 一、分层架构

```
┌──────────────────────────────────────────┐
│          应用层（main.c）                  │
│   写入数据 → 读出数据 → 串口打印验证        │
├──────────────────────────────────────────┤
│     接口层（m24c02.c / AT24C02 操作）      │  ← 本次实现
│   EEPROM_WriteByte / EEPROM_ReadByte      │
│   假写真读、页写入、连续读写等              │
├──────────────────────────────────────────┤
│     协议层（i2c.c / I²C 协议）             │  ← 已完成
│   起始/停止、收发字节、应答/非应答           │
├──────────────────────────────────────────┤
│        硬件层（GPIO + 上拉电阻）            │
└──────────────────────────────────────────┘
```

> 接口层不直接操作 GPIO 电平，而是调用协议层（I²C）的函数，按 AT24C02 的读写时序组合成完整的操作流程。

## 二、m24c02.h 头文件

### 2.1 宏定义

```c
#define W_ADDR    0xA0    // 写地址：设备地址 0x50 + 写(0) = 0xA0
#define R_ADDR    0xA1    // 读地址：设备地址 0x50 + 读(1) = 0xA1
```

### 2.2 函数声明

| 函数 | 功能 | 参数 | 返回值 |
|------|------|------|--------|
| `EEPROM_Init()` | 初始化 | void | void |
| `EEPROM_WriteByte(addr, byte)` | 写入一个字节 | 内部地址 + 数据 | void |
| `EEPROM_ReadByte(addr)` | 读取一个字节 | 内部地址 | uint8_t |
| `EEPROM_WriteBytes(addr, data, size)` | 连续写入多个字节 | 内部地址 + 数据指针 + 长度 | void |
| `EEPROM_ReadBytes(addr, buffer, size)` | 连续读取多个字节 | 内部地址 + 缓冲区 + 长度 | void |

## 三、EEPROM_WriteByte() — 写入一个字节

### 3.1 时序图

```
[S] → [0xA0(W)] → ACK → [内部地址] → ACK → [数据] → ACK → [P]
                                                          等待 5ms
```

### 3.2 步骤对照

| 步骤 | 操作 | 代码 |
|:----:|------|------|
| ① | 发出起始信号 | `I2C_Start();` |
| ② | 发送写地址 0xA0 | `I2C_SendByte(W_ADDR);` |
| ③ | 等待从设备应答 | `I2C_WaitAck();` |
| ④ | 发送内部存储地址 | `I2C_SendByte(inner_addr);` |
| ⑤ | 等待从设备应答 | `I2C_WaitAck();` |
| ⑥ | 发送写入数据 | `I2C_SendByte(byte);` |
| ⑦ | 等待从设备应答 | `I2C_WaitAck();` |
| ⑧ | 发出停止信号 | `I2C_Stop();` |
| ⑨ | 延迟 5ms（写入周期） | `Delay_ms(5);` |

### 3.3 代码实现

```c
void EEPROM_WriteByte(uint8_t inner_addr, uint8_t byte)
{
    // ① 起始信号
    I2C_Start();

    // ② 发送写地址（设备地址 + 写方向）
    I2C_SendByte(W_ADDR);

    // ③ 等待应答
    I2C_WaitAck();

    // ④ 发送内部存储地址
    I2C_SendByte(inner_addr);

    // ⑤ 等待应答
    I2C_WaitAck();

    // ⑥ 发送要写入的数据
    I2C_SendByte(byte);

    // ⑦ 等待应答
    I2C_WaitAck();

    // ⑧ 停止信号
    I2C_Stop();

    // ⑨ 等待写入周期（5ms）
    Delay_ms(5);
}
```

### 3.4 数据流

```
STM32(主) ──→ S ──→ 0xA0 ──→ inner_addr ──→ byte ──→ P
                     ←ACK──  ←ACK──      ←ACK──
AT24C02(从)
```

## 四、EEPROM_ReadByte() — 读取一个字节（假写真读）

### 4.1 时序图

```
假写阶段：
[S] → [0xA0(W)] → ACK → [内部地址] → ACK

真读阶段：
[Sr] → [0xA1(R)] → ACK → [从设备发送数据] → 主机 NACK → [P]
```

### 4.2 步骤对照

| 步骤 | 操作 | 代码 | 说明 |
|:----:|------|------|------|
| ① | 起始信号 | `I2C_Start();` | 开始假写阶段 |
| ② | 发送写地址 0xA0 | `I2C_SendByte(W_ADDR);` | 假写：用写地址保持总线控制权 |
| ③ | 等待应答 | `I2C_WaitAck();` | |
| ④ | 发送内部地址 | `I2C_SendByte(inner_addr);` | 写入地址计数器 |
| ⑤ | 等待应答 | `I2C_WaitAck();` | 假写阶段结束 |
| ⑥ | **重复起始** | `I2C_Start();` | Sr：不发 P，保持总线占用 |
| ⑦ | 发送读地址 0xA1 | `I2C_SendByte(R_ADDR);` | 真读：切换为读方向 |
| ⑧ | 等待应答 | `I2C_WaitAck();` | 总线控制权交给从设备 |
| ⑨ | 读取数据 | `byte = I2C_ReceiveByte();` | 从设备发送数据 |
| ⑩ | 发送非应答 | `I2C_SendNack();` | 只读1字节，告诉从设备"够了" |
| ⑪ | 停止信号 | `I2C_Stop();` | 结束通讯 |
| — | 返回数据 | `return byte;` | |

### 4.3 代码实现

```c
uint8_t EEPROM_ReadByte(uint8_t inner_addr)
{
    uint8_t byte;

    // ===== 假写阶段：设置地址计数器 =====

    // ① 起始信号
    I2C_Start();

    // ② 发送写地址（假写，保持主设备总线控制权）
    I2C_SendByte(W_ADDR);

    // ③ 等待应答
    I2C_WaitAck();

    // ④ 发送内部存储地址（写入地址计数器）
    I2C_SendByte(inner_addr);

    // ⑤ 等待应答
    I2C_WaitAck();

    // ===== 真读阶段：从指定地址读取数据 =====

    // ⑥ 重复起始信号（Sr，不发 P）
    I2C_Start();

    // ⑦ 发送读地址（切换为读方向）
    I2C_SendByte(R_ADDR);

    // ⑧ 等待应答（总线控制权交给从设备）
    I2C_WaitAck();

    // ⑨ 读取一个字节数据
    byte = I2C_ReceiveByte();

    // ⑩ 主机发送非应答（只读1字节，不需要更多数据）
    I2C_SendNack();

    // ⑪ 停止信号
    I2C_Stop();

    return byte;
}
```

### 4.4 数据流

```
假写（主设备控制 SDA）：
  STM32 ──→ S ──→ 0xA0(W) ──→ inner_addr
                ←ACK──       ←ACK──
  AT24C02

真读（从设备控制 SDA）：
  STM32 ──→ Sr ──→ 0xA1(R) ──→ [从设备发送 byte] ──→ NACK ──→ P
                 ←ACK──       ↑
  AT24C02 ─────────────────────┘（SDA 交给从设备）
```

## 五、WriteByte 与 ReadByte 对比

| 对比项 | WriteByte | ReadByte |
|--------|-----------|----------|
| 起始信号 | 1 个 S | 2 个（S + Sr） |
| 设备地址 | 1 次（0xA0 写） | 2 次（0xA0 假写 + 0xA1 真读） |
| 内部地址 | 发送后继续发数据 | 发送后不发数据（假写结束） |
| 数据方向 | 始终主→从 | 假写：主→从；真读：从→主 |
| 最后应答 | 从设备 ACK | 主设备 **NACK** |
| 结束 | P + 5ms 延迟 | P |
| 调用 I²C 函数数 | 3 次 SendByte | 3 次 SendByte + 1 次 ReceiveByte |

## 六、EEPROM_Init() — 初始化

```c
void EEPROM_Init(void)
{
    I2C_Init();    // 直接调用 I²C 协议层初始化（GPIO 配置）
}
```

> AT24C02 本身无需额外初始化操作，只需确保 I²C 总线（GPIO）已正确配置。

## 七、接口层与协议层的调用关系

```
EEPROM_WriteByte(inner_addr, byte)
    ├── I2C_Start()           ← 协议层
    ├── I2C_SendByte(W_ADDR)  ← 协议层
    ├── I2C_WaitAck()         ← 协议层
    ├── I2C_SendByte(addr)    ← 协议层
    ├── I2C_WaitAck()         ← 协议层
    ├── I2C_SendByte(byte)    ← 协议层
    ├── I2C_WaitAck()         ← 协议层
    ├── I2C_Stop()            ← 协议层
    └── Delay_ms(5)           ← 延时库

EEPROM_ReadByte(inner_addr)
    ├── I2C_Start()           ← 协议层（假写开始）
    ├── I2C_SendByte(W_ADDR)  ← 协议层
    ├── I2C_WaitAck()         ← 协议层
    ├── I2C_SendByte(addr)    ← 协议层
    ├── I2C_WaitAck()         ← 协议层
    ├── I2C_Start()           ← 协议层（Sr，真读开始）
    ├── I2C_SendByte(R_ADDR)  ← 协议层
    ├── I2C_WaitAck()         ← 协议层
    ├── I2C_ReceiveByte()     ← 协议层
    ├── I2C_SendNack()        ← 协议层
    ├── I2C_Stop()            ← 协议层
    └── return byte
```

## 九、关键注意点

| 注意点                | 说明                             |
| ------------------ | ------------------------------ |
| 假写真读是读取的关键         | 发读地址(R)后总线控制权立即交给从设备，必须先假写设置地址 |
| Sr（重复起始）不可省略       | 假写后不发 P，直接发 Sr 保持总线占用          |
| 只读 1 字节发 NACK      | 告诉从设备停止发送，若发 ACK 则从设备继续发下一字节   |
| 写入后必须等待 5ms        | AT24C02 写入周期，不等待则后续写入可能失败      |
| W_ADDR / R_ADDR 区别 | 仅最后一位不同（0=写，1=读）               |
| 接口层只负责时序组合         | 不操作 GPIO 电平，全部通过调用 I²C 协议层函数完成 |

# STM32 I²C 硬件外设寄存器详解 笔记

---

## 一、寄存器总览

| 寄存器 | 全称 | 位宽 | 功能 |
|--------|------|:----:|------|
| **CR1** | Control Register 1 | 16位 | 控制位：PE、ACK、START、STOP 等 |
| **CR2** | Control Register 2 | 16位 | 输入时钟频率 FREQ、中断/DMA 使能 |
| **OAR1** | Own Address Register 1 | — | 自身地址寄存器 1（从模式用） |
| **OAR2** | Own Address Register 2 | — | 自身地址寄存器 2（双地址模式） |
| **DR** | Data Register | 8位 | 数据寄存器（收发共用） |
| **SR1** | Status Register 1 | 16位 | 状态标志：SB、ADDR、BTF、TXE、RXNE 等 |
| **SR2** | Status Register 2 | 16位 | 状态标志：BUSY、MSL 等 |
| **CCR** | Clock Control Register | 16位 | SCL 时钟分频系数、标准/快速模式选择 |
| **TRISE** | Rise Time Register | — | SCL 最大上升沿时间 |

> 主模式下不需要 OAR1/OAR2（从设备才需要自己的地址）。

## 二、CR1 — 控制寄存器 1 关键位

### 2.1 位定义

| 位 | 名称 | 功能 | 备注 |
|:--:|------|------|------|
| 0 | **PE** | 外设使能 | **最后置 1** |
| 1 | SMBus | SMBus 模式选择 | 0=I²C，1=SMBus |
| 8 | **START** | 产生起始条件 | 置 1 产生，硬件自动清除 |
| 9 | **STOP** | 产生停止条件 | 配置项，字节传输结束后生效 |
| 10 | **ACK** | 应答使能 | 配置项：1=收到字节后发 ACK，0=不发(即 NACK) |
| 15 | SWRST | 软件复位 | |

### 2.2 各控制位工作方式

#### START 位

```
写 1 → 硬件产生起始信号
       ├── 主模式：重复产生起始条件，直到总线空闲
       └── 从模式：等待总线空闲后产生起始条件

硬件清除条件：
  ① 起始条件已发出（SB = 1）
  ② PE = 0（外设关闭）
```

#### STOP 位

```
写 1 → 不是立即发出停止信号！
       而是等待当前字节传输结束后再发出停止条件
       
理解：这是一个"准备停止"的配置项，而非即时按钮
```

#### ACK 位

```
ACK = 1：使能应答 → 收到一个字节后硬件自动返回 ACK（拉低 SDA）
ACK = 0：禁止应答 → 收到一个字节后不返回 ACK（等效 NACK）

理解：这是一个配置项，决定"收到字节后怎么做"
```

## 三、状态寄存器 SR1 — 关键标志位

### 3.1 关键位定义

| 位 | 名称 | 全称 | 含义 | 触发条件 |
|:--:|------|------|------|----------|
| 0 | **SB** | Start Bit | 起始条件已发出 | 硬件发出起始信号后置 1 |
| 1 | **ADDR** | Address sent | 地址已发送/匹配 | 7位：收到地址 ACK 后置 1 |
| 2 | **BTF** | Byte Transfer Finished | 字节传输完成 | 收到应答后置 1 |
| 6 | **RXNE** | RX Not Empty | 接收数据寄存器非空 | 收到一个字节后置 1 |
| 7 | **TXE** | TX Empty | 发送数据寄存器为空 | DR 数据移入移位寄存器后置 1 |

### 3.2 标志位用途

```
主设备发送流程：
  START → 检测 SB(起始已发) → 发地址 → 检测 ADDR(地址已发) 
  → 发数据 → 检测 TXE(DR空可写) → 写DR → 检测 BTF(字节传输完成+ACK)

主设备接收流程：
  START → 检测 SB → 发地址 → 检测 ADDR 
  → 检测 RXNE(收到数据) → 读DR

发送停止：
  STOP 位写 1 → 当前字节传输结束后自动发出停止信号
```

### 3.3 各标志位详解

#### SB — 起始位标志

```
置位条件：硬件成功发出起始条件
清除条件：读 SR1 + 写 DR（即开始发送数据时自动清除）

使用场景：
  START 位写 1 → while(!SB) 等待 → SB=1 说明起始信号已发出
  → 可以开始发送设备地址
```

#### ADDR — 地址发送标志

```
置位条件：
  7位地址模式：收到地址字节的 ACK 后置 1
  10位地址模式：收到第二个地址字节的 ACK 后置 1

清除条件：读 SR1 + 读 SR2

使用场景：
  发送设备地址后 → while(!ADDR) 等待 → ADDR=1 说明地址已被从设备确认
```

#### BTF — 字节传输完成标志

```
置位条件：收到应答信号后置 1（说明一个字节成功发送并被确认）
清除条件：读 SR1 + 读/写 DR

使用场景：
  发送数据后 → while(!BTF) 等待 → BTF=1 说明数据已发送且收到 ACK
```

#### TXE — 发送缓冲区空

```
置位条件：DR 中的数据已移入移位寄存器（DR 可以写入新数据）
清除条件：写 DR

使用场景：
  while(!TXE) 等待 → TXE=1 → 写入新数据到 DR
```

#### RXNE — 接收缓冲区非空

```
置位条件：移位寄存器中的数据已传送到 DR（可以读取）
清除条件：读 DR

使用场景：
  while(!RXNE) 等待 → RXNE=1 → 从 DR 读取数据
```

### 3.4 其他标志位

| 位 | 名称 | 说明 |
|:--:|------|------|
| 8 | ADD10 | 10 位地址模式下第一个字节发送完成 |
| 10 | STOPF | 从模式下检测到停止条件 |
| 11 | — | — |
| 14 | AF | 应答失败（Acknowledge Failure） |
| — | TIMEOUT | SMBus 超时 |
| — | PECERR | PEC 校验错误 |
| — | OVR | 过载/欠载错误 |

## 四、DR — 数据寄存器

| 特性 | 说明 |
|------|------|
| 位宽 | 8 位 |
| 收发 | **共用一个**（半双工，分时复用） |
| 发送 | 写入 DR → 数据自动移入移位寄存器 → 串行发出 |
| 接收 | 移位寄存器接收完成 → 自动传送到 DR → CPU 读取 |

## 五、状态位与控制位的配合关系

```
┌───────────── 操作 ─────────────┐
│                                  │
│  ① 产生起始信号                  │
│     CR1.START = 1               │
│     while(!SR1.SB) 等待          │
│                                  │
│  ② 发送设备地址                   │
│     写 DR = 设备地址              │
│     while(!SR1.ADDR) 等待        │
│     读 SR1 + 读 SR2 清除 ADDR    │
│                                  │
│  ③ 发送数据                      │
│     while(!SR1.TXE) 等待 DR 空   │
│     写 DR = 数据                  │
│     while(!SR1.BTF) 等待传输完成  │
│                                  │
│  ④ 接收数据                      │
│     while(!SR1.RXNE) 等待收到数据 │
│     读 DR                        │
│     CR1.ACK = 1 发 ACK 或 =0 发NACK│
│                                  │
│  ⑤ 产生停止信号                   │
│     CR1.STOP = 1                 │
│     （当前字节传输结束后自动生效）   │
│                                  │
└──────────────────────────────────┘
```

## 六、超时机制

为防止 while 循环卡死，每个等待都加入超时判断：

```c
uint16_t timeout = 0xFFFF;

while (!(SR1->标志位) && timeout)
{
    timeout--;
}

if (timeout == 0)
{
    return TIMEOUT;   // 超时，操作失败
}
else
{
    return OK;        // 正常完成
}
```

| 等待位置 | 检测标志 | 说明 |
|----------|----------|------|
| 起始信号后 | **SB** | 起始条件已发出 |
| 发送地址后 | **ADDR** | 地址已成功发送并收到 ACK |
| 发送数据后 | **BTF** | 字节传输完成，收到 ACK |
| 写 DR 前 | **TXE** | 发送缓冲区空，可写入新数据 |
| 读 DR 前 | **RXNE** | 接收缓冲区非空，可读取数据 |

## 七、寄存器配置汇总（初始化）

| 步骤 | 寄存器 | 操作 | 值 |
|:----:|--------|------|:--:|
| ① | RCC | 开启 GPIOB 和 I²C2 时钟 | — |
| ② | GPIO | PB10/PB11 配为复用开漏输出 | — |
| ③ | **CR2** | 输入时钟频率 | **36** |
| ④ | **CCR** | 分频系数 + 标准模式 | **180** |
| ⑤ | **TRISE** | 最大上升时间 | **37** |
| ⑥ | **CR1** | I²C 模式 + ACK 使能 | ACK=1 |
| ⑦ | **CR1** | 外设使能（**最后**） | PE=1 |

## 八、主模式 vs 从模式关键区别

| 对比项 | 主模式 | 从模式 |
|--------|--------|--------|
| 时钟控制 | **主设备控制 SCL** | 被动接收 SCL |
| 起始/停止信号 | **主设备发出** S 和 P | 检测 S 和 P |
| 检测标志 | **SB**（起始已发） | **STOPF**（收到停止） |
| 自身地址 | **不需要**配 | 需要配 OAR1/OAR2 |
| 地址发送 | 主动发送从设备地址 | 接收地址并比较匹配 |

## 九、与软件模拟 I²C 的操作对比

| I²C 操作   | 软件模拟                                 | 硬件 I²C                       |
| -------- | ------------------------------------ | ---------------------------- |
| 产生 START | `SDA_HIGH(); SCL_HIGH(); SDA_LOW();` | `CR1.START = 1; while(!SB);` |
| 产生 STOP  | `SDA_LOW(); SCL_HIGH(); SDA_HIGH();` | `CR1.STOP = 1;`（字节结束后生效）     |
| 发送 ACK   | 手动拉低 SDA + SCL 翻转                    | `CR1.ACK = 1;`（配置项）          |
| 发送 NACK  | 保持 SDA 高 + SCL 翻转                    | `CR1.ACK = 0;`（配置项）          |
| 等待 ACK   | `READ_SDA()` 读取电平                    | `while(!BTF);` 检测标志位         |
| 发送字节     | `for(8次) {设置SDA; 翻转SCL;}`            | `while(!TXE); DR = byte;`    |
| 接收字节     | `for(8次) {翻转SCL; 读SDA;}`             | `while(!RXNE); byte = DR;`   |
| 时序控制     | `Delay_us(10)` 手动延时                  | **硬件自动精确控制**                 |
| CPU 占用   | 高（每步都执行指令）                           | **低**（硬件自动处理）                |

# AT24C02 接口层实现 — 连续读写多个字节 笔记

---

## 一、EEPROM_WriteBytes() — 连续写入多个字节（页写）

### 1.1 时序图

```
[S] → [0xA0(W)] → ACK → [内部地址] → ACK
  → [数据1] → ACK → [数据2] → ACK → ... → [数据N] → ACK
  → [P]
                                              等待 5ms
```

### 1.2 与单字节写入的区别

| 对比项 | WriteByte | WriteBytes |
|--------|-----------|------------|
| 数据数量 | 1 个字节 | size 个字节 |
| 数据发送 | 直接发送 1 次 | **for 循环** size 次 |
| ACK 次数 | 3 次 | 2 + size 次 |
| 结束方式 | 数据 ACK 后 P | 循环结束后 P |
| 写入周期 | 5ms | 5ms |

### 1.3 步骤对照

| 步骤 | 操作 | 代码 | 说明 |
|:----:|------|------|------|
| ① | 起始信号 | `I2C_Start();` | |
| ② | 发送写地址 | `I2C_SendByte(W_ADDR);` | 0xA0 |
| ③ | 等待应答 | `I2C_WaitAck();` | |
| ④ | 发送内部地址 | `I2C_SendByte(inner_addr);` | 写入起始位置 |
| ⑤ | 等待应答 | `I2C_WaitAck();` | |
| ⑥~⑦ | **循环发送数据** | for 循环 | size 次 |
| ⑧ | 停止信号 | `I2C_Stop();` | |
| ⑨ | 等待写入周期 | `Delay_ms(5);` | |

### 1.4 代码实现

```c
void EEPROM_WriteBytes(uint8_t inner_addr, uint8_t *data, uint8_t size)
{
    uint8_t i;

    // ① 起始信号
    I2C_Start();

    // ② 发送写地址
    I2C_SendByte(W_ADDR);

    // ③ 等待应答
    I2C_WaitAck();

    // ④ 发送内部存储地址（写入起始位置）
    I2C_SendByte(inner_addr);

    // ⑤ 等待应答
    I2C_WaitAck();

    // ⑥~⑦ 循环发送数据
    for (i = 0; i < size; i++)
    {
        I2C_SendByte(data[i]);    // 发送第 i 个字节
        I2C_WaitAck();            // 等待应答
    }

    // ⑧ 停止信号
    I2C_Stop();

    // ⑨ 等待写入周期（5ms）
    Delay_ms(5);
}
```

### 1.5 数据流

```
STM32 ──→ S ──→ 0xA0 ──→ addr ──→ [data0] ──→ [data1] ──→ ... ──→ [dataN] ──→ P
          ↑      ↑        ↑         ↑            ↑                      ↑
        起始   写地址    内部地址    数据1        数据2                  数据N
          ←ACK─ ←ACK──  ←ACK───   ←ACK───     ←ACK───              ←ACK───
                                                                           等5ms
```

### 1.6 页写入限制

| 规则 | 说明 |
|------|------|
| 每次最多 **16 字节** | AT24C02 页大小 = 16 字节 |
| **不可跨页** | 写到页末后地址自动绕回页首 |
| 绕回后果 | 页首数据被覆盖 |

## 二、EEPROM_ReadBytes() — 连续读取多个字节

### 2.1 时序图

```
假写：
[S] → [0xA0(W)] → ACK → [起始内部地址] → ACK

真读：
[Sr] → [0xA1(R)] → ACK
  → [数据1] → 主机 ACK
  → [数据2] → 主机 ACK
  → ...
  → [数据N] → 主机 NACK
  → [P]
```

### 2.2 与单字节读取的区别

| 对比项 | ReadByte | ReadBytes |
|--------|----------|-----------|
| 数据数量 | 1 个字节 | size 个字节 |
| 数据接收 | 接收 1 次 | **for 循环** size 次 |
| 存储方式 | `return byte` | 存入 `buffer[i]` |
| 应答方式 | 直接 NACK | 前 N-1 次 **ACK**，最后 1 次 **NACK** |
| 返回值 | uint8_t | void（数据存在 buffer 中） |

### 2.3 步骤对照

| 步骤 | 操作 | 代码 | 说明 |
|:----:|------|------|------|
| **假写阶段** | | | |
| ① | 起始信号 | `I2C_Start();` | |
| ② | 发送写地址 | `I2C_SendByte(W_ADDR);` | 0xA0（假写） |
| ③ | 等待应答 | `I2C_WaitAck();` | |
| ④ | 发送内部地址 | `I2C_SendByte(inner_addr);` | 写入地址计数器 |
| ⑤ | 等待应答 | `I2C_WaitAck();` | |
| **真读阶段** | | | |
| ⑥ | 重复起始 | `I2C_Start();` | Sr |
| ⑦ | 发送读地址 | `I2C_SendByte(R_ADDR);` | 0xA1（真读） |
| ⑧ | 等待应答 | `I2C_WaitAck();` | |
| ⑨ | **循环读取数据** | for 循环 | size 次 |
| ⑩ | 停止信号 | `I2C_Stop();` | |

### 2.4 循环内部逻辑

```
for (i = 0; i < size; i++)
{
    buffer[i] = I2C_ReceiveByte();     // 读取一个字节

    if (i < size - 1)
        I2C_SendAck();                  // 非最后一字节：ACK（继续读）
    else
        I2C_SendNack();                 // 最后一字节：NACK（停止读）
}
```

### 2.5 代码实现

```c
void EEPROM_ReadBytes(uint8_t inner_addr, uint8_t *buffer, uint8_t size)
{
    uint8_t i;

    // ===== 假写阶段 =====

    // ① 起始信号
    I2C_Start();

    // ② 发送写地址（假写）
    I2C_SendByte(W_ADDR);

    // ③ 等待应答
    I2C_WaitAck();

    // ④ 发送内部存储地址（写入地址计数器）
    I2C_SendByte(inner_addr);

    // ⑤ 等待应答
    I2C_WaitAck();

    // ===== 真读阶段 =====

    // ⑥ 重复起始信号
    I2C_Start();

    // ⑦ 发送读地址（真读）
    I2C_SendByte(R_ADDR);

    // ⑧ 等待应答
    I2C_WaitAck();

    // ⑨ 循环读取数据
    for (i = 0; i < size; i++)
    {
        buffer[i] = I2C_ReceiveByte();

        if (i < size - 1)
        {
            I2C_SendAck();      // 非最后一字节：ACK，告诉从设备继续发
        }
        else
        {
            I2C_SendNack();     // 最后一字节：NACK，告诉从设备停止发
        }
    }

    // ⑩ 停止信号
    I2C_Stop();
}
```

### 2.6 ACK / NACK 判断逻辑

```
i = 0: buffer[0] = 读取 → ACK  （还有更多数据）
i = 1: buffer[1] = 读取 → ACK  （还有更多数据）
...
i = size-2: buffer[size-2] = 读取 → ACK  （还有1个）
i = size-1: buffer[size-1] = 读取 → NACK （最后一个，停止）

判断条件：i < size - 1 → ACK；i == size - 1 → NACK
```

### 2.7 数据流

```
假写：
  STM32 ──→ S ──→ 0xA0(W) ──→ inner_addr
                ←ACK──        ←ACK──

真读：
  STM32 ──→ Sr ──→ 0xA1(R) ──→ ──→ ──→ ──→ NACK ──→ P
                  ←ACK──      ↑    ↑    ↑
  AT24C02 ───────────────── [D0] [D1] ... [DN]
                             ACK  ACK ... NACK
                             ↓    ↓        ↓
                           buffer[0] buffer[1] ... buffer[N-1]
```

## 三、ACK / NACK 判断的两种写法

### 写法一：if-else 判断索引

```c
for (i = 0; i < size; i++)
{
    buffer[i] = I2C_ReceiveByte();
    if (i < size - 1)
        I2C_SendAck();
    else
        I2C_SendNack();
}
```

### 写法二：先读后判断

```c
for (i = 0; i < size; i++)
{
    buffer[i] = I2C_ReceiveByte();
    I2C_SendAck();                  // 前 N-1 次发 ACK
}
I2C_SendNack();                     // 最后补一个 NACK（覆盖上一轮的 ACK）
// 注意：此写法要求最后一个 ACK 的 SDA 已经拉高（NACK 覆盖），实现稍复杂
```

> 推荐使用**写法一**，逻辑清晰，不易出错。

## 四、连续写入注意事项

| 注意点 | 说明 |
|--------|------|
| 页大小限制 | 单次最多写入 **16 字节**（一页） |
| 不可跨页 | 地址到页末后自动绕回页首 |
| 跨页解决方案 | 分多次调用 `WriteBytes`，每次指定新的 `inner_addr` |
| 写入周期 | 每次写入后等待 **5ms** |
| 内部地址自动递增 | AT24C02 每写入一个字节后地址计数器 +1 |

### 跨页写入示例

```c
// 向地址 0x0E 开始写入 6 字节（跨越页边界）

// 第一次：写前 2 字节到第 0 页末尾（0x0E, 0x0F）
EEPROM_WriteBytes(0x0E, data, 2);

// 第二次：写后 4 字节到第 1 页开头（0x10, 0x11, 0x12, 0x13）
EEPROM_WriteBytes(0x10, data + 2, 4);
```

## 五、连续读取注意事项

| 注意点 | 说明 |
|--------|------|
| 无页大小限制 | 读取可连续，地址计数器自动递增，不绕回 |
| 必须发 NACK | 最后一个字节读完后发 NACK，否则从设备继续发送 |
| 可跨页读取 | 地址自动递增可跨越页边界 |
| 假写真读 | 随机地址读取仍需先假写设置起始地址 |

## 六、单字节 vs 多字节操作对比

| 对比项 | 写入 1 字节 | 写入 N 字节 | 读取 1 字节 | 读取 N 字节 |
|--------|:-----------:|:-----------:|:-----------:|:-----------:|
| 起始信号 | 1 个 S | 1 个 S | 2 个 (S+Sr) | 2 个 (S+Sr) |
| 设备地址发送 | 1 次 | 1 次 | 2 次 | 2 次 |
| 内部地址发送 | 1 次 | 1 次 | 1 次 | 1 次 |
| 数据收发 | 1 次 | **N 次循环** | 1 次 | **N 次循环** |
| 应答处理 | 从设备 ACK | 从设备 ACK ×N | 主机 NACK | 主机 ACK ×(N-1) + NACK |
| 结束 | P + 5ms | P + 5ms | P | P |

## 一句话定义
位操作是STM32寄存器编程的核心技能，通过位运算可以精确控制寄存器的特定位，避免影响其他位，是保证代码稳定性和可读性的基础。

## 核心内容
### 1. 基础位运算规则
| 运算符 | 名称 | 规则 | 常用场景 |
| --- | --- | --- | --- |
| & | 按位与 | 全1为1，有0则0 | 清零特定位、读取特定位 |
| \| | 按位或 | 全0为0，有1则1 | 置1特定位 |
| ^ | 按位异或 | 不同为1，相同为0 | 翻转特定位 |
| ~ | 按位取反 | 1变0，0变1 | 构建清零掩码 |
| << | 左移 | 各位左移，低位补0，高位丢弃 | 构建位掩码，数值乘以2 |
| >> | 右移 | 各位右移，高位补符号位/0，低位丢弃 | 提取特定位，数值除以2 |

### 2. 常用位操作函数（宏定义）
```c
// 把addr地址的第bit位置1
#define SET_BIT(addr, bit) (*(volatile uint32_t *)addr |= (1 << bit))
// 把addr地址的第bit位清零
#define CLEAR_BIT(addr, bit) (*(volatile uint32_t *)addr &= ~(1 << bit))
// 翻转addr地址的第bit位
#define TOGGLE_BIT(addr, bit) (*(volatile uint32_t *)addr ^= (1 << bit))
// 读取addr地址的第bit位值
#define READ_BIT(addr, bit) ((*(volatile uint32_t *)addr >> bit) & 0x1)
// 把addr地址的第bit位设置为value（0或1）
#define WRITE_BIT(addr, bit, value) do { \
    if (value) SET_BIT(addr, bit); \
    else CLEAR_BIT(addr, bit); \
} while(0)
```

### 3. 位操作实战应用
#### 3.1 置1操作（特定位设为1，其他位不变）
- 语法：`寄存器 |= (1 << 位偏移)`
- 示例：开启GPIOA时钟
  ```c
  // 错误写法：直接赋值会覆盖其他位，导致其他外设时钟关闭
  RCC->APB2ENR = 0x4; 
  // 正确写法：仅将第2位置1，其他位保持不变
  RCC->APB2ENR |= (1 << 2); 
  // 使用宏定义写法
  RCC->APB2ENR |= RCC_APB2ENR_IOPAEN;
  ```

#### 3.2 清零操作（特定位设为0，其他位不变）
- 语法：`寄存器 &= ~(1 << 位偏移)`
- 示例：PA0输出低电平
  ```c
  // 错误写法：直接赋值会将其他GPIO引脚电平置0
  GPIOA->ODR = 0xFFFE; 
  // 正确写法：仅将第0位清零，其他位保持不变
  GPIOA->ODR &= ~(1 << 0); 
  // 使用宏定义写法
  GPIOA->ODR &= ~GPIO_ODR_ODR0;
  ```

#### 3.3 翻转操作（特定位取反，其他位不变）
- 语法：`寄存器 ^= (1 << 位偏移)`
- 示例：翻转PA0电平实现LED闪烁
  ```c
  while(1) {
      GPIOA->ODR ^= (1 << 0); // 翻转PA0电平
      delay_ms(500);          // 延时500ms
  }
  ```

#### 3.4 读取操作（获取特定位的值）
- 语法：`(寄存器 >> 位偏移) & 0x1`
- 示例：读取PA1按键电平
  ```c
  uint8_t key_state = (GPIOA->IDR >> 1) & 0x1;
  if (key_state == 0) {
      // 按键按下
  }
  ```

### 4. 多比特位操作技巧
#### 4.1 连续多位同时置1
- 语法：`寄存器 |= (掩码 << 起始位偏移)`
- 示例：同时开启GPIOA、GPIOB、GPIOC时钟（位2、3、4同时置1）
  ```c
  // 掩码0x7（二进制111）左移2位得到0x1C，同时置1位2、3、4
  RCC->APB2ENR |= (0x7 << 2);
  ```

#### 4.2 连续多位同时清零
- 语法：`寄存器 &= ~(掩码 << 起始位偏移)`
- 示例：清除PA0的CNF和MODE位（4位清零）
  ```c
  // 掩码0xF（二进制1111）左移0位，取反后得到0xFFFFFFF0，与操作后清零低4位
  GPIOA->CRL &= ~(0xF << 0);
  ```

#### 4.3 连续多位设置为指定值
- 两步法：先清零目标位，再设置目标值
- 示例：将PA0的CNF和MODE位设置为0x3（推挽输出50MHz）
  ```c
  GPIOA->CRL &= ~(0xF << 0); // 第一步：清零PA0对应的4位
  GPIOA->CRL |= (0x3 << 0);  // 第二步：设置为0x3
  ```

#### 4.4 提取连续多位的值
- 语法：`(寄存器 >> 起始位偏移) & 掩码`
- 示例：提取PA0-PA3的电平值（4位）
  ```c
  uint8_t value = (GPIOA->IDR >> 0) & 0xF;
  ```

### 5. 位操作编程规范
1. **禁止直接赋值寄存器**：除非你明确知道所有位的状态，否则永远使用位操作修改寄存器
2. **优先使用宏定义**：使用ST标准库提供的位定义宏，不要直接写数字，提高代码可读性
3. **明确位偏移含义**：注释说明每一位的含义，避免魔法数字
4. **操作顺序**：修改多位时先清零再置位，避免中间状态出现错误
5. **原子操作**：位操作是原子操作，不会被中断打断，适合在中断服务函数中使用

## 注意事项&踩坑
1. 优先级问题：位反运算符`~`优先级高于左移运算符`<<`，注意括号使用，`~(1 << 2)`是正确写法，`~1 << 2`是错误写法
2. 32位整数问题：STM32是32位系统，`1`默认是32位有符号整数，取反后会出现负数，操作时使用无符号整数更安全
3. 位操作范围：寄存器是32位，位偏移不要超过31，否则会出现未定义行为
4. 保留位操作：寄存器中的保留位不要修改，保持默认值即可，否则会导致芯片工作异常

## 一句话定义

USART寄存器配置包括BRR波特率、CR1使能位、SR状态位查询和DR数据读写，轮询方式只需配置UE+TE+RE三位。

## 核心内容

### 实验需求

| 项目 | 内容 |
|------|------|
| 功能 | 电脑通过串口发送数据给 STM32，STM32 原封不动回传（Echo） |
| 通讯接口 | USART1 |
| 连接方式 | 电脑 USB → ST-Link（USB 转虚拟串口）→ 调试线 → PA9(TX) / PA10(RX) |
| 数据收发方式 | **轮询方式**（查询标志位） |

**硬件连接链路：**
```
电脑(USB) → ST-Link 2.1(USB转串口) → 调试线 → 开发板
                                        ├── PA9  = USART1_TX（复用推挽输出）
                                        └── PA10 = USART1_RX（复用输入/浮空输入）
```

### 涉及的寄存器总览

| 寄存器 | 全称 | 功能 |
|--------|------|------|
| **SR** | Status Register | 状态寄存器（标志位查询） |
| **DR** | Data Register | 数据寄存器（读/写数据，底层为 TDR + RDR 二合一） |
| **BRR** | Baud Rate Register | 波特率寄存器 |
| **CR1** | Control Register 1 | 控制寄存器 1（使能位、字长、校验、中断使能） |
| **CR2** | Control Register 2 | 控制寄存器 2（停止位、同步时钟、LIN） |
| **CR3** | Control Register 3 | 控制寄存器 3（硬件流控、DMA、IrDA） |

### 需要配置的寄存器

**BRR — 波特率寄存器：**

| 项目 | 内容 |
|------|------|
| 位宽 | 16 位（bit[15:4] 整数部分 + bit[3:0] 小数部分） |
| 115200 对应值 | **0x271** |
| 写入方式 | 直接写入计算好的值 |

**CR1 — 控制寄存器 1（本实验需要配置的位）：**

| 位 | 名称 | 功能 | 配置 |
|----|------|------|------|
| **UE** | USART Enable | 整个串口模块使能 | 置 1（必须开启） |
| **TE** | Transmitter Enable | 发送器使能 | 置 1（必须开启） |
| **RE** | Receiver Enable | 接收器使能 | 置 1（必须开启） |
| M | Word Length | 数据字长 | 0 = 8 位（默认），1 = 9 位 |
| PCE | Parity Control Enable | 校验控制使能 | 0 = 不使用校验（默认） |
| PS | Parity Selection | 校验方式选择 | 0 = 偶校验（默认），1 = 奇校验 |
| RXNEIE | RXNE Interrupt Enable | 接收缓冲区非空中断使能 | 中断方式时开启 |

> 本实验（轮询方式）只需配置 **UE + TE + RE** 三位，其余使用默认值。

**CR2 — 控制寄存器 2：**

| 可配置项 | 位置 | 默认值 | 说明 |
|----------|------|--------|------|
| **STOP[1:0]** | bit[13:12] | 00 | 停止位：00=1位, 01=0.5位, 10=2位, 11=1.5位 |

### 状态寄存器（SR）关键标志位

**常用标志位：**

| 标志位 | 全称 | 含义 | 用途 |
|--------|------|------|------|
| **TXE** | Transmit Data Register Empty | 发送数据寄存器**为空** | 写 DR 前查询，确保可写入新数据 |
| **TC** | Transmission Complete | 发送**完成**（移位寄存器也发完了） | 需要确保完整发送时使用 |
| **RXNE** | Read Data Register Not Empty | 接收数据寄存器**非空**（已收到数据） | 读 DR 前查询，确保数据已就绪 |
| **IDLE** | Idle Line Detected | 检测到**空闲帧** | 变长数据接收完毕判断 |

**TXE 与 TC 的区别：**

```
TDR 写入数据 → 数据转移到发送移位寄存器 → 移位寄存器逐位发送
                 ↑                            ↑
              TXE 置1                      TC 置1
           (TDR空,可写入下一个)         (彻底发完)
```

| 对比项 | TXE | TC |
|--------|-----|----|
| 置 1 时机 | TDR 数据交给移位寄存器后（TDR 空） | 移位寄存器全部发送完毕 |
| 含义 | 可写入下一个数据 | 上一个数据**完全发送完成** |
| 速度 | 更快（不必等移位完成） | 较慢（等待彻底发完） |
| 使用场景 | 连续发送，追求速度 | 精确控制，确保完整 |

**错误标志位：**

| 标志位 | 全称 | 含义 |
|--------|------|------|
| ORE | Overrun Error | 数据覆盖错误（上一个数据未读取，新数据已到达） |
| NE | Noise Error | 噪声检测错误 |
| FE | Framing Error | 帧格式错误（同步错误） |
| PE | Parity Error | 奇偶校验错误 |

### 数据寄存器（DR）

| 项目 | 内容 |
|------|------|
| 有效位 | bit[8:0]，共 9 位 |
| 实际使用位数 | 取决于 CR1 中 M 位的配置（8 位或 9 位） |
| 底层硬件 | TDR（发送）+ RDR（接收），共用同一地址 |
| 读操作 | 读取 RDR（接收数据） |
| 写操作 | 写入 TDR（发送数据） |

### 轮询方式收发代码逻辑

**接收数据（轮询）：**
```c
// 等待 RXNE = 1（接收缓冲区非空/已满）
while (!(SR & SR_RXNE));    // while 循环死等
uint8_t data = DR;          // 读取数据，同时硬件清除 RXNE
```

**发送数据（轮询）：**
```c
// 等待 TXE = 1（发送缓冲区为空）
while (!(SR & SR_TXE));     // while 循环死等
DR = data;                  // 写入待发送数据
```

**完整回显逻辑：**
```c
while (1)
{
    // 1. 等待接收（RXNE = 1）
    // 2. 读取数据
    // 3. 等待发送缓冲区空（TXE = 1）
    // 4. 将接收到的数据写回 DR 发送
}
```

### 寄存器配置速查表

| 配置步骤 | 寄存器          | 操作                   | 说明               |
| :--: | ------------ | -------------------- | ---------------- |
|  1   | RCC->APB2ENR | 使能 USART1 和 GPIOA 时钟 | USART1 在 APB2 总线 |
|  2   | GPIOA->CRH   | PA9 复用推挽输出，PA10 浮空输入 | 配置引脚复用功能         |
|  3   | USART1->BRR  | 写入 0x271             | 115200 波特率       |
|  4   | USART1->CR1  | UE=1, TE=1, RE=1     | 使能串口 + 收发        |
|  5   | USART1->SR   | 查询 TXE / RXNE        | 轮询等待标志位          |
|  6   | USART1->DR   | 读/写数据                | 接收读、发送写          |

## 注意事项 & 踩坑

- USART1 在 APB2 总线（72MHz），与其他串口不同
- 轮询方式只需配置 UE+TE+RE，其他使用默认值
- TXE=1 表示可写（发完），RXNE=1 表示可读（收到）
- 读 DR 自动清除 RXNE，写 DR 自动清除 TXE

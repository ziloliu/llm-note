## 一句话定义
STM32标准库对每个寄存器的每个功能位都定义了对应的宏，使用宏定义代替直接数字操作可以大幅提高代码可读性，降低出错概率，是寄存器编程的最佳实践。

## 核心内容
### 1. 宏定义命名规则
标准库宏定义遵循统一命名规则：`外设_寄存器_位功能[_位域]`
#### 示例解析
- `RCC_APB2ENR_IOPAEN`：
  - `RCC`：外设名（复位和时钟控制模块）
  - `APB2ENR`：寄存器名（APB2外设时钟使能寄存器）
  - `IOPAEN`：位功能（GPIOA端口时钟使能位）
- `GPIO_CRL_MODE0`：
  - `GPIO`：外设名（通用输入输出）
  - `CRL`：寄存器名（端口配置低寄存器）
  - `MODE0`：位功能（引脚0模式位域，共2位）
- `GPIO_CRL_MODE0_0`：
  - `MODE0_0`：位域的第0位，位域包含多位时的单独位定义

### 2. 常用宏定义分类
#### 2.1 单比特位宏定义（只有1位）
```c
// RCC_APB2ENR寄存器的GPIOA时钟使能位，位偏移2
#define RCC_APB2ENR_IOPAEN    ((uint16_t)0x0004) // 二进制0000 0000 0000 0100
// GPIO_ODR寄存器的引脚0输出位，位偏移0
#define GPIO_ODR_ODR0         ((uint16_t)0x0001) // 二进制0000 0000 0000 0001
// USART_SR寄存器的发送完成位，位偏移6
#define USART_SR_TC           ((uint16_t)0x0040) // 二进制0000 0000 0100 0000
```
- 单比特位宏的值为`1 << 位偏移`，直接用于位操作

#### 2.2 多比特位域宏定义（包含多个连续位）
```c
// GPIO_CRL寄存器的引脚0模式位域，位[1:0]，共2位
#define GPIO_CRL_MODE0        ((uint8_t)0x03) // 二进制0000 0011
// GPIO_CRL寄存器的引脚0配置位域，位[3:2]，共2位
#define GPIO_CRL_CNF0         ((uint8_t)0x0C) // 二进制0000 1100
// RCC_CFGR寄存器的系统时钟切换位域，位[1:0]，共2位
#define RCC_CFGR_SW           ((uint8_t)0x03) // 二进制0000 0011
```
- 多比特位域宏的值是对应位全1的掩码，用于清零或读取位域

#### 2.3 位域配置值宏定义
```c
// RCC_CFGR_SW位域配置值：选择HSI作为系统时钟
#define RCC_CFGR_SW_HSI       ((uint8_t)0x00)
// RCC_CFGR_SW位域配置值：选择HSE作为系统时钟
#define RCC_CFGR_SW_HSE       ((uint8_t)0x01)
// RCC_CFGR_SW位域配置值：选择PLL作为系统时钟
#define RCC_CFGR_SW_PLL       ((uint8_t)0x02)

// GPIO_CRL_CNF0位域配置值：推挽输出模式
#define GPIO_CRL_CNF0_0       ((uint8_t)0x04)
#define GPIO_CRL_CNF0_1       ((uint8_t)0x08)
#define GPIO_CRL_CNF0_PP      ((uint8_t)0x00) // 推挽输出
#define GPIO_CRL_CNF0_OD      ((uint8_t)0x04) // 开漏输出
#define GPIO_CRL_CNF0_AF_PP   ((uint8_t)0x08) // 复用推挽输出
#define GPIO_CRL_CNF0_AF_OD   ((uint8_t)0x0C) // 复用开漏输出
```
- 位域配置值宏定义了位域的不同配置选项，直接使用无需手动计算值

### 3. 宏定义使用示例
#### 3.1 单比特位置1操作
```c
// 开启GPIOA时钟，使用宏定义写法
RCC->APB2ENR |= RCC_APB2ENR_IOPAEN;
// 等价于数字写法：RCC->APB2ENR |= (1 << 2);
```

#### 3.2 单比特位清零操作
```c
// PA0输出低电平
GPIOA->ODR &= ~GPIO_ODR_ODR0;
// 等价于数字写法：GPIOA->ODR &= ~(1 << 0);
```

#### 3.3 多比特位域配置操作
```c
// 配置PA0为推挽输出50MHz
GPIOA->CRL &= ~(GPIO_CRL_CNF0 | GPIO_CRL_MODE0); // 清零PA0的CNF和MODE位
GPIOA->CRL |= GPIO_CRL_MODE0; // MODE位设为11（50MHz输出）
// CNF位保持00（推挽输出），所以不需要额外设置

// 等价于数字写法：
// GPIOA->CRL &= ~(0xF << 0);
// GPIOA->CRL |= (0x3 << 0);
```

#### 3.4 位域配置值选择操作
```c
// 选择PLL作为系统时钟
RCC->CFGR &= ~RCC_CFGR_SW; // 清零SW位域
RCC->CFGR |= RCC_CFGR_SW_PLL; // 设置为PLL时钟

// 等价于数字写法：
// RCC->CFGR &= ~(0x3 << 0);
// RCC->CFGR |= (0x2 << 0);
```

#### 3.5 状态位判断操作
```c
// 检查USART1发送是否完成
if (USART1->SR & USART_SR_TC) {
    // 发送完成，清除TC标志
    USART1->SR &= ~USART_SR_TC;
}
```

### 4. 宏定义使用规范
#### 4.1 优先级规范
1. **优先使用位功能宏**：如`RCC_APB2ENR_IOPAEN` > `(1 << 2)` > `0x04`
2. **优先使用位域宏**：如`GPIO_CRL_MODE0` > `0x3 << 0` > `0x03`
3. **优先使用配置值宏**：如`RCC_CFGR_SW_PLL` > `0x2 << 0` > `0x02`

#### 4.2 命名规范
1. 自定义宏也要遵循相同命名规则，不要使用拼音或不规范缩写
2. 宏定义全部大写，用下划线分隔单词
3. 功能相似的宏使用相同前缀，便于分组查找

#### 4.3 代码可读性规范
1. 不要使用魔法数字：代码中不要出现未定义的数字常量，所有位操作都使用宏定义
2. 必要的注释：复杂的位操作需要注释说明功能和配置效果
3. 避免复杂组合：不要在一行代码中组合过多位操作，分步骤写更清晰

### 5. 宏定义查找方法
1. 官方数据手册和参考手册：每个寄存器的位定义都有详细说明
2. 标准库头文件：`stm32f10x.h`头文件包含所有寄存器和位定义宏
3. IDE代码补全：输入外设名和寄存器名后，IDE会自动提示相关宏定义
4. 官方标准库文档：ST提供的标准库用户手册有完整的宏定义说明

## 注意事项&踩坑
1. 宏定义冲突：自定义宏不要与标准库宏重名，否则会导致编译错误
2. 位掩码位数问题：宏定义要注意是16位还是32位，避免移位时出现符号扩展问题
3. 位域跨字节问题：跨字节的位域操作要注意掩码范围，避免清除其他位
4. 版本差异：不同系列芯片的位定义可能有差异，移植时要注意核对

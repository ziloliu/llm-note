# DMA 存储器到存储器传输 — 寄存器配置详解 笔记

---

## 一、实验概述

| 项目 | 说明 |
|------|------|
| 传输方向 | **ROM（Flash）→ RAM（SRAM）** |
| DMA 控制器 | DMA1 |
| 通道 | **通道 1**（任意通道均可） |
| 模式 | **存储器到存储器**（特殊模式） |
| 时钟 | **AHB 总线**（RCC->AHBENR） |

### 1.1 为什么是 ROM → RAM

```
ROM（Flash）存储：程序代码、全局常量
RAM（SRAM）存储：变量

✅ ROM → RAM：可以读取 Flash 数据到 SRAM
❌ RAM → ROM：不可以（Flash 写入需要特定协议/烧写器）
```

### 1.2 为什么用 DMA 而不是 b=a

```
代码 b=a 的底层执行：
  ① CPU 从 Flash 读取 a 的值 → 数据总线 → CPU 寄存器
  ② CPU 将值通过数据总线 → 写入 SRAM 中 b 的地址

  → 两步都需要 CPU 参与，占用数据总线

DMA 方式：
  → Flash 和 SRAM 之间建立高速通路
  → 数据自动搬运，CPU 完全不参与
  → CPU 可以去处理其他任务
```

## 二、关键寄存器详解

### 2.1 四个核心寄存器（每个通道独立）

| 寄存器 | 全称 | 功能 |
|--------|------|------|
| **CCR** | Channel Configuration Register | 通道配置（核心） |
| **CNDTR** | Channel Number of Data to Transfer | 数据传输数量 |
| **CPAR** | Channel Peripheral Address Register | 外设地址（本实验=ROM 地址） |
| **CMAR** | Channel Memory Address Register | 存储器地址（本实验=RAM 地址） |

### 2.2 两个共用寄存器（所有通道共享）

| 寄存器 | 全称 | 功能 |
|--------|------|------|
| **ISR** | Interrupt Status Register | 中断状态标志（**只读**） |
| **IFCR** | Interrupt Flag Clear Register | 中断标志清除（**只写**，写 1 清除） |

### 2.3 寄存器存储映像

```
地址偏移    寄存器          说明
──────────────────────────────────
0x00        ISR             中断状态（所有通道共用，只读）
0x04        IFCR            中断标志清除（所有通道共用，只写）
──────────────────────────────────
0x08        CCR1            通道 1 配置
0x0C        CNDTR1          通道 1 数据数量
0x10        CPAR1           通道 1 外设地址
0x14        CMAR1           通道 1 存储器地址
──────────────────────────────────
0x1C        CCR2            通道 2 配置
...         ...             ...（每通道 4 个寄存器，间隔 0x08）
──────────────────────────────────
DMA1: 通道 1~7
DMA2: 通道 1~5
```

## 三、CCR 寄存器详解（通道配置寄存器）

### 3.1 所有位域

| 位 | 名称 | 功能 | 本实验配置 |
|:--:|------|------|:----------:|
| 0 | **EN** | 通道使能 | 最后置 1 |
| 1 | **TCIE** | 传输完成中断使能 | 可选 |
| 2 | **HTIE** | 传输过半中断使能 | 0 |
| 3 | **TEIE** | 传输错误中断使能 | 0 |
| 4 | **DIR** | 数据传输方向 | **0**（从"外设"读） |
| 5 | **CIRC** | 循环模式 | 0 |
| 6 | **PINC** | 外设地址自增 | **1** |
| 7 | **MINC** | 存储器地址自增 | **1** |
| 9:8 | **PSIZE** | 外设数据宽度 | **00**（8 位） |
| 11:10 | **MSIZE** | 存储器数据宽度 | **00**（8 位） |
| 13:12 | **PL** | 优先级 | **10**（高） |
| 14 | **MEM2MEM** | 存储器到存储器模式 | **1** |

### 3.2 各位详解

#### MEM2MEM（位 14）— 存储器到存储器模式

| 值 | 模式 |
|:--:|------|
| 0 | 正常模式（外设 ↔ 存储器） |
| **1** | **存储器到存储器模式** |

> 这是特殊标志位，必须置 1。

#### DIR（位 4）— 数据传输方向

| 值 | 方向 | 说明 |
|:--:|------|------|
| **0** | **外设 → 存储器** | 从"外设"（ROM）读 |
| 1 | 存储器 → 外设 | 向"外设"（ROM）写 |

```
本实验：ROM → RAM
  → ROM 当作"外设"（源）
  → RAM 当作"存储器"（目的）
  → DIR = 0

⚠️ MEM2MEM 模式下底层仍用"外设"和"存储器"两个概念
  → 将源存储器（ROM）当作"外设"
  → 所有配置"外设"的地方本质操作的都是 ROM
```

#### PSIZE / MSIZE（位 9:8 / 位 11:10）— 数据宽度

| 值 | 位宽 | 字节数 |
|:--:|:----:|:------:|
| 00 | **8 位** | 1 字节 |
| 01 | 16 位 | 2 字节 |
| 10 | 32 位 | 4 字节 |
| 11 | 保留 | — |

```
⚠️ PSIZE 和 MSIZE 一般设为相同值
不一致时：
  小 → 大：填充（如 8 位 0xAA → 16 位 0xAAAA）
  大 → 小：可能丢失高位
```

#### PINC / MINC（位 6 / 位 7）— 地址自增

| 位 | 名称 | 功能 |
|:--:|------|------|
| 6 | **PINC** | 外设地址自增 |
| 7 | **MINC** | 存储器地址自增 |

```
自增步长 = 数据宽度：
  PSIZE=00（8位）  → 每次地址 +1
  PSIZE=01（16位） → 每次地址 +2
  PSIZE=10（32位） → 每次地址 +4

本实验：传输多个数据 → 两个地址都需要自增
  PINC = 1, MINC = 1
```

#### PL（位 13:12）— 软件优先级

| 值 | 优先级 |
|:--:|--------|
| 00 | 低（默认） |
| 01 | 中 |
| 10 | 高 |
| 11 | 最高 |

#### CIRC（位 5）— 循环模式

| 值 | 模式 | 说明 |
|:--:|------|------|
| 0 | 正常模式 | 传输完成停止 |
| 1 | 循环模式 | 地址回到起始继续传输 |

#### EN（位 0）— 通道使能

| 值 | 状态 |
|:--:|------|
| 0 | 通道关闭 |
| 1 | 通道开启，开始数据传输 |

> 最后配置，开启后立即开始传输。

## 四、CNDTR 寄存器（数据传输数量）

### 4.1 结构

```
31:16  保留
15:0   NDT[15:0]  要传输的数据数量（0 ~ 65535）
```

### 4.2 工作机制

```
① 初始化时写入要传输的数据个数 N
② 每传输一个数据，NDT 自动减 1
③ 同时地址按数据宽度自增（如果开启了自增）
④ NDT 减到 0 → 数据传输完成
   → 产生 TC（Transfer Complete）事件
   → 如果 TCIE=1，产生传输完成中断
```

## 五、CPAR 和 CMAR 寄存器（源和目的地址）

### 5.1 功能

| 寄存器 | 全称 | 本实验 |
|--------|------|--------|
| **CPAR** | Channel Peripheral Address Register | **源地址**（ROM 中的常量数组） |
| **CMAR** | Channel Memory Address Register | **目标地址**（RAM 中的变量数组） |

```
存储器到存储器模式：
  CPAR = ROM 起始地址（源，当作"外设"）
  CMAR = RAM 起始地址（目的）
  DIR = 0（从"外设"读 → 从 ROM 读）

两个都是 32 位寄存器，直接写入地址值
```

## 六、ISR 和 IFCR 寄存器（中断标志）

### 6.1 ISR（中断状态寄存器，只读）

每个通道有 4 个标志位：

| 标志 | 全称 | 含义 |
|:----:|------|------|
| **GIFx** | Global Interrupt Flag | 全局中断标志（任何事件都置位） |
| **TCIFx** | Transfer Complete IF | 传输完成标志 |
| **HTIFx** | Half Transfer IF | 传输过半标志 |
| **TEIFx** | Transfer Error IF | 传输错误标志 |

> ISR **只读**，不能直接写入清除。

### 6.2 IFCR（中断标志清除寄存器，只写）

与 ISR 一一对应，**写 1 清除**对应标志：

| 位 | 清除目标 |
|:--:|----------|
| **CGIFx** | 清除通道 x 全局中断标志（GIFx） |
| **CTCIFx** | 清除通道 x 传输完成标志（TCIFx） |
| **CHTIFx** | 清除通道 x 传输过半标志（HTIFx） |
| **CTEIFx** | 清除通道 x 传输错误标志（TEIFx） |

### 6.3 三种中断事件

| 事件 | 缩写 | 触发条件 | ISR 标志 | CCR 使能位 |
|------|:----:|----------|:--------:|:----------:|
| 传输过半 | **HT** | NDT 减到一半 | HTIFx | HTIE |
| 传输完成 | **TC** | NDT 减到 0 | TCIFx | TCIE |
| 传输错误 | **TE** | 访问错误 | TEIFx | TEIE |

## 七、时钟使能

```c
// DMA 控制器挂在 AHB 总线上（不是外设）
RCC->AHBENR |= RCC_AHBENR_DMA1EN;    // DMA1 时钟使能
```

| 总线 | DMA 控制器 | 使能位 |
|------|-----------|--------|
| AHB | DMA1 | RCC->AHBENR bit 0 |
| AHB | DMA2 | RCC->AHBENR bit 1 |

## 八、存储器到存储器模式的特殊之处

### 8.1 通道选择

```
外设 → 存储器模式：查通道分配表（固定）
存储器 → 存储器模式：任意通道都可以

本实验选择 DMA1 通道 1（硬件优先级最高）
```

### 8.2 底层概念映射

```
DMA 底层只有两个角色："外设"和"存储器"

存储器到存储器模式是特例：
  → 源存储器（ROM）→ 当作"外设" → 配置到 CPAR
  → 目标存储器（RAM）→ 当作"存储器" → 配置到 CMAR
  → DIR = 0（从"外设"读）

所有配置"外设"的地方，本质操作的都是 ROM
```

## 九、完整配置流程

```
① 开启时钟
   RCC->AHBENR |= RCC_AHBENR_DMA1EN;

② 配置 CCR（通道配置寄存器）
   MEM2MEM = 1   → 存储器到存储器模式
   DIR     = 0   → 从"外设"（ROM）读
   PSIZE   = 00  → 8 位数据宽度
   MSIZE   = 00  → 8 位数据宽度
   PINC    = 1   → ROM 地址自增
   MINC    = 1   → RAM 地址自增
   PL      = 10  → 高优先级
   CIRC    = 0   → 正常模式
   TCIE    = 1   → 传输完成中断使能（可选）
   EN      = 0   → 暂不开启

③ 配置 CNDTR
   NDT = 传输数据个数

④ 配置 CPAR（源地址）
   CPAR = ROM 中数据起始地址

⑤ 配置 CMAR（目标地址）
   CMAR = RAM 中目标起始地址

⑥ NVIC 配置（如使用中断）
   配置 DMA1_Channel1 中断

⑦ 开启通道
   CCR.EN = 1

⑧ 等待传输完成
   查询 ISR.TCIF1 或进入中断
   清除标志：IFCR |= DMA_IFCR_CTCIF1;
```

## 十一、概念速查

| 概念 | 说明 |
|------|------|
| DMA | Direct Memory Access，直接存储器访问 |
| MEM2MEM | 存储器到存储器传输模式（特殊模式） |
| DIR | 数据方向（0=外设→存储器，1=存储器→外设） |
| PSIZE/MSIZE | 每次传输的数据位宽（8/16/32 位） |
| PINC/MINC | 每次传输后地址自动增加 |
| NDT | 剩余要传输的数据数量（自动递减） |
| TC | Transfer Complete，传输完成事件 |
| HT | Half Transfer，传输过半事件 |
| TE | Transfer Error，传输错误事件 |
| ISR | 中断状态寄存器（只读，不能清除） |
| IFCR | 中断标志清除寄存器（只写，写 1 清除） |
| GIF | Global Interrupt Flag，全局中断标志 |
| CPAR | 通道外设地址寄存器（本实验=源地址） |
| CMAR | 通道存储器地址寄存器（本实验=目标地址） |
| CCR | 通道配置寄存器（核心配置） |

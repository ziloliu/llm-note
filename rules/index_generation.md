# 索引生成规范

## 定位
索引文件是知识库的**目录树 + 内容摘要**，统一放在 `index/` 文件夹下。

### 服务对象
- **人类**：浏览知识库结构，通过链接跳转，通过摘要判断是否需要阅读原文
- **AI**：检索时先读索引，快速定位知识范围，再按摘要锁定目标文件，提高查询效率

---

## 一、索引文件组织

### 1.1 主索引文件
- 位置：`index/全文目录.md`
- 内容：完整覆盖 `raw/` 和 `core/` 的目录树，附摘要

### 1.2 可选子索引
当知识库规模较大时，可按需生成子索引，减轻主索引负担：
- `index/raw_目录.md` – 仅 raw 的树状索引
- `index/core_目录.md` – 仅 core 的树状索引
- `index/主题索引.md` – 按主题关键词的快速导航

### 1.3 索引与日志的关系
索引是当前知识库的快照，日志是操作历史。两者互补，索引本身不记录历史。

---

## 二、目录树格式

### 2.1 基本结构
使用 Markdown 的标题层级（`#` ~ `####`）和嵌套列表，完全复刻文件夹的层级关系：

```markdown
# 知识库目录

> 更新：YYYY-MM-DD HH:MM

## raw/ 原始笔记
- 📁 **计算机科学**
  - 📁 **机器学习**
    - 📄 [01_决策树基础概念]( 相对路径 ) — 摘要内容
    - 📄 [02_过拟合与正则化]( 相对路径 ) — 摘要内容
  - 📁 **编程语言**
    - 📄 [01_Python装饰器详解]( 相对路径 ) — 摘要内容

## core/ 核心笔记
- 📄 [机器学习]( 相对路径 ) — 摘要内容
- 📄 [时间管理]( 相对路径 ) — 摘要内容
```

### 2.2 图标约定
| 图标 | 含义 |
|------|------|
| 📁 | 文件夹 |
| 📄 | Markdown 文件 |

### 2.3 状态标记
在raw文件的摘要后添加提炼状态标记：
- `✅ 已提炼`：内容已整合到core笔记
- `⬜ 待提炼`：内容尚未提炼到core

示例：
```markdown
- 📄 [01_决策树基础概念](...) — 决策树的定义、信息增益原理 ✅ 已提炼
- 📄 [02_过拟合与正则化](...) — 过拟合现象与正则化方法 ⬜ 待提炼
```

### 2.4 缩进与层级
- 每层缩进使用 2 或 4 个空格
- 文件夹使用 `- 📁 **名称**` 格式
- 文件使用 `- 📄 [文件名]( 相对路径 ) — 摘要` 格式
- 最深到 5 层（对应 raw 的最大嵌套深度）

### 2.4 链接格式
- 使用**相对路径**，确保知识库整体移动后仍可跳转
- 链接指向实际文件，可直接点击打开
- 示例：`[01_决策树基础概念](../raw/计算机科学/机器学习/01_决策树基础概念.md)`

## 三、内容摘要规范

### 3.1 摘要目的
文件名有时无法充分体现内容。摘要让人类和 AI 无需打开文件即可判断：
- 这篇笔记讲的是什么
- 是否包含自己需要的信息

### 3.2 摘要来源
- **raw 文件**：AI 在入库时（`RAW_IN` 操作）生成摘要，可从文件开头或关键段落提取
- **core 文件**：AI 在提炼时（`EXTRACT` 操作）生成或更新摘要，概括该主题核心知识
- 摘要存入索引时，同时可考虑存回原文件的元数据区域（如开头的注释），但非必须

### 3.3 摘要格式
- **长度**：一句话，控制在 15~40 字
- **内容**：说明笔记的核心主题或核心观点
- **风格**：平实陈述，不评价，不啰嗦

### 3.4 摘要示例
```markdown
- 📄 [01_决策树基础概念](../raw/计算机科学/机器学习/01_决策树基础概念.md) — 决策树的定义、信息增益原理及基本构建流程
- 📄 [02_过拟合与正则化](../raw/计算机科学/机器学习/02_过拟合与正则化.md) — 过拟合现象与L1/L2正则化方法的对比
- 📄 [机器学习](../core/机器学习.md) — 整合了决策树、正则化、模型评估等机器学习核心知识
```

### 3.5 摘要的维护
- 新增笔记 → 索引追加该条目，附带摘要
- 笔记内容重大更新 → 检查摘要是否需要同步更新
- 笔记重命名或移动 → 索引中更新路径和文件名
- 删除笔记 → 索引中移除对应条目

## 四、索引生成流程

```
触发条件（raw 或 core 有变动）
    │
    ▼
┌─────────────────────────┐
│ 1. 扫描 raw/ 和 core/   │
│    递归遍历所有文件夹    │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│ 2. 构建目录树结构       │
│    按层级组织            │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│ 3. 为每个文件           │
│    读取或生成摘要        │
│    - 新文件：AI 生成摘要 │
│    - 已有文件：沿用旧摘要│
│      或检查是否需要更新  │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│ 4. 写入 index/全文目录.md│
│    包含更新时间戳        │
└───────────┬─────────────┘
            │
            ▼
┌─────────────────────────┐
│ 5. 检查：链接是否有效？ │
│    摘要是否准确？       │
└───────────┬─────────────┘
            │
            ▼
       记录日志
```

## 五、更新触发条件

以下情况应触发索引更新：
- raw/ 有新笔记入库
- raw/ 有文件被重命名、移动或删除
- core/ 有新提炼内容或文件变更
- raw/ 或 core/ 的文件夹结构变化（新建/删除/重命名主题文件夹）

---

## 六、AI 使用索引的规范

当 AI 需要查询知识库内容时，遵循以下顺序：

1. **先读索引** `index/全文目录.md`，获取全局目录树和所有文件的摘要
2. **根据摘要定位**相关文件范围，确定需要打开哪些文件
3. **按需打开具体文件**，阅读完整内容
4. **回答问题时引用来源**，注明文件路径

这比盲目遍历所有文件更高效。

---

## 七、日志记录

遵循 `log_standard.md` 规范，操作类型为 `INDEX`。

---

## 八、禁止事项

- ❌ 索引文件与实际的文件夹结构不一致
- ❌ 摘要为空或照抄文件名
- ❌ 链接使用绝对路径
- ❌ 遗漏 raw/ 或 core/ 中的文件
- ❌ 索引过期后不更新

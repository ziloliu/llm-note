## 二、中部核心 — 输入通道与通道组

### 2.1 输入通道（18 个）

| 类型 | 通道号 | 数量 | 来源 |
|------|:------:|:----:|------|
| 外部通道 | 0 ~ 15 | 16 个 | 复用 GPIO 引脚 |
| 内部通道 | 16 | 1 个 | 温度传感器 |
| 内部通道 | 17 | 1 个 | 内部参考电压 |

### 2.2 通道分组（歌单类比）

```
规则通道组（常规歌单）：最多 16 个通道（18 选 16）
注入通道组（插队歌单）：最多 4 个通道（18 选 4）

每个通道组 = 一个歌单 = 一个序列
  → 保存在序列寄存器中
  → 按配置顺序依次转换
```

### 2.3 序列寄存器（SQR）

```
规则通道序列寄存器：
  SQR1：SQ13~SQ16 + L（序列长度）
  SQR2：SQ7~SQ12
  SQR3：SQ1~SQ6

每个 SQ 用 5 位表示（通道号 0~17，需要 5 位）
总容量：16 个通道 × 5 位 = 80 位 → 3 个寄存器

L（长度）：4 位（0~15 表示 1~16 个通道）
  L = 0000 → 1 个通道
  L = 1111 → 16 个通道

配置顺序：从 SQR3 开始（SQ1 在最低位），不够再往 SQR2、SQR1 扩展
```

### 2.4 采样时间寄存器（SMPR）

```
SMPR2：通道 0~9（每个通道 3 位）
SMPR1：通道 10~17（每个通道 3 位）

采样时间选择（3 位）：
  000 = 1.5 周期
  001 = 7.5 周期
  010 = 13.5 周期
  ...
  111 = 239.5 周期
```

## 三、中部核心 — 数据寄存器与数据保护

### 3.1 规则通道 DR（1 个，16 位）

```
问题：多通道共用 1 个 DR → 后转换的结果会覆盖前面的

解决方案：
  ① 判断 EOC 标志/中断 → 及时读取 DR
  ② DMA 自动传输（推荐，效率最高）
```

### 3.2 注入通道 JDR（4 个，各自 16 位）

```
4 个通道 → 4 个 JDR → 一一对应 → 不覆盖
无需额外保护措施
```

### 3.3 ADC DMA 支持

| ADC 模块 | DMA 支持 | DMA 通道 |
|:--------:|:--------:|:--------:|
| **ADC1** | ✅ | DMA1 Channel 1 |
| **ADC2** | ❌ | 无（底层无连线） |
| **ADC3** | ✅ | DMA2 Channel 5 |

## 四、上部 — 事件与中断

### 4.1 三种事件

| 事件 | 缩写 | 标志位 | 说明 |
|------|------|:------:|------|
| 转换结束 | **EOC** | SR.EOC | 规则或注入组转换结束 |
| 注入转换结束 | **JEOC** | SR.JEOC | 仅注入组转换结束 |
| 模拟看门狗 | **AWD** | SR.AWD | 电压超出阈值范围 |

### 4.2 EOC 标志的关键细节

```
⚠️ EOC 是"组"转换结束标志，不是单个通道转换结束！

单通道时：
  转换一次 → 产生一次 EOC

多通道 + 扫描模式时：
  整组转换完成 → 才产生一次 EOC
  中间各通道转换完成不产生 EOC

→ 这就是多通道必须用 DMA 的原因
```

### 4.3 模拟看门狗

```
HTR：高阈值（上限，12 位）
LTR：低阈值（下限，12 位）

转换结果 > HTR 或 < LTR → AWD 事件 → 可产生中断
```

## 五、下部 — 触发源选择

### 5.1 触发方式

```
规则通道启动：
  方式一：ADON（简单，只能启动规则组）
    → 第一次写 1：上电唤醒
    → 再次写 1：启动转换

  方式二：SWSTART（通用，规则组和注入组都能用）
    → 前提：EXTTRIG = 1, EXTSEL = 111
    → 写 SWSTART = 1 启动规则组
    → 写 JSWSTART = 1 启动注入组

注入通道启动：
  只能用方式二：JEXTTRIG = 1, JEXTSEL = 111, JSWSTART = 1
```

### 5.2 触发配置三步走（SWSTART 方式）

```
① CR2.EXTTRIG = 1    → 开启外部触发模式（打开与门）
② CR2.EXTSEL = 111   → 选择软件触发（8 选 1 选最后一种）
③ CR2.SWSTART = 1    → 按下启动开关，开始转换

①② 是预先配置，③ 是真正的启动信号
```

### 5.3 ADON 方式（简化版）

```c
// 第一次写 1：上电唤醒
ADC1->CR2 |= ADC_CR2_ADON;

// 校准（可选但推荐）
ADC1->CR2 |= ADC_CR2_CAL;
while (ADC1->CR2 & ADC_CR2_CAL);

// 再次写 1：启动转换（只对规则组有效）
ADC1->CR2 |= ADC_CR2_ADON;
```

### 5.4 ADC3 触发电路差异

```
ADC1 + ADC2：共用一套触发电路（通用定义）
ADC3：独立的触发电路（不同的硬件连线和定义）
```

## 六、转换方式（重点！4 种组合）

### 6.1 两个控制位

| 位 | 寄存器 | 含义 |
|:--:|:------:|------|
| **SCAN** | CR1 | 扫描模式（0=关闭，1=开启） |
| **CONT** | CR2 | 连续转换（0=单次，1=连续） |

### 6.2 四种模式

| SCAN | CONT | 模式 | 歌单类比 |
|:----:|:----:|------|---------|
| 0 | 0 | 单次单通道 | **单曲播放**（播完停） |
| 1 | 0 | 单次多通道 | **顺序播放**（播完停） |
| 0 | 1 | 连续单通道 | **单曲循环** |
| 1 | 1 | 连续多通道 | **列表循环** |

## 七、特殊工作模式

### 7.1 间断模式（CR1.DISCEN）

```
间断模式 = 把歌单分成几段，每次触发播放一段

配置：
  CR1.DISCEN = 1（开启间断模式）
  CR1.DISCNUM[2:0] = N（每次转换的通道数，最多 8）

工作过程：
  第 1 次触发：转换通道 0, 1, 2（前 N 个）→ 停止
  第 2 次触发：断点续传，转换通道 3, 6, 7（接着上次）→ 停止
  第 3 次触发：转换剩余通道 9, 10 → 全组完成，产生 EOC
  第 4 次触发：回到起点重新开始

特点：
  → 不能与连续模式（CONT）同时使用
  → 一段一段播放，手动触发
  → 支持断点续传
```

### 7.2 自动注入（CR1.JAUTO）

```
JAUTO = 1：
  → 规则通道组扫描完成后
  → 自动追加注入通道组的转换
  → 相当于歌单末尾自动加入注入歌单

  规则歌单：[Ch0] [Ch1] [Ch2]
  自动追加：[Ch0] [Ch1] [Ch2] [注入Ch0] [注入Ch1]
  → 一轮完整播放后自动回到起点循环
```

## 八、校准操作

```
CR2.CAL = 1 → 开始校准
硬件自动执行 → 校准完成 → 硬件自动将 CAL 清零

判断校准完成：
  while (ADC1->CR2 & ADC_CR2_CAL);  // CAL = 0 时退出

不校准的后果：
  → 测量结果有偏差
  → 最大电压只能到约 3.25V（到不了 3.3V）

类比：天平使用前先调零
```

## 十、C 语言运算符优先级问题

### 10.1 错误写法

```c
while (ADC1->SR & ADC_SR_EOC == 0);  // ❌ 错误
```

```
优先级问题：
  == 的优先级 高于 &（按位与）
  
实际执行顺序：
  ① ADC_SR_EOC == 0  → 假（EOC 是位定义，不等于 0）
  ② ADC1->SR & 0     → 结果为 0
  ③ while(0)         → 不等待，直接退出

→ 没有真正等待转换完成！
```

### 10.2 正确写法

```c
while ((ADC1->SR & ADC_SR_EOC) == 0);  // ✅ 正确
```

```
加括号强制先执行按位与：
  ① (ADC1->SR & ADC_SR_EOC)  → 提取 EOC 位的值
  ② 结果 == 0                → 判断 EOC 是否为 0
  ③ 为 0 则继续等待，不为 0 则退出

→ 正确等待转换完成！
```

### 10.3 运算符优先级参考

```
优先级（从高到低）：
  ==  !=        （比较运算符）  ← 优先级高
  &  |  ^       （位运算符）    ← 优先级低

结论：位运算与比较运算混合使用时，必须加括号！
```

## 十一、电压转换公式

```
y = x × 3.3 / 4095

  x = DR 中的 12 位数字量（0~4095）
  y = 实际电压值（V）
  3.3 = VREF+ - VREF-
  4095 = 2^12 - 1（全 1 值）

例：
  x = 2048 → y ≈ 1.65V
  x = 4095 → y = 3.3V
  x = 0    → y = 0V
```

## 十二、完整 ADC 配置流程

```
初始化：
  ① 开 ADC 时钟（APB2ENR）
  ② ADC 时钟分频（CFGR.ADCPRE = 10，6 分频 → 12MHz）
  ③ GPIO 模拟输入配置
  ④ CR1.SCAN 配置（单通道=0，多通道=1）
  ⑤ CR2.CONT 配置（单次=0，连续=1）
  ⑥ CR2.ALIGN 配置（右对齐=0）
  ⑦ SMPR 采样时间配置
  ⑧ SQR 序列配置（L + 通道号）
  ⑨ CR2.EXTTRIG + EXTSEL 配置（软件触发=111）

启动：
  ① CR2.ADON = 1（上电唤醒）
  ② CR2.CAL = 1（校准，等待清零）
  ③ CR2.SWSTART = 1 或 CR2.ADON 再写 1

读取：
  ① while (!(SR & EOC))  → 等待转换完成
  ② 读取 DR → 计算电压
```

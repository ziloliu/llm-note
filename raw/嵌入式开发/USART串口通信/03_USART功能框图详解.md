## 一句话定义

USART模块由发送部分、接收部分、控制与状态寄存器三大部分组成，通过移位寄存器实现串并/并串转换。

## 核心内容

### 模块总览

![USART功能框图](assets/01_usart-block-diagram.png)

框图核心结构分为四大部分：

```
┌─────────────────────────────────────────────────────┐
│                    USART 模块                        │
│                                                      │
│  ┌──────────┐   ┌──────────┐   ┌──────────────────┐ │
│  │ 发送部分  │   │ 接收部分  │   │ 控制与状态寄存器  │ │
│  │ TDR→移位  │   │ 移位→RDR │   │ CR1/CR2/CR3/SR  │ │
│  └──────────┘   └──────────┘   │ + BRR 波特率      │ │
│                                 └──────────────────┘ │
└─────────────────────────────────────────────────────┘
```

### 引脚说明

**核心引脚：**

| 引脚 | 全称 | 方向 | 功能 |
|------|------|------|------|
| **TX** | Transmit | 输出 | 数据发送端，连接对方 RX |
| **RX** | Receive | 输入 | 数据接收端，连接对方 TX |

**扩展引脚：**

| 引脚 | 全称 | 功能 | 使用场景 |
|------|------|------|----------|
| SW_RX | Single-Wire RX | 单线模式/智能卡模式下的接收（内部引脚） | 半双工单线模式 |
| IrDA_OUT | Infrared Data Output | 红外发送 | IrDA 无线通信 |
| IrDA_IN | Infrared Data Input | 红外接收 | IrDA 无线通信 |
| **nRTS** | Request To Send（低有效） | 输出：告知对方"我已准备好接收" | 硬件流控 |
| **nCTS** | Clear To Send（低有效） | 输入：对方告知"你可以发送了" | 硬件流控 |
| **SCLK** | Synchronous Clock | 同步时钟输出 | 仅 USART 同步模式 |

**硬件流控说明：**
```
设备A                    设备B
  nRTS ──────────→ nCTS   （A 告诉 B：我准备好接收了）
  nCTS ←────────── nRTS   （B 告诉 A：你可以发了）
```

### 数据接收流程

```
RX 引脚（串行数据逐位输入）
    ↓
接收移位寄存器（串→并转换）
    ↓  并行传输（8/9位）
接收数据寄存器 RDR
    ↓  读取
CPU（经数据总线）
```

**接收移位寄存器工作方式：**
- 串口协议低位先发，数据依次进入**最高位**，每接收一位**右移**一次
- 移位次数：8 次后完成一个字节的接收（8 位数据时）

```
接收顺序（先收 D0，后收 D7）：
第1次：[D0, -, -, -, -, -, -, -]    D0 进入最高位
第2次：[D1, D0, -, -, -, -, -, -]   右移1位
第3次：[D2, D1, D0, -, -, -, -, -]  右移2位
...
第8次：[D7, D6, D5, D4, D3, D2, D1, D0]  完成，D0 在最低位
```

### 数据发送流程

```
CPU（经数据总线）
    ↓  写入
发送数据寄存器 TDR
    ↓  并行传输
发送移位寄存器（并→串转换）
    ↓  逐位输出
TX 引脚（串行数据逐位发出）
```

**发送移位寄存器工作方式：**
- 每次发送**最低位**（LSB 先发），发送后**右移**一次
- 最低位逐次送至 TX 引脚

```
初始：[D7, D6, D5, D4, D3, D2, D1, D0]
第1次发送：D0 → TX，右移
第2次发送：D1 → TX，右移
...
第8次发送：D7 → TX，完成
```

### 关键寄存器

**数据寄存器（DR）：**
- 底层硬件：实际为 **TDR**（写入）和 **RDR**（读取）两个独立寄存器
- 编程接口：共用同一个地址，统一命名为 **DR**
- 读操作：实际读取 RDR（接收数据寄存器）
- 写操作：实际写入 TDR（发送数据寄存器）

**状态寄存器（SR）三个关键标志位：**

| 标志位 | 全称 | 含义 | 用途 |
|--------|------|------|------|
| **RXNE** | Receive Data Register Not Empty | 接收数据寄存器**非空**（已收到数据） | 读取 DR 前判断，确保数据已就绪 |
| **TXE** | Transmit Data Register Empty | 发送数据寄存器**为空**（数据已移出） | 写入 DR 前判断，确保不会覆盖未发送数据 |
| **IDLE** | Idle Line Detected | 检测到**空闲帧**（线路持续空闲一帧长度） | 用于判断变长数据接收完毕 |

**控制寄存器（CR1 / CR2 / CR3）：**

| 寄存器 | 配置内容 |
|--------|----------|
| **CR1** | 数据位长度、校验方式、发送/接收使能、中断使能等 |
| **CR2** | 停止位长度、同步时钟配置、LIN 模式等 |
| **CR3** | 硬件流控（RTS/CTS）、DMA 使能、IrDA 模式等 |

**波特率寄存器（BRR）：**
- 功能：配置波特率分频值
- 计算公式：`波特率 = f_CK / (16 × BRR值)`
- f_CK：USART1 为 APB2 时钟（72MHz），其余为 APB1 时钟（36MHz）

### 框图整体信号流

```
发送方向：
  CPU → 数据总线 → TDR → 发送移位寄存器（并→串） → TX引脚 → 外部设备

接收方向：
  外部设备 → RX引脚 → 接收移位寄存器（串→并） → RDR → 数据总线 → CPU

控制方向：
  CR1/CR2/CR3 → 控制发送器/接收器的使能、数据格式、中断、扩展功能

状态反馈：
  发送器/接收器 → SR（TXE/RXNE/IDLE 等状态标志） → CPU 查询判断

波特率：
  外设时钟 → BRR 分频 → 驱动发送器/接收器的收发速率
```

## 注意事项 & 踩坑

- 编程时只需操作一个 DR 寄存器，底层硬件自动区分读/写对应的物理寄存器
- 接收移位寄存器是右移操作，数据从高位进入
- 发送移位寄存器也是右移操作，数据从低位取出
- RXNE 和 TXE 是最常用的状态标志位

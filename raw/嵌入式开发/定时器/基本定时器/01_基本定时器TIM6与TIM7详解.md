## 四、时基单元三大寄存器

### 4.1 总览

| 寄存器 | 全称 | 位宽 | 功能 |
|--------|------|:----:|------|
| **PSC** | Prescaler（预分频器） | 16位 | 对输入时钟进行分频 |
| **CNT** | Counter（计数器） | 16位 | 从 0 向上递增计数 |
| **ARR** | Auto-Reload Register（自动重装载） | 16位 | 设定计数上限 |

### 4.2 PSC — 预分频器

```
功能：对输入时钟进行分频，降低计数频率

分频值 = PSC + 1（PSC 为寄存器中写入的值）

示例：
  PSC = 0    → 1 分频（不分频）  → 计数频率 = 72MHz
  PSC = 1    → 2 分频           → 计数频率 = 36MHz
  PSC = 7199 → 7200 分频        → 计数频率 = 10kHz

范围：1 分频 ~ 65536 分频
```

### 4.3 CNT — 计数器寄存器

```
功能：在每个计数器时钟上升沿自动加 1

计数过程：
  0 → 1 → 2 → ... → ARR → 溢出 → 0（重新开始）

特点：
  - 只能向上计数（递增）
  - 从 0 开始
  - 达到 ARR 值后下一个时钟沿溢出
```

### 4.4 ARR — 自动重装载寄存器

```
功能：设定计数器的上限值

溢出条件：CNT 的值超过 ARR → 下一个时钟上升沿产生溢出
溢出行为：
  → CNT 归零，重新开始计数
  → 产生更新事件 (UEV)
  → 若开启中断，产生更新中断 (UI)

注意：达到 ARR 后还需一个时钟周期才溢出
      从 0 计到 ARR 共经过 (ARR + 1) 个计数周期
```

```

                      ┘└┘└┘└┘└┘└┘└┘└┘└┘└┘└┘└┘└┘└┘└┘└┘└┘└┘└

定时器时钟（PSC=1, 二分频）:
                      ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──
                      └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘
                     ↑     ↑     ↑     ↑     ↑     ↑     ↑
```

| 事件 | 说明 |
|------|------|
| CNT = 34 | 正常计数 |
| CNT = 35 | 正常计数 |
| CNT = 36 | **达到 ARR 值**（ARR = 36） |
| CNT → 37 | **下一个时钟沿** → 溢出 → 归零 → 产生更新事件 |
| CNT = 0 | 重新开始计数 |

## 六、时钟来源

### 6.1 内部时钟（CK_INT）

基本定时器**只有一种时钟源**：内部时钟。

### 6.2 定时器时钟频率（关键）

```
APB1 总线频率 = 36 MHz（72MHz / 2 分频）

定时器时钟规则：
  如果 APB1 预分频系数 = 1 → 定时器时钟 = APB1 频率
  如果 APB1 预分频系数 > 1 → 定时器时钟 = APB1 频率 × 2

本实验：
  APB1 预分频系数 = 2（> 1）
  → 定时器时钟 = 36 MHz × 2 = 72 MHz
```

### 6.3 时钟树对应关系

```
                    ┌──────────┐
   HSE(8MHz) ──→PLL │  ×9      │──→ SYSCLK = 72 MHz
                    └──────────┘         │
                                        ↓
                                    AHB = 72 MHz
                                        │
                          ┌─────────────┼──────────────┐
                          ↓             ↓              ↓
                     APB1 /2        APB2 /1       Cortex
                     = 36MHz        = 72MHz       系统时钟
                          │
                    定时器时钟 = 36 × 2 = 72 MHz
                    (TIM2~TIM7)
```

## 七、ARR 预加载机制（影子寄存器）

### 7.1 双寄存器结构

```
                    CPU 写入
                       ↓
              ┌─────────────────┐
              │   预加载寄存器    │  ← CPU 直接操作的寄存器
              │  (Preload)       │
              └────────┬────────┘
                       │ 更新事件到来时
                       ↓
              ┌─────────────────┐
              │   影子寄存器      │  ← 真正起作用的寄存器
              │  (Shadow)        │     (与 CNT 比较的值)
              └─────────────────┘
```

### 7.2 ARPE 位（CR1 寄存器）

| ARPE 值 | 模式 | ARR 更新时机 | 效果 |
|:-------:|------|-------------|------|
| **0** | **无预加载** | 写入 ARR 后**立即**更新到影子寄存器 | **即时生效** |
| **1** | **有预加载** | 等待下一次**更新事件**后才更新到影子寄存器 | **延迟生效** |

### 7.3 无预加载（ARPE = 0）

```
ARR = 0xFF（当前值）

  CPU 写入 ARR = 36
       ↓
  预加载寄存器 = 36
       ↓ （立即）
  影子寄存器 = 36    ← 立即生效
       ↓
  CNT = 32（当前）→ 33 → 34 → 35 → 36 → 溢出！

效果：写入后立刻改变比较值
```

### 7.4 有预加载（ARPE = 1）

```
ARR = 0xF5（当前值）

  CPU 写入 ARR = 36
       ↓
  预加载寄存器 = 36
       ↓ （等待更新事件）
  影子寄存器 仍为 0xF5    ← 未生效
       ↓
  CNT = F1 → F2 → F3 → F4 → F5 → 溢出！
       ↓ （产生更新事件）
  影子寄存器 = 36    ← 此时才更新
       ↓
  下一轮：CNT = 0 → 1 → ... → 36 → 溢出

效果：当前一轮按旧值计数，下一轮按新值计数
      "老人老办法，新人新办法"
```

### 7.5 两种模式对比

| 对比项 | ARPE = 0（无预加载） | ARPE = 1（有预加载） |
|--------|:-------------------:|:-------------------:|
| 写入 ARR 后 | 立即生效 | 下一轮才生效 |
| 适用场景 | 需要立即改变定时 | 运行中动态改变定时 |
| 安全性 | 较低（可能打断当前计数） | **较高**（不影响当前轮） |

## 八、定时时间计算

### 8.1 计算公式

```
定时时间 = (PSC + 1) × (ARR + 1) / 时钟频率

其中：
  PSC + 1 = 实际分频值
  ARR + 1 = 实际计数次数（从 0 计到 ARR 共 ARR+1 个周期）
  时钟频率 = 72 MHz
```

### 8.2 计算步骤

```
① 计数器时钟频率 = 时钟频率 / (PSC + 1)
② 计数器周期 = 1 / 计数器时钟频率 = (PSC + 1) / 时钟频率
③ 定时时间 = 计数器周期 × (ARR + 1)
         = (PSC + 1) × (ARR + 1) / 时钟频率
```

### 8.3 示例计算

#### 示例 1：定时 1 秒

```
目标：定时时间 = 1 秒
时钟频率 = 72 MHz

方案：
  PSC = 7199 → 分频值 = 7200
  ARR = 9999 → 计数次数 = 10000

验证：
  计数器时钟频率 = 72MHz / 7200 = 10,000 Hz
  计数器周期 = 1 / 10000 = 0.1 ms = 100 μs
  定时时间 = 100 μs × 10000 = 1,000,000 μs = 1 秒 ✅
```

#### 示例 2：定时 1 毫秒

```
目标：定时时间 = 1 ms
时钟频率 = 72 MHz

方案 A：
  PSC = 7199 → 分频值 = 7200
  ARR = 9    → 计数次数 = 10

验证：
  计数器时钟频率 = 72MHz / 7200 = 10,000 Hz
  定时时间 = 0.1ms × 10 = 1 ms ✅

方案 B：
  PSC = 71   → 分频值 = 72
  ARR = 999  → 计数次数 = 1000

验证：
  计数器时钟频率 = 72MHz / 72 = 1 MHz
  定时时间 = 1μs × 1000 = 1 ms ✅
```

#### 示例 3：PSC=7199 时的详细计算

```
内部时钟频率 = 72 MHz = 72,000,000 Hz
PSC = 7199

计数器时钟频率 = 72,000,000 / (7199 + 1)
             = 72,000,000 / 7200
             = 10,000 Hz

计数器周期 = 1 / 10,000 = 0.0001 秒 = 0.1 ms = 100 μs

即：每 100 μs 计数器加 1
```

## 九、PSC 加 1 和 ARR 加 1 的原因

### 9.1 PSC 加 1

```
PSC 存储值 = 0 → 实际分频 = 0 + 1 = 1 分频（不分频）✅
PSC 存储值 = 1 → 实际分频 = 1 + 1 = 2 分频 ✅
PSC 存储值 = 65535 → 实际分频 = 65535 + 1 = 65536 分频 ✅

如果不加 1：
  PSC = 0 表示 0 分频 → 无意义 ❌
  分频范围变成 0~65535，其中 0 无效

加 1 后：
  分频范围 1~65536，全部有效 ✅
```

### 9.2 ARR 加 1

```
ARR = 36（寄存器值）

计数过程：0 → 1 → 2 → ... → 36 → 下一个时钟沿溢出
共经过：37 个计数周期（0 到 36 共 37 个值）

从 0 计到 N 共 N+1 次（类似数组下标 0~N 共 N+1 个元素）
```

## 十、定时器连接的总线

| 定时器 | 类型 | 总线 | 定时器时钟 |
|--------|------|------|-----------|
| TIM1 | 高级 | APB2 | 72 MHz |
| TIM2 | 通用 | APB1 | 72 MHz |
| TIM3 | 通用 | APB1 | 72 MHz |
| TIM4 | 通用 | APB1 | 72 MHz |
| TIM5 | 通用 | APB1 | 72 MHz |
| **TIM6** | **基本** | **APB1** | **72 MHz** |
| **TIM7** | **基本** | **APB1** | **72 MHz** |
| TIM8 | 高级 | APB2 | 72 MHz |

> 所有定时器的时钟频率均为 72 MHz（因为 APB1 分频系数为 2，乘回 2 得 72MHz）。

## 十一、基本定时器功能总结

| 功能 | 支持 | 说明 |
|------|:----:|------|
| 定时中断 | ✅ | 更新事件产生更新中断 |
| 向上计数 | ✅ | 从 0 递增到 ARR |
| 向下计数 | ❌ | 不支持 |
| 预分频器 | ✅ | 16 位，1~65536 分频 |
| 自动重装载 | ✅ | 16 位，含预加载/影子寄存器 |
| 外部时钟输入 | ❌ | 只有内部时钟 |
| 外部 IO | ❌ | 无输入捕获/输出比较 |
| 触发 DAC | ✅ | TRGO 输出信号 |
| PWM 输出 | ❌ | 通用/高级定时器才有 |
| 输入捕获 | ❌ | 通用/高级定时器才有 |
| 通道数 | **0** | 无任何通道 |

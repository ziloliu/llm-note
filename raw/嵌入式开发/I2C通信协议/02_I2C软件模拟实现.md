# I²C 软件模拟实现（STM32）

> 本文档整合了 I²C 软件模拟的工程搭建、基础定义、底层协议函数实现的完整内容。

---

## 一、分层架构

```
┌─────────────────────────────────────────┐
│           应用层（main.c）               │
│   写入数据 → 读出数据 → 串口打印验证      │
├─────────────────────────────────────────┤
│      接口层（m24c02.c / AT24C02）        │
│   EEPROM_WriteByte / EEPROM_ReadByte     │
│   假写真读、页写入、连续读写等             │
├─────────────────────────────────────────┤
│      协议层（i2c.c / I²C 协议）           │
│   起始/停止、收发字节、应答/非应答         │
│   只负责：数据有效性、SCL/SDA 电平控制    │
├─────────────────────────────────────────┤
│         硬件层（GPIO + 上拉电阻）          │
│   PB10(SCL) / PB11(SDA) 开漏输出         │
└─────────────────────────────────────────┘
```

> I²C 协议层**不关心**设备地址、内部地址、假写真读等——这些属于接口层（AT24C02）的职责。

## 三、i2c.h 头文件 — 宏定义

### 3.1 ACK / NACK 常量

```c
#define I2C_ACK    0    // 应答：低电平
#define I2C_NACK   1    // 非应答：高电平
```

| 常量 | 值 | 对应 SDA 电平 | 含义 |
|------|:--:|:------------:|------|
| `I2C_ACK` | 0 | 低电平 | 接收成功，应答 |
| `I2C_NACK` | 1 | 高电平 | 接收失败 / 不再接收 |

### 3.2 SCL 控制宏

```c
#define SCL_HIGH()    (GPIOB->ODR |=  (1 << 10))   // PB10 拉高
#define SCL_LOW()     (GPIOB->ODR &= ~(1 << 10))   // PB10 拉低
```

| 宏 | 操作 | 对应寄存器 | 效果 |
|----|------|-----------|------|
| `SCL_HIGH()` | ODR bit10 置 1 | GPIOB->ODR | 时钟线高电平（上拉/高阻态） |
| `SCL_LOW()` | ODR bit10 清零 | GPIOB->ODR | 时钟线低电平（NMOS 接地） |

### 3.3 SDA 写入控制宏

```c
#define SDA_HIGH()    (GPIOB->ODR |=  (1 << 11))   // PB11 拉高
#define SDA_LOW()     (GPIOB->ODR &= ~(1 << 11))   // PB11 拉低
```

| 宏 | 操作 | 对应寄存器 | 效果 |
|----|------|-----------|------|
| `SDA_HIGH()` | ODR bit11 置 1 | GPIOB->ODR | 数据线高电平（释放总线） |
| `SDA_LOW()` | ODR bit11 清零 | GPIOB->ODR | 数据线低电平（占用总线） |

### 3.4 SDA 读取宏

```c
#define READ_SDA()    (GPIOB->IDR & (1 << 11))     // 读取 PB11 电平
```

| 宏 | 操作 | 对应寄存器 | 效果 |
|----|------|-----------|------|
| `READ_SDA()` | 读取 IDR bit11 | GPIOB->IDR | 非零=高电平(1)，零=低电平(0) |

> **关键：** GPIO 配置为输出模式时，IDR 仍可读取引脚实际电平，因此无需切换输入/输出模式即可读取 SDA。

### 3.5 延迟宏

```c
#define I2C_DELAY()   Delay_us(10)
```

| 宏 | 延迟 | 计算依据 |
|----|------|----------|
| `I2C_DELAY()` | 10 μs | I²C 标准模式 100kHz → 周期 10μs |

```
标准模式 100kHz 频率计算：
  周期 = 1 / 100,000 = 10 × 10⁻⁶ s = 10 μs

SCL 一个完整周期：
  高电平 ≈ 5μs + 低电平 ≈ 5μs = 10μs
  每次电平切换延迟 10μs 可保证时序满足要求
```

### 3.6 完整 i2c.h 文件

```c
#ifndef __I2C_H
#define __I2C_H

#include "stm32f10x.h"
#include "delay.h"

/* ========== ACK/NACK 常量 ========== */
#define I2C_ACK     0       // 应答：低电平
#define I2C_NACK    1       // 非应答：高电平

/* ========== SCL 控制（PB10）========== */
#define SCL_HIGH()  (GPIOB->ODR |=  (1 << 10))
#define SCL_LOW()   (GPIOB->ODR &= ~(1 << 10))

/* ========== SDA 写入控制（PB11）========== */
#define SDA_HIGH()  (GPIOB->ODR |=  (1 << 11))
#define SDA_LOW()   (GPIOB->ODR &= ~(1 << 11))

/* ========== SDA 读取（PB11）========== */
#define READ_SDA()  (GPIOB->IDR & (1 << 11))

/* ========== 延迟 ========== */
#define I2C_DELAY() Delay_us(10)

/* ========== 函数声明 ========== */
void I2C_Init(void);
void I2C_Start(void);
void I2C_Stop(void);
void I2C_SendAck(void);
void I2C_SendNack(void);
uint8_t I2C_WaitAck(void);
void I2C_SendByte(uint8_t byte);
uint8_t I2C_ReceiveByte(void);

#endif
```

### 3.7 宏定义 vs 函数

| 对比项 | 宏定义（SCL_HIGH 等） | 函数（I2C_Start 等） |
|--------|---------------------|---------------------|
| 层级 | **最底层**：控制单个引脚电平 | **协议层**：实现 I²C 协议的一个完整步骤 |
| 内容 | 一条 ODR/IDR 操作指令 | 多步电平操作 + 延迟的组合 |
| 效率 | 全局替换，无函数调用开销 | 有函数调用开销 |
| 返回值 | READ_SDA 有返回值 | I2C_WaitAck/I2C_ReceiveByte 有返回值 |

## 四、宏定义工作原理

### 4.1 ODR 位操作

```c
// 置 1（拉高）：按位或
GPIOB->ODR |= (1 << 10);    // 仅 bit10 置 1，其余不变

// 清零（拉低）：按位与取反
GPIOB->ODR &= ~(1 << 10);   // 仅 bit10 清零，其余不变
```

### 4.2 IDR 读取

```c
// 读取指定位
GPIOB->IDR & (1 << 11);     // 结果非零 = 高电平，结果为零 = 低电平
```

### 4.3 宏定义加括号的原因

```c
#define SCL_HIGH()  (GPIOB->ODR |= (1 << 10))  // 整体加括号

// 防止与其他表达式组合时出现优先级问题
// 例如：if (condition && SCL_HIGH()) 不会出错
```

## 七、I2C_Init() — GPIO 初始化

### 7.1 配置要求

| 引脚 | 功能 | GPIO 模式 | CNF | MODE |
|------|------|-----------|:---:|:----:|
| **PB10** | SCL 时钟线 | 通用开漏输出，50MHz | 01 | 11 |
| **PB11** | SDA 数据线 | 通用开漏输出，50MHz | 01 | 11 |

> CRH 寄存器（引脚 8~15），每引脚占 4 位：`[CNF1:CNF0][MODE1:MODE0]`

### 7.2 代码实现

```c
void I2C_Init(void)
{
    /* 1. 开启 GPIOB 时钟 */
    RCC->APB2ENR |= RCC_APB2ENR_IOPBEN;

    /* 2. 配置 PB10 和 PB11 为通用开漏输出，50MHz */
    // MODE[1:0] = 11（50MHz 输出）
    GPIOB->CRH |=  (GPIO_CRH_MODE10 | GPIO_CRH_MODE11);
    
    // CNF[1:0] = 01（通用开漏输出）
    GPIOB->CRH &= ~(GPIO_CRH_CNF10 | GPIO_CRH_CNF11);   // 先清零
    GPIOB->CRH |=  (GPIO_CRH_CNF10_0 | GPIO_CRH_CNF11_0); // 再置 01
}
```

### 7.3 为什么用开漏输出

| 输出值 | NMOS 状态 | 引脚状态 | 总线状态 |
|:------:|----------|----------|----------|
| 0 | 导通 | 接地 | **低电平**（占用总线） |
| 1 | 关断 | 高阻态 | **高电平**（上拉电阻拉高，释放总线） |

> 开漏输出 + 外部上拉电阻完美匹配 I²C 总线的线与逻辑。

### 7.4 为什么输出模式下仍能读取输入

```
GPIO 输出模式内部结构：
  ODR → 输出驱动器 → 引脚
              ↓
          输入锁存器 → IDR（始终连接）

即使配置为输出，IDR 仍可读取引脚实际电平
→ READ_SDA() 宏无需切换输入/输出模式即可工作
```

## 八、I2C_Start() — 发送起始信号

### 8.1 时序回顾

```
起始信号：SCL 高电平期间，SDA 产生下降沿（高→低）

SCL:  ──────────┐           ┌──────────
                 └───────────┘
SDA:  ──┐                   （数据传输...）
        └───────────────────
         ↑ 起始信号
```

### 8.2 看图说话步骤

| 步骤 | SCL | SDA | 操作 | 代码 |
|:----:|:---:|:---:|------|------|
| ① | 拉高 | 拉高 | 两线都拉高，初始空闲状态 | `SCL_HIGH(); SDA_HIGH(); I2C_DELAY();` |
| ② | 保持高 | **拉低** | SDA 产生下降沿 | `SDA_LOW(); I2C_DELAY();` |

### 8.3 代码实现

```c
void I2C_Start(void)
{
    /* ① 两线拉高（空闲状态） */
    SDA_HIGH();
    SCL_HIGH();
    I2C_DELAY();

    /* ② SCL 保持高，SDA 拉低（下降沿 = 起始信号） */
    SDA_LOW();
    I2C_DELAY();
}
```

## 九、I2C_Stop() — 发送停止信号

### 9.1 时序回顾

```
停止信号：SCL 高电平期间，SDA 产生上升沿（低→高）

SCL:  ──────────┐           ┌──────────
                 └───────────┘
SDA:  ────────────────────┐
                          └──────────
                           ↑ 停止信号
```

### 9.2 代码实现

```c
void I2C_Stop(void)
{
    /* ① SCL 拉高，SDA 拉低（准备状态） */
    SDA_LOW();
    SCL_HIGH();
    I2C_DELAY();

    /* ② SCL 保持高，SDA 拉高（上升沿 = 停止信号） */
    SDA_HIGH();
    I2C_DELAY();
}
```

### 9.3 Start 与 Stop 对比

| 操作 | 初始 SDA | 最终 SDA | SCL |
|------|:--------:|:--------:|:---:|
| **Start** | 高（拉高） | **低**（拉低） | 保持高 |
| **Stop** | 低（拉低） | **高**（拉高） | 保持高 |

## 十、I2C_SendAck() — 主机发送应答信号

### 10.1 时序回顾

```
ACK：主机在第 9 个时钟周期，将 SDA 拉低表示应答

SCL:  ─┐ ┌─┐
    ───┘ └─┘ └───
SDA: ──┐     ┌───
       └─────┘
         ↑ 低电平 = ACK
```

### 10.2 看图说话步骤

| 步骤 | SCL | SDA | 操作 | 代码 |
|:----:|:---:|:---:|------|------|
| ① | 拉低 | 拉高 | 准备状态，释放数据线 | `SCL_LOW(); SDA_HIGH(); I2C_DELAY();` |
| ② | 保持低 | **拉低** | SDA 输出低电平（ACK） | `SDA_LOW(); I2C_DELAY();` |
| ③ | **拉高** | 保持低 | SCL 拉高，开始采样 | `SCL_HIGH(); I2C_DELAY();` |
| ④ | **拉低** | 保持低 | SCL 拉低，采样结束 | `SCL_LOW(); I2C_DELAY();` |
| ⑤ | 保持低 | **拉高** | 释放数据总线 | `SDA_HIGH(); I2C_DELAY();` |

### 10.3 代码实现

```c
void I2C_SendAck(void)
{
    /* ① 准备：SCL 低，SDA 高（释放） */
    SCL_LOW();
    SDA_HIGH();
    I2C_DELAY();

    /* ② SCL 低电平期间，SDA 拉低（设置 ACK 电平） */
    SDA_LOW();
    I2C_DELAY();

    /* ③ SCL 拉高，从设备采样 SDA（读到低电平 = ACK） */
    SCL_HIGH();
    I2C_DELAY();

    /* ④ SCL 拉低，采样结束 */
    SCL_LOW();
    I2C_DELAY();

    /* ⑤ 释放数据总线 */
    SDA_HIGH();
    I2C_DELAY();
}
```

## 十一、I2C_SendNack() — 主机发送非应答信号

### 11.1 与 ACK 的区别

NACK 时 SDA **保持高电平**（不拉低），其余步骤相同。

### 11.2 代码实现

```c
void I2C_SendNack(void)
{
    /* ① 准备：SCL 低，SDA 高 */
    SCL_LOW();
    SDA_HIGH();
    I2C_DELAY();

    /* ② SDA 保持高（不做拉低操作 = NACK） */

    /* ③ SCL 拉高，从设备采样 SDA（读到高电平 = NACK） */
    SCL_HIGH();
    I2C_DELAY();

    /* ④ SCL 拉低，采样结束 */
    SCL_LOW();
    I2C_DELAY();

    /* ⑤ SDA 已经是高，无需额外操作（已释放） */
}
```

### 11.3 ACK 与 NACK 对比

| 操作 | 步骤② SDA | 采样结果 | 含义 |
|------|:---------:|:--------:|------|
| **SendAck** | **拉低** | 低电平 (0) | 应答：收到数据 |
| **SendNack** | **保持高** | 高电平 (1) | 非应答：不再接收 |

## 十二、I2C_WaitAck() — 主机等待从设备应答

### 12.1 与 SendAck/SendNack 的区别

| 对比项 | SendAck / SendNack | WaitAck |
|--------|-------------------|---------|
| 方向 | 主机**输出**到 SDA | 主机**读取** SDA |
| SDA 控制方 | **主机**控制 SDA | **从设备**控制 SDA |
| 操作前 | 主机拉低/保持高 SDA | 主机**释放** SDA（拉高） |
| 结果 | 无返回值 | **返回** ACK(0) 或 NACK(1) |

### 12.2 时序

```
主机释放 SDA（拉高）→ SCL 拉高采样 → 读取 SDA 电平 → SCL 拉低

从设备行为：
  ACK：从设备拉低 SDA → 主机读到低电平 → 返回 0
  NACK：从设备不操作（保持高） → 主机读到高电平 → 返回 1
```

### 12.3 看图说话步骤

| 步骤 | SCL | SDA | 操作 | 代码 |
|:----:|:---:|:---:|------|------|
| ① | 拉低 | 拉高 | 释放数据总线，准备接收 | `SCL_LOW(); SDA_HIGH(); I2C_DELAY();` |
| ② | **拉高** | — | SCL 拉高，开始采样 | `SCL_HIGH(); I2C_DELAY();` |
| ③ | 保持高 | **读取** | 读取 SDA 电平 | `ack = READ_SDA();` |
| ④ | **拉低** | — | SCL 拉低，采样结束 | `SCL_LOW(); I2C_DELAY();` |
| ⑤ | — | — | 返回结果 | `return ack 真值判断;` |

### 12.4 READ_SDA 返回值处理

```c
// READ_SDA() = GPIOB->IDR & (1 << 11)
// IDR 是 16 位寄存器，& (1<<11) 后：
//   SDA=高电平 → 结果 = 0x0800（非零）
//   SDA=低电平 → 结果 = 0x0000（零）

// 需要转换为 0 或 1：
//   非零 → NACK(1)
//   零   → ACK(0)
```

### 12.5 代码实现

```c
uint8_t I2C_WaitAck(void)
{
    uint16_t ack;

    /* ① 释放数据总线（主机输出高阻态） */
    SCL_LOW();
    SDA_HIGH();
    I2C_DELAY();

    /* ② SCL 拉高，开始采样 */
    SCL_HIGH();
    I2C_DELAY();

    /* ③ 读取 SDA 电平 */
    ack = READ_SDA();    // 16 位值

    /* ④ SCL 拉低，采样结束 */
    SCL_LOW();
    I2C_DELAY();

    /* ⑤ 返回真值判断（非零=1=NACK，零=0=ACK） */
    return ack ? I2C_NACK : I2C_ACK;
}
```

## 十三、I2C_SendByte() — 主机发送一个字节

### 13.1 数据有效性时序回顾

```
SCL:  ─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐
    ───┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └──
         D7  D6  D5  D4  D3  D2  D1  D0

SDA: ═══╤═══╤═══════╤═══╤═══════╤═══╤═════
        │ 1 │   0   │ 1 │   0   │ 1 │  0
        高位先行（MSB First）→ 逐位发送

每一位的传输规则：
  ① SCL 低电平期间 → SDA 设置数据（翻转）
  ② SCL 拉高        → 数据保持稳定，从设备采样
  ③ SCL 拉低        → 采样结束
  ④ 左移一位        → 准备下一位
```

### 13.2 实现思路

| 要点 | 说明 |
|------|------|
| 循环次数 | **8 次**（一个字节 = 8 位） |
| 传输顺序 | **高位先行**（MSB First），先发 bit7，最后发 bit0 |
| 取最高位 | `byte & 0x80`（0x80 = 0b10000000） |
| 结果判断 | 非零 → 高电平(1)；零 → 低电平(0) |
| 移位操作 | 每次发完一位后 `byte <<= 1`（左移，保证下次仍取最高位） |

### 13.3 每一位的详细时序

| 步骤 | SCL | SDA | 操作 |
|:----:|:---:|:---:|------|
| ① | **拉低** | — | 等待数据翻转（SCL 低电平期间才能改 SDA） |
| ② | 保持低 | **设置** | 取最高位：`byte & 0x80`，非零→拉高，零→拉低 |
| ③ | 保持低 | 保持 | I2C_DELAY()，数据保持稳定 |
| ④ | **拉高** | 保持 | SCL 高电平，从设备采样 SDA |
| ⑤ | 保持高 | 保持 | I2C_DELAY()，采样保持时间 |
| ⑥ | **拉低** | 保持 | SCL 拉低，采样结束 |
| ⑦ | 保持低 | — | I2C_DELAY() |
| ⑧ | — | — | `byte <<= 1`，左移准备下一位 |

### 13.4 代码实现

```c
void I2C_SendByte(uint8_t byte)
{
    uint8_t i;

    for (i = 0; i < 8; i++)
    {
        /* ① SCL 拉低，等待数据翻转 */
        SCL_LOW();
        SDA_LOW();      // 兼容起始信号后的初始状态
        I2C_DELAY();

        /* ② 取最高位，设置 SDA 电平 */
        if (byte & 0x80)
        {
            SDA_HIGH();  // 最高位为 1 → 拉高
        }
        else
        {
            SDA_LOW();   // 最高位为 0 → 拉低（可省略，①已拉低）
        }
        I2C_DELAY();

        /* ④ SCL 拉高，从设备采样 */
        SCL_HIGH();
        I2C_DELAY();

        /* ⑥ SCL 拉低，采样结束 */
        SCL_LOW();
        I2C_DELAY();

        /* ⑧ 左移一位，准备下一位（保证下次仍取最高位） */
        byte <<= 1;
    }
}
```

### 13.5 取最高位原理

```
byte = 0xA7 = 10100111

第 1 次：byte & 0x80 = 10000000 → 非零 → SDA=1 → 发送 "1"
         byte <<= 1 → 01001110

第 2 次：byte & 0x80 = 00000000 → 零   → SDA=0 → 发送 "0"
         byte <<= 1 → 10011100

第 3 次：byte & 0x80 = 10000000 → 非零 → SDA=1 → 发送 "1"
         byte <<= 1 → 00111000

第 4 次：byte & 0x80 = 00000000 → 零   → SDA=0 → 发送 "0"
         byte <<= 1 → 01110000

第 5 次：byte & 0x80 = 00000000 → 零   → SDA=0 → 发送 "0"
         byte <<= 1 → 11100000

第 6 次：byte & 0x80 = 10000000 → 非零 → SDA=1 → 发送 "1"
         byte <<= 1 → 11000000

第 7 次：byte & 0x80 = 10000000 → 非零 → SDA=1 → 发送 "1"
         byte <<= 1 → 10000000

第 8 次：byte & 0x80 = 10000000 → 非零 → SDA=1 → 发送 "1"
         byte <<= 1 → 00000000

发送顺序：1 → 0 → 1 → 0 → 0 → 1 → 1 → 1（MSB First）
原始字节：1 0 1 0 0 1 1 1 = 0xA7 ✓
```

## 十四、I2C_ReceiveByte() — 主机接收一个字节

### 14.1 与 I2C_SendByte() 的核心区别

| 对比项 | SendByte | ReceiveByte |
|--------|----------|-------------|
| 数据方向 | 主机 → 从设备 | 从设备 → 主机 |
| SDA 控制 | **主机**控制 SDA | **从设备**控制 SDA |
| 主机操作 SDA | 根据数据拉高/拉低 | **不操作** SDA（释放） |
| 主机操作 SCL | 控制时钟翻转 | 控制时钟翻转 |
| 移位方向 | 左移取最高位发送 | 先左移，再存最低位 |
| 参数 | `uint8_t byte`（待发送） | 无参数 |
| 返回值 | 无 | `uint8_t data`（接收到的数据） |

### 14.2 实现思路

```
初始化 data = 0
循环 8 次：
  ① SCL 拉低 → 等待从设备翻转 SDA
  ② SCL 拉高 → 开始采样
  ③ 读取 SDA 电平 → 存入 data 最低位
  ④ SCL 拉低 → 采样结束
返回 data
```

### 14.3 关键点：先左移，再存入最低位

```
接收顺序：D7 → D6 → D5 → D4 → D3 → D2 → D1 → D0（高位先行）

第 1 次：先左移(data=00000000)，存 D7 到最低位 → 0000000[D7]
第 2 次：先左移              ，存 D6 到最低位 → 000000[D7][D6]
第 3 次：先左移              ，存 D5 到最低位 → 00000[D7][D6][D5]
...
第 8 次：先左移              ，存 D0 到最低位 → [D7][D6][D5][D4][D3][D2][D1][D0]
```

**为什么先左移再存入：**
```
❌ 错误方式：先存入再左移
  第 1 次：存 D7 → 0000000[D7] → 左移 → 000000[D7]0
  第 2 次：存 D6 → 000000[D6][D7]...混乱
  第 8 次：存 D0 后再左移 → D7 被移出字节，丢失！

✅ 正确方式：先左移再存入
  第 1 次：左移(00000000) → 存 D7 → 0000000[D7]
  第 2 次：左移(000000[D7]0) → 存 D6 → 000000[D7][D6]
  第 8 次：左移 → 存 D0 → [D7][D6]...[D1][D0] ✓
  （最后一次存入后不再移位，D0 在最低位，正确）
```

### 14.4 每一位的详细时序

| 步骤 | SCL | SDA | 操作 | 代码 |
|:----:|:---:|:---:|------|------|
| ① | **拉低** | 不操作 | SCL 低电平，等待从设备翻转 SDA | `SCL_LOW(); I2C_DELAY();` |
| ② | **拉高** | — | SCL 高电平，开始采样 | `SCL_HIGH(); I2C_DELAY();` |
| ③ | 保持高 | **读取** | 读取 SDA 电平，存入 data | `if(READ_SDA()) data |= 0x01;` |
| ④ | **拉低** | — | SCL 拉低，采样结束 | `SCL_LOW(); I2C_DELAY();` |
| ⑤ | — | — | 先左移再判断（循环开始时执行） | `data <<= 1;`（下一轮开头） |

### 14.5 代码实现

```c
uint8_t I2C_ReceiveByte(void)
{
    uint8_t i;
    uint8_t data = 0;    // 用于保存接收的数据，初始为 0

    for (i = 0; i < 8; i++)
    {
        /* 先左移，新数据存入最低位 */
        data <<= 1;

        /* ① SCL 拉低，等待从设备翻转 SDA */
        SCL_LOW();
        I2C_DELAY();

        /* ② SCL 拉高，开始采样 */
        SCL_HIGH();
        I2C_DELAY();

        /* ③ 读取 SDA 电平 */
        if (READ_SDA())     // 高电平 = 1
        {
            data |= 0x01;   // 最低位存 1
        }
        // 低电平 = 0：无需操作，左移后最低位已是 0

        /* ④ SCL 拉低，采样结束 */
        SCL_LOW();
        I2C_DELAY();
    }

    return data;
}
```

假设从设备发送 `0xA7 = 10100111`（高位先行）：

|:----:|:-----------:|:--------:|:-----------:|------|

## 十五、SendByte 与 ReceiveByte 的移位对比

| 对比项 | SendByte | ReceiveByte |
|--------|----------|-------------|
| 移位时机 | **发完后**左移 | **读取前**左移 |
| 取位位置 | 每次取**最高位** (`byte & 0x80`) | 每次存入**最低位** (`data |= 0x01`) |
| 移位对象 | 参数 byte（值传递，可直接改） | 局部变量 data |
| 移位方向 | 左移 | 左移 |
| 移位目的 | 原最高位发出后，下一位成为新最高位 | 已存数据左移，为新位腾出最低位 |

```
SendByte:    [D7 D6 D5 D4 D3 D2 D1 D0] → 取 D7 → 左移 → [D6 D5 D4 D3 D2 D1 D0 0]
                                                  ↑
                                          每次取最高位发送

ReceiveByte: [0  0  0  0  0  0  0  0 ] → 左移 → [0 0 0 0 0 0 0 0] → 存 D7 → [0 0 0 0 0 0 0 D7]
                                                  ↑
                                          先左移，再存最低位
```

## 十六、所有协议函数的共同模式

每个 I²C 协议函数都遵循相同的时钟控制模式：

```
通用模式：
  ① SCL 低电平期间 → SDA 设置目标电平（或释放）
  ② I2C_DELAY()     → 保持一段时间
  ③ SCL 拉高        → 开始采样

  ⑤ SCL 拉低        → 采样结束
  ⑥ I2C_DELAY()     → 准备下一位
```

## 十九、SDA 读取注意事项

| 注意点 | 说明 |
|--------|------|
| 主机**不操作** SDA | 接收时 SDA 由从设备驱动，主机只读 |
| 不能拉低 SDA | 否则与从设备抢占总线，导致数据冲突 |
| 可以保持 SDA 拉高 | 输出高阻态 = 释放总线，不影响从设备输出 |
| READ_SDA() 返回值 | 16 位 IDR 值，需判断真值（非零=1，零=0） |
| 输出模式下可读 IDR | GPIO 输出模式时 IDR 仍可读取引脚实际电平 |

## 二十、关键注意事项

| 注意点 | 说明 |
|--------|------|
| 视角始终为主机 | 所有函数以 STM32（主设备）为出发点 |
| SCL 始终由主机控制 | 从设备不驱动 SCL（时钟扩展除外） |
| SDA 方向需明确 | 写操作：主机控制 SDA；读操作：从设备控制 SDA |
| 先拉低 SCL 再改 SDA | 数据翻转必须在 SCL 低电平期间 |
| 每步后加 I2C_DELAY | 保证电平稳定，满足时序要求 |
| 输出模式可读 IDR | 无需切换 GPIO 输入/输出模式 |
| READ_SDA 需做真值转换 | IDR 16 位值 → 0 或 1 |

# 原始笔记入库与组织规范

## 适用范围
所有待进入 `raw/` 文件夹的新笔记，以 Markdown 文件为主。

## 核心原则
1. **原文不动**：笔记正文内容、排版、链接、图片引用一律保留原样，不做实质性修改。
2. **先筛后存**：入库前必须通过基本质量检查，无意义内容不进入知识库。
3. **分类清晰**：按主题层级归档，命名规范统一，方便人直接浏览和后续脚本处理。

---

## 一、入库前检查

### 1.1 内容有效性判断
打开文件，快速判断内容是否合格。出现以下情况视为**不合格**，应拒绝入库或移至废纸篓：
- 正文为空或仅有标题
- 内容为纯乱码、无法辨认的文字
- 整篇为无意义的测试文本（如"asdfg"、"测试测试"）
- 与知识库定位完全无关的临时信息（如外卖订单、一次性的验证码）

### 1.2 结构严谨性判断
内容合格的前提下，进一步判断结构是否**可阅读、可提炼**：
- **合格标准**：有明确的主题或观点，段落基本分明，能看出表达意图
- **轻微瑕疵可接受**：偶尔的错别字、多余空行、标点不规范等，可在入库时顺手修正，但不做深度润色
- **结构混乱需标记**：如果内容本身有价值但排版极其混乱（如无任何换行、标题层级乱用），仍可入库，但应在日志中备注"需后续整理"，并考虑在提炼阶段重点处理

### 1.3 筛查结论
- **通过** → 进入下一步：命名与分类
- **不通过** → 删除或移至 `raw/_archived/`，记录日志
- **存疑** → 保留入库，日志中标记 `REVIEW`，待人工二次判断

## 二、文件命名规范

### 2.1 命名格式
```
序号_标题.md
```
- **序号**：两位数字（如 `01`、`02`），表示在该主题文件夹内的推荐阅读顺序
- **分隔符**：下划线 `_`
- **标题**：用中文或英文简洁概括笔记核心内容，避免特殊字符（`/ \ : * ? " < > |`）

### 2.2 序号规则
- **序号含义**：表示在该主题文件夹内的**推荐阅读顺序**，从基础到进阶
- **新文件递补**：查看目标文件夹中已有文件的最大序号，新文件序号递补
- **序号复用**：如果某个序号被删除，新笔记可以复用该序号，保持连续性
- **重新编号**：如需调整阅读顺序，允许重新编号已有文件，但需在日志中记录所有改名操作
- **序号连续性**：同一文件夹内序号应连续（01、02、03...），避免跳跃（01、02、08...）

### 2.2.1 序号排序原则
当需要重新编号时，按以下逻辑排序：
1. **基础概念**在前，**进阶应用**在后
2. **理论知识**在前，**实践实现**在后
3. **单点知识**在前，**综合案例**在后
4. **独立模块**在前，**依赖模块**在后

### 2.2.2 序号排序示例
```
✅ 推荐顺序：
01_协议基础与原理.md      ← 基础概念
02_底层实现方法.md        ← 底层实现
03_接口层操作A.md         ← 接口层单点
04_接口层操作B.md         ← 接口层进阶
05_综合测试.md            ← 最终验证
06_完整案例.md            ← 综合案例（可选）

❌ 避免的顺序：
01_协议基础.md
02_底层实现.md
08_接口层操作.md          ← 序号跳跃
10_测试.md
```

### 2.3 标题提炼规则
- **有现成标题**：沿用原标题，去除多余的修饰词和特殊符号
- **无标题或标题无意义**：根据全文内容总结一个 3~8 个字的标题
- **标题语言**：优先使用笔记正文的主要语言；中英文混合的主题，中文优先

### 2.4 命名示例
```
✅ 01_决策树调参经验.md
✅ 02_过拟合与正则化.md
✅ 03_Transformer架构图解.md

❌ 笔记1.md              （标题无意义）
❌ 01_笔记.md             （标题太笼统）
❌ 02_机器学习/深度学习.md （含非法字符 /）
```

## 三、主题分类规范

### 3.1 主题树结构
`raw/` 下按主题层级建立文件夹树，从宽泛到具体逐步深入：
```
raw/
└─ 一级主题/
   └─ 二级主题/
      └─ 三级主题/
         └─ 四级主题/
            └─ 笔记文件.md
```

### 3.2 层级限制
- **最少 1 层**（至少有一个一级主题文件夹）
- **最多 5 层**（一级到五级），超过 5 层说明分类过细，需要合并或重新划分
- 如果某个主题下笔记很少（≤3 篇），暂缓建立子层级，等数量积累后再细分

### 3.3 文件夹命名
- 使用**中文主题词**，不加序号
- 命名应能准确涵盖该文件夹下所有笔记的共性
- 同一层级的文件夹之间互斥，避免内容重叠
- 示例：
  ```
  ✅ 计算机科学/人工智能/机器学习
  ✅ 个人成长/时间管理
  ✅ 项目管理/敏捷开发/Scrum实践

  ❌ 学习/编程/编程语言    （概念重合）
  ❌ 01-技术/02-生活       （不需要序号）
  ```

### 3.4 归类判断流程
1. 阅读笔记，提取核心主题词
2. 在已有主题树中查找最匹配的路径
3. 如果匹配多个可能路径，选择**最直接、最具体**的那个
4. 如果现有主题都不匹配，评估是否需要新建主题文件夹：
   - 预计该主题后续还会有 ≥3 篇相关笔记 → 新建
   - 只是孤立的单篇 → 暂放入相近主题，或用 `_待归类/` 临时存放
   - 也可使用 `raw/tmp/` 作为临时存放点，待处理后再归入正式分类

## 四、分类存放示例

假设当前 `raw/` 结构如下：
```
raw/
├── 计算机科学/
│   └── 编程语言/
│       ├── 01_Python装饰器详解.md
│       └── 02_Rust所有权机制.md
├── 个人成长/
│   └── 阅读笔记/
│       └── 01_如何阅读一本书.md
```

### 场景一：有匹配路径
收到一篇关于"Python 协程"的笔记。
- 归属路径：`计算机科学/编程语言/`
- 已有文件序号最大为 `02`，新文件命名为 `03_Python协程异步编程.md`
- 放入后：
  ```
  raw/
  └── 计算机科学/
      └── 编程语言/
          ├── 01_Python装饰器详解.md
          ├── 02_Rust所有权机制.md
          └── 03_Python协程异步编程.md
  ```

### 场景二：需要新建层级
收到一篇关于"决策树算法"的笔记。
- 当前无"机器学习"相关路径，但属于"计算机科学"分支
- 判断该主题后续还会产出笔记 → 新建子文件夹
- 创建路径：`计算机科学/机器学习/`
- 命名文件：`01_决策树基础概念.md`
- 放入后：
  ```
  raw/
  └── 计算机科学/
      ├── 编程语言/
      │   └── ...
      └── 机器学习/
          └── 01_决策树基础概念.md
  ```

## 五、完整入库操作流程

```
接收笔记文件
    │
    ▼
┌─────────────────┐
│ 1. 内容有效性检查 │ → 不合格 → 删除/归档 → 记日志 → 结束
└────────┬────────┘
         │ 合格
         ▼
┌─────────────────┐
│ 2. 结构严谨性判断 │ → 轻微问题 → 顺手修正
│                  │ → 混乱有价值 → 标记 REVIEW
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 3. 确定主题路径   │ → 匹配已有 or 新建文件夹
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 4. 命名并放入     │ → 序号_标题.md
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 5. 记录日志       │ → 操作类型、路径、状态
└─────────────────┘
```

## 六、日志记录

遵循 `log_standard.md` 规范，操作类型为 `RAW_IN` 或 `RAW_ORG`。

## 七、禁止事项

- ❌ 修改原文观点、数据、表述
- ❌ 删除或更改图片、附件链接路径
- ❌ 跳过有效性检查直接入库
- ❌ 文件夹嵌套超过 5 层
- ❌ 在文件名中使用非法字符 `/\:*?"<>|`
- ❌ 将不同主题的笔记混放在同一叶子文件夹

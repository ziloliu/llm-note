# 日志记录规范

## 定位
AI 操作日志，记录所有修改操作，便于追踪和回溯。

## 存放位置
`log/YYYY-MM.md`（按月归档）

## 日志格式

每条日志一行，格式：
```
HH:MM [操作类型] [状态] 目标路径
```

### 操作类型
| 类型 | 含义 |
|------|------|
| `RAW_IN` | 新笔记入库 |
| `RAW_ORG` | raw文件重命名/移动/删除 |
| `EXTRACT` | 提炼内容至core |
| `INDEX` | 更新索引 |
| `CORE_EDIT` | 编辑core文件 |
| `RULE_EDIT` | 修改rules |
| `SCRIPT` | 运行/修改脚本 |
| `GIT` | 版本控制提交 |

### 状态标记
- `OK` – 成功
- `FAIL` – 失败（需备注原因）
- `REVIEW` – 需人工复查
- `SKIP` – 跳过

## 示例

```markdown
# 操作日志 2026年6月

## 2026-06-02
15:30 RAW_IN OK raw/计算机科学/机器学习/01_决策树基础概念.md
15:32 RAW_IN REVIEW raw/个人成长/思维模型/03_混乱笔记.md / 结构待整理
16:20 EXTRACT OK core/机器学习.md / 新增3条知识点
```

## 原则
- 每次操作后**立即记录**
- 追加写入，不覆盖
- 历史日志不删除

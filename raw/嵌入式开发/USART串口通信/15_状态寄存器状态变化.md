## 一句话定义

USART状态寄存器SR的TXE、RXNE、IDLE、TC四个状态位由硬件自动置位，通过读写DR触发硬件清零，IDLE需要软件序列清除。

## 核心内容

### SR 上电复位值

| 状态位 | 复位值 | 含义 |
|--------|:------:|------|
| **TXE** | **1** | 发送缓冲区为空（初始无数据要发，自然为空） |
| **TC** | **1** | 发送完成（移位寄存器为空，初始无数据在发送） |
| **RXNE** | **0** | 接收缓冲区为空（初始未收到任何数据） |
| **IDLE** | **0** | 未检测到空闲帧（初始状态不用于判断空闲） |

复位值二进制：`0000 0000 1100 0000`（`0x00C0`），对应 bit7=TXE=1, bit6=TC=1, bit5=RXNE=0, bit4=IDLE=0。

**各位初始值的理由：**

| 状态位 | 初始值 | 理由 |
|--------|:------:|------|
| TXE = 1 | 合理 | 初始时没有数据要发送，发送缓冲区本来就是空的 |
| TC = 1 | 合理 | 移位寄存器初始为空（没有上一个数据在发送），等价于"上一个数据已发送完成"，随时可以开始发送新数据 |
| RXNE = 0 | 合理 | 初始时没有收到任何数据，接收缓冲区为空 |
| IDLE = 0 | 合理 | IDLE 用于判断一串数据接收完毕，初始时没有任何数据收发过程，无需标记空闲 |

### IDLE 标志位的完整生命周期

**初始状态：IDLE = 0**
```
上电 → IDLE = 0
     → 进入 while(!(SR & IDLE)) 循环
     → 条件为真（IDLE=0，非空闲），开始接收字符
```

**接收过程：逐字节接收**
```
电脑发送数据 → RX 引脚收到信号 → 接收移位寄存器逐位接收
    ↓ 一个字节接收完毕
RXNE = 1（硬件自动置位）→ 读取 DR → RXNE = 0（读 DR 自动清零）
    ↓ 继续接收下一个字节
重复以上过程...
```

**最后一个字节接收完毕**
```
最后一个字节接收完成 → RXNE = 1 → 读取 DR → RXNE = 0
    ↓
线路持续高电平（无新的起始位出现）
    ↓ 经过一个完整数据帧长度
硬件检测到空闲帧 → IDLE = 1（硬件自动置位）
```

**软件处理空闲事件**
```c
// while 循环检测到 IDLE = 1，退出接收
// 保存接收长度
*size = i - 1;

// 清除 IDLE 标志位（软件序列）
temp = USART1->SR;    // 读 SR → 确认空闲事件已被软件知晓
temp = USART1->DR;    // 读 DR → 确认数据已被取走
// 硬件判定：空闲事件已处理 → IDLE = 0
```

**IDLE 清零后的行为：**

手册原文：
> IDLE 位不会再次被置高，直到 RXNE 位被置起（即又检测到一次空闲总线）。

含义拆解：
```
IDLE 清零
    ↓
此时线路可能仍然处于空闲状态（电脑还没发新数据）
    ↓
硬件不会再把 IDLE 置 1（即使持续空闲）
    ↓
为什么？因为没有新的数据收发过程，不需要重复判断"字符串结束"
    ↓
直到新一轮数据到来：RXNE 被置 1（收到新字节）
    ↓
新字节接收完毕后，线路再次空闲
    ↓
硬件才允许再次将 IDLE 置 1（检测到新一轮的空闲帧）
```

**设计理由：** IDLE 的唯一用途是判断**一串变长数据接收完毕**。既然已经处理完毕并清零了，后续的持续空闲就没有意义——你已经知道上一串数据结束了，没必要重复报告。只有当新的数据到来（RXNE 置 1）之后，IDLE 才重新具备判断"新一轮字符串是否结束"的意义。

### RXNE 标志位的生命周期

```
无数据 → RXNE = 0
    ↓ 硬件收到一个完整字节
RXNE = 1（硬件自动置位）
    ↓ 软件读取 DR
RXNE = 0（读 DR 自动清零）
    ↓ 可接收下一个字节
```

| 动作 | 执行者 | 说明 |
|------|--------|------|
| 置 1 | 硬件 | 接收移位寄存器将数据并行传入 RDR 后自动置位 |
| 清零 | 硬件（软件触发） | 软件读取 DR 时硬件自动清零 |

### TXE 标志位的生命周期

```
初始 → TXE = 1（缓冲区为空）
    ↓ 软件写入 DR
TXE = 0（写 DR 自动清零，缓冲区非空）
    ↓ 硬件将数据从 TDR 移入移位寄存器
TXE = 1（硬件自动置位，TDR 又空了）
    ↓ 可写入下一个字节
```

| 动作 | 执行者 | 说明 |
|------|--------|------|
| 置 1 | 硬件 | TDR 数据交给发送移位寄存器后自动置位 |
| 清零 | 硬件（软件触发） | 软件写入 DR 时硬件自动清零 |

### TC 标志位的生命周期

```
初始 → TC = 1（移位寄存器为空，无数据在发送）
    ↓ 软件写入 DR → 数据转移到移位寄存器 → 开始逐位发送
TC = 0（移位寄存器非空，正在发送）
    ↓ 移位寄存器最后一位发送完毕
TC = 1（硬件自动置位）
```

| 动作 | 执行者 | 说明 |
|------|--------|------|
| 置 1 | 硬件 | 移位寄存器所有位发送完毕后自动置位 |
| 清零 | 硬件 | 软件写 DR 导致新数据进入移位寄存器时自动清零 |

**TC 与 TXE 的区别：**
```
写入 DR
    ↓
数据进入 TDR → TXE = 0（TDR 非空）
    ↓
TDR 数据转入移位寄存器 → TXE = 1（TDR 空了，可写下一个）
    ↓                         但 TC 仍 = 0（移位寄存器还在逐位发送）
移位寄存器发完最后一位 → TC = 1（真正发送完成）
```

### 完整运行周期时序图

以接收并回显一个字符串 "AB" 为例：

```
时间轴 →

状态        IDLE  RXNE   TXE   TC
初始上电      0     0     1    1
                │
电脑发送 'A'   │
RX接收 'A'     0     1←    1    1     ← RXNE 硬件置 1
读 DR 取 'A'   0    →0     1    1     ← 读 DR 清零 RXNE
                │
电脑发送 'B'   │
RX接收 'B'     0     1←    1    1
读 DR 取 'B'   0    →0     1    1
                │
线路持续空闲    1←    0     1    1     ← IDLE 硬件置 1（空闲帧）
                │
软件处理：      │
  保存数据      1     0     1    1
  读 SR + 读 DR 0←    0     1    1     ← 软件序列清除 IDLE
                │
发送回显 'A'   │
写 DR 发 'A'   0     0     0←   0←    ← 写 DR 清零 TXE 和 TC
移位发完 'A'   0     0     1←   0     ← TXE 置 1（TDR 空）
                │
写 DR 发 'B'   0     0     0←   0     ← 写 DR 清零 TXE
移位发完 'B'   0     0     1←   1←    ← TXE 和 TC 均置 1
                │
等待下一轮接收  0     0     1    1     ← 回到正常等待状态
```

### 四种清零机制对比总结

| 状态位 | 置位方式 | 清零方式 | 清零触发条件 | 为什么这样设计 |
|--------|----------|----------|-------------|---------------|
| RXNE | 硬件自动 | 读 DR | 软件读走数据 = 处理完一个字节 | 一步即可证明处理完毕 |
| TXE | 硬件自动 | 写 DR | 软件写入新数据 = 缓冲区又有数据了 | 写入即非空 |
| TC | 硬件自动 | 写 DR | 新数据进入移位寄存器 = 又开始发新的了 | 有新任务即未完成 |
| IDLE | 硬件自动 | 读 SR + 读 DR（软件序列） | 两步确认：知道空闲了 + 数据取走了 | 大事件需两步确认 |

### 关键认知

| 认知点 | 说明 |
|--------|------|
| 硬件负责置位 | 所有状态位的置 1 都由硬件自动完成，软件不需要也不能手动置 1 |
| 清零需软件参与 | RXNE/TXE/TC 通过读/写 DR 触发硬件清零；IDLE 需软件序列触发硬件清零 |
| IDLE 不会重复置位 | 清零后即使线路仍空闲也不会再次置 1，必须等到新一轮数据到来（RXNE 置 1）后才允许再次检测空闲帧 |
| 所有逻辑由硬件实现 | 软件只需按规则读/写对应寄存器，底层硬件电路自动完成状态位的置位与清零 |

## 注意事项 & 踩坑

- 所有状态位都是硬件自动置位，软件不能手动置1
- RXNE/TXE/TC通过读写DR触发清零，IDLE需要软件序列
- IDLE清零后不会重复置位，直到新一轮数据到来
- 理解状态位生命周期对正确编写驱动程序至关重要

## 一、SPI 基本概念

| 项目 | 说明 |
|------|------|
| 全称 | Serial Peripheral Interface（串行外设接口） |
| 提出者 | 摩托罗拉（Motorola） |
| 本质 | 串行通信**接口标准**（不是严格意义上的"协议"） |
| 协议复杂度 | **极低**（比 UART、I²C 简单得多） |
| 通信方式 | **高速、全双工、同步**串行通信 |

**应用场景**：EEPROM、Flash、传感器、ADC/DAC、LCD 显示屏、SD 卡等

---

## 二、SPI 与 UART、I²C 对比

| 特性 | UART | I²C | **SPI** |
|------|------|-----|---------|
| 通信方式 | 异步 | 同步 | **同步** |
| 工作模式 | 全双工 | 半双工 | **全双工** |
| 线数 | 2（TX+RX） | 2（SCL+SDA） | **4** |
| 最高速率 | ~115.2kbps | 100k/400k/3.4Mbps | **18~36Mbps** |
| 地址寻址 | 无 | 有（7/10 位地址） | **无（片选线）** |
| 应答机制 | 无 | 有（ACK/NACK） | **无** |
| 协议复杂度 | 中等 | 较高 | **极低** |
| 数据帧 | 起始+数据+停止 | 起始+地址+数据+应答+停止 | **仅数据** |

### 2.1 SPI 为什么更快

SPI 的设计理念是"砍掉一切花里胡哨的，所有时间都用来传数据"：

- **两根数据线** → 同时双向传输（全双工）
- **无应答机制** → 不浪费时间等待
- **无地址寻址** → 硬件片选代替，零协议开销
- **无起始/停止位** → 有时钟就传数据

## 三、物理层连接（4 根线）

| 线名 | 全称 | 功能 | 方向 |
|------|------|------|------|
| **SCK** | Serial Clock | 串行时钟信号 | 主机→从机 |
| **MOSI** | Master Out Slave In | 主机输出，从机输入 | 主机→从机 |
| **MISO** | Master In Slave Out | 主机输入，从机输出 | 从机→主机 |
| **SS/NSS/CS** | Slave/Chip Select | 片选信号（低电平有效） | 主机→从机 |

### 3.1 信号线详解

- **SCK**：由主机产生，控制通信速率。APB2（72MHz）→ SPI 最高 36MHz，APB1（36MHz）→ SPI 最高 18MHz
- **MOSI/MISO**：两根独立数据线，实现全双工（同时收发）
- **SS**：每个从机独立一根（不能共用），低电平选中对应从机

### 3.2 连接方式

```
SCK、MOSI、MISO → 所有从机并联（总线）
SS              → 每个从机独享一根（点对点）

通信约束：
  ✅ 主机 ↔ 从机：可以通信
  ❌ 从机 ↔ 从机：不能直接通信（无从机产生时钟）
```

## 四、数据传输原理（全双工数据交换）

### 4.1 底层硬件结构

```
主机                                    从机
发送缓冲区 → 移位寄存器 ←─SCK同步─→ 移位寄存器 → 发送缓冲区
           ↓                       ↓
接收缓冲区 ← 移位寄存器 ──MISO── 移位寄存器 ← 接收缓冲区
                      ──MOSI──
```

### 4.2 数据交换过程

```
每个 SCK 时钟周期：
  → 主机移位寄存器高位移出 → MOSI 线
  → 从机移位寄存器高位移出 → MISO 线
  → 双方同时发送、同时接收
  → 8 个时钟周期 = 交换 1 字节

特点：发的同时也在收，双方移位寄存器内容互换
```

### 4.3 只收或只发时

- **只想发送**：主机发送数据，忽略 MISO 接收的内容
- **只想接收**：主机发全 0，从 MISO 读取从机数据
- 本质是全双工，不需要的方向忽略即可

## 五、SPI 工作模式（CPOL + CPHA = 4 种）

### 5.1 时钟极性（CPOL）

| CPOL | 空闲时 SCK 电平 | 第一个跳变沿 |
|:----:|:---------------:|:----------:|
| **0** | **低电平** | 上升沿 |
| **1** | **高电平** | 下降沿 |

### 5.2 时钟相位（CPHA）

| CPHA | 采样时刻 | 含义 |
|:----:|---------|------|
| **0** | **第一个跳变沿** | 空闲结束立即采样 |
| **1** | **第二个跳变沿** | 延迟半个周期后采样 |

### 5.3 四种模式总览

| 模式 | CPOL | CPHA | 空闲电平 | 采样沿 |
|:----:|:----:|:----:|:--------:|:------:|
| **Mode 0** | **0** | **0** | 低 | **上升沿**（最常用） |
| Mode 1 | 0 | 1 | 低 | 下降沿 |
| Mode 2 | 1 | 0 | 高 | 下降沿 |
| **Mode 3** | **1** | **1** | 高 | **上升沿**（次常用） |

### 5.4 模式选择

- **Mode 0（最常用）**：CPOL=0, CPHA=0，第一个上升沿采样，大多数设备默认支持
- **Mode 3（次常用）**：CPOL=1, CPHA=1，同上升沿采样但延迟半个周期
- ⚠️ 主从设备必须配置为相同模式

## 六、数据帧格式

- STM32 支持 8 位或 16 位数据帧
- 数据位序：默认高位先行（MSB First），可配置低位先行

## 七、SPI 通信流程

```
初始化：
  ① GPIO 配置（SCK/MOSI/MISO/SS 引脚）
  ② SPI 模式配置（CPOL/CPHA）
  ③ 主从模式选择
  ④ 数据帧格式（8/16 位）
  ⑤ 时钟分频（波特率）
  ⑥ 使能 SPI

数据传输：
  ① 主机拉低对应从机 SS 线（片选）
  ② 写入发送数据到缓冲区
  ③ 移位寄存器在 SCK 下逐位传输
  ④ 8 个 SCK 周期后 1 字节数据交换完成
  ⑤ 从接收缓冲区读出数据
  ⑥ 传输结束，拉高 SS 线
```

## 八、设计理念：SPI vs I²C

| | SPI | I²C |
|------|------|------|
| 设计理念 | **速度优先** | **线数优先** |
| 代价 | 多信号线（4 根） | 协议复杂（地址+应答+方向位） |
| 寻址方式 | 硬件片选（快） | 软件地址（慢） |

**SPI 核心思想**：不传地址（用片选线代替）、不需应答（有数据就传）、不等响应（有时钟就走）

## 十、W25Q32 Flash 存储器应用

### 10.1 芯片概述

W25Q32 是 Winbond 生产的 **NOR Flash 存储器**，SPI 接口：

| 项目 | 说明 |
|------|------|
| 存储容量 | **32 Mbit = 4 MB** |
| 存储类型 | **NOR Flash**（基于或非门） |
| 通信协议 | SPI |
| 引脚数量 | **8 个** |
| 支持模式 | SPI **Mode 0** 和 Mode 3 |
| 数据位序 | **高位优先**（MSB First） |
| 页大小 | **256 字节** |
| 擦除最小单位 | **段（Sector）= 4 KB** |

#### 与 EEPROM（M24C02）对比

| 对比项 | M24C02 (EEPROM) | W25Q32 (NOR Flash) |
|--------|:---------------:|:------------------:|
| 存储类型 | EEPROM | NOR Flash |
| 容量 | 256 字节 | **4 MB** |
| 通信协议 | I²C | **SPI** |
| 速度 | 较慢 | **较快** |

📄 [原文02: W25Q32芯片介绍与存储结构](../raw/嵌入式开发/SPI/02_W25Q32芯片介绍与存储结构.md)

### 10.2 引脚定义与接线

| 引脚号 | 名称 | 功能 | 接法 |
|:------:|------|------|------|
| 1 | **CS** | 片选（低电平有效） | **PC13**（GPIO） |
| 2 | **DO (IO1)** | 数据输出（MISO） | PA6（SPI1_MISO） |
| 3 | **WP (IO2)** | 写保护（低有效） | **3.3V**（禁用） |
| 4 | **GND** | 接地 | GND |
| 5 | **DI (IO0)** | 数据输入（MOSI） | PA7（SPI1_MOSI） |
| 6 | **CLK** | 时钟 | PA5（SPI1_SCK） |
| 7 | **HOLD (IO3)** | 暂停/复位（低有效） | **3.3V**（禁用） |
| 8 | **VCC** | 电源 | **3.3V** |

#### 为什么不连 NSS（PA4）

```
PA4 是 SPI1 的 NSS 引脚：
  → 如果连到 W25Q32 的 CS → 意味着 STM32 被选为从机（错误！）
  → STM32 是主机，NSS 应拉高（不被选中）
  → 片选从机用 GPIO 引脚即可

一主多从场景：
  → 每个从机的 CS 连接独立的 GPIO 引脚
  → 片选时将对应 GPIO 拉低
  → PA4 的 NSS 直连功能：广播模式（同时选中所有从机）
```

#### STM32 与 W25Q32 的 SPI 连接

```
引脚复用关系（SPI1，挂载在 APB2）：
  PA5 → SPI1_SCK  → W25Q32 CLK
  PA6 → SPI1_MISO → W25Q32 DO（从机输出）
  PA7 → SPI1_MOSI → W25Q32 DI（从机输入）
  PC13 → GPIO     → W25Q32 CS（片选）

软件模拟 SPI 时：
  → 不需要开启 SPI1 硬件模块
  → 直接操作 PA5/PA6/PA7/PC13 的 GPIO 即可
```

### 10.3 存储空间组织

```
4 MB 总存储空间
├── Block 0~63  (64 KB × 64)  共 64 个 Block
└── 每个 Block (64 KB)
    ├── Sector 0~15 (4 KB × 16)  共 16 个 Sector
    └── 每个 Sector (4 KB)
        ├── Page 0~15 (256 B × 16)  共 16 个 Page
        └── 每个 Page (256 B)
            └── 256 个字节地址
```

| 层级 | 数量 | 大小 |
|------|:----:|:----:|
| 整个芯片 | 1 | **4 MB** (2²²) |
| Block | **64** (2⁶) | 64 KB (2¹⁶) |
| Sector | **1024** (2¹⁰) | 4 KB (2¹²) |
| Page | **16384** (2¹⁴) | 256 B (2⁸) |

#### 24 位地址划分

```
24 位地址（3 字节，6 位十六进制）：
  [A23 ~ A16] [A15~A12] [A11~A8] [A7 ~ A0]
  │ Block 号  │ │Sector│  │ Page │  │ 页内偏移 │
  (0~63)    (0~15)   (0~15)   (0~255)

地址范围：
  Block 0:  0x000000 ~ 0x00FFFF
  Block 63: 0x3F0000 ~ 0x3FFFFF

完整地址 = Block号 × 64K + Sector号 × 4K + Page号 × 256 + 页内偏移
```

### 10.4 读写特性

| 操作 | 特性 |
|------|------|
| **写入** | 写入前必须**先擦除**（只能 1→0，不能 0→1） |
| 写使能 | 每次写入前必须发送**写使能指令** |
| 写入最小单位 | **页（256 字节）** |
| 页回卷 | 超出页范围，地址**回卷**覆盖同页开头 |
| 写入后状态 | 进入 **Busy 状态**，需等待完成 |
| **擦除** | 擦后所有位变为 **1**（0xFF） |
| 擦除最小单位 | **段（Sector = 4 KB）** |
| 擦除选项 | 段擦除(4KB) / 块擦除(64KB) / 整片擦除 |
| **读取** | **无需使能**，直接读取 |
| 读取限制 | **无页限制**，可连续读取任意长度 |
| 读取状态 | **不进入 Busy 状态** |

#### 写入翻转规则

```
Flash 存储位状态变化：
  1 → 0 ✅（可以直接写入）
  0 → 1 ❌（必须先擦除！）

写入流程：擦除（全变 1）→ 写入（1 变 0）

例：
  擦除后：1111 1111
  写 0xA5 = 1010 0101：1111 1111 → 1010 0101 ✅
  再写 0x5A = 0101 1010：1010 0101 → 0000 0000 ❌（0 不能变 1！）
```

### 10.5 指令集

| 指令名称 | 指令码 | 功能 | 后续数据 |
|---------|:------:|------|---------|
| **Write Enable** | **0x06** | 写使能 | 无 |
| **Page Program** | **0x02** | 页写入 | 3 字节地址 + 数据 |
| **Read Data** | **0x03** | 读取数据 | 3 字节地址 + 读出数据 |
| **Sector Erase** | **0x20** | 段擦除（4KB） | 3 字节地址 |
| **Read Status Register** | **0x05** | 读状态寄存器 | 读出 1 字节状态 |
| **JEDEC ID** | **0x9F** | 读取芯片 ID | 读出 3 字节 |

**指令要点**：
- 写使能 0x06：发送后开启写使能，可执行写入/擦除
- 页写入 0x02：发送指令 → 3 字节地址 → 1~256 字节数据；写入前必须先发 0x06；超出 256 字节会回卷；写入后进入 Busy
- 读取 0x03：发送指令 → 3 字节地址 → 接收任意长度数据；无页限制，不进 Busy
- 段擦除 0x20：发送指令 → 3 字节地址（核心是段号）；指定段全变 0xFF
- 读状态 0x05：返回 1 字节状态，**Bit 0 = BUSY**（1=忙，0=空闲），用于等待写入/擦除完成
- JEDEC ID 0x9F：返回 3 字节（制造商ID / 存储类型 / 容量），用于验证通信和识别芯片

### 10.6 完整读写流程

#### 写入流程

```
① CS 拉低（片选使能）
② 发送 0x06（写使能）
③ CS 拉高（片选失能）
④ CS 拉低
⑤ 发送 0x02（页写指令）
⑥ 发送 3 字节地址
⑦ 发送写入数据（1~256 字节）
⑧ CS 拉高
⑨ 等待写入完成：循环发送 0x05 读状态 → 检查 BUSY 位 = 0 → 退出
```

#### 读取流程

```
① CS 拉低
② 发送 0x03（读数据指令）
③ 发送 3 字节地址
④ 接收数据（任意长度，连续读取）
⑤ CS 拉高
```

#### 擦除流程

```
① CS 拉低
② 发送 0x06（写使能）
③ CS 拉高
④ CS 拉低
⑤ 发送 0x20（段擦除指令）
⑥ 发送 3 字节地址（核心是段号）
⑦ CS 拉高
⑧ 等待擦除完成（同写入流程）
```

### 10.7 软件模拟 SPI 实现

**软件模拟** = 不使用 SPI1 硬件模块，直接操作 GPIO 引脚翻转电平实现 SPI 时序。

#### 软件模拟的取舍

| 优势 | 劣势 |
|------|------|
| 不受 SPI 硬件引脚限制，可用任意 GPIO | 速度较慢（软件翻转 GPIO） |
| 便于理解底层原理 | CPU 占用高 |
| 便于调试 | — |

#### 核心函数：SPI_SwapByte

软件模拟 SPI 的核心是实现"交换 1 字节"的函数，循环 8 次交换 8 位：

```c
uint8_t SPI_SwapByte(uint8_t byte)
{
    uint8_t rByte = 0x00;

    for (int i = 0; i < 8; i++)
    {
        /* 步骤①：SCK 低电平时准备发送数据 */
        if (byte & 0x80) MOSI_HIGH();
        else             MOSI_LOW();
        byte <<= 1;

        /* 步骤②：SCK 上升沿（采样时刻） */
        SCK_HIGH();
        SPI_DELAY();

        /* 步骤③：在 MISO 上采样从设备数据 */
        rByte <<= 1;
        if (MISO_READ()) rByte |= 0x01;

        /* 步骤④：SCK 下降沿（为下一次传输准备） */
        SCK_LOW();
        SPI_DELAY();
    }

    return rByte;
}
```

**关键技巧：先移位再写入最低位**

```
错误做法（先写入再左移）：每次写入后再左移，最后一位被移出！
正确做法（先左移再写入）：第 8 次写入后不再左移，数据完整。
```

#### 软件模拟 vs 硬件 SPI

| 对比项 | 软件模拟 | 硬件 SPI |
|--------|---------|---------|
| 速度 | 较慢（GPIO 翻转） | 较快（硬件自动） |
| CPU 占用 | 高（CPU 逐位控制） | 低（配合 DMA） |
| 引脚灵活性 | **任意 GPIO** | 固定复用引脚 |
| 实现难度 | 简单（看时序图） | 需配置寄存器 |
| 学习价值 | **高（理解底层）** | 低（黑盒使用） |
| 调试方便 | **方便** | 较困难 |

### 10.8 W25Q32 关键概念速查

| 概念 | 说明 |

| W25Q32 | NOR Flash 存储器，4MB，SPI 接口 |

| 写入翻转 | 只能 1→0，不能 0→1（需先擦除） |
| SPI_SwapByte | 软件模拟核心函数（全双工交换 1 字节） |
| 先移位再写入 | 接收数据的移位技巧，避免丢失最后一位 |

### 10.9 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 写入后数据不对 | 未先擦除（0 不能变 1） | 先执行段擦除 |
| 写入无效 | 未发送写使能（0x06） | 写入前先发 0x06 |
| 数据覆盖 | 超出 256 字节页范围 | 分页写入 |
| 操作失败 | Busy 状态下操作 | 读状态寄存器等 BUSY=0 |
| 读取数据全 0xFF | 地址错误或芯片未连接 | 检查接线和地址 |
| 通信不正常 | SPI Mode 不匹配 | 使用 Mode 0 或 Mode 3 |
| CS 不生效 | CS 接到 PA4（NSS） | CS 接到独立 GPIO |
| 软件模拟数据错位 | 移位顺序错误 | 先左移再写入最低位 |
| 软件模拟从机无响应 | CS 未拉低 | 调用 SPI_Start() |
| 软件模拟 MISO 异常 | PA6 配置错误 | 确认浮空输入（CNF=01） |

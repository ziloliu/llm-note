## 三、I2C_Init() — 初始化实现

### 3.1 配置流程

```
① 开启时钟（GPIOB + I²C2）
② 配置 GPIO（PB10/PB11 复用开漏输出）
③ 配置 CR2（输入时钟频率 36MHz）
④ 配置 CCR（SCL 分频系数 180，标准模式）
⑤ 配置 TRISE（最大上升时间 37）
⑥ 配置 CR1（I²C 模式，ACK 使能）
⑦ 配置 CR1（PE 外设使能 — 最后开启）
```

### 3.2 代码实现

```c
void I2C_Init(void)
{
    /* ① 开启时钟 */
    RCC->APB2ENR |= RCC_APB2ENR_IOPBEN;     // GPIOB 时钟
    RCC->APB1ENR |= RCC_APB1ENR_I2C2EN;     // I²C2 时钟

    /* ② GPIO 配置：PB10/PB11 复用开漏输出 50MHz */
    GPIOB->CRH |= (GPIO_CRH_CNF10 | GPIO_CRH_CNF11   // CNF = 11（复用）
                  | GPIO_CRH_MODE10 | GPIO_CRH_MODE11); // MODE = 11（50MHz）

    /* ③ CR2：输入时钟频率 36MHz */
    I2C2->CR2 = 36;           // FREQ[5:0] = 36

    /* ④ CCR：标准模式，分频系数 180 */
    I2C2->CCR = 180;          // F/S=0(标准), CCR=180 → 5μs 高电平

    /* ⑤ TRISE：最大上升时间 */
    I2C2->TRISE = 37;         // 36 + 1 = 37

    /* ⑥ CR1：I²C 模式 + ACK 使能 */
    I2C2->CR1 &= ~I2C_CR1_SMBTYPE;   // SMBus 位清零 → I²C 模式
    I2C2->CR1 |= I2C_CR1_ACK;        // 使能应答

    /* ⑦ CR1：外设使能（最后） */
    I2C2->CR1 |= I2C_CR1_PE;         // PE = 1
}
```

### 3.3 各步骤详解

#### 时钟开启

| 时钟 | 寄存器 | 宏定义 | 原因 |
|------|--------|--------|------|
| GPIOB | APB2ENR | `RCC_APB2ENR_IOPBEN` | PB10/PB11 引脚 |
| I²C2 | APB1ENR | `RCC_APB1ENR_I2C2EN` | I²C2 硬件模块 |

> I²C2 连接在 **APB1**（低速外设总线），不是 APB2。

#### GPIO 配置

```
PB10 (SCL) 和 PB11 (SDA)：
  CNF[1:0]  = 11 → 复用功能输出
  MODE[1:0] = 11 → 50MHz 速度

  合并配置：CNF 和 MODE 都置 1 → 直接 OR 0xF（每引脚 4 位全 1）
```

| 引脚 | CRH 位置 | CNF | MODE | 模式 |
|------|----------|:---:|:----:|------|
| PB10 | bit[11:8] | 11 | 11 | 复用开漏输出 50MHz |
| PB11 | bit[15:12] | 11 | 11 | 复用开漏输出 50MHz |

#### 为什么是复用开漏输出

| 对比 | 推挽输出 | 复用开漏输出 |
|------|----------|-------------|
| 适用场景 | 串口 TX（点对点） | **I²C SCL/SDA**（总线） |
| 原因 | 串口不需要线与逻辑 | I²C 需要开漏 + 上拉电阻实现线与 |

#### CR2 配置

```c
I2C2->CR2 = 36;    // FREQ[5:0] = 36 → 输入时钟 36MHz
```

| 位 | 值 | 含义 |
|:--:|:--:|------|
| [5:0] | 100100 | 36MHz |
| [15:6] | 000...0 | 其他功能未使用 |

#### CCR 配置

```c
I2C2->CCR = 180;   // F/S=0, CCR=180
```

| 位 | 值 | 含义 |
|:--:|:--:|------|
| [11:0] | 180 | 分频系数 |
| [14] | 0 | DUTY（标准模式下无效） |
| [15] | 0 | F/S = 0 → **标准模式** |

```
计算过程：
  目标：100kbps → SCL 周期 10μs → 高电平 5μs
  输入时钟：36MHz → T = 1/36 μs
  CCR = 5μs / (1/36 μs) = 5 × 36 = 180
```

#### TRISE 配置

```c
I2C2->TRISE = 37;   // 36 + 1 = 37
```

```
计算过程：
  标准模式最大上升时间 = 1μs（1000ns）
  1μs / (1/36 μs) = 36 个时钟周期
  加 1（余量）→ 37
```

#### CR1 配置

```c
// I²C 模式（非 SMBus）
I2C2->CR1 &= ~I2C_CR1_SMBTYPE;    // bit1 = 0

// 使能 ACK
I2C2->CR1 |= I2C_CR1_ACK;          // bit10 = 1

// 外设使能（最后）
I2C2->CR1 |= I2C_CR1_PE;           // bit0 = 1
```

### 3.4 配置顺序的重要性

```
必须在 PE = 0 时配置以下寄存器：
  CR2（时钟频率）
  CCR（分频系数）
  TRISE（上升时间）

最后才设置 PE = 1 使能外设

⚠️ PE = 1 后，CR2/CCR/TRISE 不可修改
```

## 四、函数语义变化对比

### 4.1 ACK/NACK 从"发出"变为"设置"

| 软件模拟 | 硬件 I²C | 说明 |
|----------|----------|------|
| `I2C_SendAck()` — 立即拉低 SDA | `I2C_EnableAck()` — 设置 ACK 位 | 配置收到字节后的行为 |
| `I2C_SendNack()` — 立即保持 SDA 高 | `I2C_DisableAck()` — 清除 ACK 位 | 配置收到字节后的行为 |

```
软件模拟：调用 → 立即在 SDA 上输出电平
硬件 I²C：设置 → 等收到字节后硬件自动执行

EnableAck()：收到字节 → 硬件自动拉低 SDA（ACK）
DisableAck()：收到字节 → 硬件不拉低 SDA（NACK）
```

### 4.2 WaitAck 被删除

| 软件模拟 | 硬件 I²C |
|----------|----------|
| 单独调用 `I2C_WaitAck()` | **融入** `I2C_SendByte()` / `I2C_SendAddr()` |
| 读取 SDA 电平判断 | 检测 **BTF** / **ADDR** 标志位 |
| 返回 ACK/NACK | 返回 OK/TIMEOUT |

```
硬件 I²C 不需要单独等待应答：
  发送字节后 → while(!BTF) 等待 → BTF=1 说明字节传输完成（含收到 ACK）
  发送地址后 → while(!ADDR) 等待 → ADDR=1 说明地址发送成功（含收到 ACK）
```

### 4.3 Start 增加返回值

| 软件模拟 | 硬件 I²C |
|----------|----------|
| `void I2C_Start(void)` | `uint8_t I2C_Start(void)` |
| 总能成功（手动拉高拉低） | 可能超时（总线忙） |
| 无返回值 | 返回 OK 或 TIMEOUT |

### 4.4 Stop 语义变化

| 软件模拟 | 硬件 I²C |
|----------|----------|
| 立即执行拉高拉低 | **配置项**，字节传输结束后自动生效 |
| 瞬间完成 | 延迟生效 |

## 五、软硬件 I²C 操作方式对比

| I²C 操作 | 软件模拟 | 硬件 I²C |
|----------|----------|----------|
| GPIO 模式 | **通用**开漏输出 | **复用**开漏输出 |
| 开启时钟 | 只需 GPIOB | GPIOB + **I²C2** |
| 初始化 | 只配 GPIO | GPIO + **CR2 + CCR + TRISE + CR1** |
| 产生 START | 手动拉高拉低 SDA | `CR1.START = 1; while(!SB);` |
| 产生 STOP | 手动拉高拉低 SDA | `CR1.STOP = 1;`（延迟生效） |
| 发 ACK | 手动拉低 SDA | `CR1.ACK = 1;`（配置项） |
| 发 NACK | 手动保持 SDA 高 | `CR1.ACK = 0;`（配置项） |
| 发送字节 | `for(8次) 设SDA+翻SCL` | `while(!TXE); DR = byte; while(!BTF);` |
| 接收字节 | `for(8次) 翻SCL+读SDA` | `while(!RXNE); byte = DR;` |
| 等待应答 | `READ_SDA()` 读电平 | 检测 BTF/ADDR 标志位 |
| 时序控制 | `Delay_us(10)` | **硬件自动**（CCR 分频） |
| CPU 占用 | 高 | **低** |

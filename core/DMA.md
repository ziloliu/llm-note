## 一、DMA 基本概念

### 1.1 什么是 DMA

| 项目 | 说明 |
|------|------|
| 全称 | **D**irect **M**emory **A**ccess（直接存储器访问） |
| 本质 | 硬件数据传输通道，绕过 CPU 实现外设与存储器之间的数据交换 |
| 核心意义 | **解放 CPU**，提高数据传输效率 |

### 1.2 传统方式 vs DMA 方式

| 对比项 | 传统方式 | DMA 方式 |
|--------|----------|----------|
| 数据路径 | 外设 → CPU → 存储器 | 外设 → DMA通道 → 存储器 |
| CPU 参与 | 全程参与 | 仅初始配置 |
| 效率 | 低（等待慢速外设） | 高（硬件自动传输） |
| CPU 占用 | 高 | 低 |

**形象比喻**：
- **CPU** = 老板（能力最强，速度最快）
- **DMA 控制器** = 小秘书（专职管理数据传输）
- **外设** = 员工（串口、ADC、定时器等）

## 二、STM32 DMA 硬件架构

### 2.1 DMA 控制器

| 控制器 | 通道数 | 说明 |
|--------|:------:|------|
| **DMA1** | **7** 个通道 | 所有型号都有 |
| **DMA2** | **5** 个通道 | 仅大容量和互联型产品 |

> DMA 控制器直接挂在 **AHB 总线**上，与 CPU 共享系统数据总线，是系统核心组件，不是外设。

### 2.2 通道分配特点

| 传输类型 | 通道分配 |
|----------|----------|
| 外设 ↔ 存储器 | **固定分配**（查表确定） |
| 存储器 → 存储器 | **任意通道**（特殊模式） |

## 三、总线冲突与仲裁

### 3.1 DMA 优先原则

```
DMA 优先于 CPU！

原因：
  外设数据传输有实时性要求
  → 串口数据不及时接收 → 数据丢失
  → ADC 数据不及时读取 → 被覆盖

  CPU 稍微等待没问题
  → 有全权处理能力，可以暂停再恢复
```

### 3.2 周期挪用（实际方案）

```
CPU ══╗   ╔══╗   ╔══╗   ╔═════════ CPU
      ║DMA║  ║DMA║  ║DMA║
      ╚═══╝  ╚═══╝  ╚═══╝

DMA 传输时需要等待（外设写入周期、内存读写周期）
这段等待时间 CPU 可以使用总线
→ 真正的"插空"使用
→ 效率更高
```

**带宽保证**：总线仲裁器保证 CPU 至少可以得到 **50%** 的系统总线带宽。

## 四、DMA 传输方向

| 方向 | 缩写 | 说明 | 典型场景 |
|------|------|------|----------|
| 外设 → 存储器 | P→M | 外设读，存储器写 | 串口接收、ADC 采样 |
| 存储器 → 外设 | M→P | 存储器读，外设写 | 串口发送、DAC 输出 |
| **存储器 → 存储器** | **M→M** | ROM 读，RAM 写 | Flash 数据复制到 SRAM |

**存储器到存储器模式限制**：
- ✅ ROM → RAM（可以）
- ❌ RAM → ROM（不可以，Flash 写入需要特定协议）

## 六、DMA 优先级仲裁

### 6.1 两级仲裁

| 阶段 | 方式 | 说明 |
|------|------|------|
| **第一级** | 软件优先级 | CCR.PL 配置（4 种等级） |
| **第二级** | 硬件优先级 | 通道编号（1 > 2 > 3...） |

### 6.2 软件优先级（CCR.PL）

| PL[1:0] | 优先级 |
|:-------:|--------|
| 00 | 低（默认） |
| 01 | 中 |
| 10 | 高 |
| 11 | 最高 |

### 6.3 仲裁规则

```
① 先比较软件优先级（CCR.PL）
   → PL 值越大，优先级越高

② 软件优先级相同 → 比较硬件优先级
   → 通道编号越小，优先级越高

③ CPU vs DMA 冲突
   → DMA 优先（但保证 CPU 至少 50% 带宽）
```

## 七、DMA 寄存器详解

### 7.1 四个核心寄存器（每通道独立）

| 寄存器 | 全称 | 功能 |
|--------|------|------|
| **CCR** | Channel Configuration Register | 通道配置（核心） |
| **CNDTR** | Channel Number of Data to Transfer | 数据传输数量 |
| **CPAR** | Channel Peripheral Address Register | 外设地址（源地址） |
| **CMAR** | Channel Memory Address Register | 存储器地址（目标地址） |

### 7.2 两个共用寄存器（所有通道共享）

| 寄存器 | 全称 | 功能 |
|--------|------|------|
| **ISR** | Interrupt Status Register | 中断状态标志（**只读**） |
| **IFCR** | Interrupt Flag Clear Register | 中断标志清除（**只写**，写 1 清除） |

### 7.3 CCR 寄存器关键位域

| 位 | 名称 | 功能 |
|:--:|------|------|
| 0 | **EN** | 通道使能（最后置 1） |
| 1 | **TCIE** | 传输完成中断使能 |
| 4 | **DIR** | 数据传输方向（0=外设→存储器，1=存储器→外设） |
| 5 | **CIRC** | 循环模式（0=正常，1=循环） |
| 6 | **PINC** | 外设地址自增 |
| 7 | **MINC** | 存储器地址自增 |
| 9:8 | **PSIZE** | 外设数据宽度（00=8位，01=16位，10=32位） |
| 11:10 | **MSIZE** | 存储器数据宽度 |
| 13:12 | **PL** | 优先级（00=低，01=中，10=高，11=最高） |
| 14 | **MEM2MEM** | 存储器到存储器模式（0=正常，1=M2M） |

### 7.4 中断标志

| 标志 | 全称 | 含义 |
|:----:|------|------|
| **GIFx** | Global Interrupt Flag | 全局中断标志 |
| **TCIFx** | Transfer Complete IF | 传输完成标志 |
| **HTIFx** | Half Transfer IF | 传输过半标志 |
| **TEIFx** | Transfer Error IF | 传输错误标志 |

> ISR **只读**，清除标志需写 IFCR 对应位。

## 八、存储器到存储器模式配置

### 8.1 特殊之处

```
存储器到存储器模式是特例：
  → 源存储器（ROM）→ 当作"外设" → 配置到 CPAR
  → 目标存储器（RAM）→ 当作"存储器" → 配置到 CMAR
  → DIR = 0（从"外设"读）

通道选择：任意通道都可以（推荐通道 1，硬件优先级最高）
```

### 8.2 配置流程

```
① 开启时钟
   RCC->AHBENR |= RCC_AHBENR_DMA1EN;

② 配置 CCR
   MEM2MEM = 1   → 存储器到存储器模式
   DIR     = 0   → 从"外设"（ROM）读
   PSIZE   = 00  → 8 位数据宽度
   MSIZE   = 00  → 8 位数据宽度
   PINC    = 1   → ROM 地址自增
   MINC    = 1   → RAM 地址自增
   PL      = 10  → 高优先级
   TCIE    = 1   → 传输完成中断使能（可选）
   EN      = 0   → 暂不开启

③ 配置 CNDTR = 传输数据个数

④ 配置 CPAR = ROM 中数据起始地址

⑤ 配置 CMAR = RAM 中目标起始地址

⑥ NVIC 配置（如使用中断）

⑦ 开启通道：CCR.EN = 1

⑧ 等待传输完成：查询 ISR.TCIF1 或进入中断
   清除标志：IFCR |= DMA_IFCR_CTCIF1;
```

## 八、存储器到存储器传输实现

### 8.1 寄存器方式实现

**配置要点**：
```c
// 开启时钟
RCC->AHBENR |= RCC_AHBENR_DMA1EN;

// 配置 CCR
DMA1_Channel1->CCR = 0;                    // 先清零
DMA1_Channel1->CCR |= DMA_CCR_MEM2MEM;    // 存储器到存储器
DMA1_Channel1->CCR |= DMA_CCR_DIR;        // 从存储器读
DMA1_Channel1->CCR |= DMA_CCR_PINC;       // 外设地址自增
DMA1_Channel1->CCR |= DMA_CCR_MINC;       // 存储器地址自增
DMA1_Channel1->CCR |= DMA_CCR_TCIE;       // 传输完成中断

// 配置数据数量
DMA1_Channel1->CNDTR = 4;

// 配置地址
DMA1_Channel1->CPAR = (uint32_t)src;       // 源地址（ROM）
DMA1_Channel1->CMAR = (uint32_t)dst;       // 目标地址（RAM）

// 开启通道
DMA1_Channel1->CCR |= DMA_CCR_EN;

// 等待完成
while (!(DMA1->ISR & DMA_ISR_TCIF1));
DMA1->IFCR |= DMA_IFCR_CTCIF1;
```

📄 [原文04: DMA存储器到存储器传输寄存器实现](../raw/嵌入式开发/DMA/04_DMA存储器到存储器传输寄存器实现.md)

### 8.2 HAL 库实现

**CubeMX 配置**：
- System Core → DMA → Add → Memory To Memory
- DMA Controller: DMA1
- Channel: Channel 1
- Direction: Memory To Memory
- Mode: Normal

**代码实现**：
```c
// 启动传输
HAL_DMA_Start(&hdma_memtomem_dma1_channel1, 
              (uint32_t)src, 
              (uint32_t)dst, 
              4);

// 等待完成（阻塞方式）
HAL_DMA_PollForTransfer(&hdma_memtomem_dma1_channel1, 
                        HAL_DMA_FULL_TRANSFER, 
                        HAL_MAX_DELAY);
```

## 九、内存到串口传输

### 9.1 与存储器到存储器模式的区别

| 对比项 | 存储器到存储器 | 内存到串口 |
|--------|:-------------:|:---------:|
| MEM2MEM | **1**（开启） | **0**（不开启） |
| DIR | 0（从外设读） | **1**（从存储器读，发往外设） |
| CPAR | ROM 地址 | **USART1->DR 地址** |
| CMAR | RAM 地址 | **数组地址** |
| PINC | 1（地址自增） | **0**（串口地址不自增） |
| 通道 | 任意 | **硬件固定**（通道 4） |
| 额外配置 | 无 | **USART1->CR3.DMAT = 1** |

### 9.2 串口 DMA 使能

**额外配置**：外设端需要开启 DMA 请求
```c
// USART1 的 CR3 寄存器
USART1->CR3 |= USART_CR3_DMAT;  // 使能 DMA 发送
```

### 9.3 寄存器方式实现

```c
// 开启时钟
RCC->AHBENR |= RCC_AHBENR_DMA1EN;
RCC->APB2ENR |= RCC_APB2ENR_USART1EN;

// 配置 DMA
DMA1_Channel4->CCR = 0;
DMA1_Channel4->CCR |= DMA_CCR_DIR;        // 从存储器读
DMA1_Channel4->CCR |= DMA_CCR_MINC;       // 存储器地址自增
DMA1_Channel4->CCR |= DMA_CCR_TCIE;       // 传输完成中断

// 配置地址
DMA1_Channel4->CPAR = (uint32_t)&USART1->DR;  // 串口数据寄存器
DMA1_Channel4->CMAR = (uint32_t)txBuffer;     // 发送缓冲区
DMA1_Channel4->CNDTR = 4;                      // 4个字节

// 使能串口 DMA 发送
USART1->CR3 |= USART_CR3_DMAT;

// 开启 DMA 通道
DMA1_Channel4->CCR |= DMA_CCR_EN;
```

## 十、DMA 与中断的系统定位

### 10.1 两者对比

| 对比项 | 中断 | DMA |
|--------|------|-----|
| **角色** | 大宰相（NVIC） | 小秘书（DMA 控制器） |
| **功能** | 打断主程序，处理突发事件 | 外设与存储器之间高速数据搬运 |
| **CPU 参与** | 必须参与（执行中断服务程序） | **不参与**（硬件自动完成） |
| **处理内容** | 任意逻辑（软件实现） | 数据传输（硬件实现） |

### 10.2 系统架构中的位置

```
              ┌──────────────────────────────────┐
              │          AHB 系统高速总线          │
              │                                  │
  ┌───────┐   │  ┌────────┐   ┌───────────────┐  │
  │  CPU  │───┤  │  NVIC  │   │ DMA 控制器     │  │
  │ (CM3) │   │  │(大宰相) │   │ (小秘书)       │  │
  └───────┘   │  └────────┘   └───────┬───────┘  │
              │                       │          │
              │  ┌────────────────────┼────────┐ │
              │  │  总线矩阵           │        │ │
              └──┴────────────────────┴────────┴─┘
                   │         │         │
                ┌──┴──┐  ┌──┴──┐  ┌───┴───┐
                │ SRAM│  │Flash│  │ 外设   │
                │(RAM)│  │(ROM)│  │(串口等)│
                └─────┘  └─────┘  └───────┘
```

## 九、关键概念速查

| 概念 | 说明 |
|------|------|
| DMA | Direct Memory Access，直接存储器访问 |
| DMA 控制器 | 管理 DMA 通道的硬件模块 |
| DMA 通道 | 数据传输的硬件通路（专线专用） |
| 仲裁器 | 决定多个 DMA 请求的优先级 |
| 周期挪用 | CPU 每次让出几个时钟周期给 DMA |
| MEM2MEM | 存储器到存储器传输模式 |
| CCR | 通道配置寄存器 |
| CNDTR | 数据传输数量寄存器（自动递减） |
| CPAR | 外设地址寄存器（源地址） |
| CMAR | 存储器地址寄存器（目标地址） |
| TC | Transfer Complete，传输完成事件 |
| HT | Half Transfer，传输过半事件 |
| TE | Transfer Error，传输错误事件 |

## 十、DMA 的意义

```
① 解放 CPU
   → 数据搬运工作交给 DMA 控制器
   → CPU 专注于逻辑处理、算法计算

② 提高传输效率
   → 硬件通路直接传输
   → 不需要软件指令逐步执行

③ 保证实时性
   → 外设数据不丢失
   → 传输速度可匹配外设需求

④ 降低功耗
   → CPU 不参与数据搬运
   → 可以进入低功耗模式
```

# SPI 通信协议 — 原理与协议层 笔记

---

## 一、SPI 基本概念

| 项目 | 说明 |
|------|------|
| 全称 | **S**erial **P**eripheral **I**nterface（串行外设接口） |
| 提出者 | **摩托罗拉公司**（Motorola） |
| 起源 | MC68HC 系列处理器 |
| 本质 | 串行通信**接口标准**（不是严格意义上的"通信协议"） |
| 协议复杂度 | **接近于零**（比 UART、I²C 简单得多） |
| 通信方式 | **高速、全双工、同步**串行通信 |

### 1.1 应用场景

```
广泛应用于：
  → EEPROM、Flash 存储器
  → 传感器
  → ADC/DAC 转换器
  → 网络通信芯片
  → LCD 显示屏
  → SD 卡等
```

## 二、SPI 与 UART、I²C 对比

| 特性 | UART | I²C | **SPI** |
|------|------|-----|---------|
| 通信方式 | 异步 | 同步 | **同步** |
| 工作模式 | 全双工 | 半双工 | **全双工** |
| 连接方式 | 点对点 | 总线（一主多从） | **总线（一主多从）** |
| 线数 | 2 根（TX+RX） | 2 根（SCL+SDA） | **4 根** |
| 最高速率 | ~115.2kbps | 100k/400k/3.4Mbps | **最高 18~36Mbps** |
| 地址寻址 | 无 | 有（7/10 位地址） | **无（片选线）** |
| 应答机制 | 无 | 有（ACK/NACK） | **无** |
| 协议复杂度 | 中等 | 较高 | **极低** |
| 数据帧 | 起始+数据+停止 | 起始+地址+数据+应答+停止 | **仅数据** |

### 2.1 SPI 为什么更快

```
UART（异步全双工）：
  → 无时钟线 → 需约定波特率 → 速度受限
  → 点对点连接

I²C（同步半双工）：
  → 一根数据线 → 同一时刻只能单向传输
  → 需要应答机制 → 浪费时间
  → 需要地址寻址 → 浪费时间
  → 协议复杂（起始/地址/方向/应答/停止）

SPI（同步全双工）：
  → 两根数据线 → 同时双向传输
  → 无应答机制 → 不浪费时间
  → 无地址寻址 → 硬件片选，不浪费时间
  → 协议极简 → 有时钟就传数据
  → 设计理念：砍掉所有"花里胡哨"的，全部时间用来传数据
```

## 三、SPI 物理层连接（4 根线）

### 3.1 四条信号线

| 线名 | 全称 | 功能 | 方向 |
|------|------|------|------|
| **SCK** | Serial Clock | 串行时钟信号 | 主机→从机（主机产生） |
| **MOSI** | Master Out Slave In | 主设备输出，从设备输入 | 主机→从机 |
| **MISO** | Master In Slave Out | 主设备输入，从设备输出 | 从机→主机 |
| **SS/NSS/CS** | Slave Select / Chip Select | 片选信号（低电平有效） | 主机→从机 |

### 3.2 信号线详解

```
① SCK（时钟线）
   → 由主机产生，控制通信速率
   → 最高频率 = APB 总线频率 / 2
     APB2（72MHz）→ SPI 最高 36MHz
     APB1（36MHz）→ SPI 最高 18MHz

② MOSI（主机输出线）
   → 主机发送数据的通道
   → 对主机：OUTPUT（发送端）
   → 对从机：INPUT（接收端）
   → 方向取决于设备是主还是从

③ MISO（主机输入线）
   → 主机接收数据的通道
   → 对主机：INPUT（接收端）
   → 对从机：OUTPUT（发送端）

④ SS（片选线）
   → 低电平有效
   → 默认高电平（不选中）
   → 拉低 = 选中对应从机
   → 每个从机独立一根 SS 线（不能共用！）
```

### 3.3 连接方式图示

```
                    ┌─────────┐
              SS0 ──┤         ├── SCK ──── 总线 ──→ 所有从机 SCK
              SS1 ──┤  主机    ├── MOSI ── 总线 ──→ 所有从机 MOSI
              SS2 ──┤ (MCU)   ├── MISO ── 总线 ──→ 所有从机 MISO
                    └─────────┘
                     │    │    │
                    SS0  SS1  SS2（点对点，专线专用）
                     │    │    │
                     ↓    ↓    ↓
                  ┌────┐┌────┐┌────┐
                  │从机0││从机1││从机2│
                  └────┘└────┘└────┘

总线连接：SCK、MOSI、MISO → 所有设备并联
专线连接：SS → 每个从机独立一根（点对点）
```

### 3.4 通信约束

```
✅ 主机 ↔ 从机：可以通信
❌ 从机 ↔ 从机：不能直接通信

原因：
  → 没有从机产生时钟
  → 从机之间无 SS 片选
  → MOSI/MISO 是针对主从之间的连接设计
```

## 四、SPI 协议层 — 数据传输原理

### 4.1 底层硬件结构

```
          主机                          从机
  ┌──────────────────┐        ┌──────────────────┐
  │  发送缓冲区       │        │  发送缓冲区       │
  │  (TX Buffer)     │        │  (TX Buffer)     │
  └────────┬─────────┘        └────────┬─────────┘
           ↓                           ↓
  ┌──────────────────┐        ┌──────────────────┐
  │  移位寄存器       │◄──────►│  移位寄存器       │
  │  (Shift Register)│  SCK   │  (Shift Register)│
  └────────┬─────────┘  同步   └────────┬─────────┘
           ↓                           ↓
  ┌──────────────────┐        ┌──────────────────┐
  │  接收缓冲区       │        │  接收缓冲区       │
  │  (RX Buffer)     │        │  (RX Buffer)     │
  └──────────────────┘        └──────────────────┘

主机发送端 → MOSI → 从机接收端
从机发送端 → MISO → 主机接收端
```

### 4.2 数据交换过程（8 位 = 1 字节）

```
时钟上升沿（数据发送）：
  → 主机移位寄存器高位移出 → MOSI 线
  → 从机移位寄存器高位移出 → MISO 线
  → 双方同时发送，互不干扰

时钟下降沿（数据接收）：
  → 主机读取 MISO 线数据 → 写入移位寄存器最低位
  → 从机读取 MOSI 线数据 → 写入移位寄存器最低位
  → 双方同时接收

每个时钟周期：
  → 主机和从机交换 1 位数据
  → 8 个时钟周期 = 交换 1 个字节

特点：
  → 发的同时也在收（全双工）
  → 收的同时也在发
  → 双方移位寄存器内容互换
```

### 4.3 时序图（以 Mode 0 为例）

```
        ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐
SCK  ───┘   └───┘   └───┘   └───┘   └───┘   └───┘   └───┘   └───┘
      ↑采样  ↑采样  ↑采样  ↑采样  ↑采样  ↑采样  ↑采样  ↑采样

MOSI ──┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──
       │ D7   │ D6   │ D5   │ D4   │ D3   │ D2   │ D1   │ D0
       (主机发送：高位先行)

MISO ──┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──
       │ D7   │ D6   │ D5   │ D4   │ D3   │ D2   │ D1   │ D0
       (从机发送：高位先行)

  8 个 SCK 周期 → 交换 1 字节
  数据高位先行（MSB First）
```

### 4.4 只收或只发的情况

```
只想发送（主机）：
  → 主机发送数据，忽略接收的 MISO 数据
  → 从机随便发（全 0 或全 1），忽略接收的 MOSI 数据

只想接收（主机）：
  → 主机发送无效数据（全 0），从 MISO 读取从机数据
  → 从机忽略 MOSI 上的数据

本质：全双工，每时钟都必须同时收发
  → 不需要的功能，忽略即可
  → 单方向使用时，效果类似半双工
```

## 五、SPI 工作模式（极性 + 相位 = 4 种模式）

### 5.1 时钟极性（CPOL — Clock Polarity）

| CPOL | 空闲时 SCK 电平 | 第一个跳变沿 |
|:----:|:---------------:|:----------:|
| **0** | **低电平** | 上升沿 |
| **1** | **高电平** | 下降沿 |

```
CPOL = 0：
  空闲 ───┐         ┌───┐         ┌───
          │ 上升沿  │   │         │
          └─────────┘   └─────────┘
           ↑ 第一个跳变沿（上升沿）

CPOL = 1：
          ┌───┐         ┌───┐
  空闲 ───┘   │         │   │         ───
              │ 下降沿  │   │
              └─────────┘   └─────────
               ↑ 第一个跳变沿（下降沿）
```

### 5.2 时钟相位（CPHA — Clock Phase）

| CPHA | 采样时刻 | 含义 |
|:----:|---------|------|
| **0** | **第一个跳变沿** | 空闲结束立即采样 |
| **1** | **第二个跳变沿** | 延迟半个周期后采样 |

```
CPHA = 0（第一个跳变沿采样）：
  ┌───┐       ┌───┐
  │   │       │   │
  └───┘       └───┘
  ↑ 采样      ↑ 采样

CPHA = 1（第二个跳变沿采样）：
  ┌───┐       ┌───┐
  │   │       │   │
  └───┘       └───┘
      ↑ 采样      ↑ 采样
```

### 5.3 四种 SPI 模式

| 模式 | CPOL | CPHA | 空闲电平 | 采样沿 | 采样时机 |
|:----:|:----:|:----:|:--------:|:------:|---------|
| **Mode 0** | **0** | **0** | 低 | **上升沿** | 第一个跳变沿 |
| Mode 1 | 0 | 1 | 低 | 下降沿 | 第二个跳变沿 |
| Mode 2 | 1 | 0 | 高 | 下降沿 | 第一个跳变沿 |
| **Mode 3** | **1** | **1** | 高 | **上升沿** | 第二个跳变沿 |

### 5.4 Mode 0 和 Mode 3 对比

```
Mode 0（CPOL=0, CPHA=0）：
            ┌───┐   ┌───┐   ┌───┐
  SCK  ─────┘   └───┘   └───┘   └─────
            ↑       ↑       ↑
          采样    采样    采样（上升沿）

  数据在第一个上升沿就准备好 → 立即采样

Mode 3（CPOL=1, CPHA=1）：
          ┌───┐   ┌───┐   ┌───┐
  SCK  ───┘   └───┘   └───┘   └───────
              ↑       ↑       ↑
            采样    采样    采样（上升沿）

  数据延迟半个周期 → 在上升沿采样（同 Mode 0 效果）

共同点：都是上升沿采样
区别：Mode 3 延迟了半个周期
```

### 5.5 模式选择建议

```
最常用：Mode 0（CPOL=0, CPHA=0）
  → 数据提前准备好，第一个上升沿立即采样
  → 大多数设备默认支持

次常用：Mode 3（CPOL=1, CPHA=1）
  → 同样是上升沿采样，延迟半个周期
  → 给数据更多准备时间

Mode 0 + Mode 3 共同点：上升沿采样（更常用）
Mode 1 + Mode 2 共同点：下降沿采样（较少用）

⚠️ 主从设备必须配置为相同的 SPI Mode！
```

## 六、SPI 数据帧格式

```
STM32 支持：
  → 8 位数据帧（每次传输 1 字节）
  → 16 位数据帧（每次传输 2 字节）

数据位序：
  → 高位先行（MSB First）— 标准方式
  → 低位先行（LSB First）— 可配置
```

## 七、SPI 通信流程总结

```
初始化：
  ① 配置 GPIO（SCK/MOSI/MISO/SS 引脚）
  ② 配置 SPI 工作模式（CPOL/CPHA）
  ③ 配置主从模式
  ④ 配置数据帧格式（8/16 位）
  ⑤ 配置时钟分频（通信速率）
  ⑥ 使能 SPI

数据传输：
  ① 主机拉低对应从机 SS 线（片选）
  ② 主机写入发送数据到发送缓冲区
  ③ 移位寄存器在 SCK 驱动下逐位传输
  ④ 每个 SCK 周期交换 1 位数据
  ⑤ 8 个周期后，1 字节数据交换完成
  ⑥ 接收数据从接收缓冲区读出
  ⑦ 传输结束，主机拉高 SS 线
```

## 八、SPI 与 I²C 设计理念对比

```
I²C 的设计理念：
  → 节省线数（只需 2 根线）
  → 代价：协议复杂（地址+应答+方向位）
  → 速度受限（半双工 + 协议开销）

SPI 的设计理念：
  → 砍掉一切"花里胡哨"的
  → 代价：更多信号线（4 根线）
  → 速度极快（全双工 + 无协议开销）

核心思想：
  → 不传地址 → 用片选线代替
  → 不需应答 → 有数据就传
  → 不等响应 → 有时钟就走
  → 所有时间都用来传数据！
```

## 九、关键概念速查

| 概念 | 说明 |
|------|------|
| SPI | Serial Peripheral Interface，串行外设接口 |
| SCK | Serial Clock，串行时钟线（主机产生） |
| MOSI | Master Out Slave In，主出从入线 |
| MISO | Master In Slave Out，主入从出线 |
| SS/NSS/CS | Slave Select / Chip Select，片选信号（低电平有效） |
| CPOL | Clock Polarity，时钟极性（空闲电平：0=低，1=高） |
| CPHA | Clock Phase，时钟相位（采样沿：0=第一个，1=第二个） |
| Mode 0 | CPOL=0, CPHA=0（最常用） |
| Mode 3 | CPOL=1, CPHA=1（次常用） |
| 全双工 | 同时收发（MOSI + MISO 两根数据线） |
| 同步 | 有 SCK 时钟线同步数据传输 |
| 移位寄存器 | 底层串并转换的核心部件 |
| 片选 | 硬件选择从机（比 I²C 的地址寻址更快） |
| 高位先行 | MSB First（默认，可配置为 LSB First） |

## 十、常见问题

| 问题 | 解答 |
|------|------|
| 为什么 SPI 没有应答机制？ | 设计理念就是极简，有错就错，追求速度 |
| 从机之间能通信吗？ | 不能（无时钟、无片选、MOSI/MISO 设计不对称） |
| 片选线能共用吗？ | 不能（共用会导致同时选中多个从机） |
| 主机只想收怎么办？ | 发送无效数据（全 0），只读 MISO |
| 主机只想发怎么办？ | 忽略接收到的 MISO 数据 |
| Mode 0 和 Mode 3 有什么区别？ | 都是上升沿采样，Mode 3 延迟半个周期 |
| 最常用的模式是什么？ | **Mode 0**（CPOL=0, CPHA=0） |
| SPI 最高速率是多少？ | APB 总线频率 / 2（APB2→36MHz，APB1→18MHz） |

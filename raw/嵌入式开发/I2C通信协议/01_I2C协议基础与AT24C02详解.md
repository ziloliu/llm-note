# I²C 协议基础与 AT24C02 详解

> 本文档整合了 I²C 通信协议基础知识、AT24C02 EEPROM 芯片详解、协议时序与读写操作的完整内容。

---

## 一、I²C 基本概念

| 项目 | 内容 |
|------|------|
| 全称 | Inter-Integrated Circuit（内部集成电路） |
| 正确写法 | **I²C**（I 的平方 C） |
| 常见简写 | I2C、IIC |
| 开发者 | 飞利浦公司（20 世纪 80 年代初期） |
| 协议性质 | 开放通用协议（知识产权已公开） |
| 定位 | 系统内多个 IC 之间的**低速串行通讯** |
| 特点 | 引脚少、硬件简单、可扩展性强、通讯稳定、成本低 |

> "I2C" 中的 "2" 不是 "to" 的意思，而是 "平方"。标准英文读法为 "I square C"，口语中为简便有时读作 "I two C"。

---

## 二、I²C 与串口对比

### 2.1 相同点

| 对比项 | 说明 |
|--------|------|
| 通信方式 | 都是**串行**通信（一位一位传输） |
| 底层硬件 | 都使用**移位寄存器**实现串并转换 |
| 数据寄存器 | 收到数据后都交给内部数据缓冲寄存器（DR） |
| 速率范围 | 都属于**低速**串行通信（100kbps 量级） |

### 2.2 不同点

| 对比项 | UART（串口） | I²C |
|--------|-------------|-----|
| 信号线 | TX + RX（**两根数据线**） | SCL + SDA（**一根时钟线 + 一根数据线**） |
| 同步方式 | **异步**（无时钟线） | **同步**（有时钟线） |
| 双工模式 | **全双工**（收发各一根线） | **半双工**（一根数据线分时复用） |
| 连接方式 | **点对点**（TX↔RX 交叉连接） | **总线**（所有设备并联） |
| 设备数量 | 一对一台设备需多个串口模块 | 一条总线可挂载多个设备 |
| 输出模式 | **推挽输出** | **开漏输出 + 上拉电阻** |
| 传输顺序 | **低位先行** (LSB First) | **高位先行** (MSB First) |
| 应答机制 | **无** | **有**（ACK/NACK） |
| 寻址方式 | 无（点对点，不需要寻址） | **设备地址**（7 位或 10 位） |
| 起始/结束 | 起始位 + 停止位（协议定义） | 起始信号 S + 停止信号 P（SDA 跳变） |
| 数据帧 | 起始位 + 8 数据位 + 校验位 + 停止位 | 设备地址 + R/W + 数据 + ACK |
| 波特率控制 | 设定波特率（底层产生时钟） | 主设备控制 SCL 时钟频率 |

---

## 三、传输速率

| 模式 | 速率 | 说明 |
|------|------|------|
| 标准模式 | **100 kbps** | 与串口 115200 波特率同量级 |
| 快速模式 | **400 kbps** | 大多数设备支持 |
| 高速模式 | **3.4 Mbps** | 仅部分设备支持 |

> 标准模式 100kHz → 时钟周期 ≈ 10μs

## 四、物理层 — 硬件连接

### 4.1 两根信号线

| 线 | 全称 | 功能 | 驱动方 |
|----|------|------|--------|
| **SCL** | Serial Clock | 串行时钟线，由主设备控制，用于数据收发同步 | **主设备** |
| **SDA** | Serial Data | 串行数据线，双向传输，半双工 | 主设备或从设备（分时复用） |

### 4.2 总线拓扑

```
         VCC
          │
         [R] 上拉电阻
          │
SCL ──────┼────────────── 所有设备 SCL 引脚并联
          │
SDA ──────┼────────────── 所有设备 SDA 引脚并联
          │
   ┌──────┼──────┬──────────┐
   │      │      │          │
 主设备  从设备1  从设备2  从设备N
```

- 所有设备**并联**到同一条总线
- 总线空闲时，SDA 和 SCL 通过**上拉电阻**保持高电平
- 与串口的点对点连接不同，I²C 是**一主多从**或多主多从的总线结构

### 4.3 开漏输出与线与特性

| 项目 | 说明 |
|------|------|
| 输出模式 | 所有设备采用**开漏输出** |
| 默认状态 | 高阻态，由上拉电阻拉至高电平（逻辑 1） |
| 占用总线 | 设备输出低电平（接地），总线被拉低（逻辑 0） |
| 线与特性 | 任何一个设备输出 0，总线即为 0；所有设备输出高阻态，总线才为 1 |

```
设备输出高阻态 + 上拉电阻 → 总线为 1（高电平）
任一设备输出低电平        → 总线为 0（低电平）
```

> 开漏输出 + 上拉电阻实现了"线与"逻辑，是 I²C 总线多设备共存的基础。

### 4.4 主设备与从设备

| 角色 | 英文 | 特征 |
|------|------|------|
| **主设备** | Master | 控制 SCL 时钟信号，发起通讯，发送设备地址 |
| **从设备** | Slave | 接收时钟信号，响应主设备请求，拥有唯一地址 |

- **SCL 时钟线**：正常情况下仅由主设备驱动
- **特殊情况**：从设备可通过拉低 SCL 请求**时钟扩展**（Clock Stretching），通知主设备放慢速度
- **SDA 数据线**：主设备和从设备都可以驱动（半双工，分时复用）

### 4.5 GPIO 配置（软件模拟 I²C）

| 配置项 | 设置 | 原因 |
|--------|------|------|
| PB10 (SCL) | **通用开漏输出** | 时钟线，主机驱动 |
| PB11 (SDA) | **通用开漏输出** | 数据线，收发共用，输出模式下可读 IDR |
| 上拉电阻 | 外部 4.7KΩ 接 3.3V | 空闲时总线保持高电平 |

**开漏输出工作原理：**
```
输出 0：NMOS 导通 → 接地 → 总线拉低（占用总线）
输出 1：NMOS 关断 → 高阻态 → 上拉电阻拉高（释放总线）
```

> 输出模式下，输入回路仍然连接，可通过 IDR 寄存器读取引脚电平，满足收发双需求。

## 五、协议层 — 数据传输规则

### 5.1 传输顺序

| 项目 | 说明 |
|------|------|
| 传输方向 | **高位先行（MSB First）** |
| 与串口对比 | 串口为低位先行（LSB First），I²C 相反 |

### 5.2 起始信号（S）与停止信号（P）

**前提条件：** SCL 为**高电平**时，SDA 发生跳变

| 信号 | SCL | SDA 变化 | 简写 | 含义 | 总线状态 |
|------|:---:|----------|:----:|------|----------|
| **起始信号** | **高电平** | 高 → 低（**下降沿**） | **S** (Start) | 主设备占用总线，开始通讯 | 进入通讯 |
| **停止信号** | **高电平** | 低 → 高（**上升沿**） | **P** (Stop) | 主设备释放总线，结束通讯 | 恢复空闲 |

```
起始信号 S：        停止信号 P：
SCL: ──┐   ┌──      SCL: ──┐   ┌──
       └───┘               └───┘
SDA: ─┐                    ┌───
      └───                  ↑
       ↑                  低→高（上升沿）
    高→低（下降沿）
```

**直观记忆：**
```
空闲（高）→ 起始（拉低）→ 数据传输（高低交替）→ 停止（拉高）→ 空闲（高）
  高           低              锯齿波               高             高

类比：跳进坑（S）→ 在坑里干活（数据传输）→ 爬出坑（P）
```

### 5.3 数据有效性

| SCL 电平 | SDA 状态 | 含义 |
|----------|----------|------|
| **高电平** | **保持稳定**（高或低） | 数据采样时刻，读取 SDA 的 0 或 1 |
| **高电平** | **发生跳变** | 起始信号（高→低）或停止信号（低→高） |
| **低电平** | **可以改变** | 数据翻转时刻，为下一次采样准备新的电平 |

```
SCL:   ┌───┐   ┌───┐   ┌───┐   ┌───┐
    ───┘   └───┘   └───┘   └───┘
         采样      采样      采样      采样

SDA:  ┌────────┐           ┌────────
      │  数据1  │           │  数据2
──────┘        └───────────┘
    ↑ 变化（SCL低）  ↑ 变化（SCL低）
```

**核心规则：**
- SCL 高电平期间，SDA **必须保持稳定**（数据有效）
- SCL 低电平期间，SDA **可以改变**（准备下一位数据）

**传输一位数据的完整过程：**
```
① SCL 低电平期间 → SDA 设置好目标电平（高或低）
② SCL 低→高       → SDA 保持稳定，从设备采样
③ SCL 高→低       → 采样结束
④ SCL 低电平期间 → SDA 翻转为下一位数据的电平
```

> **关键：** SCL 高电平时发生 SDA 跳变 = 起始/停止信号；保持稳定 = 数据采样。

### 5.4 数据帧结构

一次完整的 I²C 通讯包含以下部分：

```
┌─────┬──────────────┬─────────────┬─────────┬─────┐
│  S  │ 设备地址(7位) │ R/W 方向位  │ 数据(8位)│ ACK │  → 可多次传输数据
└─────┴──────────────┴─────────────┴─────────┴─────┘
起始   从机地址        读/写控制     有效数据   应答
```

### 5.5 设备地址

#### 地址格式

```
┌────────────────────────┬─────────┐
│  设备地址（7位或10位）    │ R/W 位   │
└────────────────────────┴─────────┘
```

#### 地址位数

| 地址位数 | 可寻址设备数 | 说明 |
|----------|------------|------|
| **7 位** | 2^7 = **128** | 一般场景够用，最常用 |
| **10 位** | 2^10 = **1024** | 扩展模式，设备数量更多时使用 |

#### 读写方向位（R/W）

| 位值 | 含义 | 说明 |
|:----:|------|------|
| **0** | **写** | 主设备向从设备写入数据 |
| **1** | **读** | 主设备从从设备读取数据 |

> R/W 位是从**主设备的视角**定义的。

#### 地址传输示例（7 位地址）

```
发送字节：[A6][A5][A4][A3][A2][A1][A0][R/W]
           ├── 7位从设备地址 ──────┤  ├方向┤
           高位先行传输
```

## 六、应答机制（ACK / NACK）

### 6.1 为什么需要应答

- SDA 只有一根线，收发共用
- 发送方必须确认接收方是否正确收到数据
- 没有应答机制，后续数据传输可能混乱

### 6.2 应答时序

```
发送方                              接收方
  │                                  │
  │  传输 8 位数据（一个字节）          │
  │ ───────────────────────────────→ │
  │                                  │
  │  释放 SDA（输出高/高阻态）          │
  │                                  │
  │  ←───────────────────────────── │  接收方拉低 SDA = ACK
  │     （第 9 个时钟周期）             │
  │                                  │
```

### 6.3 应答类型

| 类型 | 英文 | SDA 电平 | 含义 |
|------|------|:--------:|------|
| **应答** | ACK (Acknowledge) | **低电平 (0)** | 接收成功，可继续传输 |
| **非应答** | NACK (Not Acknowledge) | **高电平 (1)** | 接收失败或无应答，需停止或重试 |

### 6.4 应答过程详解

```
第 1~8 个时钟周期：发送方驱动 SDA，传输 8 位数据
    ↓
发送方释放 SDA（输出高阻态，SDA 被上拉至高电平）
    ↓
第 9 个时钟周期：接收方控制 SDA
    ├── 拉低 SDA → ACK（收到数据，应答）
    └── 保持高电平 → NACK（未收到或不愿接收）
    ↓
接收方释放 SDA，准备下一轮传输
```

**总线控制权交替：**
```
发送方占线（发 8 位数据）→ 发送方释放 SDA（拉高）→ 接收方接管 SDA → 给应答信号
```

**NACK 的作用：**
- 从设备收到 NACK 后知道主机不需要更多数据
- 数据总线恢复空闲
- 主机可以发送停止信号（P）结束通讯

## 七、I²C 通讯完整流程

```
① 主设备发送起始信号（S）
        ↓
② 主设备发送 7 位从设备地址 + 1 位 R/W
        ↓
③ 从设备应答（ACK/NACK）
        ↓
④ 数据传输（每 8 位数据 + 1 位应答，可多次）
   ├── 写操作：主设备发数据 → 从设备应答
   └── 读操作：从设备发数据 → 主设备应答
        ↓
⑤ 主设备发送停止信号（P）
```

## 八、AT24C02 EEPROM 芯片详解

### 8.1 基本参数

| 参数 | 数值 |
|------|------|
| 全称 | Electrically Erasable Programmable Read-Only Memory |
| 容量 | **256 字节**（2 Kbit） |
| 页大小 | **16 字节** |
| 页数 | 16 页 |
| 写入周期 | **5 ms** |
| I²C 模式 | 标准模式 100kHz / 快速模式 400kHz |
| 供电电压 | 1.7V ~ 5.5V |
| 设备类型 ID | 1010（高 4 位固定） |

### 8.2 内部地址结构

```
8 位地址（0x00 ~ 0xFF）表示 256 个字节

地址位划分：
┌──────────────┬──────────────┐
│  高 4 位       │  低 4 位       │
│  页地址        │  页内地址       │
│  (0~15)       │  (0~15)       │
└──────────────┴──────────────┘

例：地址 0x3A → 页地址 = 0x3，页内地址 = 0xA
```

| 页号 | 地址范围 |
|:----:|----------|
| 0 | 0x00 ~ 0x0F |
| 1 | 0x10 ~ 0x1F |
| ... | ... |
| 15 | 0xF0 ~ 0xFF |

### 8.3 设备地址 vs 存储内部地址

这是两个**完全不同**的概念，必须严格区分：

| 对比项 | 设备地址 | 存储内部地址 |
|--------|----------|------------|
| 含义 | 识别总线上的**哪个设备** | 设备内部的**哪个存储单元** |
| 位数 | 7 位（+ 1 位读写方向位） | 8 位 |
| 范围 | 最多 128 个设备 | 0x00 ~ 0xFF（256 个字节） |
| 发送时机 | 通讯开始后**第一个**发送 | 设备地址之后发送 |
| 用途 | 主设备在总线上寻址从设备 | 指定从设备内部的读写位置 |

### 8.4 AT24C02 设备地址

#### 地址构成

```
7 位设备地址 = [1 0 1 0] [A2] [A1] [A0]
                 固定4位    ← 3位由引脚决定 →

本实验：A2 = A1 = A0 = 0（全接 GND）
设备地址 = 0b1010000 = 0x50
```

#### 设备码（含读写方向位）

```
完整 1 字节 = [设备地址 7 位] [R/W 1 位]

写操作：0b1010000 + 0 = 0b10100000 = 0xA0
读操作：0b1010000 + 1 = 0b10100001 = 0xA1
```

#### 代码中的宏定义

```c
#define AT24C02_WRITE_ADDR   0xA0    // 写地址
#define AT24C02_READ_ADDR    0xA1    // 读地址
```

#### 手册中的设备类型 ID

| 字段 | 位 | 值 | 说明 |
|------|-----|------|------|
| 设备类型标识 | D7~D4 | **1010** | 固定，AT24Cxx 系列统一 |
| 硬件地址 | D3~D1 | A2 A1 A0 | 由引脚电平决定 |
| 读写方向 | D0 | 0=写, 1=读 | 主设备控制 |

### 8.5 引脚功能速查

| 引脚 | 名称 | 连接 | 功能 |
|:----:|------|------|------|
| 1 | A0 | GND | 设备地址 bit0 = 0 |
| 2 | A1 | GND | 设备地址 bit1 = 0 |
| 3 | A2 | GND | 设备地址 bit2 = 0 |
| 4 | GND | GND | 电源地 |
| 5 | SDA | PB11 | I²C 数据线（经上拉电阻接 3.3V） |
| 6 | SCL | PB10 | I²C 时钟线（经上拉电阻接 3.3V） |
| 7 | WP | GND | 写保护关闭（低电平允许读写） |
| 8 | VCC | 3.3V | 电源 |

### 8.6 两个地址的发送顺序

```
主机发送顺序：

① 起始信号(S)
② 设备地址 + R/W   ← 第一个字节：告诉总线上哪个设备、读还是写
③ 存储内部地址      ← 第二个字节：告诉该设备从哪个存储单元操作
④ 数据（写入）或 接收数据（读取）
⑤ 停止信号(P)
```

```
示例：向 AT24C02 的地址 0x3A 写入数据 0x55

[S] [0xA0] [ACK] [0x3A] [ACK] [0x55] [ACK] [P]
      ↑               ↑               ↑
   设备码(写)      存储内部地址       写入数据
```

## 九、AT24C02 读写时序

### 9.1 写入一个字节（Byte Write）

#### 完整时序

```
[S] → [0xA0] → ACK → [内部地址] → ACK → [数据] → ACK → [P]
                                   等待 5ms 写入周期
```

#### 步骤详解

| 步骤 | 方向 | 内容 | 应答 |
|:----:|------|------|:----:|
| 1 | 主→从 | 起始信号 S | — |
| 2 | 主→从 | 设备码 **0xA0**（7位地址1010000 + 写0） | ACK |
| 3 | 主→从 | **内部存储地址**（0x00~0xFF，1字节） | ACK |
| 4 | 主→从 | **写入数据**（1字节） | ACK |
| 5 | 主→从 | 停止信号 P | — |
| 6 | — | **等待 5ms**（写入周期） | — |

#### 数据总线控制权

```
写入全程：数据总线始终由主设备（STM32）控制

主设备 ──S──→ [设备码+W] ──→ [内部地址] ──→ [数据] ──P──→
         ←ACK──          ←ACK──          ←ACK──
从设备（AT24C02）
```

| 步骤 | SDA 控制方 | 发送内容 | 方向 |
|:----:|-----------|----------|------|
| 1 | **主设备** | 设备地址码 + 写(0) | 主→从 |
| 2 | **主设备** | 存储内部地址 | 主→从 |
| 3 | **主设备** | 写入数据 | 主→从 |
| ACK | **从设备** | 低电平应答 | 从→主 |

> 写入过程非常顺畅：所有数据字节都由主设备发出，方向始终一致，无需切换总线控制权。

### 9.2 读取一个字节 — 两种方式

#### 当前地址读取（Current Address Read）

```
[S] → [设备地址码 + R(1)] → 从设备 ACK → [从设备发送数据] → 主设备 NACK → [P]
```

**问题：** 发出读地址后，总线控制权**立即交给从设备**，主设备无法再指定从哪个地址开始读。

**读取位置：** 从 AT24C02 内部地址计数器（Address Counter）当前指向的位置开始。

```
地址计数器示例：

上次操作后计数器指向 0x3A
    ↓
当前地址读取 → 从 0x3A 开始读
    ↓
读完后计数器自动 +1 → 0x3B
```

**特点：** 只能从上次读写位置**顺序继续**读取，不能指定任意地址。

#### 随机地址读取（Random Address Read）— 假写真读

**为什么需要假写真读：**

| 直接读的问题 | 解决方案 |
|-------------|----------|
| 给读地址(R=1)后，总线控制权立即交给从设备 | 先用**写地址**(R=0)保持主设备控制权 |
| 无法在读取前指定内部地址 | 用"假写"将目标地址写入地址计数器 |

**假写阶段（设置地址计数器）：**
```
[S] → [设备地址码 + W(0)] → ACK → [内部地址] → ACK
                                        ↑
                              写入地址计数器（非存储单元）
```

| 项目 | 说明 |
|------|------|
| 为什么用写地址？ | 写方向时主设备保持总线控制权，可以继续发送数据 |
| 发送了什么？ | 目标存储单元的地址（如 0x3A） |
| 写到了哪里？ | 写入了地址计数器（Address Counter），**非**存储单元 |
| 为什么叫"假写"？ | 只发地址，不发数据，没有真正向存储单元写入 |

**真读阶段（从指定地址读取数据）：**
```
[Sr] → [设备地址码 + R(1)] → ACK → [从设备发送数据] → 主设备 NACK → [P]
```

| 项目 | 说明 |
|------|------|
| 为什么用重复起始(Sr)？ | 不发P保持总线占用，直接重新开始新周期切换方向 |
| 总线控制权？ | 读地址(R=1)发出后，总线控制权交给从设备 |
| 从哪个地址读？ | 从地址计数器指定的地址（即假写阶段写入的地址） |
| 为什么最后发NACK？ | 只需1字节，NACK告诉从设备"够了，停止发送" |
| NACK后的行为？ | 从设备停止发送 → 主设备发P结束通讯 |

**完整流程图：**
```
┌──────────── 假写（主设备控制总线）────────────┐
│                                               │
│  [S] → [0xA0(W)] → ACK → [0x3A] → ACK       │
│   ↑       ↑                   ↑               │
│  起始   写地址             写入地址计数器       │
│                                               │
└───────────────────────────────────────────────┘
              ↓ 总线控制权不变
┌──────────── 真读（切换为从设备控制）──────────┐
│                                               │
│  [Sr] → [0xA1(R)] → ACK → [数据] → NACK → [P]│
│   ↑       ↑                   ↑       ↑       │
│  重复起始 读地址          从设备发送  够了别发了 │
│                    总线控制权交给从设备         │
└───────────────────────────────────────────────┘
```

#### 两种读取方式对比

| 对比项 | 当前地址读取 | 随机地址读取（假写真读） |
|--------|:----------:|:---------------------:|
| 起始步骤 | 直接发读地址 | 先假写（写地址+内部地址） |
| 起始信号数 | 1 个 S | 2 个（S + Sr） |
| 内部地址指定 | 不可（用计数器当前位置） | 可指定任意地址 |
| 地址计数器 | 使用当前值 | 被更新为指定值 |
| 读取位置 | 上次操作的下一位置 | 主设备指定的任意位置 |
| 适用场景 | 顺序连续读取 | **随机访问**任意存储单元 |

### 9.3 连续写入多个字节（Page Write）

#### 时序

```
[S] → [0xA0] → [ACK] → [内部地址] → [ACK]
  → [数据1] → [ACK] → [数据2] → [ACK] → ... → [数据N] → [ACK]
  → [P]
```

#### 与单字节写入的区别

| 对比项 | 单字节写入 | 多字节写入 |
|--------|-----------|-----------|
| 数据数量 | 1 个字节 | N 个字节 |
| ACK 次数 | 3 次（设备码+地址+数据） | 2 + N 次 |
| 结束方式 | 数据 ACK 后直接 P | 最后一个数据 ACK 后 P |
| 代码实现 | 直接调用 | **for 循环**逐个发送 |
| 每字节后 | 等待 5ms | 等待 5ms |

#### 页写入限制

| 规则 | 说明 |
|------|------|
| 每次最多 **16 字节** | AT24C02 页大小 = 16 字节 |
| **不可跨页** | 写到页末后地址自动**绕回**页首 |
| 绕回后果 | 页首数据被**覆盖**，相当于数据丢失 |
| 解决方案 | 分多次写入，每次不超 16 字节且不跨页 |

```
页内地址循环：
0x00 → 0x01 → ... → 0x0F → 0x00（绕回覆盖）→ 0x01 → ...
```

### 9.4 连续读取多个字节（Sequential Read）

#### 时序（随机地址连续读）

```
假写：[S] → [0xA0] → [ACK] → [起始内部地址] → [ACK]

真读：[Sr] → [0xA1] → [ACK]
  → [数据1] ← 主机 ACK（还要）
  → [数据2] ← 主机 ACK（还要）
  → ...
  → [数据N] ← 主机 NACK（够了）
  → [P]
```

#### ACK 与 NACK 的区别

| 主机应答 | 含义 | 从设备行为 |
|:--------:|------|-----------|
| **ACK** | 还要继续读 | 地址计数器 +1，发送下一字节 |
| **NACK** | 读取结束 | 停止发送，释放数据总线 |

#### 与单字节读取的对比

| 对比项 | 单字节读取 | 多字节读取 |
|--------|-----------|-----------|
| 应答方式 | 直接 NACK | 前 N-1 字节 ACK，最后 1 字节 NACK |
| 代码实现 | 直接调用 | **for 循环**，最后发 NACK |

### 9.5 AT24C02 读取模式分类

| 模式 | 英文 | 操作 | 地址来源 |
|------|------|------|----------|
| **当前地址读** | Current Address Read | 直接读 | 地址计数器当前位置 |
| **随机地址读** | Random Address Read | 假写真读 | 主机指定地址 |
| **当前地址连续读** | Sequential Read (Current) | 当前地址读 + 多次 ACK | 从计数器位置开始 |
| **随机地址连续读** | Sequential Read (Random) | 假写真读 + 多次 ACK | 从主机指定地址开始 |

## 十、地址计数器（Address Counter）

```
AT24C02 内部维护一个 8 位地址计数器：

写入操作时：
  → 计数器被设置为写入的内部地址
  → 写入完成后计数器 +1

读取操作时：
  → 从计数器指向的地址开始读取
  → 读取完成后计数器 +1

假写操作时：
  → 计数器被设置为假写指定的地址
  → 不执行真正的存储写入
```

**作用：**
- 当前地址读取时，从计数器指向的位置开始
- 连续读取时，自动递增访问下一个存储单元
- 假写操作实质是设置计数器的值

## 十一、主设备与从设备的角色分工

| 操作 | 主设备（STM32） | 从设备（AT24C02） |
|------|----------------|------------------|
| 时钟信号 SCL | **唯一控制方** | 被动接收 |
| 起始/停止信号 | **发出 S 和 P** | 被动接收 |
| 设备选择 | **发送设备地址码** | 地址比较，判断是否选中自己 |
| 写入数据 | **控制 SDA 发送** | 接收并返回 ACK |
| 读取数据 | 接收并返回 ACK/NACK | **控制 SDA 发送** |
| 总线释放 | 输出高阻态（SDA 拉高） | 输出高阻态（SDA 拉高） |

## 十二、从设备地址匹配机制

```
主设备发出：[S] → [1 0 1 0 A2 A1 A0 R/W]
                        ↓
所有从设备同时接收前 7 位地址
                        ↓
每个从设备内部硬件比较器：
  收到的地址 == 自身地址？
    ├── 是 → 返回 ACK，参与后续通讯
    └── 否 → 不响应，保持沉默
```

## 十三、数据总线控制权转换

### 13.1 写入操作 — 无需转换

```
全程主设备控制 SDA：
  S → 设备码+W → 内部地址 → 数据 → P
  └──────── 主设备始终控制 ────────┘
```

### 13.2 读取操作 — 必须转换

```
假写阶段：主设备控制 SDA
  S → 设备码+W → 内部地址
  └── 主设备控制 ──┘

         ↓ 控制权转换点：Sr + 设备码+R

真读阶段：从设备控制 SDA
  Sr → 设备码+R → [数据从设备发] → NACK → P
       └── 从设备控制 ──┘
```

### 13.3 控制权转换的关键

| 触发条件 | 说明 |
|----------|------|
| 发出读地址(R=1) + 从设备ACK | 总线控制权交给从设备 |
| 从设备收到 NACK | 从设备停止发送，总线空闲 |
| 主设备发出 P | 总线恢复空闲状态 |

## 十四、写入与读取完整对比

| 对比项 | 写入一个字节 | 读取一个字节（随机地址） |
|--------|-------------|----------------------|
| 起始 | S | S |
| 设备码 | 0xA0（写） | 第一步 0xA0（假写），第二步 0xA1（真读） |
| 内部地址 | 直接发送 | 通过假写发送 |
| 数据方向 | 始终主→从 | 假写：主→从；真读：从→主 |
| 最后应答 | 从设备 ACK | 主设备 NACK |
| 结束 | P | P |
| 中间过渡 | 无 | **Sr（重复起始）** |
| 写入等待 | 5ms | 无 |

## 十五、完整通讯对比总结

| 操作 | 信号数 | 总线切换 | 复杂度 |
|------|:------:|:--------:|:------:|
| 写入 1 字节 | S + 3 字节 + P | 无 | 简单 |
| 当前地址读 1 字节 | S + 1 字节 + P | 1 次 | 较简单 |

| 连续读 N 字节 | S + 2 字节 + Sr + N 字节 + P | 1 次 | for 循环 |

## 十六、I²C 协议特点总结

| 特点 | 说明 |
|------|------|
| 硬件简单 | 仅 SCL + SDA 两根线 + 上拉电阻 |
| 协议复杂 | 每次读写需多个步骤（起始→设备码→地址→数据→应答→停止） |
| 半双工 | 单根数据线，收发分时复用 |
| 低速 | 标准 100kHz / 快速 400kHz |
| 总线结构 | 多设备并联，通过设备地址区分 |
| 开漏输出 | 线与逻辑，支持多设备共存 |
| 应答机制 | 保证收发双方同步确认 |
| 广泛应用 | 简单、稳定、成本低 |

## 十七、关键记忆口诀

| 口诀 | 含义 |
|------|------|
| **SCL 高时看 SDA** | 高电平采样数据或检测起始/停止跳变 |
| **SCL 低时改 SDA** | 低电平时才能翻转数据线电平 |
| **起始：高拉低** | SCL 高时 SDA 下降沿 |
| **停止：低拉高** | SCL 高时 SDA 上升沿 |
| **ACK 是 0，NACK 是 1** | 应答拉低，非应答保持高 |
| **高位先行 MSB** | I²C 先发最高位，与串口相反 |
| **假写真读** | 用写地址设置计数器，再用读地址读数据 |
| **ACK 继续，NACK 停止** | 连续读取时主机控制从设备是否继续发送 |

# AT24C02 接口层实现 — 硬件 I²C 版本 笔记

---

## 一、接口层变更要点

### 1.1 头文件（m24c02.h）— 无需修改

接口层的函数声明完全不变，主函数调用方式也不变：

| 函数 | 签名 | 与软件模拟版本一致 |
|------|------|:------------------:|
| `EEPROM_Init()` | `void` | ✅ |
| `EEPROM_WriteByte()` | `(uint8_t addr, uint8_t byte)` | ✅ |
| `EEPROM_ReadByte()` | `(uint8_t addr) → uint8_t` | ✅ |
| `EEPROM_WriteBytes()` | `(uint8_t addr, uint8_t *data, uint8_t size)` | ✅ |
| `EEPROM_ReadBytes()` | `(uint8_t addr, uint8_t *buffer, uint8_t size)` | ✅ |

### 1.2 核心改动：调用方式变化

| 软件模拟调用 | 硬件 I²C 调用 | 变化 |
|-------------|--------------|------|
| `I2C_SendByte(0xA0)` + `I2C_WaitAck()` | `I2C_SendAddr(0xA0)` | **合并**，地址发送内置等待 |
| `I2C_SendByte(data)` + `I2C_WaitAck()` | `I2C_SendByte(data)` | **删除** WaitAck，内置 BTF 等待 |
| `I2C_SendAck()` | `I2C_EnableAck()` | 改名（配置项） |
| `I2C_SendNack()` | `I2C_DisableAck()` | 改名（配置项） |

### 1.3 总体修改原则

```
所有改动总结为两点：
  ① 等待应答步骤全部删除（硬件自动处理）
  ② 发送设备地址改用 I2C_SendAddr()（专用函数）
```

## 二、EEPROM_WriteByte() — 写入一个字节

### 2.1 软件模拟 vs 硬件 I²C

| 步骤 | 软件模拟 | 硬件 I²C |
|:----:|----------|----------|
| ① | `I2C_Start()` | `I2C_Start()` |
| ② | `I2C_SendByte(W_ADDR)` | `I2C_SendAddr(W_ADDR)` |
| ③ | `I2C_WaitAck()` | **删除** |
| ④ | `I2C_SendByte(inner_addr)` | `I2C_SendByte(inner_addr)` |
| ⑤ | `I2C_WaitAck()` | **删除** |
| ⑥ | `I2C_SendByte(byte)` | `I2C_SendByte(byte)` |
| ⑦ | `I2C_WaitAck()` | **删除** |
| ⑧ | `I2C_Stop()` | `I2C_Stop()` |
| ⑨ | `Delay_ms(5)` | `Delay_ms(5)` |

### 2.2 代码实现

```c
void EEPROM_WriteByte(uint8_t inner_addr, uint8_t byte)
{
    // ① 起始信号
    I2C_Start();

    // ② 发送写地址（内置等待 ADDR 标志）
    I2C_SendAddr(W_ADDR);

    // ③ 发送内部存储地址
    I2C_SendByte(inner_addr);

    // ④ 发送写入数据
    I2C_SendByte(byte);

    // ⑤ 停止信号
    I2C_Stop();

    // ⑥ 等待写入周期
    Delay_ms(5);
}
```

### 2.3 应答处理说明

| 软件模拟 | 硬件 I²C |
|----------|----------|
| 每次发完数据后单独调用 `I2C_WaitAck()` | `I2C_SendAddr()` 内部等待 ADDR |
| 手动读取 SDA 电平 | `I2C_SendByte()` 内部等待 BTF |
| ACK/NACK 由软件判断 | 硬件自动检测，标志位反映结果 |

## 三、EEPROM_ReadByte() — 读取一个字节（假写真读）

### 3.1 关键变化：ACK/NACK 和 STOP 的设置顺序

**软件模拟中**：ACK/NACK 和 STOP 是即时执行的

**硬件 I²C 中**：ACK、NACK、STOP 都是**配置项/设置位**，在**下一个字节传输完成后**才生效

```
CR1.ACK = 0（DisableAck）→ 设置：下一个收到的字节后不返回 ACK
CR1.STOP = 1             → 设置：下一个字节传输结束后发出停止信号
```

### 3.2 顺序调整逻辑

```
❌ 错误顺序（软件模拟思路）：
  DisableAck()     ← 设置非应答
  I2C_Stop()       ← 设置停止
  byte = I2C_ReceiveByte()  ← 读取字节
  问题：设置在读取之前，但读取时字节还没到，设置提前生效逻辑错误

✅ 正确顺序（硬件 I²C 思路）：
  DisableAck()     ← 设置：下一个字节收到后不返回 ACK
  I2C_Stop()       ← 设置：下一个字节传输结束后发出停止
  byte = I2C_ReceiveByte()  ← 真正读取字节 → 硬件自动执行上述设置
```

### 3.3 为什么这样设置才正确

```
DisableAck() + I2C_Stop() 设置后：
  ↓
I2C_ReceiveByte() 读取最后一个字节
  ↓
硬件自动行为：
  → 收到字节后不返回 ACK（因为 ACK 位已被清除）
  → 字节传输结束后发出停止信号（因为 STOP 位已被设置）
```

### 3.4 手册依据（主接收器传送序列）

```
事件序列：
  S → EV5(SB=1) → 发送地址 → EV6(ADDR=1)
  → EV7(RXNE=1, 读DR) → EV7_1(设置ACK=0, 设置STOP)
  → 读取最后一个字节 → 硬件自动发NACK + STOP

关键事件 EV7_1（适用于只接收一个字节）：
  → 在接收最后一个字节之前
  → 先设置 ACK = 0（禁用应答）
  → 再设置 STOP = 1（准备停止）
  → 然后读取 DR 接收数据
  → 硬件自动执行 NACK + STOP
```

### 3.5 代码实现

```c
uint8_t EEPROM_ReadByte(uint8_t inner_addr)
{
    uint8_t byte;

    // ===== 假写阶段 =====

    // ① 起始信号
    I2C_Start();

    // ② 发送写地址（假写）
    I2C_SendAddr(W_ADDR);

    // ③ 发送内部地址（写入地址计数器）
    I2C_SendByte(inner_addr);

    // ===== 真读阶段 =====

    // ④ 重复起始
    I2C_Start();

    // ⑤ 发送读地址
    I2C_SendAddr(R_ADDR);

    // ⑥ 设置非应答（下一个字节收到后不返回 ACK）
    I2C_DisableAck();

    // ⑦ 设置停止信号（下一个字节传输结束后发出）
    I2C_Stop();

    // ⑧ 读取数据（硬件自动执行 NACK + STOP）
    byte = I2C_ReceiveByte();

    return byte;
}
```

### 3.6 执行流程图

```
① I2C_Start()          → 产生起始信号
② I2C_SendAddr(W_ADDR) → 发送写地址，等待 ADDR
③ I2C_SendByte(addr)   → 发送内部地址，等待 BTF
④ I2C_Start()          → 重复起始
⑤ I2C_SendAddr(R_ADDR) → 发送读地址，等待 ADDR
⑥ I2C_DisableAck()     → 设置：收到字节后不返回 ACK
⑦ I2C_Stop()           → 设置：字节传输结束后发出 STOP
⑧ I2C_ReceiveByte()    → 读取数据 → 硬件自动执行 NACK + STOP
```

## 四、EEPROM_WriteBytes() — 连续写入多个字节

### 4.1 代码实现

```c
void EEPROM_WriteBytes(uint8_t inner_addr, uint8_t *data, uint8_t size)
{
    uint8_t i;

    // ① 起始信号
    I2C_Start();

    // ② 发送写地址
    I2C_SendAddr(W_ADDR);

    // ③ 发送内部地址
    I2C_SendByte(inner_addr);

    // ④ 循环发送数据
    for (i = 0; i < size; i++)
    {
        I2C_SendByte(data[i]);
    }

    // ⑤ 停止信号
    I2C_Stop();

    // ⑥ 等待写入周期
    Delay_ms(5);
}
```

### 4.2 变化对比

| 步骤 | 软件模拟 | 硬件 I²C |
|:----:|----------|----------|
| 发地址 | `I2C_SendByte(W_ADDR)` + `I2C_WaitAck()` | `I2C_SendAddr(W_ADDR)` |
| 发内部地址 | `I2C_SendByte(addr)` + `I2C_WaitAck()` | `I2C_SendByte(addr)` |
| 发数据 | `I2C_SendByte(data[i])` + `I2C_WaitAck()` | `I2C_SendByte(data[i])` |

> 所有 `I2C_WaitAck()` 调用均删除，应答检测内置在 `SendAddr` 和 `SendByte` 中。

## 五、EEPROM_ReadBytes() — 连续读取多个字节

### 5.1 关键：ACK/NACK/STOP 的设置位置

与单字节读取不同，多字节读取中：

| 情况 | 设置行为 |
|------|----------|
| 非最后一字节（`i < size-1`） | `I2C_EnableAck()` → 收到字节后返回 ACK |
| 最后一字节（`i == size-1`） | `I2C_DisableAck()` + `I2C_Stop()` → 收到字节后发 NACK + STOP |

**设置必须在读取之前**，这样硬件才能在收到字节后自动执行对应操作。

### 5.2 代码实现

```c
void EEPROM_ReadBytes(uint8_t inner_addr, uint8_t *buffer, uint8_t size)
{
    uint8_t i;

    // ===== 假写阶段 =====

    // ① 起始信号
    I2C_Start();

    // ② 发送写地址（假写）
    I2C_SendAddr(W_ADDR);

    // ③ 发送内部地址
    I2C_SendByte(inner_addr);

    // ===== 真读阶段 =====

    // ④ 重复起始
    I2C_Start();

    // ⑤ 发送读地址
    I2C_SendAddr(R_ADDR);

    // ⑥ 循环读取数据
    for (i = 0; i < size; i++)
    {
        if (i == size - 1)
        {
            // 最后一个字节：设置非应答 + 设置停止
            I2C_DisableAck();
            I2C_Stop();
        }
        else
        {
            // 非最后一个字节：保持应答使能
            I2C_EnableAck();
        }

        // 读取字节（放在设置之后）
        buffer[i] = I2C_ReceiveByte();
    }
}
```

### 5.3 为什么 ACK/NACK/STOP 设置必须在 ReceiveByte 之前

```
❌ 错误顺序：
  buffer[i] = I2C_ReceiveByte();   ← 先读取（硬件已自动发了 ACK）
  I2C_DisableAck();                ← 后设置（来不及了）
  I2C_Stop();                      ← 后设置（来不及了）

✅ 正确顺序：
  I2C_DisableAck();                ← 先设置
  I2C_Stop();                      ← 先设置
  buffer[i] = I2C_ReceiveByte();   ← 后读取 → 硬件自动执行设置的行为
```

### 5.4 执行流程图（3 字节读取示例）

```
假写：
  ① I2C_Start()
  ② I2C_SendAddr(W_ADDR)
  ③ I2C_SendByte(inner_addr)

真读：
  ④ I2C_Start()
  ⑤ I2C_SendAddr(R_ADDR)

循环 i=0：
  EnableAck()          ← 保持 ACK
  buffer[0] = ReceiveByte()  ← 收到后硬件发 ACK

循环 i=1：
  EnableAck()          ← 保持 ACK
  buffer[1] = ReceiveByte()  ← 收到后硬件发 ACK

循环 i=2（最后一个）：
  DisableAck()         ← 设置 NACK
  I2C_Stop()           ← 设置 STOP
  buffer[2] = ReceiveByte()  ← 收到后硬件发 NACK + STOP
```

## 六、硬件 I²C 接口层与软件模拟接口层对比

| 函数 | 软件模拟版本 | 硬件 I²C 版本 |
|------|-------------|--------------|
| **WriteByte** | 8 步（含 3 次 WaitAck） | **5 步** |
| **ReadByte** | 11 步（含 3 次 WaitAck） | **8 步** |
| **WriteBytes** | 8 步（含 N+2 次 WaitAck） | **5 步** |
| **ReadBytes** | 11 步（含 N+2 次 WaitAck） | **8 步** |

> 每个函数减少的步骤数 = 删除的 WaitAck 调用次数。

## 七、主函数 — 无需修改

```c
int main(void)
{
    // 初始化
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    MX_USART1_UART_Init();
    EEPROM_Init();

    printf("I2C 硬件方式实验开始\n");

    // 单字节读写
    EEPROM_WriteByte(0x00, 'A');
    EEPROM_WriteByte(0x01, 'B');
    EEPROM_WriteByte(0x02, 'C');
    uint8_t b1 = EEPROM_ReadByte(0x00);
    uint8_t b2 = EEPROM_ReadByte(0x01);
    uint8_t b3 = EEPROM_ReadByte(0x02);
    printf("byte1=%c\tbyte2=%c\tbyte3=%c\n", b1, b2, b3);

    // 多字节读写
    EEPROM_WriteBytes(0x00, "123456", 6);
    uint8_t buffer[100] = {0};
    EEPROM_ReadBytes(0x00, buffer, 6);
    printf("buffer=%s\n", buffer);

    // 超 16 字节测试
    memset(buffer, 0, sizeof(buffer));
    EEPROM_WriteBytes(0x00, "1234567890abcdefjhijk", 21);
    EEPROM_ReadBytes(0x00, buffer, 21);
    printf("buffer=%s\n", buffer);

    while (1) {}
}
```

> 与软件模拟版本完全一致，无需任何修改。

## 九、硬件 I²C 完整工程总结

### 9.1 分层结构

```
┌─────────────────────────────────────┐
│        应用层（main.c）              │
│   写入/读出/串口打印（无需修改）      │
├─────────────────────────────────────┤
│     接口层（m24c02.c/h）             │
│   假写真读、页写入、连续读写          │
│   调用 I2C_SendAddr/SendByte 等      │
├─────────────────────────────────────┤
│     协议层（i2c.c/h）                │
│   硬件 I²C 寄存器操作                │
│   START/STOP/ACK/D收发字节           │
├─────────────────────────────────────┤
│     硬件层（STM32 I²C2 外设）        │
│   CR1/CR2/CCR/TRISE/SR1/DR          │
│   PB10(SCL) PB11(SDA) 复用开漏输出   │
└─────────────────────────────────────┘
```

### 9.2 三种 I²C 实现方式对比

| 对比项     | 软件模拟 I²C | 硬件 I²C（寄存器） | 硬件 I²C（HAL 库） |
| ------- | -------- | ----------- | ------------- |
| GPIO 模式 | 通用开漏输出   | 复用开漏输出      | CubeMX 配置     |
| 时序控制    | 手动 Delay | 硬件自动（CCR）   | 硬件自动          |
| CPU 占用  | 高        | 低           | 低             |
| 代码量     | 大        | 中           | 小             |
| 引脚灵活性   | 任意 GPIO  | 固定复用引脚      | 固定复用引脚        |
| 可靠性     | 依赖软件     | 硬件保证        | 硬件保证          |

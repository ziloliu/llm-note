# DMA 原理、寄存器与存储器到存储器传输 笔记

---

## 一、DMA 基本概念

### 1.1 什么是 DMA

| 项目 | 说明 |
|------|------|
| 全称 | Direct Memory Access（直接存储器访问） |
| 本质 | 硬件数据传输通道，替代 CPU 执行数据搬运 |
| 核心优势 | 解放 CPU，高速数据传输 |

### 1.2 传统方式 vs DMA 方式

```
传统方式（CPU 参与）：
  ROM → 数据总线 → CPU → 数据总线 → RAM
  CPU 需要：读取源数据 → 暂存内部寄存器 → 写入目标地址
  每一步都占用 CPU 时间和数据总线

DMA 方式（CPU 不参与）：
  ROM → DMA 通道 → RAM
  CPU 只负责配置，之后完全由 DMA 硬件自动完成
  CPU 可以去处理其他任务
```

## 二、STM32 的 DMA 控制器

### 2.1 DMA1 和 DMA2

| 控制器 | 通道数 | 说明 |
|--------|:------:|------|
| **DMA1** | 7 个通道 | 挂在 AHB 总线 |
| **DMA2** | 5 个通道 | 挂在 AHB 总线 |

> DMA 控制器是系统核心部件，挂在 **AHB 总线**上，不是外设。

### 2.2 通道分配特点

| 传输类型 | 通道分配 |
|----------|----------|
| 外设 ↔ 存储器 | **固定分配**（查表确定） |
| 存储器 → 存储器 | **任意通道**（特殊模式） |

## 三、存储器到存储器传输

### 3.1 传输方向

```
ROM（Flash）→ RAM（SRAM）

限制：
  ✅ ROM → RAM（可以）
  ❌ RAM → ROM（不可以，Flash 需要特定协议写入）

ROM 存放：程序代码、全局常量
RAM 存放：变量
```

### 3.2 为什么用 DMA 而不是赋值语句

```
代码：b = a;    // a 在 ROM，b 在 RAM

CPU 执行过程：
  ① 从 ROM 读取 a 的值 → CPU 内部寄存器
  ② 通过数据总线将值写入 RAM 中 b 的地址
  
  每一步都占用 CPU 和数据总线

DMA 方式：
  DMA 控制器在 ROM 和 RAM 之间建立高速通路
  数据自动搬运，CPU 完全不参与
```

### 3.3 本实验选择

```
使用 DMA1 通道 1（任意通道都可以）
原因：存储器到存储器模式不查通道分配表
默认硬件优先级最高的通道
```

## 四、关键寄存器详解

### 4.1 四个核心寄存器（每个通道独立）

| 寄存器 | 全称 | 功能 |
|--------|------|------|
| **CCR** | Channel Configuration Register | 通道配置（核心） |
| **CNDTR** | Channel Number of Data to Transfer | 数据传输数量 |
| **CPAR** | Channel Peripheral Address Register | 外设地址 |
| **CMAR** | Channel Memory Address Register | 存储器地址 |

### 4.2 两个共用寄存器（所有通道共享）

| 寄存器 | 全称 | 功能 |
|--------|------|------|
| **ISR** | Interrupt Status Register | 中断状态（只读） |
| **IFCR** | Interrupt Flag Clear Register | 中断标志清除（只写） |

### 4.3 寄存器存储映像

```
地址偏移    寄存器          功能
─────────────────────────────────
0x00        ISR             中断状态（所有通道共用）
0x04        IFCR            中断标志清除（所有通道共用）
─────────────────────────────────
0x08        CCR1            通道 1 配置
0x0C        CNDTR1          通道 1 数据数量
0x10        CPAR1           通道 1 外设地址
0x14        CMAR1           通道 1 存储器地址
─────────────────────────────────
0x1C        CCR2            通道 2 配置
...         ...             ...
─────────────────────────────────
每 4 个寄存器为一组，对应一个通道
DMA1: 通道 1~7
DMA2: 通道 1~5
```

## 五、CCR 寄存器详解（通道配置寄存器）

### 5.1 所有位域

| 位 | 名称 | 功能 |
|:--:|------|------|
| 0 | **EN** | 通道使能 |
| 1 | **TCIE** | 传输完成中断使能 |
| 2 | **HTIE** | 传输过半中断使能 |
| 3 | **TEIE** | 传输错误中断使能 |
| 4 | **DIR** | 数据传输方向 |
| 5 | **CIRC** | 循环模式 |
| 6 | **PINC** | 外设地址自增 |
| 7 | **MINC** | 存储器地址自增 |
| 9:8 | **PSIZE** | 外设数据宽度 |
| 11:10 | **MSIZE** | 存储器数据宽度 |
| 13:12 | **PL** | 优先级 |
| 14 | **MEM2MEM** | 存储器到存储器模式 |

### 5.2 MEM2MEM（存储器到存储器模式）

| 值 | 模式 |
|:--:|------|
| 0 | 正常模式（外设 ↔ 存储器） |
| **1** | **存储器到存储器模式**（本实验使用） |

> 这是存储器到存储器传输的**特殊标志位**，必须置 1。

### 5.3 DIR（数据传输方向）

| 值 | 方向 | 说明 |
|:--:|------|------|
| **0** | 外设 → 存储器 | 从外设**读** |
| 1 | 存储器 → 外设 | 向外设**写** |

```
本实验：ROM → RAM
  → ROM 当作"外设"（源）
  → RAM 当作"存储器"（目的）
  → DIR = 0（从外设读）
```

### 5.4 PSIZE / MSIZE（数据宽度）

| 值 | 位宽 | 字节数 |
|:--:|:----:|:------:|
| 00 | 8 位 | 1 字节 |
| 01 | 16 位 | 2 字节 |
| 10 | 32 位 | 4 字节 |
| 11 | 保留 | — |

```
⚠️ PSIZE 和 MSIZE 一般设为相同值
如果不一致：
  小 → 大：填充（如 8 位 0xAA → 16 位 0xAAAA）
  大 → 小：可能丢失高位
```

### 5.5 PINC / MINC（地址自增）

| 位 | 名称 | 功能 |
|:--:|------|------|
| PINC | 外设地址自增 | 每次传输后外设地址 +PSIZE |
| MINC | 存储器地址自增 | 每次传输后存储器地址 +MSIZE |

```
自增步长 = 数据宽度：
  PSIZE=00（8位） → 每次地址 +1
  PSIZE=01（16位） → 每次地址 +2
  PSIZE=10（32位） → 每次地址 +4

本实验：
  传输多个数据 → 两个地址都需要自增
  PINC = 1, MINC = 1
```

### 5.6 PL（优先级）

| 值 | 优先级 |
|:--:|--------|
| 00 | 低 |
| 01 | 中 |
| 10 | 高 |
| 11 | 最高 |

### 5.7 CIRC（循环模式）

| 值 | 模式 |
|:--:|------|
| 0 | 正常模式（传输完成停止） |
| 1 | 循环模式（地址回到起始，继续传输） |

### 5.8 EN（通道使能）

| 值 | 状态 |
|:--:|------|
| 0 | 通道关闭 |
| 1 | 通道开启，开始数据传输 |

## 六、CNDTR 寄存器（数据传输数量）

### 6.1 寄存器结构

```
31:16  保留
15:0   NDT[15:0]  要传输的数据数量（0 ~ 65535）
```

### 6.2 工作机制

```
① 初始化时写入要传输的数据个数
② 每传输一个数据，NDT 自动减 1
③ 当 NDT 减到 0 时，数据传输完成

传输完成时：
  → 产生 TC（Transfer Complete）事件
  → 如果 TCIE=1，产生传输完成中断
```

---

## 七、CPAR 和 CMAR 寄存器（源和目的地址）

### 7.1 功能

| 寄存器 | 功能 | 本实验 |
|--------|------|--------|
| **CPAR** | 外设基地址 | ROM 起始地址（源） |
| **CMAR** | 存储器基地址 | RAM 起始地址（目的） |

```
存储器到存储器模式：
  CPAR = 源数据地址（ROM/Flash 中的常量数组）
  CMAR = 目标数据地址（RAM 中的变量数组）
  DIR = 0（从"外设"读取）
```

## 八、中断机制

### 8.1 三种中断事件

| 事件 | 缩写 | 触发条件 | ISR 标志位 | CCR 使能位 |
|------|:----:|----------|:----------:|:----------:|
| 传输过半 | **HT** | NDT 减到一半 | HTIF | HTIE |
| 传输完成 | **TC** | NDT 减到 0 | TCIF | TCIE |
| 传输错误 | **TE** | 访问错误 | TEIF | TEIE |

### 8.2 全局中断标志

```
GIF = Global Interrupt Flag
  → 任何一个事件（HT/TC/TE）发生都会置位
  → 统一管理标志
```

### 8.3 ISR 寄存器（中断状态，只读）

| 每个通道 4 位 | 功能 |
|:---:|------|
| GIFx | 全局中断标志 |
| TCIFx | 传输完成标志 |
| HTIFx | 传输过半标志 |
| TEIFx | 传输错误标志 |

```
ISR 只读，不能直接写入清除
需要通过 IFCR 寄存器清除
```

### 8.4 IFCR 寄存器（中断标志清除，只写）

| 位 | 功能 |
|:--:|------|
| CGIFx | 清除通道 x 全局中断标志 |
| CTCIFx | 清除通道 x 传输完成标志 |
| CHTIFx | 清除通道 x 传输过半标志 |
| CTEIFx | 清除通道 x 传输错误标志 |

```
IFCR 与 ISR 一一对应
写 1 清除对应标志位
```

## 九、时钟使能

```c
// DMA 控制器挂在 AHB 总线上
RCC->AHBENR |= RCC_AHBENR_DMA1EN;    // 开启 DMA1 时钟
```

| 总线 | DMA 控制器 | 使能位 |
|------|-----------|--------|
| AHB | DMA1 | RCC->AHBENR bit 0 |
| AHB | DMA2 | RCC->AHBENR bit 1 |

## 十、寄存器配置流程（存储器到存储器）

### 10.1 配置顺序

```
① 开启时钟
   RCC->AHBENR |= RCC_AHBENR_DMA1EN;

② 配置 CCR（通道配置）
   MEM2MEM = 1   → 存储器到存储器模式
   DIR     = 0   → 从"外设"（ROM）读
   PSIZE   = 00  → 8 位数据宽度
   MSIZE   = 00  → 8 位数据宽度
   PINC    = 1   → 外设地址自增
   MINC    = 1   → 存储器地址自增
   PL      = 10  → 高优先级
   TCIE    = 1   → 传输完成中断使能（可选）
   EN      = 0   → 暂不开启

③ 配置 CNDTR
   NDT = 传输数据个数

④ 配置 CPAR
   源地址（ROM）

⑤ 配置 CMAR
   目标地址（RAM）

⑥ 开启通道
   CCR.EN = 1

⑦ 等待传输完成
   查询 ISR.TCIF 或进入中断
```

### 10.2 本实验参数

| 参数 | 值 | 说明 |
|------|:--:|------|
| MEM2MEM | 1 | 存储器到存储器 |
| DIR | 0 | 从 ROM 读 |
| PSIZE | 00 | 8 位 |
| MSIZE | 00 | 8 位 |
| PINC | 1 | ROM 地址自增 |
| MINC | 1 | RAM 地址自增 |
| CNDTR | N | 数据个数 |
| CPAR | ROM 地址 | 源地址 |
| CMAR | RAM 地址 | 目标地址 |

## 十一、概念速查

| 概念 | 说明 |
|------|------|
| DMA | 直接存储器访问，硬件数据搬运 |
| MEM2MEM | 存储器到存储器传输模式 |
| DIR | 数据方向（外设→存储器 / 存储器→外设） |
| PSIZE/MSIZE | 每次传输的数据位宽 |
| PINC/MINC | 每次传输后地址自动增加 |
| NDT | 剩余要传输的数据数量 |
| TC | 传输完成事件 |
| HT | 传输过半事件 |
| ISR | 中断状态寄存器（只读） |
| IFCR | 中断标志清除寄存器（只写） |
| CPAR | 通道外设地址寄存器 |
| CMAR | 通道存储器地址寄存器 |
| CCR | 通道配置寄存器（核心） |

## 十二、与传统方式对比

| 对比项 | CPU 赋值 | DMA 传输 |
|--------|:--------:|:--------:|
| CPU 参与 | ✅ 全程参与 | ❌ 只负责配置 |
| 传输速度 | 受 CPU 指令周期限制 | 硬件高速传输 |
| CPU 负载 | 高 | **几乎为零** |
| 适用场景 | 少量数据 | 大量数据高速传输 |
| 编程复杂度 | 简单（b=a） | 需要配置多个寄存器 |

---

## 十三、存储器到存储器模式要点

```
⚠️ 这是 DMA 的特殊模式
  → 不查通道分配表，任意通道都可以
  → 必须设置 MEM2MEM = 1
  → 底层仍用"外设"和"存储器"两个概念
  → 将源存储器（ROM）当作"外设"
  → 目标存储器（RAM）当作"存储器"
  → DIR = 0（从"外设"读）
```

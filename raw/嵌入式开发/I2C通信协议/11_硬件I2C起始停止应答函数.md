# 硬件 I²C 寄存器实现 — 起始/停止/应答函数 笔记

---

## 一、I2C_Start() — 产生起始信号

### 1.1 核心逻辑

```
① CR1.START = 1 → 请求产生起始信号
② while 循环等待 SR1.SB = 1（起始位已发出）
③ 加入超时机制防止卡死
④ 返回 OK 或 TIMEOUT
```

### 1.2 涉及的寄存器

| 寄存器 | 位 | 操作 | 含义 |
|--------|:--:|------|------|
| **CR1** | START (bit8) | 写 1 | 请求产生起始信号 |
| **SR1** | SB (bit0) | 读取 | 起始位已发出标志（1=已发出） |

### 1.3 代码实现

```c
uint8_t I2C_Start(void)
{
    uint16_t timeout = 0xFFFF;

    /* ① 请求产生起始信号 */
    I2C2->CR1 |= I2C_CR1_START;

    /* ② 等待起始信号真正发出（SB = 1） */
    while (!(I2C2->SR1 & I2C_SR1_SB) && timeout)
    {
        timeout--;
    }

    /* ③ 返回结果 */
    if (timeout)
        return OK;       // SB = 1，起始信号已发出
    else
        return TIMEOUT;  // 超时，起始信号发出失败
}
```

### 1.4 START 位工作原理

```
CR1.START = 1
    ├── 主模式：硬件重复尝试产生起始条件，直到总线空闲
    └── 从模式：等待总线空闲后产生起始条件

成功发出后：
    → SR1.SB = 1（硬件自动置位）
    → CR1.START = 0（硬件自动清除）
```

### 1.5 超时机制

```c
while (!(I2C2->SR1 & I2C_SR1_SB) && timeout)
//      ↑ SB = 0 时继续等       ↑ timeout ≠ 0 时继续等

退出条件（满足任一）：
  ① SB = 1 → 起始信号已发出 → 返回 OK
  ② timeout = 0 → 超时 → 返回 TIMEOUT
```

| 条件 | 结果 |
|------|------|
| SB = 1（起始已发） | 退出循环，返回 OK |
| timeout 减到 0（超时） | 退出循环，返回 TIMEOUT |
| 两者同时为真 | 继续循环等待 |

## 二、I2C_Stop() — 产生停止信号

### 2.1 代码实现

```c
void I2C_Stop(void)
{
    I2C2->CR1 |= I2C_CR1_STOP;   // 设置产生停止条件
}
```

### 2.2 STOP 位工作原理

```
CR1.STOP = 1
    → 不是立即发出停止信号！
    → 等待当前字节传输完成后，自动发出停止条件
    → 相当于一个"准备停止"的配置项

主模式下：字节传输结束后发出停止信号
从模式下：当前字节传输结束后释放 SCL 和 SDA
```

### 2.3 与软件模拟的区别

| 对比项 | 软件模拟 | 硬件 I²C |
|--------|----------|----------|
| 执行方式 | 立即拉高拉低 SDA | 设置配置位，延迟生效 |
| 时机 | 手动控制 | 硬件自动在字节传输结束后执行 |
| 是否需要等待 | 无需等待 | 内部自动处理 |

## 三、I2C_EnableAck() — 使能应答

### 3.1 代码实现

```c
void I2C_EnableAck(void)
{
    I2C2->CR1 |= I2C_CR1_ACK;    // ACK 位置 1 → 使能应答
}
```

### 3.2 ACK 位工作原理

```
CR1.ACK = 1（使能）
    → 每收到一个字节后，硬件自动拉低 SDA 返回 ACK

CR1.ACK = 0（禁止）
    → 收到字节后不拉低 SDA → 等效于返回 NACK
```

### 3.3 与软件模拟的区别

| 对比项 | 软件模拟 | 硬件 I²C |
|--------|----------|----------|
| ACK | 调用后**立即**拉低 SDA | 设置配置位，收到字节后**自动**返回 |
| 本质 | 一次性的输出操作 | **配置项**，决定后续收到字节的行为 |

## 四、I2C_DisableAck() — 使能非应答

### 4.1 代码实现

```c
void I2C_DisableAck(void)
{
    I2C2->CR1 &= ~I2C_CR1_ACK;   // ACK 位清零 → 禁止应答（等效 NACK）
}
```

### 4.2 工作原理

```
CR1.ACK = 0
    → 收到字节后硬件不操作 SDA
    → SDA 保持高电平（上拉电阻拉高）
    → 等效于返回 NACK

使用场景：读取最后一个字节前调用，告知从设备停止发送
```

## 五、信号产生方式对比

### 5.1 软件模拟 vs 硬件 I²C

| 信号 | 软件模拟 | 硬件 I²C |
|------|----------|----------|
| **START** | `SDA_HIGH(); SCL_HIGH(); SDA_LOW();` | `CR1.START = 1; while(!SB);` |
| **STOP** | `SDA_LOW(); SCL_HIGH(); SDA_HIGH();` | `CR1.STOP = 1;` |
| **ACK** | `SDA_LOW(); SCL翻转; SDA_HIGH();` | `CR1.ACK = 1;` |
| **NACK** | `SDA保持高; SCL翻转;` | `CR1.ACK = 0;` |

### 5.2 硬件 I²C 控制位特性总结

| 控制位 | 类型 | 生效时机 | 是否需要等待 |
|--------|------|----------|:------------:|
| **START** | 即时触发 | 重复尝试直到总线空闲 | 需等待 SB |
| **STOP** | 配置项 | 当前字节传输结束后 | 无需等待 |
| **ACK** | 配置项 | 收到字节后自动执行 | 无需等待 |

## 六、WaitAck 被删除的原因

| 软件模拟 | 硬件 I²C |
|----------|----------|
| 需要单独调用 `I2C_WaitAck()` | 应答检测融入 `SendByte()` 和 `SendAddr()` |
| 手动读取 SDA 电平 | 通过检测 **BTF** 或 **ADDR** 标志位 |
| 返回 ACK/NACK 状态 | 返回 OK/TIMEOUT |

```
硬件 I²C 中应答的检测方式：

发送字节后：
  → while(!BTF) 等待
  → BTF = 1 说明字节传输完成（已收到 ACK）

发送地址后：
  → while(!ADDR) 等待
  → ADDR = 1 说明地址发送成功（已收到 ACK）

BTF/ADDR 标志位本身就包含了"收到应答"的含义
→ 无需单独的 WaitAck 函数
```

## 七、CR1 寄存器关键位汇总

| 位 | 名称 | 读写 | 用途 |
|:--:|------|:----:|------|
| 0 | **PE** | R/W | 外设使能 |
| 1 | SMBUS | R/W | SMBus/I²C 模式选择 |
| 8 | **START** | R/W | 产生起始条件 |
| 9 | **STOP** | R/W | 产生停止条件 |
| 10 | **ACK** | R/W | 应答使能 |
| 15 | SWRST | R/W | 软件复位 |

## 八、SR1 寄存器关键位汇总

| 位 | 名称 | 含义 | 清除方式 |
|:--:|------|------|----------|
| 0 | **SB** | 起始位已发出 | 读 SR1 + 写 DR |
| 1 | **ADDR** | 地址已发送/匹配 | 读 SR1 + 读 SR2 |
| 2 | **BTF** | 字节传输完成（收到 ACK） | 读 SR1 + 读/写 DR |
| 6 | **RXNE** | 接收数据寄存器非空 | 读 DR |
| 7 | **TXE** | 发送数据寄存器为空 | 写 DR |

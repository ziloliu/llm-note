## 二、寄存器总览

SysTick 共有 **4 个寄存器**：

| 寄存器 | 全称 | 位宽 | 功能 |
|--------|------|:----:|------|
| **CTRL** | Control and Status | 32位 | 控制 + 状态（只用低 3 位 + bit16） |
| **LOAD** | Reload Value | 32位 | 重装载值（用 bit[23:0]） |
| **VAL** | Current Value | 32位 | 当前计数值（用 bit[23:0]） |
| **CALIB** | Calibration | 32位 | 校准寄存器（只读，出厂固定） |

---

## 三、CTRL — 控制和状态寄存器

### 3.1 位定义

| 位 | 名称 | 类型 | 功能 |
|:--:|------|:----:|------|
| 0 | **ENABLE** | 控制 | 计数器使能 |
| 1 | **TICKINT** | 控制 | 中断使能 |
| 2 | **CLKSOURCE** | 控制 | 时钟源选择 |
| 16 | **COUNTFLAG** | 状态 | 计数到零标志 |

> 高 16 位中只用 bit16，低 16 位中只用 bit[2:0]，共 4 位有效。

### 3.2 三位控制位详解

#### ENABLE — 位 0（计数器使能）

```
ENABLE = 0：计数器关闭（复位默认值）
ENABLE = 1：计数器开启，开始递减计数

⚠️ 必须先配置好 LOAD，再开启 ENABLE
```

#### TICKINT — 位 1（中断使能）

```
TICKINT = 0：计数到零时 不产生中断（仅置 COUNTFLAG 标志）
TICKINT = 1：计数到零时 产生 SysTick 异常中断

之前基础篇配置：101（TICKINT = 0，不使用中断）
若要使用中断：111（TICKINT = 1）
```

#### CLKSOURCE — 位 2（时钟源选择）

```
CLKSOURCE = 0：AHB 时钟的 1/8（72MHz / 8 = 9MHz）
CLKSOURCE = 1：AHB 处理器时钟（72MHz）

复位默认值 = 0 → 默认使用 9MHz
实际使用建议 = 1 → 选择 72MHz，计时精度更高
```

### 3.3 时钟源与 RCC 时钟树的对应关系

```
RCC 时钟树中的对应位置：

  AHB 总线时钟 = 72MHz
       │
       ├──→ APB1 分频 → APB1 外设（36MHz）
       ├──→ APB2 分频 → APB2 外设（72MHz）
       ├──→ Cortex 系统时钟 ──→ 除以 8 ──→ SysTick（9MHz）
       │                      ↑
       │                   CLKSOURCE = 0 时
       │
       └──→ Cortex 系统时钟 ──→ SysTick（72MHz）
                              ↑
                           CLKSOURCE = 1 时
```

### 3.4 CTRL 配置值汇总

| 配置值（二进制） | ENABLE | TICKINT | CLKSOURCE | 含义 |
|:---------------:|:------:|:-------:|:---------:|------|
| 000 | 0 | 0 | 0 | 关闭（复位默认） |
| **101** | 1 | 0 | 1 | 开启，**不产生中断**，72MHz |
| **111** | 1 | 1 | 1 | 开启，**产生中断**，72MHz |

## 四、LOAD — 重装载值寄存器

### 4.1 位定义

| 位 | 名称 | 说明 |
|:--:|------|------|
| **[23:0]** | RELOAD | 24 位重装载值 |
| [31:24] | — | 保留 |

### 4.2 工作原理

```
① 写入初始计数值到 LOAD
② ENABLE 后，计数器从 LOAD 值开始向下递减
③ 减到 0 时：
   → COUNTFLAG = 1
   → 产生中断（若 TICKINT = 1）
   → 自动将 LOAD 的值重新装载到计数器
④ 继续递减（循环往复）

因此名为"重装载值"（Reload Value）
```

### 4.3 装载值计算

```
目标定时时间 × 时钟频率 = 装载值

示例（时钟频率 = 72MHz）：
  定时 1 μs  → 72MHz × 1μs  = 72
  定时 10 μs → 72MHz × 10μs = 720
  定时 1 ms  → 72MHz × 1ms  = 72000
  定时 10 ms → 72MHz × 10ms = 720000

24 位最大值 = 2^24 - 1 = 16,777,215
最大定时 = 16,777,215 / 72μs ≈ 233 ms（72MHz 时）
```

### 4.4 LOAD 寄存器容量

| 参数 | 数值 |
|------|------|
| 位宽 | 24 位 |
| 最大装载值 | 16,777,215（0xFFFFFF） |
| 72MHz 下最大定时 | ≈ 233 ms |
| 9MHz 下最大定时 | ≈ 1.865 s |

## 五、VAL — 当前计数值寄存器

### 5.1 位定义

| 位 | 名称 | 说明 |
|:--:|------|------|
| **[23:0]** | CURRENT | 当前计数器的值 |
| [31:24] | — | 保留 |

### 5.2 读写特性

```
读操作：返回当前计数器的值（可随时读取）
写操作：无论写入什么值，计数器都会被清零
       → 写入后 COUNTFLAG 也被清零
       → 相当于重新开始计数
```

### 5.3 使用场景

```
主要用途：读取当前计数值
  → 判断计数到哪一步了
  → 计算已经过去的时间

写入用途：
  → 立即清零计数器（重新开始计时）
  → 清除 COUNTFLAG 标志位
```

### 5.4 示例：读取剩余时间

```c
uint32_t remaining = SysTick->VAL;    // 读取当前计数值
// remaining = 当前剩余的计数周期数
// 已过去时间 = (LOAD - remaining) / 时钟频率
```

## 六、CALIB — 校准寄存器

### 6.1 特性

| 特性 | 说明 |
|------|------|
| 读写属性 | **只读** |
| 复位值 | **0x0000BB80**（特殊，非零） |
| 用途 | 出厂校准值，一般不使用 |

### 6.2 位定义

| 位 | 名称 | 说明 |
|:--:|------|------|
| [23:0] | TENMS | 10ms 时间间隔对应的参考值 |
| [30] | SKEW | 校准值是否精确（1=不精确） |
| [31] | NOREF | 是否有参考时钟（1=无参考时钟） |

> 一般无需关心此寄存器，出厂时已固定。

## 八、SysTick 工作流程图

```
                    ┌──────────┐
                    │ 配置 LOAD │  写入重装载值
                    └────┬─────┘
                         ↓
                    ┌──────────┐
                    │ 配置 CTRL │  设置 TICKINT、CLKSOURCE
                    └────┬─────┘
                         ↓
                    ┌──────────┐
                    │ ENABLE=1 │  开启计数器
                    └────┬─────┘
                         ↓
              ┌──→ 计数器从 LOAD 值开始递减
              │          ↓
              │    每个时钟周期减 1
              │          ↓
              │    计数器 == 0 ?
              │     ↓是
              │    COUNTFLAG = 1
              │     ├── TICKINT=1 → 产生 SysTick 中断
              │     └── TICKINT=0 → 仅置标志
              │          ↓
              │    自动重新装载 LOAD 值
              │          ↓
              └── 继续递减（循环往复）

关闭条件：
  ENABLE = 0 → 计数器停止
```

## 九、寄存器配置示例

### 9.1 延时函数（不使用中断，基础篇）

```c
// 配置：101 = ENABLE(1) + TICKINT(0) + CLKSOURCE(1)
SysTick->LOAD = 720 - 1;       // 10μs 装载值
SysTick->VAL  = 0;              // 清零当前值
SysTick->CTRL = 0x05;           // 101: 使能 + 无中断 + 72MHz

while (!(SysTick->CTRL & (1 << 16)));  // 等待 COUNTFLAG = 1

SysTick->CTRL = 0x00;           // 关闭定时器
```

### 9.2 使用中断（操作系统时基）

```c
// 配置：111 = ENABLE(1) + TICKINT(1) + CLKSOURCE(1)
SysTick->LOAD = 72000 - 1;     // 1ms 装载值
SysTick->VAL  = 0;              // 清零当前值
SysTick->CTRL = 0x07;           // 111: 使能 + 中断 + 72MHz

// SysTick 中断服务函数
void SysTick_Handler(void)
{
    // 每 1ms 执行一次
    // 操作系统任务调度 / 时间计数等
}
```

## 十、SysTick 定时器特点总结

| 特点 | 说明 |
|------|------|
| 位宽 | 24 位 |
| 计数方向 | 向下递减 |
| 自动重装载 | 支持（LOAD 寄存器） |
| 中断 | 支持（TICKINT 控制） |
| 时钟源 | 可选 72MHz 或 9MHz |
| 最大定时 | ≈ 233ms（72MHz）或 ≈ 1.86s（9MHz） |
| 寄存器数量 | 4 个（CTRL、LOAD、VAL、CALIB） |
| 复位状态 | 关闭（ENABLE = 0） |
| 主要用途 | 操作系统时基、延时函数、闹钟 |

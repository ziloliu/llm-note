## 四、ADC 初始化实现

### 4.1 配置步骤总览

```
① 开启 ADC1 时钟（APB2）
② ADC 时钟分频（72MHz / 6 = 12MHz）
③ GPIO 配置（PC0 模拟输入）
④ CR1：禁用扫描模式
⑤ CR2：开启连续转换、右对齐
⑥ SMPR1：配置通道 10 采样时间
⑦ SQR1：配置序列长度 L = 0（1 个通道）
⑧ SQR3：配置 SQ1 = 通道 10
⑨ CR2：配置软件触发（EXTTRIG + EXTSEL = 111）
```

### 4.2 ① 开启时钟

```c
// ADC1 连接在 APB2 高速外设总线上
RCC->APB2ENR |= RCC_APB2ENR_ADC1EN;
```

```
⚠️ GPIOC 时钟可以不开启

原因：
  → PC0 配置为模拟输入模式
  → 模拟输入只需要一根导线连通
  → 不需要施密特触发器、上下拉等数字电路
  → 这些电路不需要时钟驱动
  → 因此 GPIO 时钟可以省略

验证：不配置 GPIOC 时钟，ADC 依然正常工作
```

### 4.3 ② ADC 时钟分频

```c
// RCC->CFGR 中 ADCPRE[1:0] 配置分频系数
// 72MHz / 6 = 12MHz（< 14MHz）
RCC->CFGR |= (1 << 15);   // ADCPRE 高位 = 1
RCC->CFGR &= ~(1 << 14);  // ADCPRE 低位 = 0
// ADCPRE[1:0] = 10 → 6 分频
```

### 4.4 ③ GPIO 配置

```c
// PC0 配置为模拟输入
// CNF[1:0] = 00, MODE[1:0] = 00 → 模拟输入
GPIOC->CRL &= ~(0xF << 0);  // PC0 配置位全部清零
```

```
模拟输入模式：
  → MODE = 00（输入模式）
  → CNF = 00（模拟输入）
  → 四位全零 = 模拟输入
  → 引脚直接连通 ADC 输入，无额外电路
```

### 4.5 ④ 禁用扫描模式

```c
// CR1.SCAN = 0（单通道不需要扫描）
ADC1->CR1 &= ~ADC_CR1_SCAN;
```

```
SCAN = 0：
  → 只转换配置的第一个通道
  → 单通道模式（单曲播放）
```

### 4.6 ⑤ CR2 基本配置

```c
// 连续转换模式（单曲循环）
ADC1->CR2 |= ADC_CR2_CONT;      // CONT = 1

// 数据右对齐（默认）
ADC1->CR2 &= ~ADC_CR2_ALIGN;    // ALIGN = 0
```

### 4.7 ⑥ 采样时间配置

```c
// 通道 10 在 SMPR1 中（SMP10 = [2:0]）
// 选择 7.5 周期（001）
ADC1->SMPR1 &= ~(0x7 << 0);    // 清零 SMP10 三位
ADC1->SMPR1 |= (0x1 << 0);     // SMP10 = 001（7.5 周期）
```

```
SMPR1 位分配（通道 10~17）：
  [2:0]   = SMP10  ← 本实验
  [5:3]   = SMP11
  ...
  [23:21] = SMP17

采样时间选择：
  000 = 1.5 周期
  001 = 7.5 周期    ← 选择此项
  010 = 13.5 周期
  ...
  111 = 239.5 周期

总转换时间 = 7.5 + 12.5 = 20 个 ADC 时钟周期
```

### 4.8 ⑦ 序列长度配置

```c
// SQR1[23:20] = L（序列长度 - 1）
// L = 0 → 1 个通道
ADC1->SQR1 &= ~(0xF << 20);   // L = 0000
```

```
L 值含义：
  0000 = 1 个通道 ← 本实验
  0001 = 2 个通道
  ...
  1111 = 16 个通道

默认全零，本实验不配也可以
```

### 4.9 ⑧ 通道号配置

```c
// SQR3[4:0] = SQ1（第一个要转换的通道号）
// SQ1 = 10（通道 10）
ADC1->SQR3 &= ~(0x1F << 0);   // 清零 SQ1 五位
ADC1->SQR3 |= (10 << 0);      // SQ1 = 10
```

```
SQR3 位分配（第 1~6 个通道）：
  [4:0]   = SQ1（第 1 个转换的通道号）← 10
  [9:5]   = SQ2
  [14:10] = SQ3
  [19:15] = SQ4
  [24:20] = SQ5
  [29:25] = SQ6

SQ1 存放在 SQR3 的最低 5 位
通道号 10 = 0b01010
```

### 4.10 ⑨ 软件触发配置

```c
// 开启外部触发
ADC1->CR2 |= ADC_CR2_EXTTRIG;  // EXTTRIG = 1

// 选择软件触发（111）
ADC1->CR2 |= ADC_CR2_EXTSEL;   // EXTSEL[2:0] = 111
```

```
软件触发配置链：
  EXTTRIG = 1（开启外部触发选择）
  EXTSEL = 111（选择 SWSTART 作为触发源）
  → 之后写 SWSTART = 1 即可启动转换
```

## 五、启动转换实现

```c
void ADC1_StartConvert(void)
{
    // ① 上电唤醒
    ADC1->CR2 |= ADC_CR2_ADON;      // ADON = 1（唤醒）

    // ② 执行校准
    ADC1->CR2 |= ADC_CR2_CAL;       // CAL = 1（开始校准）

    // ③ 等待校准完成（硬件自动清零 CAL）
    while (ADC1->CR2 & ADC_CR2_CAL);

    // ④ 软件触发启动转换
    ADC1->CR2 |= ADC_CR2_SWSTART;   // SWSTART = 1
}
```

### 5.1 ADON 操作说明

```
第一次写 1（ADON 从 0→1）：
  → 从断电模式唤醒
  → ADC 上电初始化
  → 不启动转换

第二次写 1（ADON 已经为 1，再写 1）：
  → 启动规则通道 AD 转换
  → 每次写 1 都启动一次新转换
  → 但本实验使用 SWSTART 方式启动
```

### 5.2 校准操作

```
CR2.CAL 校准过程：
  ① 软件写 CAL = 1 → 开始校准
  ② 硬件执行校准（类似天平调零）
  ③ 校准完成 → 硬件自动将 CAL 清零
  ④ 判断 CAL = 0 → 校准完成

等待校准完成：
  while (ADC1->CR2 & ADC_CR2_CAL);
  → CAL = 1：校准中，继续等待
  → CAL = 0：校准完成，退出循环
```

### 5.3 SWSTART 操作

```
前提条件：
  ① EXTTRIG = 1（已配置）
  ② EXTSEL = 111（已配置）

操作：
  ADC1->CR2 |= ADC_CR2_SWSTART;  // 启动规则通道转换

注意：
  → 硬件开始转换后会自动清除 SWSTART 位
  → SWSTART 清零 ≠ 转换完成
  → 转换完成要看 SR.EOC 标志
```

## 六、读取转换结果

```c
double ADC1_ReadV(void)
{
    // 等待转换完成（EOC = 1）
    while (!(ADC1->SR & ADC_SR_EOC));

    // 读取数据寄存器（读 DR 自动清除 EOC）
    // 转换公式：V = ADC值 × 3.3 / 4095
    return ADC1->DR * 3.3 / 4095;
}
```

### 6.1 EOC 标志位

```
SR.EOC（End Of Conversion）：
  → 规则通道转换完成时由硬件置 1
  → 读取 DR 寄存器时自动清除
  → 或软件写 0 清除

判断方式：
  while (!(ADC1->SR & ADC_SR_EOC));
  → EOC = 0：转换中，继续等待
  → EOC = 1：转换完成，退出循环
```

### 6.2 SWSTART vs EOC

```
SWSTART 位：
  → 写 1 开始转换
  → 硬件立即清除（开始就清零）
  → SWSTART = 0 不代表转换完成！

EOC 位：
  → 转换完成才置 1
  → 是判断转换完成的正确标志
```

### 6.3 电压转换公式

```
V = ADC值 × 3.3 / 4095

  ADC值 = DR 中的 12 位数字量（0 ~ 4095）
  3.3   = VREF+ - VREF-（参考电压范围）
  4095  = 2^12 - 1（12 位最大值）

例：
  ADC值 = 2048 → V = 2048 × 3.3 / 4095 ≈ 1.65V
  ADC值 = 4095 → V = 3.3V
  ADC值 = 0    → V = 0V
```

## 七、主函数实现

```c
#include "stm32f10x.h"
#include "usart.h"
#include "adc.h"
#include "delay.h"
#include <stdio.h>

int main(void)
{
    USART1_Init(115200);
    Delay_init();
    ADC1_Init();

    printf("Hello World!\n");

    // 启动 AD 转换（上电 + 校准 + 软件触发）
    ADC1_StartConvert();

    while (1)
    {
        // 读取并打印电压值
        printf("V = %.2f V\n", ADC1_ReadV());
        Delay_ms(1000);              // 每秒打印一次
    }
}
```

## 九、完整配置流程

```
初始化（ADC1_Init）：
  ① RCC->APB2ENR |= ADC1EN          → 开 ADC 时钟
  ② RCC->CFGR |= ADCPRE_DIV6        → 72MHz/6 = 12MHz
  ③ GPIOC->CRL &= ~(0xF)            → PC0 模拟输入
  ④ CR1 &= ~SCAN                     → 禁用扫描（单通道）
  ⑤ CR2 |= CONT                      → 连续转换
  ⑥ CR2 &= ~ALIGN                    → 右对齐
  ⑦ SMPR1 |= 0x1                     → 通道 10 采样 7.5 周期
  ⑧ SQR1 &= ~(0xF<<20)              → L = 0（1 个通道）
  ⑨ SQR3 |= 10                       → SQ1 = 通道 10
  ⑩ CR2 |= EXTTRIG                   → 开启外部触发
  ⑪ CR2 |= EXTSEL                    → 选择软件触发（111）

启动（ADC1_StartConvert）：
  ① CR2 |= ADON                      → 上电唤醒
  ② CR2 |= CAL                       → 开始校准
  ③ while(CR2 & CAL)                 → 等待校准完成
  ④ CR2 |= SWSTART                   → 软件触发转换

读取（ADC1_ReadV）：
  ① while(!(SR & EOC))               → 等待转换完成
  ② return DR * 3.3 / 4095           → 返回电压值
```

## 十一、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 读数恒为 0 | ADC 未上电 | 先写 ADON = 1 唤醒 |
| 读数恒为 0 | 未启动转换 | CR2.SWSTART = 1 |
| 读数恒为 4095 | 通道号配错 | 检查 SQR3 中 SQ1 = 10 |
| 读数不变化 | CONT 未开启 | CR2.CONT = 1 |
| 读数不稳定 | 采样时间太短 | 增大 SMP10 值 |
| 电压值偏差大 | 未校准 | 启动前执行 CAL 校准 |
| GPIO 时钟未开 | 不影响 | 模拟输入不需要 GPIO 时钟 |
| ADC 时钟超 14MHz | 分频配错 | 确认 ADCPRE = 10（6 分频） |
| SWSTART 清零但未完成 | 误判标志 | 等 EOC，不是等 SWSTART |

## 十二、SWSTART vs EOC 重要区分

```
SWSTART 位：
  写 1 → 开始转换 → 硬件立即清除
  SWSTART = 0 只表示"已经开始"，不表示"已经完成"

EOC 位：
  转换完成 → 硬件置 1
  读 DR → 硬件清除
  EOC = 1 才表示转换完成

时序：
  写 SWSTART = 1
    ↓ （硬件立即清零 SWSTART）
    ↓ （进行 12 位逐次逼近转换）
    ↓ （转换完成）
  EOC = 1
    ↓ （读取 DR）
  EOC = 0
```

## 十三、为什么 GPIO 时钟可以省略

```
GPIO 模拟输入模式下的内部电路：

  引脚 ────────────────────→ ADC 输入
          （一根导线直连）

  不需要的电路：
    ❌ 施密特触发器（数字输入用）
    ❌ 上拉/下拉电阻
    ❌ 输出驱动器

  这些电路都关闭 → 不需要时钟驱动
  → GPIO 时钟可以不开启
  → 但开启也不影响功能
```

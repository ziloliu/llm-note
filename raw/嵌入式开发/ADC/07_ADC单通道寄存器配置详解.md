# ADC 独立模式单通道采集 — 寄存器配置详解 笔记

---

## 一、实验需求

| 项目 | 说明 |
|------|------|
| 功能 | 采集可变电阻器电压，串口打印到电脑 |
| ADC 模块 | ADC1（独立模式） |
| 通道 | 通道 10（ADC123_IN10） |
| 输入引脚 | **PC0**（第 26 号引脚） |
| 信号源 | 可变电阻器（3.3V ~ GND 之间） |
| 工作模式 | 连续转换（单曲循环） |
| 触发方式 | 软件触发 |
| 数据对齐 | 右对齐（默认） |

## 二、硬件原理

### 2.1 可变电阻电路

```
3.3V ───┬───────────── PC0 (ADC_IN10)
        │
        │   ← 旋钮位置（滑动触点）
        │     向左：电压↑（接近 3.3V）
        │     向右：电压↓（接近 0V）
        │
GND ────┘
R51 = 0Ω（相当于导线）
```

### 2.2 引脚复用

```
PC0 = GPIO 端口 C 的第 0 引脚
PC0 = ADC123_IN10
  → ADC1、ADC2、ADC3 都可使用此引脚作为通道 10
  → 三个 ADC 模块共用同一个输入通道
```

## 四、配置步骤详解

### 4.1 配置总览

```
① 开启时钟（APB2: ADC1）
② ADC 时钟分频（RCC->CFGR: ADCPRE = /6 → 12MHz）
③ 配置工作模式（CR1: SCAN, CR2: CONT/ALIGN）
④ 配置触发方式（CR2: EXTTRIG + EXTSEL = 111）
⑤ 配置采样时间（SMPR1: SMP10）
⑥ 配置序列（SQR1: L=0, SQR3: SQ1=10）
⑦ 校准（CR2: CAL）
⑧ 启动转换（CR2: ADON 或 SWSTART）
⑨ 读取数据（SR: EOC → DR）
```

## 五、CR1 控制寄存器 1

### 5.1 关键位

| 位 | 名称 | 功能 | 本实验 |
|:--:|------|------|:------:|
| 0 | AWDEN | 规则组模拟看门狗使能 | — |
| 1 | JAWDEN | 注入组模拟看门狗使能 | — |
| 8 | **SCAN** | 扫描模式 | **0**（单通道） |
| 9 | JEOCIE | 注入完成中断使能 | — |
| 10 | AWDIE | 看门狗中断使能 | — |
| 11 | EOCIE | 规则完成中断使能 | — |
| 12 | AWDSGL | 看门狗单通道模式 | — |
| 15:13 | AWDCH | 看门狗通道选择 | — |
| 25 | JAUTO | 自动注入 | — |
| 12:11 | DISCNUM | 间断模式计数 | — |
| 11 | DISCEN | 规则间断模式 | — |

### 5.2 SCAN 位（扫描模式）

```
CR1.SCAN = 0（默认）：只转换序列中第一个通道（单曲播放）
CR1.SCAN = 1：按序列顺序依次转换所有通道（顺序播放）

本实验：只有 1 个通道 → SCAN = 0（不配也行）
```

### 5.3 双模式（DUALMOD）

```
CR1.DUALMOD[3:0]：选择双 ADC 工作模式

  → 可选两个 ADC 模块配合工作
  → 本实验使用独立模式，不涉及双模式
  → 只用一个 ADC1 即可
```

## 六、CR2 控制寄存器 2（核心）

### 6.1 关键位总览

| 位 | 名称 | 功能 | 本实验值 |
|:--:|------|------|:--------:|
| 0 | **ADON** | ADC 开关/启动 | 先 1 上电，再 1 启动 |
| 1 | **CONT** | 连续转换模式 | **1**（开启） |
| 2 | **CAL** | 校准启动 | 写 1 启动，硬件清零 |
| 3 | RSTCAL | 校准复位 | — |
| 11 | **ALIGN** | 数据对齐 | **0**（右对齐） |
| 12 | DMA | DMA 模式使能 | — |
| 17 | **EXTTRIG** | 外部触发使能 | **1** |
| 19:17 | **EXTSEL** | 触发源选择 | **111**（SWSTART） |
| 22 | **SWSTART** | 规则组软件启动 | 写 1 启动 |
| 21 | JSWSTART | 注入组软件启动 | — |

### 6.2 工作模式配置

**CONT 连续转换（单曲循环）**
```c
// CR2.CONT = 1：转换完成后自动重新开始
ADC1->CR2 |= ADC_CR2_CONT;
```

**ALIGN 数据对齐（右对齐）**
```c
// CR2.ALIGN = 0：右对齐（默认，不配也行）
ADC1->CR2 &= ~ADC_CR2_ALIGN;
```

### 6.3 ADON 双重作用

```
第一次写 1（ADON 从 0→1）：
  → 从断电模式唤醒（上电）
  → ADC 初始化准备
  → 不启动转换

第二次写 1（ADON 已经为 1，再写 1）：
  → 启动规则通道 AD 转换
  → 每次写 1 都启动一次新转换

⚠️ ADON 方式只能启动规则组，不能启动注入组
```

### 6.4 校准（CAL）

```
CR2.CAL 校准过程：
  ① 软件写 CAL = 1 → 开始校准（类似天平调零）
  ② 硬件执行校准
  ③ 校准完成 → 硬件自动将 CAL 清零

等待校准完成：
  while (ADC1->CR2 & ADC_CR2_CAL);
  → CAL = 1：校准中
  → CAL = 0：校准完成
```

### 6.5 触发方式配置

**配置链：EXTTRIG → EXTSEL → SWSTART**

```
步骤 ①：EXTTRIG = 1（开启外部触发选择）
  → 相当于打开"与门"

步骤 ②：EXTSEL[2:0] = 111（选择软件触发）
  → 八选一多路选择器选择 SWSTART

步骤 ③：SWSTART = 1（启动转换）
  → 按下"开始按钮"

三者都配置完成后，写 SWSTART = 1 即可启动规则组转换
```

**触发源选择表（EXTSEL[2:0]）**

| EXTSEL | 触发源 |
|:------:|--------|
| 000 | TIM1 CC1 |
| 001 | TIM1 CC2 |
| 010 | TIM1 CC3 |
| 011 | TIM2 CC2 |
| 100 | TIM3 TRGO |
| 101 | TIM4 CC4 |
| 110 | EXTI 线 11 |
| **111** | **SWSTART（软件触发）** |

**注入组对应配置**

```
规则组：EXTTRIG + EXTSEL + SWSTART
注入组：JEXTTRIG + JEXTSEL + JSWSTART

注入组触发源 111 → JSWSTART
```

### 6.6 ADON vs SWSTART 启动方式对比

| 方式 | 操作 | 适用 | 限制 |
|------|------|:----:|------|
| **ADON 再写 1** | `CR2 \|= ADON` | 规则组 | 只能规则组 |
| **SWSTART** | 配 EXTTRIG=1, EXTSEL=111, SWSTART=1 | 规则+注入 | 需要额外配置 |

## 七、SMPR 采样时间寄存器

### 7.1 寄存器分配

```
SMPR2[29:0]：通道 0~9（每个通道 3 位，共 30 位）
  [2:0]   = SMP0
  [5:3]   = SMP1
  ...
  [29:27] = SMP9

SMPR1[23:0]：通道 10~17（每个通道 3 位，共 24 位）
  [2:0]   = SMP10  ← 本实验（通道 10）
  [5:3]   = SMP11
  ...
  [23:21] = SMP17
```

### 7.2 采样时间选择

| SMP[2:0] | 采样周期 | 总转换时间（12MHz） |
|:--------:|:--------:|:------------------:|
| **000** | **1.5 周期** | (1.5+12.5)/12MHz ≈ 1.17μs |
| **001** | **7.5 周期** | (7.5+12.5)/12MHz ≈ 1.67μs |
| **010** | **13.5 周期** | (13.5+12.5)/12MHz ≈ 2.17μs |
| 011 | 28.5 周期 | (28.5+12.5)/12MHz ≈ 3.42μs |
| 100 | 41.5 周期 | (41.5+12.5)/12MHz ≈ 4.50μs |
| 101 | 55.5 周期 | (55.5+12.5)/12MHz ≈ 5.67μs |
| 110 | 71.5 周期 | (71.5+12.5)/12MHz ≈ 7.00μs |
| 111 | 239.5 周期 | (239.5+12.5)/12MHz ≈ 21.0μs |

```c
// 通道 10 采样时间 = 7.5 周期
ADC1->SMPR1 &= ~(0x7 << 0);   // 清零 SMP10
ADC1->SMPR1 |= (0x1 << 0);    // SMP10 = 001
```

### 7.3 总转换时间公式

```
总转换时间 = 采样时间 + 12.5 个 ADC 时钟周期

  采样时间：可配置（1.5 ~ 239.5 周期）
  12.5 周期：12 位逐次逼近 + 0.5 周期余量

默认（SMP=000, 14MHz）：
  1.5 + 12.5 = 14 周期 → 14/14MHz = 1μs
```

## 八、SQR 规则序列寄存器

### 8.1 三个寄存器排布

```
80 位 = 16 个通道 × 5 位/通道 + 4 位 L

SQR3[32位]：SQ1 ~ SQ6（第 1~6 个转换通道）
  [4:0]   = SQ1（第 1 个要转换的通道号）
  [9:5]   = SQ2
  [14:10] = SQ3
  [19:15] = SQ4
  [24:20] = SQ5
  [29:25] = SQ6
  [31:30] = 保留

SQR2[32位]：SQ7 ~ SQ12
  [4:0]   = SQ7
  ...
  [29:25] = SQ12
  [31:30] = 保留

SQR1[32位]：SQ13 ~ SQ16 + L
  [4:0]   = SQ13
  [9:5]   = SQ14
  [14:10] = SQ15
  [19:15] = SQ16
  [23:20] = L（序列长度）
  [31:24] = 保留
```

### 8.2 L 值（序列长度）

| L[3:0] | 实际通道数 | 说明 |
|:------:|:---------:|------|
| 0000 | **1** | 本实验 |
| 0001 | 2 | — |
| ... | ... | — |
| 1111 | 16 | 最大 |

```
实际通道数 = L + 1
本实验：L = 0 → 1 个通道（通道 10）
```

### 8.3 通道号位宽

```
每个通道号 = 5 位（能表示 0~31）
  → 通道号范围：0~17（共 18 个通道）
  → 5 位最多表示 31，足够覆盖
  → 4 位最多表示 15，不够（需要 0~17）

为什么 SQR 需要三个寄存器：
  16 通道 × 5 位 = 80 位
  一个 32 位寄存器存不下
  需要 3 个 32 位寄存器（96 位 > 80 位）
```

### 8.4 配置顺序

```
一般从 SQR3 开始配置（倒序排布）：
  SQR3 → SQ1~SQ6（第 1~6 个转换通道）
  SQR2 → SQ7~SQ12（第 7~12 个转换通道）
  SQR1 → SQ13~SQ16 + L（最后 4 个通道 + 长度）

本实验：
  SQR1.L = 0（1 个通道）
  SQR3.SQ1 = 10（通道 10）
```

### 8.5 JSQR（注入组序列寄存器）

```
只有 1 个 JSQR 寄存器（32 位够用）：
  → 注入组最多 4 个通道
  → 每个通道 5 位 → 4 × 5 = 20 位
  → 加上 2 位长度 → 22 位
  → 32 位寄存器足够
```

## 九、状态寄存器（SR）

| 位 | 名称 | 说明 | 清除方式 |
|:--:|------|------|---------|
| 0 | **AWD** | 模拟看门狗事件标志 | 软件写 0 |
| 1 | **EOC** | 规则通道转换完成 | **读 DR** 或软件写 0 |
| 2 | **JEOC** | 注入通道转换完成 | 软件写 0 |
| 3 | **STRT** | 规则通道开始标志 | 软件写 0 |
| 4 | **JSTRT** | 注入通道开始标志 | 软件写 0 |

```
EOC（End Of Conversion）：
  → 规则通道转换完成时硬件置 1
  → 注入通道转换完成时也会置 1（通用）
  → 读取 DR 寄存器时自动清除
  → 或软件写 0 清除

JEOC（End Of Injected Conversion）：
  → 仅注入通道转换完成时置 1

AWD（Analog WatchDog）：
  → 电压超出阈值范围时置 1
```

## 十、HTR / LTR（模拟看门狗阈值）

```
HTR（高阈值寄存器）：12 位上限
LTR（低阈值寄存器）：12 位下限

转换结果 > HTR 或 < LTR → AWD 事件

可开启中断：CR1.AWDIE = 1
可指定单一通道：CR1.AWDSGL = 1 + CR1.AWDCH = 通道号
```

---

## 十一、JOFR（注入组偏移量寄存器）

```
JOFR1 ~ JOFR4：4 个 12 位寄存器
  → 每个注入通道对应一个偏移量
  → 转换结果 - 偏移量 = 最终值
  → 结果可能为负数（需要符号位）
  → 注入组最多 4 个通道 → 4 个偏移量寄存器
```

---

## 十二、数据寄存器（DR / JDR）

```
DR（规则通道数据寄存器）：
  → 16 位，只有 1 个
  → 多通道共用 → 会被覆盖
  → 及时读取或用 DMA

JDR（注入通道数据寄存器）：
  → 16 位，有 4 个（JDR1~JDR4）
  → 一一对应，不会覆盖
```

## 十三、特殊工作模式

### 13.1 自动注入（JAUTO）

```
CR1.JAUTO = 1：
  → 规则通道扫描完成后
  → 自动转换注入通道组
  → 相当于歌单自动切换
```

### 13.2 间断模式（DISCEN）

```
CR1.DISCEN = 1：
  → 规则通道序列不完整扫描
  → 每次只转换 N 个通道就停止
  → N 由 CR1.DISCNUM[2:0] 配置
  → 需要再次触发才能继续
```

### 13.3 模拟看门狗单通道模式（AWDSGL）

```
CR1.AWDSGL = 1：
  → 看门狗只监控一个通道
  → 通道号由 CR1.AWDCH[4:0] 指定

CR1.AWDSGL = 0：
  → 看门狗监控所有规则通道
```

## 十五、完整配置代码

```c
void ADC1_Init(void)
{
    // ① 开启 ADC1 时钟
    RCC->APB2ENR |= RCC_APB2ENR_ADC1EN;

    // ② ADC 时钟分频：72MHz / 6 = 12MHz
    RCC->CFGR |= (1 << 15);
    RCC->CFGR &= ~(1 << 14);

    // ③ PC0 模拟输入
    GPIOC->CRL &= ~(0xF << 0);

    // ④ 工作模式
    ADC1->CR1 &= ~ADC_CR1_SCAN;       // SCAN = 0（单通道）
    ADC1->CR2 |= ADC_CR2_CONT;        // CONT = 1（连续转换）
    ADC1->CR2 &= ~ADC_CR2_ALIGN;      // ALIGN = 0（右对齐）

    // ⑤ 采样时间：通道 10，7.5 周期
    ADC1->SMPR1 &= ~(0x7 << 0);
    ADC1->SMPR1 |= (0x1 << 0);

    // ⑥ 序列配置
    ADC1->SQR1 &= ~(0xF << 20);       // L = 0（1 个通道）
    ADC1->SQR3 &= ~(0x1F << 0);
    ADC1->SQR3 |= (10 << 0);          // SQ1 = 通道 10

    // ⑦ 触发方式：软件触发
    ADC1->CR2 |= ADC_CR2_EXTTRIG;     // 开启外部触发
    ADC1->CR2 |= ADC_CR2_EXTSEL;      // EXTSEL = 111

    // ⑧ 上电唤醒
    ADC1->CR2 |= ADC_CR2_ADON;

    // ⑨ 校准
    ADC1->CR2 |= ADC_CR2_CAL;
    while (ADC1->CR2 & ADC_CR2_CAL);

    // ⑩ 启动转换
    ADC1->CR2 |= ADC_CR2_SWSTART;
}
```

## 十六、SWSTART vs EOC 重要区分

```
SWSTART 位：
  写 1 → 开始转换 → 硬件立即清除
  SWSTART = 0 只表示"已经开始"，不表示"已经完成"

EOC 位：
  转换完成 → 硬件置 1
  读 DR → 硬件清除
  EOC = 1 才表示转换完成

时序：
  写 SWSTART = 1
    ↓ （硬件立即清零 SWSTART）
    ↓ （12 位逐次逼近转换中...）
    ↓ （转换完成）
  EOC = 1
    ↓ （读取 DR）
  EOC = 0
```

## 十七、CR2 配置位分类

```
CR2 配置分类：

工作模式：
  → CONT（连续转换）
  → ALIGN（数据对齐）

校准相关：
  → CAL（启动校准，硬件清零）
  → RSTCAL（校准复位）

DMA 相关：
  → DMA（DMA 使能）

触发相关：
  → EXTTRIG（外部触发使能）
  → EXTSEL[2:0]（触发源选择）
  → SWSTART（规则组启动）
  → JSWSTART（注入组启动）

电源控制：
  → ADON（上电唤醒 + 启动转换）
```

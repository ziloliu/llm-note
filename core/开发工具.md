## 一、VSCode 变量机制

### ${ExecutableName} 变量

`${ExecutableName}` 代表"当前可执行文件名称（不含扩展名）"，常用于嵌入式项目中生成同名的 `.hex` 固件文件。

**注意**：VSCode 本身没有名为 `ExecutableName` 的预定义变量，它可能来自以下来源：

- **自定义输入变量**（最常见）— 在 `tasks.json` 里通过 `inputs` 自定义
  ```json
  "inputs": [
    {
      "id": "ExecutableName",
      "type": "promptString",
      "description": "输入可执行文件名"
    }
  ]
  ```
  引用时用 `${input:ExecutableName}`，但有些模板直接写成 `${ExecutableName}`（需插件支持）

- **环境变量** — 系统或终端里设置了 `ExecutableName`，写成 `${env:ExecutableName}`

- **扩展注入的变量** — CMake Tools、PlatformIO 等扩展可能在上下文中提供类似变量

- **Shell / Makefile 变量** — 任务直接执行 Shell 命令时，由 Shell 替换

## 二、.hex 文件格式

`.hex` 文件是嵌入式开发中常见的**固件烧录文件**，全称 Intel HEX。

- **本质**：把编译生成的机器码用 ASCII 十六进制文本表示，并带有地址信息
- **用途**：方便烧录器/调试器（如 J-Link、ST-Link、OpenOCD）写入单片机

**典型使用场景**：嵌入式 C/C++ 项目中，先编译出 `.elf`，再通过 `objcopy` 转换为 `.hex`

```bash
arm-none-eabi-objcopy -O ihex ${ExecutableName}.elf ${ExecutableName}.hex
```

## 三、更推荐的替代方案

可以直接利用 VSCode 的内置变量，不必自己另造 `ExecutableName`。

**推荐**：使用 `${fileBasenameNoExtension}`，自动取当前文件名（不含扩展名），适合单文件项目

```json
"command": "arm-none-eabi-objcopy -O ihex ${fileBasenameNoExtension}.elf ${fileBasenameNoExtension}.hex"
```

### 嵌入式项目常用内置变量

| 变量 | 示例值 | 说明 |
|------|--------|------|
| `${fileBasenameNoExtension}` | `main` | 当前文件名，不含扩展名 |
| `${workspaceFolder}` | `/home/user/stm32-project` | 工作区根目录 |
| `${file}` | `/home/user/stm32-project/main.c` | 当前文件完整路径 |
| `${fileDirname}` | `/home/user/stm32-project` | 当前文件所在目录 |
| `${fileBasename}` | `main.c` | 当前文件名，含扩展名 |
| `${input:ExecutableName}` | 用户输入值 | 自定义 promptString 输入 |

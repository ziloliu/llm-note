# ADC 工作原理 — 逐次逼近型 笔记

---

## 一、ADC 常见类型对比

| 类型 | 原理 | 速度 | 精度 | 集成度 | 典型应用 |
|------|------|:----:|:----:|:------:|---------|
| **逐次逼近型（SAR）** | 逐位比较 | **中等** | **中等** | **高** | STM32、通用采集 |
| **Sigma-Delta（Σ-Δ）** | 差值累加积分 | 慢 | **高** | 中等 | 高精度测量、音频 |
| **双积分型** | 两次积分 | 慢 | **高** | 低 | 高抗干扰测量 |
| **Flash（闪速）型** | 并行比较 | **极快** | **低** | 低 | 高速采集、视频 |

### 1.1 各类型核心特点

```
逐次逼近型（SAR）：
  → 二分法逐位比较
  → 速度与精度均衡
  → STM32 选用此类型

Sigma-Delta 型：
  → 采样差值 → 累加积分
  → 高精度、抗干扰强
  → 转换速度慢

Flash 型：
  → 多个比较器并行工作
  → 所有位同时得出
  → 速度极快，但精度低、电路复杂
```

### 1.2 STM32 选择逐次逼近型的原因

```
精度够用：12 位分辨率
速度适中：1 MSPS（每秒百万次采样）
集成度高：电路规模合理，适合片上集成

→ 精度与速度之间的最佳平衡
```

## 二、逐次逼近型 ADC 工作原理

### 2.1 核心类比：天平称重

```
逐次逼近 = 天平称重

天平称重：             ADC 转换：
  左盘：待测物体          正端：待测电压 Ux
  右盘：砝码             负端：DAC 输出电压
  操作：逐个加减砝码       操作：逐位判断 0/1
  结果：平衡时读取砝码值    结果：逐位确定二进制数
```

### 2.2 天平称重过程

```
砝码组：5g, 4g, 3g, 2g, 1g（共 15g）
待测物体：6g

步骤：
  ① 放 5g → 不够重 → 保留 5g
  ② 放 4g → 超重了 → 取走 4g
  ③ 放 3g → 超重了 → 取走 3g
  ④ 放 2g → 超重了 → 取走 2g
  ⑤ 放 1g → 刚好平衡 → 保留 1g

结果：5g + 1g = 6g ✅

原则：先大后小，逐个尝试，大的保留，小的调整
```

## 三、逐次逼近型 ADC 结构框图

```
                    ┌──────────────────────────────┐
                    │        逻辑控制电路           │
                    │  （控制逐位判断：保留/清除）    │
                    └─────┬──────────────┬─────────┘
                          │              │
                     控制信号          比较结果
                          ↓              ↑
   VREF ──→ ┌────────────────────┐   ┌───────────┐
            │   8位DAC转换器      │   │           │
            │ (数字量→模拟量)     │   │   比较器   │──→ 输出
            └────────┬───────────┘   │           │
                     │               └─────┬─────┘
                     ↓                     ↑
              ┌──────────────┐        Ux（待测电压）
              │ 8位寄存器     │
              │(存储当前砝码)  │
              └──────────────┘
                     ↓
              输出缓冲区 → 转换结果
```

### 3.1 核心组件说明

| 组件 | 功能 | 类比 |
|------|------|------|
| **比较器** | 比较待测电压与 DAC 输出 | 天平 |
| **8 位寄存器** | 存储当前二进制砝码值 | 砝码组 |
| **DAC 转换器** | 将数字量转为模拟电压 | 砝码重量计算 |
| **逻辑控制电路** | 控制逐位判断保留/清除 | 操作砝码的人 |
| **VREF** | 参考电压，确定量程 | 砝码最大量程 |
| **输出缓冲区** | 存储最终结果 | 称量结果 |

## 四、比较器工作原理

```
比较器有两个输入端和一个输出端：

  正端（+）：待测电压 Ux
  负端（-）：DAC 输出电压（砝码电压）

比较结果：
  Ux > DAC 输出 → 正端更大 → 输出高电平（1）
                  → 意味着砝码偏小 → 保留当前砝码，继续试下一位

  Ux < DAC 输出 → 负端更大 → 输出低电平（0）
                  → 意味着砝码偏大 → 去除当前砝码，继续试下一位

  Ux = DAC 输出 → 平衡     → 当前值即为转换结果
```

## 五、8 位寄存器与砝码对应关系

### 5.1 位权砝码表

| 位 | 二进制 | 位权（砝码重量） | 对应电压 |
|:--:|:------:|:--------------:|:--------:|
| Bit 7 | 1000 0000 | **128** | 128V |
| Bit 6 | 0100 0000 | **64** | 64V |
| Bit 5 | 0010 0000 | **32** | 32V |
| Bit 4 | 0001 0000 | **16** | 16V |
| Bit 3 | 0000 1000 | **8** | 8V |
| Bit 2 | 0000 0100 | **4** | 4V |
| Bit 1 | 0000 0010 | **2** | 2V |
| Bit 0 | 0000 0001 | **1** | 1V |

### 5.2 与天平砝码的区别

```
天平砝码：5, 4, 3, 2, 1（递减 1）
ADC 砝码：128, 64, 32, 16, 8, 4, 2, 1（递减一半）

ADC 砝码是 2 的幂次递减 → 每次砍一半 → 二分法
```

### 5.3 模拟电压转换

```
假设 VREF = 255V（为了方便计算），8 位分辨率

数字量 D（0~255）→ 模拟电压 V = D × 255/255 = D（伏）

寄存器值 = 0b10000000 (128) → DAC 输出 = 128V
寄存器值 = 0b01000000 (64)  → DAC 输出 = 64V
寄存器值 = 0b00100000 (32)  → DAC 输出 = 32V
...
```

```
├──────┼──────────┼──────────┼──────────┼──────────┼──────────┤

│      │          │          │          │ 输出 1   │ Bit5 = 1 │
├──────┼──────────┼──────────┼──────────┼──────────┼──────────┤

│      │          │          │          │ 输出 0   │ Bit4 = 0 │
├──────┼──────────┼──────────┼──────────┼──────────┼──────────┤
│  5   │ 00101000 │    40    │   40V    │ 33 < 40  │ 砝码偏大  │
│      │          │          │          │ 输出 0   │ Bit3 = 0 │
├──────┼──────────┼──────────┼──────────┼──────────┼──────────┤
│  6   │ 00100100 │    36    │   36V    │ 33 < 36  │ 砝码偏大  │
│      │          │          │          │ 输出 0   │ Bit2 = 0 │
├──────┼──────────┼──────────┼──────────┼──────────┼──────────┤
│  7   │ 00100010 │    34    │   34V    │ 33 < 34  │ 砝码偏大  │

└──────┴──────────┴──────────┴──────────┴──────────┴──────────┘

最终结果：0b00100001 = 33 → 对应 33V ✅
```

### 6.2 逐步逼近过程

```
取值范围变化（二分法）：

初始范围：0V ~ 255V
    ↓ Bit7=0（128V 偏大）
范围：0V ~ 127V
    ↓ Bit6=0（64V 偏大）
范围：0V ~ 63V
    ↓ Bit5=1（32V 偏小）
范围：32V ~ 63V
    ↓ Bit4=0（48V 偏大）
范围：32V ~ 47V
    ↓ Bit3=0（40V 偏大）
范围：32V ~ 39V
    ↓ Bit2=0（36V 偏大）
范围：32V ~ 35V
    ↓ Bit1=0（34V 偏大）
范围：32V ~ 33V
    ↓ Bit0=1（33V 刚好）
结果：33V ✅

每次范围缩小一半 → 8 次比较后确定结果
```

## 七、逐次逼近型 ADC 工作流程

```
① 初始化：8 位寄存器清零，准备最高位（Bit7）

② 从最高位开始，将当前位 置 1

③ DAC 将寄存器值转换为模拟电压

④ 比较器比较待测电压与 DAC 输出：
   → 高电平（Ux > DAC）：砝码偏小，保留当前位 = 1
   → 低电平（Ux < DAC）：砝码偏大，清除当前位 = 0

⑤ 移到下一位，重复 ②③④

⑥ 所有位判断完毕，寄存器中的值即为转换结果

时间复杂度：n 位需要 n 次比较（本例 8 位 = 8 次）
```

## 八、从 8 位推广到 12 位（STM32）

```
STM32 的 ADC 为 12 位分辨率：

  VREF = 3.3V
  分辨率 = 2^12 = 4096 级
  最小变化量 = 3.3V / 4096 ≈ 0.806mV

转换过程：
  → 从 Bit11（最高位）开始
  → 逐位判断到 Bit0（最低位）
  → 共需 12 次比较
  → 每次范围缩小一半

例：输入电压 2.65V
  数字量 D = 2.65 × 4096 / 3.3 ≈ 3289
  二进制：0b110011011001
```

## 九、关键公式

```
① 分辨率：离散级别数 = 2^n
   n = 8  → 256 级
   n = 12 → 4096 级

② 最小分辨率（量化步长）：
   ΔV = VREF / 2^n
   n = 12, VREF = 3.3V → ΔV ≈ 0.806mV

③ 数字量 → 电压：
   V = D × VREF / 2^n

④ 电压 → 数字量：
   D = V × 2^n / VREF

⑤ 逐次逼近比较次数：
   N = n（n 位需要 n 次比较）
```

## 十一、三种 ADC 类型核心差异

```
逐次逼近型（SAR）：
  1 个比较器 + 1 个 DAC
  逐位比较，n 次得出结果
  → 速度与精度均衡 ⭐

Sigma-Delta 型：
  1 个比较器 + 积分器 + 数字滤波器
  通过累积差值提高精度
  → 高精度、低速度

Flash 型：
  2^n - 1 个比较器并行工作
  1 个时钟周期得出结果
  → 极高速、低精度、高成本
```

## 十二、概念速查

| 概念 | 说明 |
|------|------|
| 逐次逼近 | 从高位到低位逐位比较确定值 |
| 比较器 | 比较两个输入电压大小，输出高低电平 |
| DAC | 数字模拟转换器，将寄存器值转为模拟电压 |
| VREF | 参考电压，确定 ADC 量程范围 |
| 量化步长 | 最小分辨率，ΔV = VREF / 2^n |
| 位权 | 每一位对应的权重（2 的幂次） |
| 逻辑控制电路 | 控制逐位判断，决定保留或清除当前位 |
| 输出缓冲区 | 存储最终转换结果 |
| 二分法 | 每次比较将搜索范围缩小一半 |
| 天平类比 | 左盘=待测电压，右盘=DAC输出，砝码=位权 |

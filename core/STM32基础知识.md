## 一、DMA（直接存储器访问）

### 1.1 基本概念

DMA（Direct Memory Access）是一种硬件数据传输通道，用于替代 CPU 执行数据搬运任务。其核心优势是解放 CPU，实现高速数据传输。

**传统方式 vs DMA 方式**：
- **传统方式**：CPU 参与整个数据搬运过程，占用 CPU 时间和数据总线
- **DMA 方式**：CPU 只负责配置，之后完全由 DMA 硬件自动完成，CPU 可以处理其他任务

📄 [原文09: DMA原理与存储器到存储器传输](09_DMA原理与存储器到存储器传输.md)

### 1.2 STM32 的 DMA 控制器

STM32 有两个 DMA 控制器：
- **DMA1**：7 个通道，挂在 AHB 总线
- **DMA2**：5 个通道，挂在 AHB 总线

DMA 控制器是系统核心部件，挂在 AHB 总线上，不是外设。

**通道分配特点**：
- 外设 ↔ 存储器：固定分配（查表确定）
- 存储器 → 存储器：任意通道（特殊模式）

📄 [原文09: DMA原理与存储器到存储器传输](09_DMA原理与存储器到存储器传输.md)

### 1.3 存储器到存储器传输

**传输方向**：ROM（Flash）→ RAM（SRAM）

**限制**：
- ✅ ROM → RAM（可以）
- ❌ RAM → ROM（不可以，Flash 需要特定协议写入）

**为什么用 DMA 而不是赋值语句**：
- CPU 执行 `b = a` 需要多个步骤，占用 CPU 和数据总线
- DMA 方式在 ROM 和 RAM 之间建立高速通路，数据自动搬运，CPU 完全不参与

**本实验选择**：使用 DMA1 通道 1（任意通道都可以），因为存储器到存储器模式不查通道分配表。

📄 [原文09: DMA原理与存储器到存储器传输](09_DMA原理与存储器到存储器传输.md)

### 1.4 关键寄存器详解

#### 四个核心寄存器（每个通道独立）

| 寄存器 | 全称 | 功能 |
|--------|------|------|
| **CCR** | Channel Configuration Register | 通道配置（核心） |
| **CNDTR** | Channel Number of Data to Transfer | 数据传输数量 |
| **CPAR** | Channel Peripheral Address Register | 外设地址 |
| **CMAR** | Channel Memory Address Register | 存储器地址 |

#### 两个共用寄存器（所有通道共享）

| 寄存器 | 全称 | 功能 |
|--------|------|------|
| **ISR** | Interrupt Status Register | 中断状态（只读） |
| **IFCR** | Interrupt Flag Clear Register | 中断标志清除（只写） |

📄 [原文09: DMA原理与存储器到存储器传输](09_DMA原理与存储器到存储器传输.md)

### 1.5 CCR 寄存器详解（通道配置寄存器）

CCR 寄存器是 DMA 配置的核心，包含以下重要位域：

| 位 | 名称 | 功能 |
|:--:|------|------|
| 0 | **EN** | 通道使能 |
| 1 | **TCIE** | 传输完成中断使能 |
| 2 | **HTIE** | 传输过半中断使能 |
| 3 | **TEIE** | 传输错误中断使能 |
| 4 | **DIR** | 数据传输方向 |
| 5 | **CIRC** | 循环模式 |
| 6 | **PINC** | 外设地址自增 |
| 7 | **MINC** | 存储器地址自增 |
| 9:8 | **PSIZE** | 外设数据宽度 |
| 11:10 | **MSIZE** | 存储器数据宽度 |
| 13:12 | **PL** | 优先级 |
| 14 | **MEM2MEM** | 存储器到存储器模式 |

**关键位域说明**：

- **MEM2MEM**：存储器到存储器模式的特殊标志位，必须置 1
- **DIR**：数据传输方向，0 表示从外设读，1 表示向外设写
- **PSIZE/MSIZE**：数据宽度，00=8位，01=16位，10=32位
- **PINC/MINC**：地址自增，每次传输后地址自动增加
- **PL**：优先级，00=低，01=中，10=高，11=最高

📄 [原文09: DMA原理与存储器到存储器传输](09_DMA原理与存储器到存储器传输.md)

### 1.6 寄存器配置流程（存储器到存储器）

配置 DMA 存储器到存储器传输的步骤：

1. **开启时钟**：`RCC->AHBENR |= RCC_AHBENR_DMA1EN;`
2. **配置 CCR**：
   - MEM2MEM = 1（存储器到存储器模式）
   - DIR = 0（从"外设"读）
   - PSIZE = 00（8 位数据宽度）
   - MSIZE = 00（8 位数据宽度）
   - PINC = 1（外设地址自增）
   - MINC = 1（存储器地址自增）
   - PL = 10（高优先级）
   - TCIE = 1（传输完成中断使能，可选）
   - EN = 0（暂不开启）
3. **配置 CNDTR**：设置传输数据个数
4. **配置 CPAR**：设置源地址（ROM）
5. **配置 CMAR**：设置目标地址（RAM）
6. **开启通道**：CCR.EN = 1
7. **等待传输完成**：查询 ISR.TCIF 或进入中断

📄 [原文09: DMA原理与存储器到存储器传输](09_DMA原理与存储器到存储器传输.md)

### 1.7 中断机制

DMA 支持三种中断事件：

| 事件 | 缩写 | 触发条件 | ISR 标志位 | CCR 使能位 |
|------|:----:|----------|:----------:|:----------:|
| 传输过半 | **HT** | NDT 减到一半 | HTIF | HTIE |
| 传输完成 | **TC** | NDT 减到 0 | TCIF | TCIE |
| 传输错误 | **TE** | 访问错误 | TEIF | TEIE |

**全局中断标志**：GIF = Global Interrupt Flag，任何一个事件发生都会置位。

**ISR 寄存器**：只读，不能直接写入清除。
**IFCR 寄存器**：只写，写 1 清除对应标志位。

📄 [原文09: DMA原理与存储器到存储器传输](09_DMA原理与存储器到存储器传输.md)

### 1.8 使用要点

**存储器到存储器模式要点**：
- 这是 DMA 的特殊模式，不查通道分配表，任意通道都可以
- 必须设置 MEM2MEM = 1
- 底层仍用"外设"和"存储器"两个概念
- 将源存储器（ROM）当作"外设"，目标存储器（RAM）当作"存储器"
- DIR = 0（从"外设"读）

**与传统方式对比**：

| 对比项 | CPU 赋值 | DMA 传输 |
|--------|:--------:|:--------:|
| CPU 参与 | ✅ 全程参与 | ❌ 只负责配置 |
| 传输速度 | 受 CPU 指令周期限制 | 硬件高速传输 |
| CPU 负载 | 高 | **几乎为零** |
| 适用场景 | 少量数据 | 大量数据高速传输 |
| 编程复杂度 | 简单（b=a） | 需要配置多个寄存器 |

# 高级定时器寄存器详解 — RCR、BDTR 及新增功能 笔记

---

## 一、实验需求

| 项目 | 说明 |
|------|------|
| 定时器 | **TIM1**（高级定时器） |
| 功能 | 输出 **5 个周期**的 PWM 方波 |
| PWM 频率 | **2Hz**（周期 0.5s） |
| 占空比 | 50%（高电平关断，低电平点亮） |
| 输出引脚 | **PA8**（TIM1_CH1） |
| 控制对象 | **LED3**（低电平点亮） |
| 关键技术 | **RCR（重复计数器）** |
| 总时长 | 5 × 0.5s = **2.5 秒** |

## 二、RCR（重复计数器）

### 2.1 寄存器结构

```
RCR 寄存器（16位）：
  ┌────────────────────────────────────┐
  │  15:8  │  7:0                      │
  │  保留   │  REP（重复计数值）          │
  └────────┴──────────────────────────┘

REP 取值范围：0 ~ 255（8 位）
实际周期数 = RCR + 1（最大 256）
```

### 2.2 RCR 工作原理

```
普通定时器（无 RCR）：
  CNT 溢出 → 立即产生更新事件

高级定时器（有 RCR）：
  CNT 溢出 → RCR 减 1（不产生更新事件）
  CNT 再溢出 → RCR 再减 1
  ...
  RCR 减到 0 → 再次溢出时 → 产生更新事件
```

### 2.3 本实验 RCR 配置

```
RCR = 4（rep = 4）

溢出次数:  第1次   第2次   第3次   第4次   第5次
           ↓      ↓      ↓      ↓      ↓
RCR 递减:   4→3    3→2    2→1    1→0    0→更新事件
PWM 输出:   ✅     ✅     ✅     ✅     ✅

总 PWM 周期数 = 4 + 1 = 5
总时长 = 5 × 0.5s = 2.5s
```

### 2.4 RCR 与闪烁次数对照

| RCR 值 | PWM 周期数 | LED 闪烁次数 | ARR=4999 时总时长 |
|:------:|:----------:|:------------:|:----------------:|
| 0 | 1 | 1 | 0.5s |
| 1 | 2 | 2 | 1.0s |
| 2 | 3 | 3 | 1.5s |
| 3 | 4 | 4 | 2.0s |
| **4** | **5** | **5** | **2.5s** |
| 9 | 10 | 10 | 5.0s |
| 255 | 256 | 256 | 128.0s |

## 三、BDTR（刹车和死区时间寄存器）

### 3.1 寄存器结构

```
BDTR 寄存器（16位）：
┌──────┬────┬────┬────┬──────┬────────────────┐
│ MOE  │AOE │BKE │BKP │ 保留  │  DTG[7:0]      │
│ bit15│14  │13  │12  │ 11:9 │  死区时间        │
│主输出 │自动 │刹车 │刹车 │      │  发生器         │
│ 使能 │输出 │使能 │极性 │      │                │
└──────┴────┴────┴────┴──────┴────────────────┘
```

### 3.2 关键位详解

| 位 | 名称 | 功能 | 本实验 |
|:--:|------|------|:------:|
| **15** | **MOE** | **主输出使能** | **必须置 1** |
| 14 | AOE | 自动输出使能（MOE 可在更新事件时自动置位） | — |
| 13 | BKE | 刹车使能 | 不使用 |
| 12 | BKP | 刹车极性 | 不使用 |
| 7:0 | DTG | 死区时间发生器 | 不使用 |

### 3.3 MOE（主输出使能）详解

```
MOE = 1：主输出使能
  → 所有输出通道正常输出 PWM
  → 高级定时器的"总开关"

MOE = 0：主输出关闭
  → 所有输出引脚进入"空闲状态"
  → 空闲状态由 CR2.OISx 位定义

⚠️ 高级定时器必须设置 MOE=1 才能输出 PWM
   通用定时器不需要此配置
```

### 3.4 使能位层级关系

```
高级定时器输出需要三层使能：

① CEN（CR1）    → 定时器总开关（计数器使能）
② CCxE（CCER）  → 通道使能（通道输出使能）
③ MOE（BDTR）   → 主输出使能（高级定时器特有）

三层都必须开启，缺一不可：
  CEN=1, CC1E=1, MOE=1 → 正常输出 PWM

通用定时器只需两层：
  CEN=1, CC1E=1 → 正常输出 PWM
```

## 四、CR2（控制寄存器 2）新增位

### 4.1 空闲状态位

```
CR2 中的空闲状态位：
  OIS1  / OIS1N   → 通道 1 / 互补通道 1 的空闲状态
  OIS2  / OIS2N   → 通道 2 / 互补通道 2 的空闲状态
  OIS3  / OIS3N   → 通道 3 / 互补通道 3 的空闲状态
  OIS4             → 通道 4 的空闲状态

空闲状态定义：
  MOE = 0（主输出关闭）时输出引脚的默认电平

OISx = 0（Reset）：空闲时输出低电平
OISx = 1（Set）：  空闲时输出高电平
```

### 4.2 空闲状态应用场景

```
LED 电路（低电平点亮）：
  → 希望关闭后 LED 不亮
  → 空闲状态应为高电平
  → OISx = 1（Set）

电机驱动电路：
  → 希望关闭后电机停止
  → 空闲状态需根据电路设计决定
  → 确保不会出现短路或失控
```

```

           + CR2 空闲状态位
           + 互补输出（CHxN）
           + MOE 主输出使能
```

## 六、死区时间（Dead Time）详解

### 6.1 为什么需要死区时间

```
H 桥电机驱动电路：

  VCC
   │
  MOS管A ──── 电机 ──── MOS管B
   │                       │
  CH1 输出               CH1N 输出（互补）

理想情况：CH1=高时 CH1N=低，CH1=低时 CH1N=高
实际问题：MOS管关断比开通慢
  → 切换瞬间两者可能同时导通
  → 短路！烧毁！

解决方案：插入死区时间
  → 关断后延迟一段时间再开通互补管
  → 确保两个管子不会同时导通
```

### 6.2 死区时间配置（DTG[7:0]）

| DTG[7:5] | 格式 | 死区时间计算 |
|:--------:|------|-------------|
| 0xx | DTG[7:0] × Tdtg | Tdtg = Tdts（分频后时钟周期） |
| 10x | (64 + DTG[5:0]) × 2×Tdtg | Tdtg = 2×Tdts |
| 110 | (32 + DTG[4:0]) × 8×Tdtg | Tdtg = 8×Tdts |
| 111 | (32 + DTG[4:0]) × 16×Tdtg | Tdtg = 16×Tdts |

```
DTG 配置规则：
  前 3 位决定计算格式
  后 5~7 位决定具体数值
  Tdts = 经过 CKD 分频后的时钟周期

配置非常复杂，使用时需要查手册
本实验不使用死区时间功能
```

## 七、刹车功能（Break）详解

### 7.1 工作原理

```
外部信号 → BKE（刹车使能）→ BKP（极性检测）
  → 检测到指定边沿
  → MOE 自动清零
  → 所有输出立即进入空闲状态
  → 实现紧急关断

应用场景：
  → 电机过流保护
  → 外部紧急停止按钮
  → 安全联锁电路
```

### 7.2 刹车配置位

| 位 | 功能 |
|:--:|------|
| BKE | 刹车使能（0=禁用，1=使能） |
| BKP | 刹车极性（0=低电平有效，1=高电平有效） |
| AOE | 自动输出使能（刹车解除后 MOE 是否自动恢复） |

## 八、本实验寄存器配置汇总

### 8.1 需要配置的寄存器

| 寄存器 | 配置内容 | 值 |
|--------|----------|:--:|
| RCC.APB2ENR | 开启 TIM1 时钟 | — |
| GPIOA.CRH | PA8 复用推挽输出 | CNF=10, MODE=11 |
| TIM1.PSC | 预分频 | 7199 |
| TIM1.ARR | 自动重装载 | 4999 |
| TIM1.RCR | 重复计数器 | **4** |
| TIM1.CCMR1 | 输出模式 | CC1S=00, OC1M=110 |
| TIM1.CCER | 通道使能+极性 | CC1E=1, CC1P=0 |
| TIM1.CR1 | 定时器使能 | CEN=1 |
| TIM1.BDTR | **主输出使能** | **MOE=1** |

### 8.2 PWM 参数计算

```
PSC = 7199 → 分频系数 = 7200
计数频率 = 72MHz / 7200 = 10kHz
ARR = 4999 → 计数 5000 次溢出
溢出周期 = 5000 / 10000 = 0.5s
PWM 频率 = 1 / 0.5 = 2Hz

RCR = 4 → 5 次溢出后停止
总时长 = 5 × 0.5 = 2.5s
LED 闪烁 5 次
```

### 8.3 占空比

```
CCR1 = 2500
占空比 = 2500 / 5000 = 50%

高电平 0.25s + 低电平 0.25s = 周期 0.5s
```

## 九、BDTR 相关的 OSSI/OSSR 位

### 9.1 概念

| 位 | 全称 | 功能 |
|:--:|------|------|
| OSSR | Off-State Selection for Run mode | 运行模式关闭状态选择 |
| OSSI | Off-State Selection for Idle mode | 空闲模式关闭状态选择 |

### 9.2 关闭状态（Off-State）

```
两种关闭状态场景：

① 运行模式关闭状态（OSSR）：
   MOE = 1（主输出开启）
   但定时器被关闭（CEN=0）
   → 输出无效电平（由极性 CCxP 定义）

② 空闲模式关闭状态（OSSI）：
   MOE = 0（主输出关闭）
   → 输出空闲电平（由 CR2.OISx 定义）
```

## 十、本实验代码实现要点

### 10.1 额外需要配置的寄存器

```
相比通用定时器 PWM 输出，高级定时器额外需要：

① RCR = 4（重复计数值）
   → 控制输出 5 个周期后停止

② BDTR.MOE = 1（主输出使能）
   → 高级定时器必须开启

③ CR2.OIS1 = 1（空闲状态为高电平）
   → 停止后 LED 熄灭

④ CCER.CC1P = 1（低电平有效）可选
   → 配合空闲状态确保 LED 关闭
```

### 10.2 使能顺序

```
正确的使能顺序：

① 配置时基（PSC, ARR, RCR）
② 配置输出通道（CCMR1, CCER, CCR）
③ 配置空闲状态（CR2.OISx）
④ MOE = 1（BDTR 主输出使能）
⑤ CEN = 1（CR1 定时器使能）
```

## 十一、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 无 PWM 输出 | MOE 未使能 | BDTR.MOE = 1 |
| LED 停止后常亮 | 空闲状态为低电平 | CR2.OIS1 = 1 |
| LED 停止后常亮 | 极性未配低电平有效 | CCER.CC1P = 1 或 OIS1=1 |
| 闪烁次数不对 | RCR 值配错 | RCR = 4（5 次） |
| 闪烁次数不对 | 混淆 RCR 和周期数 | 周期数 = RCR + 1 |
| 定时器不计数 | CEN 未使能 | CR1.CEN = 1 |
| 第一次就进中断 | 初始化产生更新事件 | 清除 UIF 标志 |

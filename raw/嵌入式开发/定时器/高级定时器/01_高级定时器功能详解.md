# STM32 高级定时器（TIM1/TIM8）详解 笔记

---

## 一、高级定时器概述

### 1.1 定位

| 项目 | 说明 |
|------|------|
| 编号 | TIM1、TIM8（一头一尾） |
| 继承 | 具有通用定时器的**所有功能** |
| 总线 | APB2（高速外设总线） |
| 时钟 | 72MHz（与通用定时器相同） |

### 1.2 新增三大功能

| 功能 | 说明 |
|------|------|
| **重复计数器（RCR）** | 控制更新事件产生频率，可生成有限个周期波形 |
| **互补输出 + 死区时间** | CH1/CH1N 互补信号，带可编程死区延迟 |
| **刹车输入（Break）** | 外部信号强制关断所有输出 |

## 二、重复计数器（RCR）

### 2.1 与通用定时器的区别

```
通用定时器（无 RCR）：
  CNT 溢出 → 直接产生更新事件（UEV）

高级定时器（有 RCR）：
  CNT 溢出 → 重复计数器减 1（不产生更新事件）
  重复计数器溢出（0 再减）→ 才产生更新事件（UEV）
```

### 2.2 工作原理

```
重复计数器初始值 = REP（RCR 寄存器中的值）

例：REP = 2

  CNT 第 1 次溢出 → REP: 2→1（不产生更新事件）
  CNT 第 2 次溢出 → REP: 1→0（不产生更新事件）
  CNT 第 3 次溢出 → REP: 0→溢出 → 产生更新事件！

总溢出次数 = REP + 1 = 3
```

### 2.3 核心公式

```
更新事件产生频率 = CNT 溢出频率 / (REP + 1)

PWM 周期输出个数 = REP + 1

例：REP = 4 → 输出 5 个 PWM 周期后产生更新事件
```

### 2.4 RCR 寄存器

| 项目 | 说明 |
|------|------|
| 寄存器名 | **RCR**（Repetition Counter Register） |
| 核心位 | **REP[7:0]**（8 位，0~255） |
| 功能 | 设定重复计数器的初始值 |

### 2.5 应用场景

```
生成有限个周期的 PWM 波形：

① 设置 RCR = N-1（输出 N 个周期）
② 启动 PWM 输出
③ CNT 每溢出一次，RCR 减 1
④ RCR 减到 0 后再溢出 → 产生更新事件
⑤ 在更新中断中关闭定时器
⑥ 输出停止

例：LED 闪烁 5 次
  RCR = 4 → 5 次溢出 → 更新事件 → 关闭定时器
```

### 2.6 更新事件产生对比

| 定时器类型 | CNT 溢出 → | REP 减到 0 → |
|-----------|-----------|-------------|
| 基本/通用 | **直接产生更新事件** | — |
| 高级 | **REP 减 1** | **产生更新事件** |

## 三、互补输出与死区时间

### 3.1 互补输出

```
高级定时器通道 1~3 可输出两路互补信号：

通道 1：CH1   + CH1N（互补）
通道 2：CH2   + CH2N（互补）
通道 3：CH3   + CH3N（互补）
通道 4：CH4（无互补输出）

互补信号特征：
  → 频率和周期相同
  → 相位相差 180°（完全反相）
  → CHx 高 ↔ CHxN 低
  → CHx 低 ↔ CHxN 高
```

### 3.2 互补信号示意

```
CH1：   ┌────────┐                 ┌────────┐
        │        │                 │        │
   ─────┘        └─────────────────┘        └─────

CH1N：          ┌────────┐                 ┌────────┐
                │        │                 │        │
   ─────────────┘        └─────────────────┘        └───

占空比：CH1 = 80%      CH1N = 20%
```

### 3.3 H 桥电路

```
        VCC
         │
    ┌────┼────┐
    │    │    │
   Q1   │   Q3
    │    │    │
    ├─[电机]─┤
    │    │    │
   Q2   │   Q4
    │    │    │
    └────┼────┘
         │
        GND

CH1  有效 → Q1+Q4 导通 → 电机正转
CH1N 有效 → Q2+Q3 导通 → 电机反转
```

### 3.4 死区时间的必要性

```
问题：MOSFET 开通快、关断慢

理想情况：
  CH1  高 → 低    （立刻关断 Q1+Q4）
  CH1N 低 → 高    （立刻开通 Q2+Q3）

实际情况：
  CH1  高 → 低    （关断较慢，Q1+Q4 还没完全关断）
  CH1N 低 → 高    （开通较快，Q2+Q3 已经导通）
                      ↓
              Q1+Q2+Q3+Q4 同时导通 → 电源短路！→ 烧毁！
```

### 3.5 死区时间原理

```
死区时间 = 在开通前插入的一段延迟

原则：
  → 由高变低（关断）：立即执行
  → 由低变高（开通）：延迟一段时间再执行

效果：
  → 保证任何时刻都不会有两个管子同时导通
  → 在死区时间内，所有管子都关断
  → 负载不工作（"死"掉了 → 死区时间）
```

### 3.6 死区时间波形图

```
OCxREF（参考信号）：
  ┌────────────────┐                 ┌────────────────┐
  │                │                 │                │
──┘                └─────────────────┘                └──

OCx（实际输出）：
       ┌──────────────┐                 ┌──────────────┐
       │              │                 │              │
──────┘              └─────────────────┘              └──
  ↑延迟                                    ↑延迟
  死区时间                                  死区时间

OCxN（互补输出）：
  ┌──────────────┐                       ┌──────────────┐
  │              │                       │              │
──┘              └───────────────────────┘              └──
                 ↑延迟                                    ↑延迟
                 死区时间                                  死区时间

关键：
  高电平延迟变化（开通延迟）
  低电平立即变化（关断立即）
  → 任何时刻 OCx 和 OCxN 不会同时为高
```

### 3.7 死区时间配置

| 配置项 | 说明 |
|--------|------|
| 寄存器 | **BDTR**（Break and Dead-Time Register） |
| 位域 | **DTG[7:0]**（Dead-Time Generator） |
| 范围 | 可编程，从几 ns 到几百 μs |
| 步长 | 取决于配置位，参考手册具体定义 |

## 四、刹车输入（Break Input）

### 4.1 概念

```
刹车信号 = 紧急关断信号
  → 一旦触发，所有输出立即关断
  → 用于保护电路和设备安全
```

### 4.2 信号来源

```
TIMx_BKIN 引脚：
  → 外部引脚直接输入刹车信号
  → 可配置极性（上升沿/下降沿触发）

时钟失效事件：
  → 内部时钟异常时自动触发刹车
```

### 4.3 在框图中的位置

```
功能框图中：

TIMx_BKIN ──→ 极性选择 ──→ 刹车检测
                              │
                              ↓
                         输出控制（在 OCx 输出前）
                              │
                              ↓
                    所有输出强制关断
                    MOE 位自动清零
```

### 4.4 刹车触发后的输出状态

| 配置位 | 输出状态 |
|--------|----------|
| OSSI=0, OSSR=0 | 输出高阻态 |
| OSSI=1, OSSR=0 | 输出空闲电平（由 OISx 位定义） |
| OSSI=x, OSSR=1 | 输出空闲电平（使能运行模式） |

### 4.5 应用场景

```
→ 电机过流保护：电流传感器检测到过流 → 刹车信号 → 关断输出
→ 紧急停止按钮：人工按下停止按钮 → 刹车信号 → 关断输出
→ 电源异常：电压/电流异常 → 刹车信号 → 关断输出
```

## 五、BDTR 寄存器（刹车和死区时间）

### 5.1 寄存器结构

| 位域 | 名称 | 功能 |
|------|------|------|
| [7:0] | **DTG[7:0]** | 死区时间生成器 |
| [10:8] | LOCK | 锁定配置（防止误改） |
| [11] | **BKE** | 刹车使能 |
| [12] | **BKP** | 刹车极性 |
| [13] | **AOE** | 自动输出使能 |
| [14] | **MOE** | **主输出使能（总开关）** |
| [15] | **OSSI** | 空闲模式下关闭状态输出 |

### 5.2 MOE 位（主输出使能）

```
MOE = 0：所有通道输出关断
MOE = 1：所有通道输出使能

特殊：
  → 刹车触发时 MOE 自动清零 → 输出关断
  → AOE=1 时，更新事件自动恢复 MOE=1
  → AOE=0 时，需要软件手动恢复 MOE=1
```

## 六、高级定时器框图变化总结

```
与通用定时器的框图对比：

相同部分：
  → 时钟源选择（内部/外部/ITR/ETR）
  → 触发控制器（TRGI/TRGO）
  → 预分频器 PSC
  → 计数器 CNT
  → 自动重装载 ARR
  → 输入捕获通道（与通用相同）
  → 输出比较通道（基础部分相同）

新增部分：
  → 重复计数器 RCR（在 CNT 和更新事件之间）
  → 互补输出 CHxN（通道 1~3）
  → 死区时间插入 DTG（在 OCx 输出前）
  → 刹车输入 Break（强制关断输出）
  → BDTR 寄存器（配置死区和刹车）
```

## 七、三大新功能对比

| 功能 | 寄存器 | 核心位 | 作用 |
|------|--------|--------|------|
| 重复计数器 | RCR | REP[7:0] | 控制更新事件频率 |
| 死区时间 | BDTR | DTG[7:0] | 延迟互补信号开通时间 |
| 刹车输入 | BDTR | BKE/BKP | 外部信号强制关断输出 |
| 主输出使能 | BDTR | MOE | 所有输出的总开关 |

## 八、应用场景

| 功能 | 应用 |
|------|------|
| 重复计数器 | LED 有限次数闪烁、有限周期 PWM 波形 |
| 互补输出 | 电机驱动（H 桥电路） |
| 死区时间 | H 桥电路防短路保护 |
| 刹车输入 | 电机过流保护、紧急停止 |

## 九、概念速查

| 概念        | 说明                      |
| --------- | ----------------------- |
| **互补输出**  | CHx 和 CHxN 频率相同、相位相反的信号 |
| **死区时间**  | 互补信号切换时插入的延迟，防止同时导通     |
| **H 桥电路** | 四个 MOSFET 组成的电机正反转驱动电路  |
| **刹车信号**  | 外部输入的紧急关断信号（Break）      |
| **MOE**   | 主输出使能位，所有输出的总开关         |
| **RCR**   | 重复计数寄存器，控制更新事件产生频率      |
| **REP**   | RCR 中的重复计数值             |
| **BDTR**  | 刹车和死区时间寄存器              |

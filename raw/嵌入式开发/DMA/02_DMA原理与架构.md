# DMA（直接存储器访问）原理与架构 笔记

---

## 一、DMA 概述

### 1.1 基本概念

| 项目 | 说明 |
|------|------|
| 全称 | **D**irect **M**emory **A**ccess（直接存储器访问） |
| 本质 | 硬件数据传输通道，绕过 CPU 实现外设与存储器之间的数据交换 |
| 核心意义 | **解放 CPU**，提高数据传输效率 |

### 1.2 传统方式 vs DMA 方式

```
传统方式（CPU 参与）：
  外设 ←→ 数据总线 ←→ CPU ←→ 数据总线 ←→ 存储器
  每次数据传输都需要 CPU 全程参与
  CPU 大部分时间在等待慢速外设/存储器

DMA 方式（CPU 不参与）：
  外设 ←→ DMA 通道（高速通路）←→ 存储器
  CPU 只负责初始配置，之后完全由 DMA 控制器自动完成
  CPU 可以去处理其他任务
```

### 1.3 形象比喻

```
老板 = CPU（能力最强，速度最快）
小秘书 = DMA 控制器（专职管理数据传输）
员工 = 外设（串口、ADC、定时器等）
公司资源 = 系统数据总线
特殊资源 = 存储器（RAM/ROM）

传统方式：
  员工 A 和员工 B 交接工作 → 必须通过老板传话
  → 效率低，浪费老板时间

DMA 方式：
  员工 A 和员工 B 交接工作 → 直接找小秘书
  → 小秘书建立高速通路，自动完成
  → 老板完全不参与，去做更重要的事
```

## 二、STM32 DMA 硬件架构

### 2.1 DMA 控制器

| 控制器 | 通道数 | 说明 |
|--------|:------:|------|
| **DMA1** | **7** 个通道 | 所有型号都有 |
| **DMA2** | **5** 个通道 | 仅大容量和互联型产品 |

```
总通道数 = 7 + 5 = 12 个通道

小容量芯片：可能没有 DMA2
大容量芯片：DMA1 + DMA2 都有
```

### 2.2 系统架构中的位置

```
┌──────────────────────────────────────────────┐
│               AHB 系统高速总线                 │
│                                              │
│   ┌──────┐    ┌──────┐    ┌──────────┐      │
│   │ CPU  │    │ SRAM │    │ DMA 控制器│      │
│   │(CM3) │    │(RAM) │    │(DMA1/DMA2)│      │
│   └──┬───┘    └──┬───┘    └────┬─────┘      │
│      │           │              │             │
│   ═══╧═══════════╧══════════════╧═══         │
│                总线矩阵                        │
│         ┌──────┴──────┐                       │
│         ↓             ↓                       │
│      APB1 桥       APB2 桥                    │
│         ↓             ↓                       │
│    低速外设总线    高速外设总线                 │
│   (TIM2~7,UART)  (TIM1,ADC,SPI)              │
└──────────────────────────────────────────────┘

DMA 控制器直接挂在 AHB 总线上
与 CPU 共享系统数据总线
是系统核心组件，不是外设
```

### 2.3 每个通道管理多个外设

```
DMA1 通道 1 ─┬─ ADC1
              ├─ TIM2_CH3
              └─ TIM4_CH1

DMA1 通道 2 ─┬─ SPI1_RX
              ├─ USART3_TX
              └─ TIM1_CH1
              ...

多个外设 → 同一通道（或的关系）
同时请求时只能响应一个（仲裁器决定）
```

## 三、总线冲突与仲裁

### 3.1 总线冲突场景

```
当 CPU 和 DMA 同时想要访问同一个目标（RAM 或外设）时
→ 产生总线冲突
→ 需要仲裁器决定谁优先
```

### 3.2 DMA 优先原则

```
DMA 优先于 CPU！

原因：
  外设数据传输有实时性要求
  → 串口数据不及时接收 → 数据丢失
  → ADC 数据不及时读取 → 被覆盖
  → 数据传输节奏被打乱 → 系统异常

  CPU 稍微等待没问题
  → 有全权处理能力，可以暂停再恢复
  → 不会导致数据丢失
```

### 3.3 两种总线移交方案

| 方案 | 名称 | 方式 | 效率 |
|------|------|------|:----:|
| 方案一 | 整块让出 | CPU 完全让出总线，DMA 结束后归还 | 低 |
| **方案二** | **周期挪用** | CPU 每次让出几个时钟周期，用完立即归还 | **高** |

```
方案一（整块让出）：
  CPU ═══════════╗         ╔═══════════ CPU
                  ║  DMA   ║
                  ╚═══════╝
  CPU 长时间无法使用总线

方案二（周期挪用/周期窃取）：
  CPU ══╗   ╔══╗   ╔══╗   ╔═════════ CPU
        ║DMA║  ║DMA║  ║DMA║
        ╚═══╝  ╚═══╝  ╚═══╝

  DMA 传输时需要等待（外设写入周期、内存读写周期）
  这段等待时间 CPU 可以使用总线
  → 真正的"插空"使用
  → 效率更高

✅ 实际系统中使用的是方案二（周期挪用）
```

### 3.4 带宽保证

```
总线仲裁器做循环调度
→ 保证 CPU 至少可以得到 50% 的系统总线带宽
→ 防止 DMA 请求过于频繁导致 CPU 饿死
```

## 四、DMA 传输方向

### 4.1 三种传输方向

| 方向 | 缩写 | 说明 | 典型场景 |
|------|------|------|----------|
| 外设 → 存储器 | P→M | 外设读，存储器写 | 串口接收、ADC 采样 |
| 存储器 → 外设 | M→P | 存储器读，外设写 | 串口发送、DAC 输出 |
| **存储器 → 存储器** | **M→M** | ROM 读，RAM 写 | Flash 数据复制到 SRAM |

### 4.2 存储器到存储器（特殊模式）

```
ROM（Flash）→ RAM（SRAM）

限制：
  ✅ ROM → RAM（可以）
  ❌ RAM → ROM（不可以）
     Flash 存储程序和固件，随意写入会破坏系统

本质：
  DMA 底层只有"外设"和"存储器"两个概念
  存储器到存储器模式是特例
  → 将源存储器（ROM）当作"外设"
  → 目标存储器（RAM）当作"存储器"
```

## 六、DMA 请求与应答流程

```
① 外设向 DMA 控制器发出 DMA 请求

② DMA 控制器收到请求后，给外设一个应答信号
   → 表示：我收到了，准备处理

③ 外设收到应答后，释放请求（不再持续请求）

④ DMA 控制器通过仲裁器判断优先级

⑤ 获取总线控制权后，在外设和存储器之间建立高速通路

⑥ 数据自动传输

⑦ 传输完成后，产生传输完成事件/中断

注意事项：
  → 如果通道已被占用，新请求不被响应
  → 外设会持续请求直到被响应
  → 每个通道同一时刻只能处理一个外设
```

## 七、DMA 优先级仲裁

### 7.1 两级仲裁

| 阶段 | 方式 | 说明 |
|------|------|------|
| **第一级** | 软件优先级 | CCR.PL 配置（4 种等级） |
| **第二级** | 硬件优先级 | 通道编号（1 > 2 > 3...） |

### 7.2 软件优先级（CCR.PL）

| PL[1:0] | 优先级 |
|:-------:|--------|
| 00 | 低（默认） |
| 01 | 中 |
| 10 | 高 |
| 11 | 最高 |

### 7.3 仲裁规则

```
① 先比较软件优先级（CCR.PL）
   → PL 值越大，优先级越高

② 软件优先级相同 → 比较硬件优先级
   → 通道编号越小，优先级越高
   → 通道 1 > 通道 2 > 通道 3 > ...

③ CPU vs DMA 冲突
   → DMA 优先（但保证 CPU 至少 50% 带宽）
```

## 八、关键概念速查

| 概念 | 说明 |
|------|------|
| DMA | Direct Memory Access，直接存储器访问 |
| DMA 控制器 | 管理 DMA 通道的硬件模块（小秘书） |
| DMA 通道 | 数据传输的硬件通路（专线专用） |
| DMA 请求 | 外设向 DMA 控制器发出的传输请求 |
| DMA 应答 | DMA 控制器对请求的确认信号 |
| 仲裁器 | 决定多个 DMA 请求的优先级 |
| 周期挪用 | CPU 每次让出几个时钟周期给 DMA |
| CCR | Channel Configuration Register，通道配置寄存器 |
| PL | Priority Level，软件优先级 |
| MEM2MEM | 存储器到存储器传输模式 |

## 九、DMA 控制器与 CPU 对比

| 对比项 | CPU (CM3) | DMA 控制器 |
|--------|:---------:|:---------:|
| 角色 | 老板 | 小秘书 |
| 功能 | 执行所有指令 | 专职管理数据传输 |
| 速度 | 最快 | 高速但次于 CPU |
| 连接总线 | AHB | AHB |
| 能力 | 全能 | 仅数据搬运 |
| 参与数据传输 | 需要全程参与 | 自动完成，无需干预 |

## 十、DMA 的意义总结

```
① 解放 CPU
   → 数据搬运工作交给 DMA 控制器
   → CPU 专注于逻辑处理、算法计算

② 提高传输效率
   → 硬件通路直接传输
   → 不需要软件指令逐步执行

③ 保证实时性
   → 外设数据不丢失
   → 传输速度可匹配外设需求

④ 降低功耗
   → CPU 不参与数据搬运
   → 可以进入低功耗模式
```

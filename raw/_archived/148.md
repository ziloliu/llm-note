# ADC 工作原理 — 逐次逼近型详解 笔记

---

## 一、ADC 三种常见类型

### 1.1 类型总览

| 类型 | 核心原理 | 速度 | 精度 | 电路复杂度 | 典型场景 |
|------|---------|:----:|:----:|:---------:|---------|
| **逐次逼近型（SAR）** | 逐位二分法比较 | **中等** | **中等** | **中等** | STM32、通用MCU |
| **Sigma-Delta（Σ-Δ）** | 差值累加积分 | 慢 | **高** | 较高 | 高精度测量、音频 |
| **Flash（闪速）型** | 多比较器并行比较 | **极快** | 低 | 高 | 高速采样、视频 |

### 1.2 各类型原理简介

**逐次逼近型（SAR）**
```
原理：二分法，从最高位到最低位逐位尝试
  → 取中间值与待测电压比较
  → 大了就往下找，小了就往上找
  → 每次缩小一半范围
  → n 位需要 n 次比较

特点：速度与精度的均衡选择
```

**Sigma-Delta 型**
```
原理：对相邻采样值的差值（Δ）进行累加（Σ）
  → 底层有积分器
  → 通过累积消除误差

名称来源：
  Σ（Sigma）= 累加符号
  Δ（Delta）= 差值/增量

特点：高精度，但转换速度慢
  → 需要做累加、做积分
  → 排除外界干扰能力强

衍生：双积分型（两次积分，抗干扰更强，精度更高，速度更慢）
```

**Flash 型**
```
原理：电阻分压 + 多个比较器并行比较
  → 参考电压通过电阻串分出多个电位
  → 每个电位与输入电压同时比较
  → 所有位同时得出结果

特点：速度极快（1个时钟周期完成）
  → 但精度低（位数少）
  → 电路规模大（每个电位需要一个比较器）
  → 精度提高 → 电路规模指数增长
```

### 1.3 STM32 选择逐次逼近型的原因

```
精度要求：12 位分辨率 → 够用
速度要求：适中 → 满足多数应用
集成度要求：片上集成 → 电路规模合理

逐次逼近型 = 精度与速度的最佳平衡点
```

## 二、天平称重类比

### 2.1 天平称重过程

```
砝码组：5g, 4g, 3g, 2g, 1g
测量范围：1g ~ 15g（全部加在一起）
待测物体：6g（未知重量）

左盘 = 物体（待测电压 Ux）
右盘 = 砝码（DAC 输出电压）

操作规则：
  → 先放最大的砝码
  → 不够重 → 保留，加下一个
  → 超重了 → 去掉刚加的，换更小的
  → 直到平衡
```

### 2.2 逐步操作

```
步骤 1：放 5g    → 左盘重（物体 > 5g）→ 保留 5g
步骤 2：放 4g    → 右盘重（5+4=9 > 6g）→ 去掉 4g
步骤 3：放 3g    → 右盘重（5+3=8 > 6g）→ 去掉 3g
步骤 4：放 2g    → 右盘重（5+2=7 > 6g）→ 去掉 2g
步骤 5：放 1g    → 平衡！（5+1=6 = 6g）→ 保留 1g

结果：5g + 1g = 6g ✅
```

### 2.3 口诀

```
先上大，后上小
超重减，不足留
逐个尝试到平衡
```

## 三、逐次逼近型 ADC 工作原理

### 3.1 简化模型参数

```
为了方便计算，做以下假设：

  参考电压 VREF = 255V（实际不可能，纯粹为了好算）
  分辨率 = 8 位
  数字量范围 = 0 ~ 255
  全零（0）→ 对应 0V
  全一（255）→ 对应 255V

  数字值 = 模拟电压值（一对一对应）

待测电压：Ux = 33V
```

### 3.2 核心组件

```
┌─────────────────────────────────────────────┐
│                                             │
│   Ux(33V) ──→ (+)  ┌──────────┐            │
│                     │  比较器    │──→ 比较结果 │
│   DAC输出 ──→ (-)  └──────────┘    (高/低)  │
│                          ↑                   │
│                          │                   │
│                 ┌────────┴────────┐          │
│                 │   逻辑控制电路    │          │
│                 │ (决定保留/清除)   │          │
│                 └────────┬────────┘          │
│                          ↑                   │
│                 ┌────────┴────────┐          │
│                 │   8位寄存器       │          │
│                 │  (存储砝码值)     │          │
│                 └────────┬────────┘          │
│                          ↓                   │
│                 ┌────────┴────────┐          │
│                 │   8位DAC转换器    │←─ VREF   │
│                 │ (数字量→模拟量)   │          │
│                 └─────────────────┘          │
│                          ↓                   │
│                   输出缓冲区 → 结果           │
└─────────────────────────────────────────────┘
```

### 3.3 组件功能说明

| 组件 | 功能 | 天平类比 |
|------|------|---------|
| **比较器** | 比较 Ux 与 DAC 输出，输出高/低电平 | 天平（左盘=物体，右盘=砝码） |
| **8 位寄存器** | 存储当前二进制砝码值 | 当前放在天平上的砝码组 |
| **DAC 转换器** | 将寄存器数字量转为模拟电压 | 计算砝码总重量 |
| **逻辑控制电路** | 根据比较结果决定保留/清除当前位 | 操作砝码增减的人 |
| **VREF** | 参考电压，确定量程 | 砝码最大量程 |
| **输出缓冲区** | 存储最终转换结果 | 称量结果 |

## 四、比较器工作原理

```
两个输入端：
  (+) 正端：待测电压 Ux
  (-) 负端：DAC 输出电压（砝码电压）

输出规则：
  Ux > DAC 输出 → 正端更大 → 输出高电平（1）
    → 含义：砝码偏小 → 保留当前位，继续试下一位

  Ux < DAC 输出 → 负端更大 → 输出低电平（0）
    → 含义：砝码偏大 → 清除当前位，继续试下一位

  Ux = DAC 输出 → 平衡 → 当前值即为结果

本质：模拟电压 → 高低电平（0/1）→ 控制数字逻辑
```

## 五、8 位寄存器与砝码对应

### 5.1 位权表

| 位 | 寄存器置1时的值 | 十进制（砝码重量） | 模拟电压 |
|:--:|:--------------:|:----------------:|:--------:|
| Bit 7（最高位） | 1000 0000 | **128** | 128V |
| Bit 6 | 0100 0000 | **64** | 64V |
| Bit 5 | 0010 0000 | **32** | 32V |
| Bit 4 | 0001 0000 | **16** | 16V |
| Bit 3 | 0000 1000 | **8** | 8V |
| Bit 2 | 0000 0100 | **4** | 4V |
| Bit 1 | 0000 0010 | **2** | 2V |
| Bit 0（最低位） | 0000 0001 | **1** | 1V |

### 5.2 砝码特点

```
天平砝码：5, 4, 3, 2, 1（等差递减 1）
ADC 砝码：128, 64, 32, 16, 8, 4, 2, 1（等比递减一半）

ADC 砝码 = 2 的幂次递减 → 每次缩小一半 → 二分法
```

### 5.3 DAC 转换

```
寄存器中的数字量 → DAC → 模拟电压

例：寄存器 = 0b10000001 = 128 + 1 = 129
    DAC 输出 = 129V（因为 VREF=255V 刚好与数值一一对应）
    
公式：V = D × VREF / 2^n = 129 × 255 / 256 ≈ 129V
```

## 六、逻辑控制电路

```
逻辑控制电路 = 操作砝码增减的人

判断逻辑：
  比较器输出 = 高电平（Ux > DAC）→ 砝码偏小
    → 保留当前位（1）
    → 移到下一位，继续置 1 尝试

  比较器输出 = 低电平（Ux < DAC）→ 砝码偏大
    → 清除当前位（0）
    → 移到下一位，继续置 1 尝试

关键：只操作当前正在尝试的那一位
  → 上一位已确定，不再修改
  → 每次只试一个新的位
```

## 八、逐次逼近型 ADC 工作流程总结

```
① 初始化：寄存器清零，准备最高位（Bit n-1）

② 将当前位 置 1

③ DAC 将寄存器值转换为模拟电压

④ 比较器比较 Ux 与 DAC 输出：
   → 高电平（Ux > DAC）：砝码偏小 → 保留当前位 = 1
   → 低电平（Ux < DAC）：砝码偏大 → 清除当前位 = 0

⑤ 移到下一位（n-2），重复 ②③④

⑥ 直到最低位（Bit 0）判断完毕

⑦ 寄存器中的值即为最终转换结果

⑧ 结果放入输出缓冲区，通过数据总线输出

时间复杂度：n 位分辨率 → n 次比较
  8 位 → 8 次比较
  12 位（STM32）→ 12 次比较
```

## 九、从 8 位推广到 12 位（STM32）

```
STM32 参数：
  分辨率：12 位
  VREF+ = 3.3V
  VREF- = 0V
  离散级别：2^12 = 4096
  最小分辨率：3.3V / 4096 ≈ 0.806mV

转换过程：
  → 从 Bit11（最高位，2048 对应电压）开始
  → 逐位判断到 Bit0（最低位，1 对应电压）
  → 共需 12 次比较
  → 每次范围缩小一半

例：输入电压 2.65V
  D = 2.65 × 4096 / 3.3 ≈ 3289
  二进制：0b110011011001
  
12 次比较确定结果
```

| ADC 术语 | 天平类比 | 说明 |
|---------|---------|------|

| 清除当前位 | 砝码偏大，取走 | 当前砝码太重 |
| 逐位尝试 | 先大后小加砝码 | 从最高位到最低位 |
| 最终寄存器值 | 平衡时砝码值 | 转换结果 |

## 十一、关键公式

```
① 离散级别数 = 2^n
   n = 8  → 256
   n = 12 → 4096

② 最小分辨率 ΔV = VREF / 2^n

③ 数字量 → 电压：V = D × VREF / 2^n

④ 电压 → 数字量：D = V × 2^n / VREF

⑤ 转换比较次数 = n（n 位需要 n 次比较）

⑥ 逐次逼近本质 = 二分搜索
   每次比较将搜索范围缩小一半
```

## 十二、三种类型核心差异总结

```
逐次逼近型（SAR）：
  1 个比较器 + 1 个 DAC + 逻辑控制
  逐位比较，n 次完成
  速度与精度均衡 ⭐（STM32 选用）

Sigma-Delta 型：
  1 个比较器 + 积分器 + 数字滤波器
  累积差值，高精度低速
  Σ=累加，Δ=差值

Flash 型：
  2^n - 1 个比较器并行
  1 个时钟周期完成
  速度快但精度低、电路庞大
  每个电位需一个比较器+编码器
```

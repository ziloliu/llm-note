# 输入捕获测量 PWM 周期和频率 — 中断方式实现 笔记

---

## 一、全局变量定义

```c
volatile uint16_t cycle = 0;    // 保存周期值（μs）
volatile uint8_t  count = 0;    // 上升沿次数计数
```

| 变量 | 类型 | 作用 | 初始值 |
|------|------|------|:------:|
| **cycle** | uint16_t | 保存捕获到的周期（CCR1 的值） | 0 |
| **count** | uint8_t | 记录第几次上升沿到来 | 0 |

> cycle 最大值 = 65535（ARR 最大值），不会溢出。

---

## 二、中断服务函数（初始版本）

### 2.1 完整代码

```c
void TIM4_IRQHandler(void)
{
    if (TIM4->SR & TIM_SR_CC1IF)         // 判断是否通道 1 捕获中断
    {
        TIM4->SR &= ~TIM_SR_CC1IF;       // 清除中断标志位
        count++;                           // 上升沿次数加一

        if (count == 1)                    // 第一次上升沿
        {
            TIM4->CNT = 0;                 // 清零计数器，从零开始计数
        }
        else if (count == 2)              // 第二次上升沿
        {
            cycle = TIM4->CCR1;            // 读取 CCR1 = 周期（μs）
            count = 0;                     // 清零 count，准备下一次
        }
    }
}
```

### 2.2 处理流程

```
第 1 个上升沿 → count=1 → CNT=0（开始计数）
        ··· CNT 递增 ···
第 2 个上升沿 → count=2 → cycle=CCR1（读取周期）→ count=0
        ··· CNT 递增 ···
第 3 个上升沿 → count=1 → CNT=0（重新开始）
        ··· 循环往复 ···
```

### 2.3 关键操作

| 操作 | 说明 |
|------|------|
| 清除 SR.CC1IF | **必须手动**，否则反复进入中断 |
| `TIM4->CNT = 0` | 第一次上升沿时清零计数器 |
| `cycle = TIM4->CCR1` | 第二次上升沿时读取捕获值 |
| `count = 0` | 重置状态，准备下一次测量 |

## 三、获取周期和频率函数

### 3.1 获取周期

```c
double TIM4_GetPWMPeriod(void)
{
    return (double)cycle / 1000.0;    // μs → ms
}
```

```
cycle 单位：μs
返回值单位：ms
转换：ms = μs / 1000
```

### 3.2 获取频率

```c
double TIM4_GetPWMFrequency(void)
{
    if (cycle == 0)
        return 0;
    return 1000000.0 / (double)cycle;  // μs → Hz
}
```

```
⚠️ 不能用 1000.0/cycle（得到 kHz，不是 Hz）
✅ 用 1000000.0/cycle（μs 倒数 → Hz）

推导：
  T = cycle μs = cycle × 10⁻⁶ s
  f = 1/T = 10⁶ / cycle Hz = 1,000,000 / cycle Hz
```

### 3.3 单位换算

| 量 | 原始值 | 公式 | 结果单位 |
|----|--------|------|----------|
| 周期 | cycle (μs) | cycle / 1000.0 | **ms** |
| 频率 | cycle (μs) | 1000000.0 / cycle | **Hz** |

## 四、主函数

```c
int main(void)
{
    USART1_Init(115200);
    Delay_init();

    TIM5_Init();             // PWM 输出初始化
    TIM5_Start();            // 启动 PWM
    TIM5_SetDutyCycle(50);   // 50% 占空比

    TIM4_Init();             // 输入捕获初始化（⚠️ 不能忘！）
    TIM4_Start();            // 启动捕获

    printf("Hello World!\n");

    while (1)
    {
        printf("T=%.2fms, F=%.2fHz\n",
               TIM4_GetPWMPeriod(),
               TIM4_GetPWMFrequency());

        Delay_ms(1000);
    }
}
```

## 五、调试过程中的问题

### 5.1 变量名冲突

```
错误：count 重定义
原因：systick.c 中已有 count 变量
解决：将 tim4.c 中的 count 改名（如 cnt）
     或删除 systick.c
```

### 5.2 忘记初始化 TIM4

```
现象：输出周期和频率都为 0
原因：只调用了 TIM4_Start()，没有调用 TIM4_Init()
解决：在主函数中补上 TIM4_Init()
```

### 5.3 检查清单

```
✅ TIM5 初始化 + 启动（PWM 信号源）
✅ TIM4 初始化（输入捕获配置）
✅ TIM4 启动（CR1.CEN = 1）
✅ 跳线帽连接（PA1 → PB6）
✅ 中断配置（DIER.CC1IE=1 + NVIC）
✅ 中断服务函数中清除标志位
✅ TIM4_Init() 在 TIM4_Start() 之前调用
```

## 七、后续优化方向

| 优化项 | 说明 |
|--------|------|
| 去掉 cycle 全局变量 | 直接读 CCR1（但需配合从模式复位） |
| 去掉 count 变量 | 使用从模式复位自动清零 CNT |
| 去掉中断 | 使用 SMCR 复位模式，硬件自动处理 |
| 增加占空比测量 | PWM 输入模式，双通道 + 从模式 |

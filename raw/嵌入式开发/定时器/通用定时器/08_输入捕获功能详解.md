# 通用定时器输入捕获功能详解 笔记

---

## 一、输入捕获功能概述

### 1.1 功能定义

| 项目 | 说明 |
|------|------|
| 功能 | 捕获输入通道上信号的**上升沿**或**下降沿** |
| 核心操作 | 在信号边沿到来时，**记录当前计数器 CNT 的值到 CCR** |
| 本质 | 对计数器做"快照"——边沿到来时按下"快门" |
| 主要应用 | 测量信号的**周期/频率**、**高电平宽度**、**占空比** |

### 1.2 能测量的参数

| 测量项 | 方法 | 说明 |
|--------|------|------|
| **周期 T** | 连续两个上升沿之间的 CNT 差值 | T = CCR₂ - CCR₁ |
| **频率 f** | f = 1/T | 周期的倒数 |
| **高电平宽度** | 上升沿到下降沿之间的 CNT 差值 | 高电平持续时间 |
| **低电平宽度** | 下降沿到上升沿之间的 CNT 差值 | 低电平持续时间 |
| **占空比** | 高电平宽度 / 周期 | D = t_high / T |

## 三、输入捕获信号处理流程

### 3.1 信号通路

```
① 外部信号通过 CHx 引脚输入 → TIx（原始输入信号）

② 输入滤波器 → 滤除毛刺噪声
   → 滤波后的信号：TIxF（F = Filter）

③ 边沿检测器 → 选择上升沿或下降沿
   → 极性由 CCER.CCxP 控制
   → 处理后的信号：TIxFP（P = Polarity）

④ 多路选择器 → 选择信号来源
   → TIxFP1（直通，来自本通道）
   → TIxFP2（交叉，来自相邻通道）
   → TRC（触发输入，特殊用途）
   → 输出：ICxPS

⑤ 输入预分频器 → 对捕获事件分频
   → ICxPSC 控制
   → 输出信号触发捕获事件

⑥ 捕获事件 → 将 CNT 的当前值锁存到 CCR
```

### 3.2 信号命名规则

| 名称 | 含义 |
|------|------|
| TI1 | 通道 1 原始输入信号 |
| TI1F | 通道 1 经过滤波后的信号（F = Filter） |
| TI1FP1 | 通道 1 经过滤波 + 极性处理后，直通映射到 CCR1 |
| TI1FP2 | 通道 1 经过滤波 + 极性处理后，交叉映射到 CCR2 |
| IC1 | 输入捕获 1 的最终信号（多路选择器输出） |
| IC1PS | IC1 经过预分频后的信号（PS = Prescaler） |

## 四、捕获原理

### 4.1 "快照"类比

```
捕获过程就像拍照：

  输入信号的上升沿 = 按下快门
  计数器 CNT 的值 = 被拍摄的对象
  CCR 寄存器 = 存储照片的地方

  每次上升沿到来 → 按下快门 → 将 CNT 值复制到 CCR
  下次上升沿到来 → 再次按下快门 → CCR 被新值覆盖（只保留最新的）
```

### 4.2 测量周期原理

```
输入信号：
      ┌────┐         ┌────┐         ┌────┐
      │    │         │    │         │    │
──────┘    └─────────┘    └─────────┘    └─────
      ↑              ↑              ↑
    捕获①          捕获②          捕获③

CNT 计数过程（计数器时钟周期 = 1μs）：
  捕获①时刻：CCR1 = 0     （第一次上升沿，CNT 归零开始计数）
  捕获②时刻：CCR2 = 1500  （第二次上升沿，CNT 值锁存到 CCR）
  捕获③时刻：CCR3 = 3000  （第三次上升沿，覆盖 CCR）

周期 T = CCR2 - CCR1 = 1500 - 0 = 1500 μs = 1.5 ms
频率 f = 1/T = 1/1.5ms ≈ 666.7 Hz
```

### 4.3 测量高电平宽度

```
捕获上升沿时刻：CCR1 = 0
捕获下降沿时刻：CCR2 = 900

高电平宽度 = CCR2 - CCR1 = 900 μs

占空比 = 高电平宽度 / 周期 = 900 / 1500 = 60%
```

## 五、CCMR1 中的输入模式配置

### 5.1 通道 1 输入模式关键位（低 8 位）

| 位 | 名称 | 功能 | 配置 |
|:--:|------|------|:----:|
| 1:0 | **CC1S** | 通道方向 | **01** = 输入（映射到 TI1） |
| 3:2 | IC1PSC | 输入预分频 | 00 = 不分频 |
| 7:4 | IC1F | 输入滤波 | 0000 = 不滤波 |

### 5.2 通道 2 输入模式关键位（高 8 位）

| 位 | 名称 | 功能 | 配置 |
|:--:|------|------|:----:|
| 9:8 | **CC2S** | 通道方向 | **01** = 输入（映射到 TI2） |
| 11:10 | IC2PSC | 输入预分频 | 00 = 不分频 |
| 15:12 | IC2F | 输入滤波 | 0000 = 不滤波 |

### 5.3 CCS 选择说明

```
CC1S = 00 → 输出比较模式
CC1S = 01 → 输入模式，IC1 映射到 TI1（直通）
CC1S = 10 → 输入模式，IC1 映射到 TI2（交叉）
CC1S = 11 → 输入模式，IC1 映射到 TRC（触发输入）

⚠️ 与输出模式不同：
  输出时 CCS = 00
  输入时 CCS ≠ 00（通常 = 01）
```

## 六、CCER 寄存器配置

### 6.1 通道 1 关键位

| 位 | 名称 | 功能 | 配置 |
|:--:|------|------|:----:|
| 0 | **CC1E** | 捕获/比较使能 | **1** = 使能 |
| 1 | **CC1P** | 输入极性/边沿选择 | **0** = 上升沿捕获，**1** = 下降沿 |

### 6.2 CC1P 在输入模式下的含义

```
CC1P = 0：捕获上升沿（默认）
CC1P = 1：捕获下降沿

输入极性 vs 输出极性：
  输入模式：决定捕获哪种边沿
  输出模式：决定高/低电平有效
  → 两者含义完全不同！
```

## 七、ARR 设置与测量范围

### 7.1 ARR 为什么设为最大值

```
ARR = 65535（16 位最大值）

目的：避免计数器在测量过程中溢出
  → 在一个信号周期内，CNT 从 0 开始递增
  → 如果 CNT 还没到第二个上升沿就溢出了 → 测量失败
  → ARR 设为最大值 → 最大化测量范围

前提条件：信号周期 ≤ 65535 个计数周期
```

### 7.2 测量范围计算

```
计数器时钟周期 = 1μs（72MHz / 72 = 1MHz）

最小可测周期 = 1μs（1 个计数周期）
最大可测周期 = 65535μs ≈ 65.5ms（不溢出）

对应频率范围：
  最大频率 = 1MHz（信号频率不能超过计数器频率）
  最小频率 = 1/65.5ms ≈ 15.26Hz
```

### 7.3 超出范围的解决方案

| 问题 | 现象 | 解决方案 |
|------|------|----------|
| 频率太高 | CNT 还没加到 1，周期已结束 | ① 提高计数器时钟频率 ② 测量 N 个周期取平均 |
| 频率太低 | CNT 溢出，无法完整测量 | ① 降低计数器时钟频率 ② 记录溢出次数累加 |

### 7.4 高频信号处理

```
信号频率 > 计数器频率时：

方案①：提高计数器时钟（减小 PSC）
  → 但有上限（72MHz）

方案②：测量 N 个周期取平均
  → 测量第 1 个上升沿到第 N+1 个上升沿的间隔
  → 周期 = 测量值 / N
  → 适用于频率特别高的场景
```

### 7.5 低频信号处理

```
信号周期 > 65535μs 时：

方案①：降低计数器时钟（增大 PSC）
  → 每次计数时间更长，可测范围更大

方案②：记录溢出次数
  → 每次溢出时中断计数 +1
  → 最终值 = 溢出次数 × 65536 + 当前 CCR 值
  → 适用于周期很长的信号
```

## 八、输入捕获测量周期配置总结

### 8.1 配置流程

```
① 开启定时器时钟 + GPIO 时钟

② 配置 GPIO：复用浮空输入（或上拉输入）

③ 配置 CCMR1：
   → CCxS = 01（输入模式）
   → ICxF = 0000（不滤波，或根据需要滤波）
   → ICxPSC = 00（不分频）

④ 配置 CCER：
   → CCxP = 0（上升沿捕获）或 1（下降沿）
   → CCxE = 1（使能捕获）

⑤ 配置 PSC 和 ARR：
   → PSC 根据精度需求选择（如 72 分频 → 1μs 计数）
   → ARR = 65535（最大值，避免溢出）

⑥ 使能定时器（CR1.CEN = 1）

⑦ 开启捕获中断（DIER.CC1IE = 1）+ NVIC
   → 在中断中读取 CCR 值，计算周期

⑧ 主循环中读取并处理测量结果
```

### 8.2 本实验参数

| 参数 | 值 | 说明 |
|------|:--:|------|
| 计数器时钟 | 72MHz | APB1 × 2 |
| PSC | 71 | 72 分频 → **1MHz** → 1μs/次 |
| ARR | 65535 | 最大值 |
| 捕获边沿 | 上升沿 | CCxP = 0 |
| 滤波 | 不滤波 | ICxF = 0 |
| 输入预分频 | 不分频 | ICxPSC = 0 |
| 测量范围 | 1μs ~ 65535μs | 对应频率 15.26Hz ~ 1MHz |

## 九、输入捕获 vs 输出比较

| 对比项 | 输入捕获 | 输出比较 |
|--------|----------|----------|
| 信号方向 | **外部 → 内部** | **内部 → 外部** |
| CCR 用途 | **保存** CNT 的快照值 | 作为与 CNT **比较**的基准 |
| 引脚配置 | 复用浮空/上拉**输入** | 复用推挽**输出** |
| CCMR.CCS | **01**（输入） | **00**（输出） |
| 边沿/极性 | CCxP 选择捕获边沿 | CCxP 选择输出有效电平 |
| CCxE | 使能捕获 | 使能输出 |
| 典型应用 | 测量信号周期/频率 | 产生 PWM 波形 |

## 十、CCMR1 配置对比（输入 vs 输出）

### 10.1 输出模式（之前呼吸灯）

```c
// 通道 2 输出模式
TIM5->CCMR1 &= ~TIM_CCMR1_CC2S_0;    // CC2S = 00（输出）
TIM5->CCMR1 &= ~TIM_CCMR1_CC2S_1;    //
TIM5->CCMR1 |=  TIM_CCMR1_OC2M_2;    // OC2M = 110（PWM 模式 1）
TIM5->CCMR1 |=  TIM_CCMR1_OC2M_1;    //
TIM5->CCMR1 &= ~TIM_CCMR1_OC2M_0;    //
```

### 10.2 输入模式（本实验）

```c
// 通道 1 输入模式
TIM->CCMR1 |=  TIM_CCMR1_CC1S_0;     // CC1S = 01（输入，映射到 TI1）
TIM->CCMR1 &= ~TIM_CCMR1_CC1S_1;     //
TIM->CCMR1 &= ~TIM_CCMR1_IC1F;       // IC1F = 0000（不滤波）
TIM->CCMR1 &= ~TIM_CCMR1_IC1PSC;     // IC1PSC = 00（不分频）
```

## 十一、输入捕获中断处理流程

```
中断服务函数（TIMx_IRQHandler）：

① 检查捕获中断标志（SR.CC1IF）
② 读取 CCR1 的值（捕获到的计数器快照）
③ 计算周期 = 当前 CCR1 - 上次 CCR1
④ 更新"上次 CCR1"变量
⑤ 清除中断标志（SR.CC1IF = 0）
⑥ 返回
```

## 十二、总结

| 项目 | 说明 |
|------|------|
| 输入捕获 | 信号边沿到来时，硬件自动将 CNT 值锁存到 CCR |
| 测量周期 | 两次上升沿捕获的 CCR 差值 × 计数器时钟周期 |
| 测量占空比 | 上升沿到下降沿的 CCR 差值 / 周期 |
| ARR 设为最大 | 65535，避免溢出 |
| PSC 选择 | 根据精度需求，通常 72 分频 → 1μs 精度 |
| CCER 配置 | CCxP 选边沿，CCxE 使能 |
| CCMR 配置 | CCxS = 01（输入），ICxF 和 ICxPSC 按需设置 |
| GPIO 配置 | 复用浮空输入（或上拉输入） |
